# 第82章：最终自检清单与补充篇总结

这一章是大模型工程师面试补充篇的收束章。

从第 60 章到第 82 章，我们讨论了从项目包装、面试表达、入职成长、技术判断、跨团队影响、路线图、指标复盘、安全风险、成本 ROI、长期成长，到求职材料、模拟面试、offer 选择和完整求职路线。

这套内容的目标不是制造焦虑，而是帮助你把“大模型工程师”从一个模糊岗位，拆成一套可训练、可证明、可成长的能力系统。

## 82.1 项目能力自检

你应该能回答：

- 我最核心的 1-2 个项目是什么？
- 每个项目解决了什么真实问题？
- 我的具体贡献是什么？
- 系统架构是什么？
- 数据链路是什么？
- 使用了哪些模型和工具？
- 如何评测？
- 指标如何变化？
- 失败 case 是什么？
- 安全和成本如何处理？
- 如果重做一次，我会怎么改？

如果这些问题答不清，先补项目复盘。

## 82.2 技术能力自检

你应该能解释：

- Transformer 和 attention。
- tokenization 和 position encoding。
- pretraining、SFT、RLHF、DPO。
- LoRA 和参数高效微调。
- RAG 的完整链路。
- embedding、hybrid retrieval、rerank。
- Agent、function calling、tool use。
- LLM eval 和 LLM-as-a-judge。
- KV cache、prefill、decode。
- vLLM、SGLang、PagedAttention、continuous batching。
- prompt injection、权限、审计和安全边界。

不要求每个方向都源码级深入。

但至少要能讲清核心机制、适用场景和 trade-off。

## 82.3 工程能力自检

你应该能讲清：

- 如何设计一个生产级 RAG 系统。
- 如何设计一个安全 Agent 系统。
- 如何设计一个推理服务。
- 如何设计评测平台。
- 如何做灰度和回滚。
- 如何做日志和 trace。
- 如何排查质量下降。
- 如何排查延迟上升。
- 如何做成本归因。
- 如何处理线上事故。

大模型工程师不是只调 prompt。

你要能把模型放进真实系统。

## 82.4 面试表达自检

你应该准备：

- 30 秒自我介绍。
- 2 分钟自我介绍。
- 30 秒项目介绍。
- 2 分钟项目介绍。
- 10 分钟项目深挖版本。
- 5 个行为面故事。
- 3 个系统设计模板。
- 5 个 debug 排查模板。
- 1 份反向提问清单。
- 1 份英语项目介绍。

所有回答都要结构化。

不要临场组织。

## 82.5 求职材料自检

你应该有：

- 一份目标岗位简历。
- 一个项目一页纸。
- 一个项目复盘文档。
- 一个 GitHub README 或代码样例。
- 一个面试题库。
- 一个公司投递跟踪表。
- 一个 offer 对比表。
- 一个 30-60-90 天计划。

材料不是一次性准备。

每次面试和项目后都要更新。

## 82.6 职业判断自检

你应该能判断：

- 这个岗位是真大模型工程，还是 API 包装？
- 这个团队是否有真实用户？
- 是否有评测闭环？
- 是否有安全和成本意识？
- 是否有技术 owner？
- 是否有资源支持？
- 我能否负责完整问题？
- 未来 1-2 年能否形成职业资产？

好的职业选择，不只是拿到 offer。

而是进入一个能持续成长和交付的环境。

## 82.7 最后一版求职前检查

投递前，检查：

- 简历是否匹配目标岗位。
- 项目是否有指标。
- 是否有一页纸。
- 是否能讲失败和取舍。
- 是否准备反向问题。
- 是否知道自己的薪资底线。
- 是否准备好面试复盘表。

面试前，检查：

- 核心项目是否能 2 分钟讲完。
- 高频原理题是否能讲清。
- 系统设计是否有模板。
- debug 是否有排查顺序。
- 行为故事是否有结果和反思。
- 英语表达是否练过。

拿 offer 前，检查：

- 岗位职责是否清楚。
- 前 90 天目标是否清楚。
- 汇报对象是否清楚。
- 薪酬结构是否完整。
- 成长空间是否真实。
- 风险是否可接受。

## 82.8 补充篇总结

大模型工程师的竞争力，不是一组孤立技能。

它是一套系统能力：

- 懂模型原理。
- 会工程落地。
- 能设计评测。
- 能处理线上问题。
- 有安全和成本意识。
- 能跨团队沟通。
- 能做技术判断。
- 能沉淀资产。
- 能持续成长。

面试只是这套能力的一次展示。

求职只是职业路径的一段。

真正长期有效的，是你能否不断把真实问题转化成项目，把项目转化成复盘，把复盘转化成能力，把能力转化成更大的影响力。

如果你已经读完这套补充篇，下一步不是继续无限读材料。

下一步应该是行动：

- 整理简历。
- 复盘项目。
- 写一页纸。
- 做模拟面试。
- 补一个真实 demo。
- 写一篇技术复盘。
- 投递几个目标岗位。
- 每次面试后更新题库。

学习最终要落到行动。

大模型工程师的成长，也只能在真实问题、真实反馈和真实交付中完成。
