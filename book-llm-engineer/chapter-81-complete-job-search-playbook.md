# 第81章：完整求职作战路线图

前面几章分别讨论了材料、模拟面试、offer 选择和失败案例。

这一章把所有内容整合成一条完整求职路线图。

目标是让你知道：从准备到投递，从面试到 offer，从入职到成长，每一步应该做什么。

## 81.1 第一阶段：定位目标岗位

先不要急着投简历。

先确定目标岗位。

大模型岗位至少分几类：

- 大模型应用工程师。
- RAG 工程师。
- Agent 工程师。
- 大模型算法工程师。
- 推理部署工程师。
- AI Infra 工程师。
- 评测平台工程师。
- 大模型产品工程师。

不同岗位准备重点不同。

应用工程师重 RAG、Agent、工具调用、上线和反馈。

算法工程师重模型原理、训练、微调、数据和实验。

推理工程师重 serving、KV cache、batching、并行、延迟和成本。

负责人岗位重路线图、跨团队协作、风险、成本和业务价值。

定位不清，简历和面试都会散。

## 81.2 第二阶段：补齐核心能力

根据目标岗位补能力。

最低公共能力包括：

- Transformer 和 LLM 基础。
- Prompt、RAG、Agent 基础。
- Python 和工程能力。
- 评测和实验方法。
- API、服务化和日志。
- 安全、权限和成本意识。

不要无限学习。

围绕目标岗位补最短路径。

例如投 RAG 岗位，就优先补：文档解析、chunk、embedding、hybrid retrieval、rerank、citation、评测、bad case。

例如投推理岗位，就优先补：prefill、decode、KV cache、PagedAttention、continuous batching、TTFT、TPOT、vLLM/SGLang。

## 81.3 第三阶段：做一个可讲项目

至少准备一个主项目。

主项目要能讲 30 分钟。

它应该包括：

- 背景。
- 问题。
- 架构。
- 数据。
- 模型。
- 评测。
- 指标。
- 风险。
- 成本。
- 失败和改进。

如果没有公司项目，可以做个人项目。

但个人项目要尽量生产化：

- 有 README。
- 有评测。
- 有错误分析。
- 有日志。
- 有安全边界说明。
- 有后续计划。

一个完整项目胜过五个浅 demo。

## 81.4 第四阶段：准备求职材料包

材料包包括：

- 一份目标岗位简历。
- 一个项目一页纸。
- 一个项目复盘。
- 一个 GitHub 或代码样例。
- 一个面试题库。
- 一个反向提问清单。

如果目标岗位不同，准备不同版本。

不要一份简历投所有岗位。

## 81.5 第五阶段：投递和内推

投递时要记录。

表格字段包括：

- 公司。
- 岗位。
- JD 要点。
- 投递渠道。
- 简历版本。
- 当前状态。
- 面试轮次。
- 面试问题。
- 复盘。
- 结果。

内推时不要只发简历。

最好附上：

- 目标岗位。
- 你和岗位匹配的三点。
- 一页纸项目摘要。
- GitHub 或作品链接。

让内推人更容易帮你。

## 81.6 第六阶段：面试准备

按轮次准备。

HR 面：讲动机、背景、期望、稳定性。

技术一面：讲原理和项目细节。

技术二面：讲系统设计、debug、实验和工程深度。

交叉面：讲协作、安全、成本、业务理解。

主管面：讲成长、owner 意识、路线图和影响力。

每轮都要准备反向问题。

面试是双向选择。

## 81.7 第七阶段：面试复盘和迭代

每场面试后立刻复盘。

记录：

- 问了什么。
- 答得如何。
- 追问在哪里。
- 暴露了什么短板。
- 团队有什么信息。
- 下一场如何改。

复盘后更新题库和标准回答。

求职过程本身就是训练过程。

越复盘，越稳定。

## 81.8 第八阶段：Offer 选择和谈薪

offer 选择看：

- 岗位真实性。
- 项目阶段。
- 团队能力。
- 个人职责。
- 资源支持。
- 成长空间。
- 组织稳定性。
- total compensation。

谈薪前明确底线。

拿到完整结构后再比较。

不要只看 base。

也不要只看短期薪资，忽略长期职业资产。

## 81.9 第九阶段：入职前准备

接受 offer 后，准备：

- 复习岗位相关技术。
- 整理 30-60-90 天计划。
- 确认汇报对象和第一个项目。
- 了解团队技术栈。
- 准备 onboarding 问题。

入职不是求职结束。

真正的职业收益要靠入职后兑现。

## 81.10 小结

完整求职路线不是“投简历、刷题、面试”。

它应该是：

```text
定位岗位 -> 补能力 -> 做项目 -> 准备材料 -> 投递内推 -> 模拟面试 -> 复盘迭代 -> offer 选择 -> 入职计划
```

每一步都要形成资产。

当你把求职当成系统工程，而不是临时冲刺，成功率和长期收益都会更高。
