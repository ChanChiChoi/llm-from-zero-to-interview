# 第十二章：多模态面试题

本章把第十五册的多模态与生成模型知识整理成面试问答。多模态面试常见特点是问题跨度大：前半段可能问 CLIP 和 VLM，后半段突然切到 diffusion、视频生成、语音、deepfake 和安全评估。回答时要抓住主线：模态表示、跨模态对齐、架构连接、训练目标、评估指标和安全风险。

建议练习方式：每题先用 1 分钟回答核心，再准备 3 分钟扩展版。如果面试官追问，就从结构、数据、loss、工程 trade-off 和 bad case 展开。

## 12.1 多模态总览

### 问题 1：什么是多模态模型？

参考回答：

```text
多模态模型是能处理多种信息形式的模型，例如文本、图像、语音和视频。核心问题是把不同模态编码成模型可处理的 token 或 embedding，并通过对齐、融合或生成目标，让模型能跨模态理解、检索、推理和生成。
```

### 问题 2：多模态模型的核心难点是什么？

参考回答：

```text
核心难点包括不同模态表示不同、信息密度不同、对齐困难、训练目标不统一、评估复杂和安全风险更大。比如图像需要视觉 encoder，语音需要 audio encoder 或 codec，视频还要处理时间一致性。
```

### 问题 3：多模态任务有哪些类型？

参考回答：

```text
可以分为理解、检索和生成。理解包括 VQA、OCR、图表问答和视频问答；检索包括 text-to-image 和 image-to-text retrieval；生成包括文生图、图生图、文生视频、TTS 和 speech-to-speech。
```

## 12.2 CLIP 与图文对齐

### 问题 4：CLIP 的核心思想是什么？

参考回答：

```text
CLIP 使用图文对比学习训练 image encoder 和 text encoder，把图片和文本映射到同一个语义空间。匹配图文对相似度高，不匹配图文对相似度低。训练好后可以做图文检索和 zero-shot 图像分类。
```

### 问题 5：CLIP 的 loss 怎么算？

参考回答：

```text
一个 batch 有 N 对图文，分别得到 N 个 image embeddings 和 N 个 text embeddings，归一化后计算 N x N 相似度矩阵。对角线是正样本，非对角线是 batch 内负样本。loss 通常是 image-to-text 和 text-to-image 两个方向的 cross entropy 平均。
```

### 问题 6：CLIP 如何做 zero-shot 分类？

参考回答：

```text
把类别名称写成 prompt，例如 a photo of a dog，用 text encoder 编码所有类别 prompt；再用 image encoder 编码待分类图片；计算图片向量和类别文本向量的相似度，选择相似度最高的类别。
```

### 问题 7：CLIP 的局限是什么？

参考回答：

```text
CLIP 强在全局语义对齐和检索，但弱在细粒度计数、OCR、空间关系、复杂组合关系和长回答。它输出的是相似度，不是对话模型，也容易受 prompt 和训练数据偏差影响。
```

## 12.3 Vision Encoder 与 VLM 架构

### 问题 8：Vision encoder 在 VLM 中有什么作用？

参考回答：

```text
Vision encoder 把原始图像像素编码成视觉特征。VLM 通常使用 patch-level visual tokens，再通过 projector 映射到 LLM hidden size，让 LLM 能基于图像信息进行问答和推理。
```

### 问题 9：ViT 的 patch embedding 是什么？

参考回答：

```text
Patch embedding 是把图片切成固定大小的 patch，并把每个 patch 映射成向量。实现上常用 kernel size 和 stride 都等于 patch size 的 Conv2d，把 [B,3,H,W] 转成 [B,N,d] 的视觉 token 序列。
```

### 问题 10：VLM 的基本架构是什么？

参考回答：

```text
常见 VLM 由 vision encoder、connector 和 LLM 组成。图片先经过 vision encoder 得到 visual tokens，再通过 projector、Q-Former、resampler 或 cross-attention 接入 LLM，最后 LLM 结合文本 prompt 和视觉信息生成回答。
```

### 问题 11：LLaVA 风格如何接入图片？

参考回答：

```text
LLaVA 风格通常用 CLIP vision encoder 提取 patch-level visual tokens，再用 MLP projector 映射到 LLM hidden size，然后把 visual tokens 插入文本 token 序列中，让 LLM 通过 self-attention 同时处理视觉和文本上下文。
```

### 问题 12：Q-Former 的作用是什么？

参考回答：

```text
Q-Former 使用一组 learnable query tokens 去查询 vision encoder 输出，从大量视觉特征中抽取少量和语言相关的视觉表示。它是 frozen vision encoder 和 LLM 之间的桥，可以减少送入 LLM 的视觉 token 数。
```

## 12.4 多模态 Instruction Tuning

### 问题 13：多模态 instruction tuning 的目标是什么？

参考回答：

```text
目标是让模型学会按照用户指令使用视觉信息。它训练模型做图片描述、VQA、OCR、图表理解、多轮图文对话和资料不足拒答。训练时通常只对 assistant 回复计算 loss。
```

### 问题 14：多模态 SFT 样本包含哪些字段？

参考回答：

```text
通常包含图片路径或 pixel values、messages 对话、image token 占位符和 assistant 回复。训练时还会构造 input_ids、labels、attention_mask、pixel_values，并确保图片数量和 image token 数量一致。
```

### 问题 15：为什么只对 assistant 部分算 loss？

参考回答：

```text
因为训练目标是让模型基于图片和用户指令生成助手回答，而不是预测用户输入、system prompt 或 image token。非 assistant 部分通常 mask 成 -100，否则模型会学会复述输入或预测占位符。
```

## 12.5 Diffusion 与文生图

### 问题 16：Diffusion 的核心思想是什么？

参考回答：

```text
Diffusion 的核心是加噪和去噪。训练时把真实样本按不同时间步加噪，让模型学习预测噪声或去噪方向；生成时从随机噪声开始，按 scheduler 多步去噪，逐渐得到清晰样本。文生图中，文本 prompt 作为条件控制去噪过程。
```

### 问题 17：Noise scheduler 的作用是什么？

参考回答：

```text
Scheduler 定义每个时间步的噪声强度，以及采样时如何从当前 noisy sample 更新到下一步。它影响训练加噪过程、推理采样路径、生成质量和速度，是 diffusion 系统中的关键组件。
```

### 问题 18：Stable Diffusion 的核心模块有哪些？

参考回答：

```text
Stable Diffusion 主要包括 VAE、text encoder、denoising U-Net 或 DiT、noise scheduler。VAE 负责像素和 latent 之间转换，text encoder 编码 prompt，U-Net 在 latent space 根据文本条件预测噪声，scheduler 控制采样过程。
```

### 问题 19：为什么 Stable Diffusion 在 latent space 里扩散？

参考回答：

```text
因为像素空间维度高，直接扩散计算成本大。Latent diffusion 先用 VAE 把图片压缩到低维 latent，在 latent 中去噪，最后再解码成图片，能显著降低训练和推理成本。
```

### 问题 20：Classifier-free guidance 是什么？

参考回答：

```text
CFG 让模型同时学习有条件和无条件去噪。生成时比较条件预测和无条件预测，并放大朝向条件的方向，使结果更符合 prompt。guidance scale 太低可能不跟 prompt，太高可能失真和多样性下降。
```

### 问题 21：ControlNet 解决什么问题？

参考回答：

```text
ControlNet 给 diffusion 模型加入额外结构条件，例如边缘、深度、姿态和分割图。Prompt 主要控制语义，ControlNet 可以更精确控制布局、姿态和结构，解决纯文本 prompt 难以精确控制生成的问题。
```

## 12.6 视频与语音

### 问题 22：视频生成为什么比图像生成难？

参考回答：

```text
视频比图像多了时间维度。模型不仅要保证每帧质量，还要保持帧间一致、物体身份稳定、动作连续、镜头运动合理和物理关系可信。很多失败来自闪烁、身份漂移、纹理变化和运动不连续。
```

### 问题 23：World model 和视频生成有什么关系？

参考回答：

```text
视频生成要求模型预测世界随时间变化，因此可能学到部分物体持久性、运动、遮挡和物理规律。但能生成合理视频不等于具备完整 world model。真正的 world model 还需要可交互、可预测行动后果，并能在任务中验证。
```

### 问题 24：ASR、TTS 和 speech-to-speech 有什么区别？

参考回答：

```text
ASR 是语音转文本，TTS 是文本转语音，speech-to-speech 是语音输入直接生成语音输出。工程上 speech-to-speech 可以用 ASR -> LLM -> TTS 的级联系统，也可以用端到端语音 token 模型。
```

### 问题 25：Whisper 的基本结构是什么？

参考回答：

```text
Whisper 是 encoder-decoder 语音模型。输入音频先转换成 log-mel spectrogram，由 audio encoder 编码声学信息，再由 text decoder 自回归生成文本 token。大规模多语言弱监督数据让它具备较强鲁棒性。
```

### 问题 26：Audio codec 和 speech token 的作用是什么？

参考回答：

```text
Audio codec 把连续 waveform 编码成离散 audio tokens，并能解码回音频。Speech token 让语音可以像文本 token 一样被 Transformer 建模，从而支持 TTS、speech-to-speech 和统一多模态语言模型。
```

## 12.7 统一多模态

### 问题 27：什么是统一多模态模型？

参考回答：

```text
统一多模态模型是能用统一架构、表示或接口处理文本、图像、语音、视频等多种模态的模型或系统。目标是支持多模态理解、推理和生成，例如图文问答、语音对话、文生图、视频理解和工具操作。
```

### 问题 28：统一多模态有哪些技术路线？

参考回答：

```text
可以分成接口统一、表示统一和架构统一。接口统一是多个模块组成一个助手；表示统一是把不同模态映射到共享 embedding 空间；架构统一是把文本、图像、语音、视频都 token 化，用统一 Transformer 建模。
```

### 问题 29：端到端统一模型和模块化系统怎么取舍？

参考回答：

```text
端到端模型信息流更自然，可以联合优化，但训练成本高、可控性和安全评估难。模块化系统能复用成熟 ASR、OCR、VLM、TTS 和生成模型，易调试、易上线，但有误差传播和信息损失。实际产品通常折中。
```

## 12.8 评估与安全

### 问题 30：如何评估一个 VLM？

参考回答：

```text
我会从 VQA、OCR、图表理解、grounding、多轮对话、视觉幻觉、安全和文本能力回归几个维度评估。除了最终准确率，还要做错误归因，区分感知错误、OCR 错误、grounding 错误和推理错误。
```

### 问题 31：多模态幻觉有哪些来源？

参考回答：

```text
来源包括 vision encoder 没捕捉到细节、OCR 错误、视觉 token 压缩丢信息、训练数据偏差、LLM 语言先验过强和 prompt 诱导过度猜测。模型可能生成流畅答案，但答案不被图像证据支持。
```

### 问题 32：什么是多模态 prompt injection？

参考回答：

```text
它是把恶意指令隐藏在图片、文档、网页截图、音频或视频中，诱导模型把外部内容当成系统指令执行。防护原则是外部内容只能作为数据，不应覆盖系统指令；工具调用还要做权限检查和二次确认。
```

### 问题 33：视频和语音生成有哪些安全风险？

参考回答：

```text
主要风险是 deepfake、虚假新闻、诈骗、伪造证据、未授权声音或肖像克隆，以及违法有害内容生成。需要授权机制、水印、provenance、深伪检测、输入输出审核和高风险请求拒绝。
```

## 12.9 项目追问

### 问题 34：如何讲一个 CLIP 图文检索项目？

参考回答：

```text
我会先说明数据集和任务，使用 CLIP image/text encoder 生成归一化 embedding，通过 cosine similarity 或 FAISS 做 text-to-image 和 image-to-text 检索。评估用 Recall@K、MRR 和 Top-1 Accuracy，并分析 OCR、中文 prompt、细粒度识别和空间关系 bad case。
```

### 问题 35：如何设计一个 VLM 图片问答评测集？

参考回答：

```text
我会覆盖图片描述、物体识别、属性、计数、空间关系、OCR、图表、多轮问答和资料不足拒答。每个样本标注标准答案和证据区域，并记录错误类型，区分感知错误、OCR 错误、推理错误和幻觉。
```

### 问题 36：如何设计多模态 RAG？

参考回答：

```text
多模态 RAG 可以把图片 OCR、caption、metadata、文本 chunk 和图像 embedding 统一索引。查询时做文本检索、图像检索或 hybrid retrieval，再用 reranker 和 LLM 基于证据回答。评估要看 retrieval recall、citation accuracy、groundedness 和 unsupported claim rate。
```

## 12.10 开放研究题

### 问题 37：下一代多模态模型可能怎么发展？

参考回答：

```text
我认为会朝更统一的 Any-to-Any 模型、更强长视频和语音实时交互、更好的 grounding 和工具使用、更低成本的多模态 token 化、更可靠的安全评估发展。同时会更强调可验证证据、隐私保护和端侧多模态能力。
```

### 问题 38：多模态模型是否真的理解世界？

参考回答：

```text
要谨慎判断。视频生成和跨模态推理说明模型可能学到部分世界统计规律，例如物体外观、运动和遮挡。但真正理解世界需要可交互、可预测行动后果、能泛化到新场景，并能通过任务验证。生成看起来合理不等于具备完整 world model。
```

## 12.11 本章总结

多模态面试的回答要围绕六条主线：

1. 表示：不同模态如何变成 token 或 embedding。
2. 对齐：CLIP、SigLIP、contrastive learning。
3. 架构：vision encoder、connector、LLM、Q-Former、resampler。
4. 生成：diffusion、Stable Diffusion、DALL·E、视频、语音。
5. 评估：VQA、OCR、grounding、幻觉、安全。
6. 工程：数据、loss mask、延迟、成本、上线监控。

真正好的回答不是背名词，而是能说清楚输入输出、模型结构、训练目标、指标、失败模式和改进方向。到这里，第十五册《多模态与生成模型专题》的正文第一版已经完成。
