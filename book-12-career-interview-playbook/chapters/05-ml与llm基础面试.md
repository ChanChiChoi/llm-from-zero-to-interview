# 第五章：ML 与 LLM 基础面试

ML 与 LLM 基础面试考的不是你能不能背出很多名词，而是你能不能把大模型背后的基本假设、训练目标、优化机制和模型结构讲清楚。

很多候选人在这一关失分，不是因为完全不会，而是因为知识是碎的。比如能背交叉熵公式，但讲不清它和最大似然的关系；知道 Transformer，但讲不清 attention 为什么有效；听过 overfitting，但解释不了大模型为什么在巨量参数下仍能泛化；会说 tokenization，但不知道 tokenizer 会影响训练效率、压缩率、跨语言表现和安全边界。

基础面试的目标，是证明你不是只会调库和背论文，而是真的理解大模型是如何从数据中学习、如何被优化、如何生成文本、为什么会出错。

本章重点：概率、优化、泛化、Transformer、LLM 训练目标、tokenization、loss。

## 5.1 基础面试到底考什么

大模型算法岗的基础面试通常围绕六条主线：

1. 概率与统计：概率分布、条件概率、最大似然、交叉熵、KL 散度。
2. 优化：梯度下降、Adam、learning rate、batch size、正则化、收敛与稳定性。
3. 泛化：过拟合、欠拟合、数据规模、模型规模、正则化、训练集与测试集差距。
4. Transformer：self-attention、位置编码、残差、归一化、MLP、decoder-only 架构。
5. LLM 训练目标：next-token prediction、causal LM loss、SFT、偏好学习。
6. Tokenization：BPE、SentencePiece、词表大小、特殊 token、长度和多语言问题。

这些主题看似分散，其实可以用一句话串起来：大模型用 tokenizer 把文本变成 token 序列，用 Transformer 表示上下文，通过 next-token prediction 最大化训练数据的似然，再用优化器在大规模数据和算力上学习参数，最后通过评估和对齐让模型更有用、更安全。

面试时最重要的是建立这种整体图景。

## 5.2 回答基础题的三层结构

基础题不要只给定义。推荐用三层结构回答：

1. 直觉：这个概念解决什么问题。
2. 机制：它具体怎么工作，可以给公式或流程。
3. 边界：它有什么假设、缺点和常见坑。

例如问“什么是交叉熵”，差的回答是：

```text
交叉熵是分类任务常用损失，用来衡量两个分布的差异。
```

更好的回答是：

```text
交叉熵可以理解为用模型分布 q 去编码真实分布 p 时的平均代价。在分类或语言模型里，真实标签通常是 one-hot，所以交叉熵等价于最大化正确 token 的 log probability。它和最大似然直接相关：最小化负 log likelihood 就是在最小化训练数据上的交叉熵。边界是它依赖标签质量，如果数据有噪声或 label smoothing、mask 处理不当，loss 解释会变化。
```

这个回答有直觉、机制和边界。

## 5.3 概率基础：语言模型到底在建模什么

语言模型本质上建模的是 token 序列的概率。

给定一个 token 序列：

```text
x_1, x_2, ..., x_T
```

自回归语言模型把联合概率分解为：

```text
P(x_1, x_2, ..., x_T) = P(x_1) P(x_2 | x_1) ... P(x_T | x_1, ..., x_{T-1})
```

也就是每一步根据前面的上下文预测下一个 token。

这就是 next-token prediction 的概率基础。

面试官可能问：为什么语言模型可以只预测下一个 token？

可以这样回答：

```text
因为根据概率链式法则，任意序列的联合概率都可以分解为一系列条件概率的乘积。自回归语言模型选择按从左到右的顺序建模这些条件概率。训练时最大化每个真实下一个 token 的概率，推理时反复采样下一个 token，就可以生成完整序列。
```

需要注意，next-token prediction 看似简单，但它迫使模型学习大量隐含能力：语法、事实、推理模式、代码结构、对话风格和世界知识。因为要预测下一个 token，模型必须从上下文中提取尽可能多的信息。

## 5.4 最大似然、负对数似然和交叉熵

最大似然估计的目标是：找到一组模型参数，让训练数据出现的概率尽可能大。

对语言模型来说，就是最大化训练集中每个真实 token 的条件概率。由于很多概率相乘会非常小，实际优化通常最大化 log probability，等价于最小化负 log likelihood。

直观地说：

1. 模型给正确 token 的概率越高，loss 越低。
2. 模型给正确 token 的概率越低，loss 越高。
3. 如果模型非常自信地预测错，loss 会很大。

在 one-hot 标签下，cross entropy 和 negative log likelihood 是等价的。

面试常见追问：为什么不用 accuracy 作为训练目标？

可以回答：

```text
Accuracy 不可导，而且只关心预测是否排第一，不关心概率分布的置信度。交叉熵是可导的，会鼓励模型提高正确 token 的概率，并且对“自信地错”惩罚更大，更适合作为梯度优化目标。
```

另一个常见追问：perplexity 是什么？

Perplexity 可以理解为语言模型平均每一步有多少有效选择。它通常等于平均负 log likelihood 的指数形式。loss 越低，perplexity 越低，说明模型对真实序列越不困惑。

但 perplexity 也有边界：

1. 不同 tokenizer 下 perplexity 不一定可直接比较。
2. 低 perplexity 不一定代表回答更有用。
3. 对齐后的模型可能牺牲部分 perplexity，但交互质量更好。
4. 如果测试集被污染，perplexity 会虚高或虚低。

## 5.5 KL 散度和交叉熵的关系

KL 散度衡量一个分布 q 相对于真实分布 p 的偏离程度。它不是对称距离，因为 KL(p || q) 和 KL(q || p) 含义不同。

交叉熵、熵和 KL 散度之间有一个重要关系：

```text
H(p, q) = H(p) + KL(p || q)
```

如果真实分布 p 固定，最小化交叉熵就等价于最小化 KL(p || q)。

在大模型里，KL 经常出现在：

1. 蒸馏：让学生模型接近教师模型。
2. RLHF：限制 policy 不要偏离 reference model 太远。
3. DPO：隐式地控制 policy 相对 reference 的偏移。
4. 分布漂移分析：比较训练分布和测试分布。

面试时不用把公式推得很复杂，但要讲清方向。KL(p || q) 更关注 q 是否覆盖 p 中高概率的区域；KL(q || p) 更关注 q 自己生成的区域在 p 下是否合理。不同方向会带来不同优化偏好。

## 5.6 优化基础：梯度下降在做什么

深度学习训练本质上是在高维参数空间里最小化 loss。梯度告诉我们 loss 增长最快的方向，所以负梯度方向通常能让 loss 下降。

最基础的更新形式是：

```text
参数新值 = 参数旧值 - 学习率 * 梯度
```

学习率是最重要的超参之一。

1. 学习率太大，训练可能震荡、发散或 loss spike。
2. 学习率太小，训练很慢，可能停在较差区域。
3. warmup 可以避免训练初期梯度不稳定导致参数剧烈变化。
4. decay 可以在后期更细致地收敛。

面试官可能问：为什么大模型训练常用 warmup？

可以回答：

```text
训练初期模型参数还没有形成稳定表示，梯度统计也不稳定。如果一开始使用较大学习率，容易造成 loss spike 或发散。warmup 让学习率从小逐渐升高，给优化器和模型表示一个稳定启动过程，尤其在大 batch、混合精度和分布式训练中很重要。
```

## 5.7 SGD、Adam 和 AdamW

SGD 使用当前 batch 的梯度更新参数。它简单、泛化性质好，但对学习率敏感，收敛可能慢。

Adam 在 SGD 基础上引入一阶矩和二阶矩估计。直观地说，它会根据梯度均值和梯度方差为不同参数自适应调整步长，因此在 Transformer 和大模型训练中非常常用。

AdamW 是 Adam 的重要改进，核心是 decoupled weight decay，也就是把权重衰减和梯度更新解耦。

面试常见问题：Adam 和 AdamW 有什么区别？

可以回答：

```text
Adam 中如果直接把 L2 正则加到梯度里，权重衰减会和 Adam 的自适应学习率耦合，不同参数受到的衰减不再是统一的。AdamW 把 weight decay 从梯度更新中解耦，直接对参数做衰减，更符合权重衰减的原始含义，也通常带来更稳定的 Transformer 训练效果。
```

还要知道 weight decay 不是对所有参数都应该加。bias、LayerNorm 或 RMSNorm 的缩放参数通常不做 weight decay，因为它们的作用和普通权重不同。

## 5.8 Batch Size、梯度累积和泛化

Batch size 影响训练速度、显存、梯度噪声和泛化。

大 batch 的优点：

1. 吞吐更高。
2. 梯度估计更稳定。
3. 分布式训练效率更好。

大 batch 的缺点：

1. 显存需求高。
2. 可能泛化变差。
3. 需要更仔细的学习率和 warmup 调整。

梯度累积可以在显存有限时模拟更大的 global batch。比如每张卡 micro batch 是 2，8 张卡，累积 4 步，则 global batch 是 2 * 8 * 4 = 64。

面试官可能问：梯度累积和真正大 batch 完全等价吗？

大多数情况下，如果没有 dropout 随机性差异、batch norm 统计差异、优化器 step 时机差异等问题，梯度累积可以近似等价于更大的 batch。但在实际系统中，随机性、数值误差和并行实现会带来细微差别。

## 5.9 泛化：为什么大模型不只是记忆

传统直觉认为，参数太多容易过拟合。但大模型常常参数极多，却在很多任务上表现出很强泛化能力。这背后有多个原因：

1. 训练数据规模巨大，覆盖了丰富模式。
2. 自监督任务提供了密集训练信号。
3. Transformer 架构有强表达能力。
4. 优化过程和正则化会影响最终解。
5. 预训练学到的表示可以迁移到下游任务。

但大模型并不是不会记忆。它可能记住训练数据中的事实、代码片段、隐私信息或 benchmark 样本。因此泛化和记忆同时存在。

面试时可以这样回答“大模型是不是在记忆训练数据”：

```text
大模型既有记忆，也有泛化。记忆体现在它可能复现高频或重复训练样本，尤其是罕见字符串、代码和个人信息。泛化体现在它能组合训练中学到的模式处理新问题。判断两者需要看数据去重、污染检测、held-out 评估、反事实测试和任务迁移表现。
```

这个回答比简单说“不是记忆，是理解”更成熟。

## 5.10 Bias-Variance、过拟合和欠拟合

过拟合是训练集表现好、测试集表现差。欠拟合是训练集和测试集都表现差。

在大模型中，过拟合不只发生在小模型或小数据上。SFT、DPO、小规模领域微调、评估集迭代过多时都可能过拟合。

常见现象：

1. 训练 loss 持续下降，验证 loss 上升。
2. 固定评估集提升，线上样本不提升。
3. 模型学会固定格式，但真实任务能力没提升。
4. 对安全数据过拟合，导致大量误拒。

应对方法包括：

1. 增加和清洗数据。
2. 使用验证集和早停。
3. 降低学习率或训练轮数。
4. 加强数据多样性。
5. 做交叉评估和 held-out 测试。
6. 对评估集做污染检查。

## 5.11 Transformer 的核心直觉

Transformer 的核心思想是：让序列中每个位置根据上下文中其他位置的信息更新自己的表示。

RNN 按时间顺序处理 token，长距离依赖难训练且并行性差。CNN 局部感受野强，但要堆很多层才能建模长距离关系。Self-attention 允许任意两个 token 直接交互，因此更适合长文本建模，并且训练时更容易并行。

Transformer block 通常包含：

1. Self-attention：聚合上下文信息。
2. MLP：对每个位置做非线性变换。
3. 残差连接：帮助梯度传播和保留原始信息。
4. LayerNorm 或 RMSNorm：稳定训练。
5. 位置编码：让模型知道 token 顺序。

面试官问“attention 为什么有效”，可以回答：

```text
Self-attention 为每个 token 动态选择上下文中最相关的信息。和固定窗口或顺序递归不同，attention 权重依赖当前输入，因此同一个 token 在不同上下文中可以关注不同位置。这让模型能灵活建模长距离依赖、指代关系、代码结构和多步推理线索。
```

## 5.12 Self-Attention 的机制

Self-attention 中，每个 token 的 hidden state 会投影成 query、key、value。

直觉如下：

1. query 表示“我想找什么信息”。
2. key 表示“我能提供什么索引”。
3. value 表示“如果被关注，我提供什么内容”。

计算流程：

1. query 和 key 做点积，得到相关性分数。
2. 分数除以维度平方根，避免数值过大。
3. 加 mask，屏蔽未来 token 或 padding。
4. softmax 得到 attention 权重。
5. 用权重加权 value，得到上下文表示。

为什么要除以维度平方根？

如果 query 和 key 维度很大，点积的方差会随维度增大，softmax 输入过大后容易饱和，梯度变小。除以 sqrt(d) 可以稳定分数尺度。

为什么 decoder-only LLM 需要 causal mask？

因为训练目标是根据过去预测未来。如果不加 causal mask，模型在训练时可以看到答案 token，造成信息泄漏，训练 loss 会虚低，推理时无法复现这种条件。

## 5.13 Multi-Head Attention 为什么需要多头

单头 attention 只能在一个表示子空间里计算相关性。多头 attention 把 hidden state 分成多个 head，让不同 head 学习不同关系。

例如：

1. 有的 head 关注局部语法。
2. 有的 head 关注长距离依赖。
3. 有的 head 关注括号或代码结构。
4. 有的 head 关注实体指代。

当然，不要把每个 head 的功能解释得过于神秘。实际模型中 head 的可解释性有限，有些 head 可能冗余，可以被剪枝或合并。GQA 和 MQA 就是在推理效率和表示能力之间做取舍。

面试时可以补充：多头不是简单增加参数，而是让模型在多个子空间并行建模关系；但更多 head 不一定总是更好，因为 head_dim、显存、通信和推理成本也会变化。

## 5.14 位置编码：为什么 Transformer 需要顺序信息

Self-attention 本身对输入顺序不敏感。如果不加入位置信息，模型很难区分“狗咬人”和“人咬狗”。

常见位置方法包括：

1. 绝对位置编码。
2. 相对位置编码。
3. RoPE。
4. ALiBi。

大模型中 RoPE 很常见。它通过旋转 query 和 key，把相对位置信息融入 attention 分数。RoPE 的优势是适合自回归模型，能较好表达相对位置关系，并且在一定程度上支持长度外推。但长度外推不是无条件可靠，通常还需要训练长度、缩放策略和评估配合。

面试官可能问：位置编码会影响长上下文能力吗？

可以回答：

```text
会。位置编码决定模型如何表示 token 间距离。训练时见过的最大长度、RoPE scaling、attention 实现、数据分布和评估任务都会影响长上下文表现。简单扩大 context window 不一定让模型真正会用长上下文，还需要长上下文数据和评估验证。
```

## 5.15 LayerNorm、RMSNorm 和残差连接

归一化和残差连接是深层 Transformer 稳定训练的关键。

残差连接让每层学习对表示的增量修改，而不是完全重写表示。它有助于梯度传播，也让深层网络更容易优化。

LayerNorm 会对每个 token 的 hidden 维度做归一化，减均值、除标准差，再乘可学习缩放和偏置。RMSNorm 不减均值，只按均方根缩放，计算更简单，在很多 LLM 中常见。

Pre-Norm 和 Post-Norm 也是高频考点：

1. Post-Norm 是先子层，再残差，再归一化。
2. Pre-Norm 是先归一化，再子层，再残差。

大模型常用 Pre-Norm，因为训练更稳定，梯度更容易流过残差路径。但 Post-Norm 在某些设置下也有优势。

## 5.16 MLP、激活函数和 SwiGLU

Transformer block 里除了 attention，还有 MLP。Attention 负责 token 间信息交互，MLP 负责对每个位置的表示进行非线性变换和特征组合。

早期 Transformer 常用 ReLU 或 GELU。很多现代 LLM 使用 SwiGLU 或 GeGLU 这类门控激活。

SwiGLU 的直觉是：一部分投影产生候选特征，另一部分投影产生门控信号，用门控控制哪些特征通过。这通常能提升表达能力，但会增加一定计算量。

面试时可以这样回答“MLP 在 Transformer 中有什么作用”：

```text
Attention 主要做跨 token 的信息混合，MLP 主要做每个 token 位置上的非线性特征变换。没有 MLP，模型只有线性投影和 attention 加权，表达能力会受限。现代 LLM 中 MLP 往往占据大量参数和计算，是模型知识和能力的重要承载部分。
```

## 5.17 Decoder-only、Encoder-only 和 Encoder-Decoder

三类 Transformer 架构要能区分。

| 架构 | 代表 | 适合任务 |
| --- | --- | --- |
| Encoder-only | BERT | 理解、分类、检索、表示学习 |
| Decoder-only | GPT 系列 | 自回归生成、对话、代码、通用 LLM |
| Encoder-Decoder | T5 | 条件生成、翻译、摘要 |

Decoder-only LLM 用 causal mask，从左到右预测下一个 token。它特别适合统一成“给定上下文，继续生成”的接口，所以成为通用大模型主流架构。

Encoder-only 不适合直接自回归生成，因为它通常双向看上下文。Encoder-decoder 在条件生成上很自然，但架构和推理缓存更复杂。

面试时不要说某一种架构绝对更好。不同架构服务不同任务和历史阶段。

## 5.18 Tokenization：文本如何变成模型输入

模型不能直接处理字符串，必须先通过 tokenizer 把文本变成 token id。

常见方法包括：

1. Word-level：按词切分，词表大，遇到未知词麻烦。
2. Character-level：词表小，但序列很长。
3. BPE：从字符开始合并高频片段。
4. WordPiece：类似子词方法，常见于 BERT。
5. SentencePiece：把文本当作原始字符流处理，常用于多语言。

大模型常用子词 tokenizer，因为它在词表大小和序列长度之间折中。

Tokenizer 会影响：

1. 训练效率：同样文本被切成多少 token。
2. 多语言表现：不同语言压缩率不同。
3. 代码能力：空格、缩进、符号切分很重要。
4. 安全边界：恶意字符串、Unicode、特殊 token 可能造成异常行为。
5. 上下文长度：token 数不等于字符数或词数。

面试官可能问：为什么中文或某些语言 token 更长会有问题？

可以回答：

```text
如果 tokenizer 对某种语言压缩率差，同样语义需要更多 token，训练和推理成本更高，有效上下文更短，模型在该语言上的数据利用效率也更低。因此 tokenizer 的多语言覆盖和压缩率会影响模型能力和公平性。
```

## 5.19 特殊 Token 和 Padding

常见特殊 token 包括 BOS、EOS、PAD、UNK，以及对话模型中的 system、user、assistant 分隔 token。

这些 token 看起来是工程细节，但会影响训练和生成。

1. EOS 学不好，模型可能不知道何时停止。
2. PAD 如果参与 loss，会污染训练目标。
3. 对话模板不一致，会导致训练和推理分布不匹配。
4. 特殊 token 泄漏或误用，可能造成模型行为异常。

面试中常见问题：padding token 为什么要在 loss 中 ignore？

因为 padding 不是自然文本的一部分，只是为了 batch 对齐。如果把 padding 当成真实目标训练，模型会被鼓励预测 PAD，影响语言建模目标。

## 5.20 LLM 训练目标：Pretraining、SFT 和偏好学习

LLM 训练通常可以分为几个阶段。

预训练阶段：用大规模文本做 next-token prediction，学习语言、知识、代码和推理模式。

SFT 阶段：用指令数据训练模型按照人类指令回答，让模型学会对话格式、任务遵循和回答风格。

偏好学习阶段：用人类或模型偏好数据，让模型更偏向有用、安全、诚实的回答。常见方法包括 RLHF、RLAIF、DPO 等。

这三个阶段目标不同：

1. 预训练让模型“会语言、会知识”。
2. SFT 让模型“会按指令回答”。
3. 偏好学习让模型“更符合人类偏好和安全要求”。

面试官可能问：为什么不只做预训练？

可以回答：

```text
预训练模型学到的是续写分布，不一定知道如何作为助手回答问题。它可能补全文档、模仿网页、输出不稳定格式。SFT 用指令数据把模型行为转向问答和任务完成，偏好学习进一步优化回答质量、安全性和人类偏好。
```

## 5.21 Causal LM Loss 的细节

Causal LM loss 的关键是 shift。

输入 token 是：

```text
[x_1, x_2, x_3, ..., x_T]
```

模型在位置 1 的输出预测 x_2，在位置 2 的输出预测 x_3，以此类推。因此训练时通常用 logits 的前 T-1 个位置对齐 labels 的后 T-1 个 token。

常见错误：

1. logits 和 labels 没有 shift。
2. padding 没有 ignore。
3. prompt 部分和 response 部分 mask 错误。
4. 多轮对话模板和训练 labels 不匹配。
5. 评估 loss 时把不该算的 token 算进去。

SFT 中经常只对 assistant response 计算 loss，而不对 user prompt 计算 loss。原因是我们希望模型学习如何回答，而不是学习复述用户输入。

但这也不是绝对规则。有些训练设置会对完整序列算 loss，取决于数据格式和目标。

## 5.22 面试高频基础题与回答要点

### 5.22.1 为什么 Transformer 比 RNN 更适合大模型

回答要点：

1. Transformer 训练并行性更好。
2. Self-attention 更容易建模长距离依赖。
3. 残差、归一化和大规模优化配合成熟。
4. 但 attention 的序列长度平方复杂度是代价。

### 5.22.2 为什么 attention 要加 mask

回答要点：

1. Causal mask 防止看到未来 token。
2. Padding mask 防止关注无意义 padding。
3. mask 语义要明确 True 是保留还是屏蔽。

### 5.22.3 为什么 softmax 后能表示概率

回答要点：

1. softmax 输出非负。
2. 所有类别概率和为 1。
3. 输入 logit 差值决定相对概率。
4. temperature 可以调节分布尖锐程度。

### 5.22.4 为什么需要 validation set

回答要点：

1. 训练集 loss 不能反映泛化。
2. 验证集用于调参、早停和模型选择。
3. 过度使用验证集也会导致评估过拟合。

### 5.22.5 为什么模型会 hallucinate

回答要点：

1. 训练目标是预测 plausible token，不是真实性验证。
2. 预训练数据有噪声和矛盾。
3. 模型缺少外部事实校验。
4. 解码策略可能放大不确定性。
5. SFT 可能鼓励模型回答而不是承认不知道。

### 5.22.6 为什么 instruction tuning 有用

回答要点：

1. 预训练学会续写，不等于学会听指令。
2. 指令数据把任务格式和回答风格对齐到用户需求。
3. 它可以显著提升零样本和多任务指令遵循能力。
4. 数据质量比数量更关键。

## 5.23 如何表现出“基础扎实”

基础扎实不是把定义背得很流利，而是能在不同概念之间建立连接。

例如：

1. 从 next-token prediction 讲到最大似然。
2. 从交叉熵讲到 KL 散度和 perplexity。
3. 从 attention 讲到 mask、复杂度和 KV cache。
4. 从 tokenizer 讲到上下文长度和多语言公平性。
5. 从 SFT loss 讲到 prompt mask 和 assistant-only loss。
6. 从泛化讲到数据污染和评估可信度。

面试中如果你能主动补充边界条件，通常是加分项。比如讲 perplexity 时提到 tokenizer 不同不能直接比较，讲 long context 时提到扩大窗口不等于会用长上下文，讲 hallucination 时提到训练目标和事实校验之间的差距。

## 5.24 常见失分点

### 5.24.1 背概念但讲不出关系

比如知道 cross entropy、KL、maximum likelihood，但不知道它们在语言模型训练中如何连接。

### 5.24.2 公式会背但不会解释直觉

面试官更关心你能否把公式和模型行为联系起来。

### 5.24.3 忽略工程细节

基础题也会追到实现，比如 labels shift、padding mask、dtype、tokenizer 特殊 token。

### 5.24.4 绝对化表达

例如“模型参数越大一定越好”“perplexity 越低模型一定越强”“DPO 一定比 RLHF 好”。大模型里几乎所有结论都有条件。

### 5.24.5 不承认不确定性

遇到不确定问题，可以说出自己的理解和假设，而不是硬编。强候选人会区分确定知识、经验判断和需要实验验证的部分。

## 5.26 本章小结

ML 与 LLM 基础面试考的是知识结构，而不是名词数量。

你需要把概率、优化、泛化、Transformer、tokenization 和训练目标连成一张图：文本经过 tokenizer 变成 token，Transformer 用 attention 和 MLP 建模上下文，训练目标通过 next-token prediction 最大化数据似然，优化器在大规模数据上更新参数，泛化能力来自数据、架构和优化共同作用，而 SFT 和偏好学习进一步改变模型行为。

准备这一章的关键，是每个概念都能讲清直觉、机制和边界。下一章进入 Training 面试，会把这些基础进一步落到大规模训练中的数据、分布式、loss spike、checkpoint、评估和成本估算问题上。
