# 第二十四册：大模型推理框架与 Serving Engine 实战

## 本书定位

本书是“大模型算法岗系统面试二十四部曲”的第二十四册，专门讲大模型推理框架、LLM Serving Engine、vLLM/SGLang 架构和从 0 实现推理系统。

第六册偏“会部署、会调参、会优化”，第二十三册偏“平台承载、资源调度、可观测性和治理”。本书偏“懂引擎内部、能实现核心模块、能改造 serving 架构”。

本书重点回答：

1. LLM serving engine 到底由哪些模块组成。
2. 从 naive generate 到生产级推理框架，中间需要补哪些系统能力。
3. vLLM 为什么能提升吞吐，PagedAttention 和 continuous batching 如何工作。
4. SGLang 为什么强调 runtime、RadixAttention、prefix sharing 和 structured generation。
5. PD 分离为什么出现，prefill 和 decode 的资源画像有什么不同。
6. 如何从 nano-vLLM、tiny-llm、mini-sglang 这类教学项目学习推理框架。
7. 面试中如何设计一个 vLLM-like 或 SGLang-like 推理系统。

## 与其他书的关系

1. 第六册讲推理部署与优化总览，本书讲推理框架和 serving engine 内部实现。
2. 第二十三册讲 AI Infra、GPU 集群和推理平台治理，本书讲具体推理引擎架构。
3. 第十一册讲系统设计面试，本书提供推理框架类系统设计题素材。
4. 第三册讲项目实战，本书可沉淀从 0 实现 mini serving engine 的实战项目。
5. 第十九册讲真实工程坑，本书讲推理性能和 serving 事故背后的框架级原因。
