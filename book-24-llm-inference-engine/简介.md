# 第二十四册简介：大模型推理框架与 Serving Engine 实战

## 本书定位

第二十四册专门讲大模型推理框架、LLM Serving Engine 和从 0 实现推理系统。

第六册讲“大模型部署与推理工程”，重点是如何使用和优化推理服务；第二十三册讲“AI Infra”，重点是平台化承载、调度、治理和可观测性。本书则下沉到推理框架内部，系统讲解 vLLM、SGLang、PagedAttention、continuous batching、KV Cache 管理、scheduler、prefix cache、radix cache、speculative decoding、PD 分离和分布式 serving 架构。

本书也会把 `nano-vLLM`、`tiny-llm`、`mini-sglang` 这类教学项目作为源码学习路线，帮助读者从最小可运行推理框架逐步升级到生产级 serving engine。

## 为什么需要这本书

大模型推理已经不只是“调用 generate API”。真实线上系统要同时处理高并发、低延迟、高吞吐、显存约束、多模型路由、流式输出、长上下文、prefix 复用、KV Cache 管理、节点故障和成本控制。

vLLM、SGLang 等框架之所以重要，是因为它们把这些问题系统化为一个推理引擎：请求进入队列，scheduler 组织 batch，runtime 执行 prefill 和 decode，KV cache manager 管理显存块，streaming 组件返回 token，metrics 系统持续记录 TTFT、TPOT、吞吐和显存。

如果只会部署框架，而不理解框架内部，就很难回答高级面试问题，也很难在真实项目中定位性能瓶颈、设计 PD 分离、改造调度策略或评估新 serving 架构。

## 适合读者

1. 已理解 Transformer 推理、KV Cache 和基础部署，但想深入推理框架内部的读者。
2. 想准备 vLLM、SGLang、LLM serving、推理系统和系统设计面试的候选人。
3. 想从 0 写一个 mini serving engine，理解 scheduler、KV cache 和 batching 的工程师。
4. 想研究 PD 分离、disaggregated serving、prefix cache、speculative decoding 和分布式推理架构的人。

## 学完后应具备的能力

1. 能讲清 LLM serving engine 的核心模块和请求生命周期。
2. 能从 0 实现一个最小推理框架，包括 generate loop、KV Cache、batching、scheduler 和 streaming。
3. 能解释 vLLM 的 PagedAttention、block manager、continuous batching 和 engine 架构。
4. 能解释 SGLang 的 runtime、RadixAttention、prefix sharing、structured generation 和调度思路。
5. 能分析 PD 分离为什么出现、解决什么问题、带来什么新瓶颈。
6. 能设计 KV Cache manager、continuous batching scheduler、prefix cache 和推理性能压测方案。
7. 能把 nano-vLLM、tiny-llm、mini-sglang 这类教学项目和生产级框架建立对应关系。

## 推荐参考项目

1. `nano-vLLM`：https://github.com/GeeeekExplorer/nano-vllm
2. `tiny-llm`：https://github.com/skyzh/tiny-llm
3. `mini-sglang`：https://github.com/sgl-project/mini-sglang

这些项目适合作为源码阅读和实战改造入口。阅读时重点关注：engine 如何组织请求，scheduler 如何形成 batch，KV cache 如何分配和释放，prefill/decode 如何执行，streaming 如何返回 token，以及教学实现和生产实现之间差了哪些工程模块。
