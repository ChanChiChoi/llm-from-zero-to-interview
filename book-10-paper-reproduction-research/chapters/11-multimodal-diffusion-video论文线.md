# 第十一章：Multimodal、Diffusion、Video 论文线

这一章讨论多模态、扩散生成和视频生成论文线。它们是大模型从“只处理文本”走向“理解和生成现实世界信号”的关键路线。

如果说前面章节里的 Transformer、scaling、alignment、RAG 和 Agent 主要围绕语言模型展开，那么这一章要回答的是另一个问题：模型如何把图像、语音、视频这些连续世界信号接入统一的学习框架？

多模态不是简单地“给语言模型加个图片输入”。它背后有几组核心矛盾：

1. 图像、音频、视频是连续信号，语言是离散符号，表示方式不同。
2. 图文、音文、视频文本之间的对齐数据规模大但噪声也大。
3. 理解任务和生成任务的训练目标不同。
4. 图像生成要看清晰度和文本一致性，视频生成还要看时间一致性、物理合理性和镜头连贯性。
5. 多模态模型既要继承语言模型的推理能力，又要避免视觉幻觉、听觉错译和生成滥用风险。

本章按论文线展开：先讲图文对齐，以 CLIP 为代表；再讲视觉语言模型，以 Flamingo、BLIP、LLaVA 为代表；再讲扩散生成，以 DALL-E、Latent Diffusion、Stable Diffusion、EDM 为代表；再讲语音和视频，以 Whisper、VideoPoet、Sora 类方向为代表；最后讨论如何复现和面试回答。

## 11.1 先建立总图：多模态模型到底在学什么

可以先用一个小白例子。

假设你教一个孩子认识世界。只给他文字，他可以学会“猫是一种动物，有胡须，会喵喵叫”。但如果他从没看过猫的图片，看到真实猫时未必能识别。只给他图片，他能认出猫，但很难回答“猫和老虎有什么亲缘关系”。只给他视频，他能看到猫跳跃、奔跑、躲避障碍，但未必能总结成语言知识。

多模态学习就是把这些信息源对齐：让模型知道一张图、一段声音、一段视频和一句话之间的关系。

从任务上看，多模态研究可以分成两大类：

| 类别 | 代表任务 | 核心问题 |
| --- | --- | --- |
| 多模态理解 | 图像分类、图文检索、VQA、OCR、视频问答、语音识别 | 如何把非文本信号映射到可理解语义 |
| 多模态生成 | 文生图、图生图、文生视频、语音合成、音乐生成 | 如何根据条件生成高质量非文本信号 |

从模型结构上看，也有几条路线：

1. 双塔对齐：图像编码器和文本编码器分别编码，然后在共同空间对齐，CLIP 是代表。
2. Encoder-Decoder：视觉编码器输出特征，语言模型或解码器生成文本，captioning 和 VQA 常见。
3. 视觉 token 接入 LLM：把图像转成若干视觉 token，通过 projector 或 cross-attention 接到语言模型，LLaVA、Flamingo 等属于这个大方向。
4. 扩散模型：从噪声逐步去噪生成图像或视频，Stable Diffusion 是代表。
5. 自回归多模态 token 模型：把图像、视频、音频离散化成 token，再用类似语言模型的方法建模，VideoPoet 等属于这条线。

面试时最重要的是不要把“多模态”说成一个单一技术。更准确的说法是：多模态是一组关于表示、对齐、融合、生成和评估的技术问题。

## 11.2 CLIP：从固定类别监督到自然语言监督

### 11.2.1 CLIP 之前的问题

传统视觉模型通常基于固定标签训练，比如 ImageNet 的 1000 类。这个设定很成功，但有明显限制。

第一，类别集合固定。模型学会了“金毛犬”“校车”“草莓”，但如果你问它“戴红帽子的猫”“破损的交通标志”“像 19 世纪油画风格的房间”，固定标签体系很难覆盖。

第二，标注成本高。高质量图像分类数据需要人工标注，扩展到所有视觉概念很困难。

第三，下游迁移依赖微调。每来一个新任务，都要重新收集标签并训练分类头。

CLIP 的出发点是：互联网上有大量图像和自然语言描述，能否直接用自然语言作为监督？

### 11.2.2 CLIP 的核心机制

CLIP 使用图像编码器和文本编码器。训练数据是一批图文对。模型要把匹配的图像和文本拉近，把不匹配的图像和文本推远。

简单说，一个 batch 中有 N 张图和 N 段文字，正确配对在对角线上。模型计算所有图像和所有文本的相似度，然后做对比学习。训练完成后，图像和文本被映射到同一个语义空间。

这带来一个重要能力：zero-shot classification。

比如要做猫狗分类，不需要重新训练分类器。可以写两个文本提示：

```text
a photo of a cat
a photo of a dog
```

把图片 embedding 和这两个文本 embedding 比较，相似度更高的就是预测类别。

### 11.2.3 CLIP 的历史意义

CLIP 的意义不只是“图文检索效果好”。它改变了视觉模型的监督方式。

过去视觉模型学的是固定标签；CLIP 学的是语言描述和视觉内容之间的开放词汇对齐。它让自然语言成为视觉概念的接口。

这也是后续文生图和视觉语言模型的重要基础。很多扩散模型使用 CLIP 或类似文本编码器作为条件；很多多模态 LLM 使用 CLIP 的视觉编码器作为视觉表征来源。

### 11.2.4 CLIP 的边界

CLIP 也不是万能视觉理解模型。

它擅长全局语义对齐，但对细粒度空间关系、计数、OCR、复杂推理不一定可靠。比如“左边的人拿着红色杯子，右边的人拿着蓝色杯子”，CLIP 的整体相似度可能不足以精确判断。

它还依赖大规模互联网图文数据，数据噪声、偏见和版权问题都会影响模型行为。

面试中可以这样概括：

CLIP 的本质贡献是用大规模自然语言监督训练可迁移视觉表示，并通过图文对比学习实现开放词汇的 zero-shot 视觉识别。它强在语义对齐和迁移，弱在细粒度结构、局部定位和复杂视觉推理。

## 11.3 Flamingo、BLIP、LLaVA：把视觉接入语言模型

### 11.3.1 从图文对齐到视觉对话

CLIP 可以判断图文是否匹配，也可以做图文检索和 zero-shot 分类。但用户往往想问更复杂的问题：

1. 这张图里发生了什么？
2. 图中表格第三行是什么意思？
3. 这张医学图像可能有什么异常？
4. 根据这张设计图写一段说明。
5. 比较两张图的差异。

这些任务需要语言生成和多轮对话能力。于是研究重点从“图文 embedding 对齐”转向“视觉语言模型”。

视觉语言模型通常要解决两个问题：

1. 视觉信息如何变成语言模型能消费的 token 或特征。
2. 语言模型如何基于视觉特征进行问答、推理和生成。

### 11.3.2 Flamingo：连接预训练视觉模型和语言模型

Flamingo 的目标是构建能够 few-shot 适应多种图像和视频任务的视觉语言模型。它的关键思路不是从零训练一个巨大的多模态模型，而是桥接已有的强视觉模型和强语言模型。

Flamingo 引入了处理交错图文输入的结构，可以让模型在 prompt 中看到图片、文字示例，再回答新的视觉问题。这很像语言模型的 in-context learning，只是上下文中包含图像。

它的重要性在于：

1. 支持图像和文本交错输入。
2. 支持 few-shot multimodal learning。
3. 展示了强语言模型可以通过适当接口扩展到视觉任务。

对于面试，Flamingo 常被用来讨论“如何把冻结的视觉编码器和冻结的语言模型连接起来”。关键点是视觉特征压缩、cross-attention 或 adapter 机制，以及多模态数据格式。

### 11.3.3 BLIP 系列：围绕图文理解和生成的数据闭环

BLIP 系列关注图文预训练、caption 生成、过滤和视觉语言理解。它的重要启发是：多模态模型不只是结构问题，数据质量同样关键。

互联网上的图文对噪声很大。图片旁边的文字不一定描述图片，可能是广告、导航、评论或无关上下文。BLIP 类工作强调用模型生成或筛选 caption，构造更好的图文训练数据。

这个思想后来在多模态 instruction tuning 中也很常见：先用强模型生成或清洗视觉指令数据，再训练视觉助手模型。

### 11.3.4 LLaVA：Visual Instruction Tuning

LLaVA 的核心意义在于把 instruction tuning 的思想引入视觉语言模型。

它使用视觉编码器提取图像特征，通过一个连接模块投影到语言模型可以理解的空间，再用多模态指令数据训练模型回答视觉问题。

LLaVA 的贡献不只是架构简单，更重要的是证明了一个路线：用已有强语言模型作为大脑，用视觉 encoder 作为眼睛，用 projector 做接口，再通过高质量视觉指令数据让模型学会“看图对话”。

一个典型 LLaVA 类模型可以拆成：

1. Vision encoder：通常来自 CLIP 或类似模型。
2. Projector：把视觉特征映射到 LLM embedding 空间。
3. LLM：负责语言理解、推理和生成。
4. Instruction data：图像问答、描述、推理、多轮对话。

训练通常分阶段：先做图文对齐或 projector 预训练，再做视觉 instruction tuning。

### 11.3.5 视觉语言模型的失败模式

VLM 常见失败包括：

1. 视觉幻觉：图里没有的物体被说成存在。
2. 细节误读：颜色、数量、位置、文字识别出错。
3. 语言先验过强：模型根据常识猜，而不是看图。
4. OCR 不稳定：图片中文字、公式、表格识别错误。
5. 多图比较困难：跨图对应关系和差异定位不稳定。
6. 坐标和定位弱：很难精确指出区域或框的位置。

因此评估 VLM 不能只看回答流畅度。要看 grounding、OCR、counting、spatial reasoning、chart understanding、multi-image reasoning 等细项。

## 11.4 Diffusion：从噪声到图像的生成范式

### 11.4.1 生成模型路线的背景

图像生成早期有几条重要路线：VAE、GAN、自回归模型和扩散模型。

GAN 曾经在图像清晰度上非常强，但训练不稳定，模式覆盖和文本控制也有难点。自回归图像模型可以把图像离散化后像语言一样生成，但高分辨率图像 token 很多，成本高。扩散模型的兴起，是因为它在生成质量、训练稳定性和条件控制之间取得了很好的平衡。

扩散模型可以用一个直观比喻理解：

训练时，把真实图像一步步加噪，直到变成接近纯噪声；模型学习如何从带噪图像预测去噪方向。生成时，从噪声开始，反复去噪，逐步得到清晰图像。

### 11.4.2 DALL-E 与文本到图像生成

DALL-E 系列代表了文本生成图像的重要路线。早期 DALL-E 使用离散图像 token 和自回归建模，展示了语言条件可以控制图像生成。后续 DALL-E 2 等系统结合了图文表示和生成模型，使文本到图像质量大幅提高。

这条线的重要意义是：文本不再只是图像的标签，而是生成过程的控制接口。用户可以用自然语言描述内容、风格、场景和关系，模型生成对应图像。

但文生图难点也随之出现：

1. 文本一致性：图像是否真的满足 prompt。
2. 组合泛化：多个对象和关系能否正确组合。
3. 细节控制：文字、手指、空间关系、数量是否准确。
4. 安全和版权：是否生成不当内容或模仿特定受保护风格。

### 11.4.3 Latent Diffusion 和 Stable Diffusion

Latent Diffusion Models 的关键贡献是：不要直接在像素空间做扩散，而是在预训练 autoencoder 的 latent space 中做扩散。

像素空间分辨率高，计算很贵。Latent space 更紧凑，保留主要语义和视觉细节，同时大幅降低计算成本。Stable Diffusion 正是基于 latent diffusion 路线而被广泛使用。

一个简化流程是：

1. 用 VAE encoder 把图像压缩到 latent。
2. 在 latent 上加噪并训练 U-Net 去噪。
3. 用文本编码器提供条件。
4. 通过 cross-attention 让图像生成过程关注文本条件。
5. 生成完成后，用 VAE decoder 把 latent 解码成图像。

Latent Diffusion 的意义在于把高质量文生图从极高算力门槛拉低到更可复现、更可部署的范围。

### 11.4.4 Classifier-free guidance 和条件控制

扩散模型生成时常用 guidance 来增强条件一致性。直观理解是：模型一方面知道“无条件生成”会往哪里走，另一方面知道“按照文本条件生成”会往哪里走。通过调节 guidance scale，可以让生成结果更贴合 prompt。

guidance scale 太低，图像可能不听话；太高，图像可能过饱和、失真、细节变差。这个 trade-off 是文生图调参中非常常见的问题。

除了文本条件，扩散模型还可以接入边缘图、深度图、姿态图、分割图、草图、参考图等控制信号。ControlNet、image-to-image、inpainting 等方向都来自这个思想：生成模型不仅要“自由创作”，还要“可控生成”。

### 11.4.5 EDM：把扩散设计空间讲清楚

EDM 的价值在于系统梳理扩散模型的设计空间，把训练、采样、噪声调度、网络预条件等因素拆开分析。很多扩散论文效果提升来自多个细节叠加，如果不拆分，很难知道到底是哪一部分起作用。

EDM 给研究者的启发是：扩散模型不是一个单点算法，而是一组设计选择。复现时要认真记录 noise schedule、sampler、prediction target、preconditioning、loss weighting、number of function evaluations 等细节。

### 11.4.6 扩散模型的评估问题

图像生成评估很难，因为“好图”不等于单一数值。

常见指标包括：

1. FID：衡量生成图像分布和真实图像分布差异，但不一定反映文本一致性。
2. CLIP score：衡量图文相似度，但可能被 prompt hack 或 CLIP 偏差影响。
3. 人类偏好：更贴近真实感受，但成本高且主观。
4. 美学评分：可反映审美质量，但容易偏向特定风格。
5. 专项评估：文字生成、计数、空间关系、组合属性等。

面试中如果被问“如何评估文生图模型”，不要只说 FID。应该分成视觉质量、文本一致性、多样性、可控性、安全性和成本。

## 11.5 Whisper：语音大模型与弱监督扩展

Whisper 是语音识别方向的代表工作之一。它的核心思想和 CLIP 有相似之处：利用大规模弱监督数据训练一个鲁棒模型，而不是只依赖小规模精标数据。

Whisper 使用大量互联网音频和转录文本进行训练，覆盖多语言、多任务和多场景。它展示了一个重要趋势：语音系统也可以通过规模化弱监督获得很强的 zero-shot 迁移能力。

语音任务和图像任务不同。语音包含时间结构、口音、噪声、语速、音乐背景、多人说话等复杂因素。Whisper 的价值在于鲁棒性：在多语言、多口音、多噪声条件下仍能工作。

但 Whisper 类模型也有边界：

1. 专有名词、低资源语言和强噪声场景仍可能出错。
2. 时间戳对齐可能不稳定。
3. 对长音频的分段策略会影响结果。
4. 语音翻译和语音识别混合时需要明确任务提示。
5. 对口音和方言的公平性仍需评估。

面试中可以把 Whisper 和 CLIP 放在一起理解：两者都说明，大规模弱监督数据加上合适预训练目标，可以得到跨任务、跨域迁移能力强的基础模型。

## 11.6 Video：从图像生成到世界动态建模

### 11.6.1 视频为什么更难

视频不是多张图片简单拼接。它多了时间维度，因此难点显著增加。

视频生成至少要满足：

1. 单帧质量：每一帧都要清晰合理。
2. 时间一致性：同一对象在不同帧中外观稳定。
3. 运动合理性：动作连续，速度和轨迹符合直觉。
4. 物理一致性：遮挡、碰撞、重力、光照变化合理。
5. 镜头语言：视角、景别、镜头切换符合叙事。
6. 文本条件一致性：整个视频都要符合 prompt，而不是只有第一帧符合。

图像生成主要解决空间结构，视频生成同时解决空间和时间结构。

### 11.6.2 视频模型的主要路线

视频生成大致有几种路线：

1. 2D diffusion 扩展到时间维度：在 U-Net 或 Transformer 中加入时序模块。
2. 先生成关键帧，再插帧或补全运动。
3. 图像模型加 temporal adapter，把预训练图像生成能力迁移到视频。
4. 自回归多模态 token 建模，把视频离散成 token 后用 Transformer 生成。
5. Diffusion Transformer，把扩散和 Transformer 结合，统一处理长视频 token。

不同路线的 trade-off 不同。扩散路线通常视觉质量强，自回归 token 路线更接近语言模型统一建模范式，但压缩器和 token 设计很关键。

### 11.6.3 VideoPoet：用语言模型范式做视频生成

VideoPoet 展示了一条很有代表性的路线：使用 decoder-only Transformer 处理多模态 token，通过预训练和任务适配支持文本、图像、视频、音频等条件，完成视频生成相关任务。

它的重要性在于：视频生成不一定只能走传统扩散 U-Net 路线，也可以把多模态信号 token 化后，用类似 LLM 的方式统一建模。

这种路线的优势是统一：文本到视频、图像到视频、视频补全、音频相关生成可以放进同一个序列建模框架。难点是 tokenization、长序列成本、生成稳定性和高保真解码。

### 11.6.4 Sora 类方向：视频生成向世界模拟靠近

Sora 类方向代表了更大规模视频生成模型的趋势：模型不仅生成“看起来像视频”的短片，还试图学习物体、场景、镜头和运动之间的规律。

公开信息显示，这类模型通常会强调：

1. 使用大规模视频和图像数据训练。
2. 支持文本到视频、图像到视频、视频延展等任务。
3. 采用适合高维视觉时空数据的表示方式。
4. 在长时长、一致性和复杂场景上取得明显进展。

但需要谨慎：很多 Sora 类系统没有完整公开训练数据、模型规模、损失函数和工程细节。因此在论文复现或面试中，不能把未公开细节当成确定事实。更合适的表述是：Sora 类工作表明大规模视频生成正在向“可扩展时空生成模型”演进，但具体实现需要以公开技术报告和可复现论文为准。

### 11.6.5 视频评估为什么难

视频评估比图像更难。单帧好看不代表视频好。常见评价维度包括：

1. 视觉质量：清晰度、细节、伪影。
2. 文本一致性：是否符合 prompt 的主体、动作、场景和风格。
3. 时间一致性：对象身份是否稳定，背景是否闪烁。
4. 运动质量：动作是否自然，轨迹是否合理。
5. 物理合理性：交互、遮挡、碰撞是否符合直觉。
6. 多样性：同一 prompt 能否生成不同但合理的视频。
7. 安全性：是否生成不当、侵权或误导性内容。

因此视频论文里单一指标很难服众，通常需要自动指标、人类偏好和案例分析结合。

## 11.7 多模态论文的共性：表示、对齐、融合、生成

读多模态论文时，可以用四个关键词做框架。

### 11.7.1 表示

每种模态如何变成模型能处理的表示？图像可以用 patch embedding、CNN feature、ViT token、VAE latent；语音可以用 log-mel spectrogram 或 codec token；视频可以用时空 patch 或离散 token。

表示决定了后续模型能看到什么，也决定了计算成本。

### 11.7.2 对齐

不同模态之间如何建立关系？CLIP 用对比学习对齐图文；VLM 用图像特征和语言 token 对齐；文生图用文本条件控制生成过程；语音识别把音频对齐到文本。

对齐不充分，就会出现图文不一致、听错、看错、答非所问。

### 11.7.3 融合

模态信息在哪里融合？早融合是在输入层直接拼接；中间融合通过 cross-attention 或 adapter；晚融合则分别编码后再匹配或决策。

CLIP 是典型双塔晚融合；Flamingo 和 LLaVA 更偏中间融合；扩散模型中的 cross-attention 是生成过程中的条件融合。

### 11.7.4 生成

生成任务要考虑输出空间。文本是离散 token，图像和音频是连续信号，视频是高维时空信号。不同输出空间对应不同建模选择：自回归、扩散、flow、codec language model 等。

## 11.8 复现建议：如何做一个靠谱的多模态小实验

多模态大模型复现成本高，不建议一开始就复现完整 CLIP、Stable Diffusion 或视频模型。更现实的路线是做缩小版、模块化复现。

### 11.8.1 复现 CLIP 小实验

可以使用小规模图文数据集，训练一个图像 encoder 和文本 encoder，用对比学习做图文检索。

重点观察：

1. batch size 对对比学习效果的影响。
2. temperature 对相似度分布的影响。
3. 文本 prompt 变化对 zero-shot 分类的影响。
4. 图文数据噪声对检索结果的影响。

### 11.8.2 复现 LLaVA 类小实验

可以冻结 vision encoder 和小型 LLM，只训练 projector，再用少量视觉问答数据做 instruction tuning。

重点观察：

1. projector 结构是否影响收敛。
2. 只做 caption 数据和做 instruction 数据的差异。
3. 模型是否真的看图，还是依赖语言先验。
4. OCR、计数、空间关系分别表现如何。

### 11.8.3 复现 Stable Diffusion 相关实验

不建议从零训练大模型。可以基于开源权重做小型实验，比如比较 sampler、steps、guidance scale、prompt 改写、LoRA 微调、ControlNet 条件控制。

重点观察：

1. guidance scale 对文本一致性和图像质量的影响。
2. sampler 和 steps 对速度与质量的影响。
3. 训练数据规模和风格一致性对 LoRA 的影响。
4. 过拟合、风格泄漏和身份相似风险。

### 11.8.4 复现语音实验

可以用公开语音数据评估 Whisper 类模型在噪声、口音、语言切换和长音频分段下的表现。

重点观察 WER、分段策略、时间戳稳定性、低资源语言和专有名词错误。

### 11.8.5 视频实验从评估开始

视频生成训练成本更高。更建议从评估开始：收集不同模型生成结果，建立一组 prompt，比较时间一致性、文本一致性、运动质量和失败类型。

如果做训练实验，可以从短视频、低分辨率、有限类别开始，重点理解时序模块和数据处理，而不是追求视觉效果。

## 11.9 面试高频问题

### 问题 1：CLIP 和传统图像分类模型最大的区别是什么？

传统图像分类模型学习固定标签集合，CLIP 学习图像和自然语言之间的开放词汇对齐。CLIP 可以用文本 prompt 做 zero-shot 分类和检索，因此迁移性更强，但在细粒度空间关系、计数和复杂推理上仍有边界。

### 问题 2：CLIP 为什么能做 zero-shot 分类？

因为图像和文本被训练到同一个 embedding 空间。分类时把类别写成自然语言 prompt，比较图像 embedding 和各个文本 embedding 的相似度，就可以在没有该数据集训练的情况下预测类别。

### 问题 3：Flamingo 和 LLaVA 的共同点是什么？

共同点是都把视觉信息接入语言模型，让模型基于图像进行语言生成。区别是 Flamingo 强调交错图文输入和 few-shot 多模态学习，LLaVA 强调视觉 instruction tuning 和开源可复现路线。

### 问题 4：视觉语言模型为什么会幻觉？

可能因为视觉特征不足、训练数据中语言先验太强、图像分辨率或 OCR 能力不够、指令数据偏向流畅回答，以及模型缺乏显式 grounding。它可能根据常见模式猜答案，而不是严格依据图像。

### 问题 5：Latent Diffusion 为什么重要？

它把扩散过程从高维像素空间转移到更紧凑的 latent space，在保留视觉质量的同时显著降低训练和推理成本。Stable Diffusion 的广泛传播很大程度上依赖这一路线。

### 问题 6：Stable Diffusion 中文本如何控制图像？

文本经过文本编码器得到条件表示，扩散 U-Net 在去噪过程中通过 cross-attention 等机制使用文本条件。生成时 guidance 可以增强 prompt 一致性，但过强 guidance 可能导致失真。

### 问题 7：FID 能否完整评价文生图模型？

不能。FID 主要衡量生成分布和真实分布的差异，不直接衡量文本一致性、细粒度属性、可控性、安全性和人类偏好。文生图评估需要多维指标和人工评估结合。

### 问题 8：Whisper 的核心启发是什么？

大规模弱监督音频转录数据可以训练出鲁棒语音识别模型，使其在多语言、多任务和 zero-shot 场景下表现较好。它说明语音领域也可以从规模化弱监督中获得强迁移能力。

### 问题 9：视频生成比图像生成难在哪里？

视频多了时间维度，需要保持对象身份、运动轨迹、场景结构和物理交互的一致性。单帧质量高不代表视频质量高，时间闪烁、动作不连续和物理不合理都是常见问题。

### 问题 10：Sora 类模型应该如何谨慎表述？

可以说它代表了大规模视频生成向可扩展时空建模和世界动态模拟方向发展，但很多训练数据、模型结构和工程细节未完全公开。讨论时应区分公开事实、合理推测和未验证传闻。

## 11.10 本章小结

多模态、扩散和视频生成共同推动大模型从文本智能走向现实世界信号建模。

CLIP 的关键贡献是自然语言监督和图文对比学习，让视觉模型具备开放词汇迁移能力。

Flamingo、BLIP、LLaVA 等视觉语言模型把视觉特征接入语言模型，使模型可以进行看图问答、描述和多模态对话。

扩散模型通过逐步去噪成为图像生成主流路线，Latent Diffusion 进一步降低高分辨率生成成本，Stable Diffusion 让文生图进入广泛应用。

Whisper 展示了大规模弱监督在语音识别中的威力。

VideoPoet 和 Sora 类方向则说明视频生成正在从短片合成走向更复杂的时空建模。

读这类论文时，要始终抓住四个问题：输入如何表示，不同模态如何对齐，信息在哪里融合，输出如何生成和评估。只要这四个问题清楚，多模态论文线就不会散。
