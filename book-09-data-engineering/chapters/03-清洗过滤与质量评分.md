# 第三章：清洗过滤与质量评分

如果说上一章的 Web-scale 数据采集解决的是“从哪里拿到足够多的数据”，那么本章要解决的是“哪些数据值得进入训练集”。这一步看似像脏活累活，实际上决定了大模型预训练的上限、稳定性、安全性和产品体验。

很多初学者会以为：只要模型足够大、数据足够多，模型自然能从噪声中学到规律。这个想法只对了一半。大模型确实有很强的统计归纳能力，但它不会自动知道哪些网页是正文、哪些是广告、哪些是模板、哪些是乱码、哪些是低质量 SEO 内容、哪些是隐私信息、哪些是测试集污染、哪些是重复复制的水文。模型会把训练数据中的一切都当作“世界的一部分”去拟合。

所以，清洗过滤不是训练前的边角流程，而是大模型工程的核心环节之一。

本章重点讨论：乱码、模板页、广告、低质量文本、毒性、PII、质量分类器和规则过滤。

合规边界：本章讨论数据质量治理、安全过滤、隐私保护和工程评估，不提供规避网站限制、绕过访问控制、恢复或利用个人敏感信息的方法。

---

## 1. 先建立直觉：清洗过滤到底在过滤什么？

把训练语料想象成给模型读的“图书馆”。采集阶段像把全世界的书、网页、笔记、论坛帖子、代码仓库搬进仓库。清洗过滤阶段则像图书管理员、编辑、法律顾问和安全审核员一起做筛选。

他们要问的问题不是一个，而是一组：

1. 这段文本是不是正常可读？
2. 这段文本是不是正文，还是网页导航、版权声明、广告、按钮、评论区垃圾文本？
3. 这段文本有没有重复？
4. 这段文本有没有极低信息密度？
5. 这段文本有没有明显机器生成垃圾、采集错误或格式损坏？
6. 这段文本属于什么语言、领域、风格？
7. 这段文本是否包含个人敏感信息？
8. 这段文本是否包含过高风险的有害内容？
9. 这段文本是否可能污染评测集或 benchmark？
10. 这段文本是否适合当前模型的目标能力？

注意，清洗过滤不是简单地“删脏数据”。更准确地说，它是在做训练分布塑形。

如果你过滤掉大量口语和论坛数据，模型可能更像一本书，但聊天感下降。如果你保留太多网页评论，模型可能更接地气，但事实性、格式稳定性和安全性可能下降。如果你过度过滤少数语言，模型会在这些语言上能力薄弱。如果你只保留“教科书质量”数据，小模型可能受益很大，但大模型的世界知识覆盖可能不足。

这就是数据清洗的难点：它不是一个单目标优化问题，而是质量、覆盖率、多样性、安全性、版权合规、成本和下游目标之间的权衡。

## 2. 来龙去脉：从 NLP 数据预处理到大模型数据治理

早期 NLP 任务的数据处理比较直接。比如情感分类、机器翻译、命名实体识别，数据集通常由研究者或标注团队提前整理好。常见预处理包括分词、去特殊字符、统一大小写、去停用词、长度截断。这些操作服务于一个固定任务，规模通常从几千到几百万样本。

预训练语言模型出现后，数据处理的目标发生了变化。BERT、GPT 这类模型不再只面向一个任务，而是要从大规模无标注文本中学习通用语言能力。于是，过滤不再只是为了让某个分类器输入更干净，而是为了构造一个能支撑通用能力的语料宇宙。

到了 GPT-3、Gopher、Chinchilla 之后，数据规模进一步变大。数据质量开始和 scaling law 直接相关：同样的算力和参数量，如果数据噪声更低、重复更少、领域配比更合理，模型可以获得更好的 loss 和下游能力。反过来，低质量数据会浪费 token budget，让模型在重复模板、垃圾页面和无意义文本上消耗训练算力。

再后来，phi-1 等工作强调“textbook quality”数据的重要性。它提醒大家：对于某些目标，尤其是代码、数学、推理和教学型能力，高质量、小规模、结构清晰的数据可能比巨量杂乱 web 数据更有效。这不意味着大规模数据不重要，而是说明数据质量会改变效率曲线。

从历史上看，数据清洗经历了三个阶段：

1. 规则预处理阶段：主要靠长度、字符、语言、重复、黑名单等启发式规则。
2. 统计过滤阶段：加入语言模型困惑度、n-gram 分布、词频、压缩率等统计特征。
3. 模型化治理阶段：使用质量分类器、毒性分类器、PII 检测器、领域分类器、embedding 聚类和人工审计闭环。

今天的大模型数据工程通常不是单一过滤器，而是一套多级流水线。

## 3. 为什么“更多数据”不等于“更好数据”？

大模型训练有一个重要事实：每个 token 都要消耗算力。一个 1T token 的训练集，不只是硬盘上更大而已，它意味着训练过程中模型会在这些 token 上反复做前向、反向和参数更新。

如果其中大量 token 是重复页脚、模板导航、乱码、SEO 堆词、低质量问答、采集错误，那么模型就会把昂贵算力花在学习这些噪声上。

更糟的是，低质量数据不是“无影响”。它可能带来具体坏处：

1. 降低训练效率：loss 下降慢，有效信息密度低。
2. 诱导坏格式：模型学会输出网页残留、版权页脚、无意义列表。
3. 增加幻觉：低可信网页和拼接文本会污染事实知识。
4. 增加重复记忆：大量重复文本让模型更容易记忆原文。
5. 放大偏见和毒性：低治理语料可能包含攻击性、歧视性或高风险内容。
6. 影响下游风格：模型输出可能变得啰嗦、营销化、标题党或论坛化。
7. 污染评测：如果 benchmark 或答案泄漏进训练集，评估结果会虚高。

因此，一个成熟团队不会只汇报“我们有多少 TB 数据”，而会汇报：有效 token 数、来源构成、重复率、语言分布、质量分布、安全过滤策略、PII 处理策略、版本和审计信息。

## 4. 清洗过滤流水线总览

一个典型的大模型数据清洗流水线可以拆成下面几层：

1. 解析层：从 HTML、PDF、代码仓库、文档格式中抽取文本和结构。
2. 基础有效性过滤：去空文本、乱码、异常字符、极短或极长样本。
3. 正文抽取和模板去除：去导航栏、菜单、页脚、版权声明、广告块。
4. 语言识别和领域识别：判断语言、领域、文体和来源类型。
5. 规则质量过滤：基于长度、字符比例、重复行、标点比例、词汇分布等规则。
6. 模型质量评分：用分类器或小模型判断文本是否高质量。
7. 安全和隐私过滤：处理毒性、敏感内容、PII、政策风险内容。
8. 去重和污染检测：文档级、段落级、近重复、benchmark contamination。
9. 抽样审计：人工检查过滤前后样本，估计误删和漏删。
10. 版本化产出：保存过滤规则、阈值、模型版本、数据 lineage。

这条流水线不是一次性脚本，而是反复迭代的系统。每次训练结果、人工审计、下游评测和安全评估都会反过来修改过滤策略。

## 5. 基础有效性过滤：先把明显坏数据挡在外面

基础过滤的目标很简单：排除“明显不能训练”的样本。

常见规则包括：

1. 文本长度过短：比如只有几个词、一个标题、一个按钮文本。
2. 文本长度异常过长：可能是拼接错误、日志倾倒、重复模板堆叠。
3. 非文本字符比例过高：大量乱码、控制字符、二进制残留。
4. 数字或符号比例异常：例如满屏电话号码、价格、ID、乱码编码。
5. 重复字符过多：例如“哈哈哈哈哈”或同一符号重复数千次。
6. 行重复率过高：网页菜单、商品列表、采集错误经常出现这种模式。
7. Unicode 异常：混入不可见字符、方向控制符、错误编码。
8. 语言不匹配：目标语料是中文，样本却是其他语言或混合乱码。

这些规则看起来朴素，但非常有效。因为大规模 web 数据里，最先要处理的不是微妙质量问题，而是大量显而易见的脏数据。

一个常见误区是：规则越多越好。实际上，规则过滤要非常谨慎。比如“数字比例高”可能过滤掉财报、科学论文、数学题、代码日志；“符号比例高”可能过滤掉代码、公式、配置文件；“重复行高”可能过滤掉诗歌、歌词或结构化文档。

所以规则不能脱离数据源和目标任务。对通用自然语言、代码、数学、法律、医学、金融，规则阈值应该不同。

## 6. 乱码和编码问题

乱码是 web-scale 数据中最常见的问题之一。它来自编码识别错误、网页解析错误、PDF 抽取错误、压缩解压错误、复制粘贴损坏和混合语言环境。

乱码对模型的伤害主要有三类：

1. 浪费 token：Tokenizer 会把异常字符拆成大量罕见 token。
2. 干扰分布：模型学习到无意义字符组合。
3. 影响稳定性：异常长序列或奇怪字符可能放大训练和推理中的边缘问题。

常见检测方法包括：

1. 统计不可打印字符比例。
2. 统计 Unicode replacement character 的比例。
3. 检测异常 byte pattern。
4. 用语言识别模型判断文本是否像自然语言。
5. 统计 tokenizer 后的 token/character 比例。

其中 token/character 比例很实用。如果一段中文正常文本 1000 个字符会变成相对合理的 token 数，但一段乱码可能被切成大量碎片 token。异常高的比例说明这段文本对训练很不友好。

但也要注意：代码、数学公式、多语言文本和表格可能天然 token 密度较高，不能简单一刀切。

## 7. 模板页、导航栏和 boilerplate 去除

网页不是为大模型训练设计的。一个网页里真正有价值的正文，可能只占 HTML 的一小部分。其他内容包括导航栏、推荐链接、版权信息、cookie 提示、页脚、侧边栏、评论框、广告位、分享按钮等。

如果不去除 boilerplate，模型会大量学习这些模板：

1. “点击这里查看更多”
2. “版权所有，未经许可不得转载”
3. “上一篇 / 下一篇”
4. “分享到微信微博”
5. “热门推荐”
6. “接受 Cookie”

这些文本本身不一定有害，但信息密度很低，而且重复度极高。大规模保留会稀释正文质量。

常见方法包括：

1. DOM 结构分析：识别正文节点、导航节点、页脚节点。
2. 文本密度方法：正文区域通常文本密度高、链接密度低。
3. 链接比例过滤：链接文本占比过高的段落更像菜单或推荐栏。
4. 重复模板检测：同一站点大量页面中重复出现的段落可能是模板。
5. 规则和模型结合：先用规则粗筛，再用模型判断正文质量。

面试中经常会问：“为什么不能直接从 HTML 提取所有文本？”标准回答是：因为 HTML 中包含大量非正文信息，直接提取会引入模板噪声、广告噪声和重复文本，降低有效 token 密度，并改变模型输出风格。

## 8. 广告、SEO 垃圾和低信息密度文本

广告和 SEO 垃圾是 web 数据里非常难处理的一类问题。它们往往是正常自然语言，不一定乱码，也不一定短，但训练价值很低。

低质量 SEO 文本常见特征包括：

1. 关键词堆砌。
2. 标题党。
3. 大量同义改写。
4. 内容空泛，信息密度低。
5. 多段结构高度相似。
6. 夹杂商品、服务、联系方式或推广话术。
7. 句子语义连贯性弱，但表面语法正常。

这类文本很难用单条规则识别。因为它们不是“坏字符”，而是“低价值语言”。

常见处理方式有三类：

1. 规则特征：关键词密度、链接密度、重复 n-gram、广告词表、联系方式模式。
2. 统计特征：困惑度、压缩率、句子相似度、段落重复率。
3. 质量分类器：训练一个模型判断文本是否像高质量文章、教材、问答或文档。

但广告词表不能无限扩大。因为很多领域文本天然包含价格、地址、电话、产品名、品牌名，比如电商评论、商业案例、法律合同和新闻报道。过度过滤会损失真实世界知识。

更稳妥的策略是多信号融合：单一广告词不直接删除，而是与链接比例、重复程度、文本质量分、来源信誉、人工审计结果一起判断。

## 9. 质量评分：从二元过滤到连续打分

早期过滤常常是二元决策：保留或删除。但在大模型训练中，更常见的是给每个样本一个质量分。

质量分可以用于多个目的：

1. 删除最低质量样本。
2. 按质量分采样，高质量样本更高概率进入训练。
3. 对不同来源设置不同阈值。
4. 做 data mixture 配比时作为权重。
5. 支持后续审计和版本回溯。

一个简单质量评分系统可能包含：

1. 可读性分：语言是否自然、句子是否完整。
2. 信息密度分：是否包含实质内容。
3. 结构分：段落、标题、列表是否清晰。
4. 来源分：来源是否可靠、领域是否重要。
5. 重复惩罚：是否与已有样本高度重复。
6. 安全惩罚：是否包含高风险或不适合训练的内容。
7. 隐私惩罚：是否疑似包含个人敏感信息。

质量评分不是绝对真理。它反映的是团队对“什么是好训练数据”的定义。

例如，百科、教材、论文和技术文档通常在事实性和结构上得分高；论坛、聊天和社交媒体在结构上可能得分低，但对对话、口语和真实用户表达很重要。因此质量分不能替代数据配比策略。

## 10. 规则过滤 vs 模型过滤

数据过滤常见两大路线：规则过滤和模型过滤。

规则过滤的优点：

1. 可解释。
2. 便宜。
3. 容易并行。
4. 适合处理明显错误。
5. 便于审计和复现。

规则过滤的缺点：

1. 覆盖有限。
2. 容易误伤边缘数据。
3. 很难识别语义低质。
4. 阈值依赖经验。

模型过滤的优点：

1. 能捕捉语义质量。
2. 能识别复杂模式。
3. 可以适配不同领域。
4. 可以输出连续分数。

模型过滤的缺点：

1. 成本更高。
2. 需要标注或弱监督数据。
3. 可能继承标注偏见。
4. 可解释性较差。
5. 分布外数据容易误判。

成熟系统通常不是二选一，而是组合使用：先用规则过滤掉显然坏的数据，再用模型评分处理更微妙的质量问题，最后用人工抽样审计校准阈值。

## 11. 质量分类器怎么训练？

质量分类器的目标是判断一段文本是否适合进入训练集，或给出质量分。常见训练方式包括：

1. 人工标注：让标注员按质量标准打分。
2. 弱监督：把高质量来源当正例，把低质量来源当负例。
3. 对比学习：让模型区分教材式文本和垃圾网页。
4. LLM-as-judge：用更强模型辅助生成质量标签，再人工抽检。
5. 下游反馈：根据训练后模型表现反推数据质量策略。

弱监督很常见。例如，可以把高质量教材、百科、论文摘要、技术文档作为正例，把采集错误页、低质 SEO 页、模板页、垃圾评论作为负例。分类器学到的不是“真理”，而是这些来源之间的统计差异。

这会带来一个风险：分类器可能把“文风像教材”误认为“质量高”，从而偏好正式、标准、主流语言，压低口语、方言、少数语言、社区讨论和新兴表达。

因此，质量分类器需要配套审计：

1. 分语言审计。
2. 分领域审计。
3. 分来源审计。
4. 分长度审计。
5. 分人群和风格审计。

否则，一个看似提升平均质量的分类器，可能会系统性删除某些重要数据。

## 12. 困惑度过滤：用语言模型判断“像不像正常文本”

困惑度过滤是一种经典方法。直觉是：如果一个参考语言模型觉得某段文本非常“不像正常语言”，它可能是乱码、拼接错误、机器垃圾或异常文本。

困惑度低，说明文本在参考模型看来更可预测；困惑度高，说明文本更异常。

但困惑度过滤有明显局限：

1. 高质量专业文本可能困惑度高，因为参考模型不熟悉该领域。
2. 低质量套话可能困惑度低，因为语言模式很常见。
3. 少数语言可能被误判为异常。
4. 代码和数学公式不适合用普通自然语言困惑度直接过滤。

所以困惑度更适合作为一个特征，而不是最终裁判。

一个更稳健的用法是：对每个语言、领域、来源分别统计困惑度分布，只过滤极端异常样本，而不是用统一阈值处理所有数据。

## 13. 毒性和有害内容过滤

大模型训练语料中不可避免会出现攻击性语言、仇恨表达、骚扰、暴力、成人内容、自伤相关内容、危险行为描述等。是否过滤、过滤多少、如何保留上下文，是非常复杂的问题。

如果完全不过滤，模型可能更容易复现有害表达。如果过度过滤，模型可能不了解这些内容，导致安全分类、风险识别和拒答能力变弱。很多安全训练需要模型见过风险语义，但以受控方式学习如何识别、拒绝和安全回应。

因此，安全过滤通常不是简单删除所有敏感内容，而是分层处理：

1. 明显非法或高风险内容：严格移除或隔离。
2. 攻击性和仇恨内容：根据用途过滤、降权或放入安全专用数据池。
3. 新闻、研究、教育语境中的风险讨论：保留时需要上下文判断。
4. 安全问答和拒答示例：用于后续 SFT/RLHF/安全训练，而不是混入普通预训练池。

这里的关键是上下文。比如一篇讨论网络安全治理的政策文章和一段鼓励攻击行为的文本，表面关键词可能相似，但训练用途完全不同。

所以安全过滤需要模型分类、规则、人工审核和数据分层共同完成。

---

## 14. PII 和隐私过滤

PII 指 personally identifiable information，即可以识别个人身份的信息。常见类型包括姓名、地址、电话号码、邮箱、身份证件号、账号、地理位置、医疗记录、财务信息等。

大模型训练中处理 PII 的原因有两个：

1. 合规和伦理：训练数据不应不当包含个人敏感信息。
2. 记忆风险：模型可能记住罕见、重复或结构化的敏感文本。

PII 过滤常见方法包括：

1. 正则和规则：识别邮箱、电话、证件号、银行卡号等结构化模式。
2. NER 模型：识别人名、地名、机构名等实体。
3. 上下文分类器：判断实体是否构成隐私风险。
4. 哈希或脱敏：对某些字段做不可逆处理或替换。
5. 来源级策略：对高风险来源直接排除或隔离。

PII 检测的难点在于误报和漏报。

误报例子：小说里的人名、公开名人信息、公司地址、论文作者邮箱、代码示例中的 fake email。

漏报例子：非标准格式的联系方式、图片 OCR 后的个人信息、论坛中的口语化地址、多个字段组合后可识别个人。

工程上通常不会只依赖一个 PII detector，而是用多层策略，并保留审计记录。对高风险数据源，宁可更保守；对公开、专业、低风险来源，可以结合上下文处理。

## 15. 低质量机器生成数据

随着生成式 AI 普及，web 上的机器生成文本越来越多。训练新模型时，如果大量采集到旧模型生成的低质量内容，可能出现“模型吃模型输出”的问题。

机器生成数据不一定坏。高质量 synthetic data 可以用于数学、代码、指令跟随和安全训练。但未经筛选的大量机器生成垃圾会带来风险：

1. 信息多样性下降。
2. 事实错误被放大。
3. 语言风格同质化。
4. 模型过度学习模板化表达。
5. 长尾真实表达被稀释。

检测机器生成文本很难，尤其当生成模型越来越强时。更实际的策略不是追求完美检测，而是从来源、时间、结构和质量信号综合控制。

例如：

1. 对已知低质内容农场降权。
2. 监控重复模板和语义相似段落。
3. 对疑似批量生成内容抽样审计。
4. 保留高质量合成数据，但明确标注来源和生成方式。
5. 不把 synthetic data 和自然语料混在一起不可追踪地使用。

关键不是“机器生成一定不能用”，而是“不能无标识、无评估、无配比地用”。

---

## 16. 不同数据类型的过滤策略不同

同一套规则不能处理所有数据。

### 16.1 网页正文

重点是正文抽取、模板去除、广告过滤、语言识别、质量评分和去重。

### 16.2 代码数据

代码数据不能用普通文本规则直接过滤。符号多、缩进多、重复结构多都是正常现象。代码过滤更关注：

1. 文件是否可解析。
2. 许可证是否可用。
3. 是否是自动生成文件。
4. 是否包含 secrets 或凭据。
5. 是否重复 fork 或 vendor 代码。
6. 是否包含测试答案或 benchmark 泄漏。

### 16.3 数学数据

数学文本可能包含大量公式、符号、编号和短句。普通困惑度或符号比例过滤容易误伤。更合适的方法是领域分类、结构识别和人工抽样。

### 16.4 论文和技术文档

这类数据质量通常较高，但 PDF 抽取容易产生页眉页脚、断行、参考文献噪声、公式损坏和表格错乱。

### 16.5 对话数据

对话数据要关注角色标注、轮次完整性、隐私、攻击性内容和 consent。不能把真实用户对话简单当普通 web 文本处理。

### 16.6 多语言数据

多语言过滤要防止主流语言规则误伤低资源语言。语言识别模型在短文本、混合文本和低资源语言上容易不准，需要抽样审计。

## 17. 阈值怎么定？

过滤系统里最常见的问题是：阈值怎么定？比如质量分低于多少删除？重复率高于多少删除？毒性分高于多少隔离？

成熟做法不是拍脑袋，而是建立验证闭环。

常见步骤：

1. 从原始数据中抽样。
2. 对不同阈值过滤后的样本做人工审计。
3. 估计误删率和漏删率。
4. 训练小规模模型或做数据消融实验。
5. 观察验证 loss、下游 benchmark、安全评估和生成样例。
6. 按语言、领域和来源分别调整阈值。
7. 固化版本并记录规则。

阈值选择要服务于目标。训练通用 base model、代码模型、数学模型、对话模型、安全模型，阈值可能完全不同。

比如，代码模型不能因为符号多就过滤；安全模型不能完全删除所有风险语义；对话模型需要保留真实口语；教材型小模型可能更偏好高质量结构化文本。

---

## 18. 误删和漏删：过滤系统的两个错误

过滤系统有两个基本错误：

1. False Positive：把好数据删掉。
2. False Negative：把坏数据留下。

很多团队只关注漏删，因为坏数据留下来容易被看到。但误删同样危险，因为它会改变训练分布。

误删的后果包括：

1. 某些语言能力下降。
2. 某些领域知识缺失。
3. 口语表达能力下降。
4. 少数群体或低资源文化表达被削弱。
5. 模型变得过于书面、僵硬或单一。

漏删的后果包括：

1. 训练效率下降。
2. 输出风格变差。
3. 安全风险上升。
4. 隐私记忆风险上升。
5. benchmark 污染。

一个好答案不能只说“加强过滤”，而要说清楚如何平衡误删和漏删，如何用审计和实验评估。

---

## 19. 过滤后如何评估数据质量？

数据过滤是否有效，不能只看删掉了多少。删掉 50% 数据不代表质量提升，可能只是删掉了某些语言或领域。

常见评估指标包括：

1. 保留率：每个来源、语言、领域保留了多少。
2. 重复率：文档级、段落级、近重复率是否下降。
3. 质量分分布：过滤前后质量评分是否改善。
4. PII 命中率：敏感信息风险是否下降。
5. 毒性分布：高风险内容比例是否下降。
6. 语言分布：是否意外损失低资源语言。
7. 领域分布：是否过度偏向百科、新闻或技术文档。
8. 人工审计通过率：抽样人工检查是否更好。
9. 小模型验证：用过滤前后数据训练小模型比较 loss 和下游表现。
10. 生成样例：观察模型是否输出模板、广告、乱码、隐私信息或不良风格。

其中，小模型验证很重要。因为有些过滤规则看起来合理，但对最终模型效果未必好。先用较小训练预算做 ablation，可以避免把错误策略放大到昂贵的大训练中。

---

## 20. 数据清洗和训练目标的关系

清洗过滤必须服从模型目标。

如果目标是通用 base model，需要覆盖广泛世界知识，不能只保留高雅书面文本。如果目标是代码模型，要保留大量高质量代码、文档、issue、问答和测试，但严格处理 license、secrets 和重复。如果目标是数学模型，要保留题目、解答、教材、证明、公式和合成推理数据。如果目标是企业助手，要重点治理隐私、安全、事实性和指令风格。

这说明“数据质量”不是抽象概念，而是相对于目标能力定义的。

一段论坛讨论对百科问答可能质量低，但对训练用户意图理解很有价值。一段严谨论文对聊天产品可能不够自然，但对专业问答很重要。一段安全风险说明对普通预训练可能需要隔离，但对安全分类器训练很关键。

所以面试中不要说“低质量数据全部删除”。更专业的说法是：分层、打标、降权、隔离和按用途进入不同数据池。

---

## 21. 面向专家：清洗过滤改变模型的归纳偏置

从专家视角看，数据清洗不是中性的。它会改变模型的归纳偏置。

模型最终学到的是训练分布的压缩表示。你过滤掉什么，模型就少学什么；你保留什么，模型就多拟合什么；你给什么更高采样权重，模型就把什么视作更重要的模式。

例如：

1. 过度偏好教科书数据，模型可能更会解释概念，但对真实用户混乱输入鲁棒性下降。
2. 过度过滤社交媒体，模型可能更安全、更正式，但对流行表达、讽刺、错别字和多轮口语理解变弱。
3. 过度清理代码重复，可能减少记忆风险，但也可能删除常见 API 用法模式。
4. 过度过滤毒性语料，可能减少有害输出，但也可能削弱模型识别和处理有害内容的能力。
5. 过度依赖英文质量分类器，可能系统性压低非英语和低资源语言。

因此，过滤系统应该和 data mixture、训练目标、评估体系一起设计，而不是由数据团队孤立完成。

## 22. 一个可落地的清洗过滤方案

如果面试官问：“给你 100TB 原始 web 数据，你怎么清洗成预训练语料？”可以按下面框架回答。

第一步，建立数据 schema。为每条样本记录来源 URL、时间、语言、解析方式、域名、长度、hash、许可证信息、过滤状态和质量分。

第二步，做解析和正文抽取。对 HTML、PDF、文档、代码分别用不同 parser，保留结构信息，不只保留纯文本。

第三步，基础过滤。删除空文本、乱码、异常字符、极端长度、明显模板和解析失败样本。

第四步，语言和领域分类。识别语言、领域和内容类型，为后续分桶阈值做准备。

第五步，规则质量过滤。按语言和领域设置长度、符号比例、重复行、链接比例、低信息密度等规则。

第六步，模型质量评分。训练或使用质量分类器，输出连续质量分，而不是简单删除。

第七步，安全和隐私处理。使用 PII 检测、毒性分类、安全分类和来源级策略，做删除、脱敏、隔离或降权。

第八步，去重和污染检测。做文档级去重、近重复检测，并与 benchmark、评测集和高风险泄漏集比对。

第九步，人工抽样审计。分来源、语言、质量分桶抽样，估计误删和漏删。

第十步，小规模训练验证。用不同过滤策略训练小模型，对比验证 loss、下游任务、安全评估和生成质量。

第十一步，版本化发布。记录所有规则、阈值、分类器版本、数据 hash、过滤统计和审计报告。

这套回答的重点是：不是写一个脚本清洗，而是构建可复现、可审计、可迭代的数据治理系统。

## 23. 常见面试题

### 23.1 为什么大模型训练前要做数据清洗？

回答要点：

1. 提升有效 token 密度。
2. 减少乱码、模板、广告、重复和低质内容。
3. 降低隐私和安全风险。
4. 减少模型记忆和 benchmark 污染。
5. 改善训练效率、下游能力和生成风格。

一句话回答：大模型会拟合训练分布，脏数据不是无害噪声，而会消耗算力、改变模型行为并带来安全和隐私风险。

### 23.2 规则过滤和模型过滤怎么取舍？

回答要点：

1. 规则过滤便宜、可解释、适合明显坏样本。
2. 模型过滤能识别语义质量，但成本高、可能有偏差。
3. 实际系统通常先规则粗筛，再模型评分，最后人工审计校准。
4. 不同语言、领域、数据类型需要不同阈值。

### 23.3 如何设计质量分类器？

回答要点：

1. 定义质量标准：可读性、信息密度、结构、来源、安全和隐私。
2. 构造训练数据：人工标注、弱监督、正负来源对比、LLM 辅助标注。
3. 输出连续分数，而不是只做二分类。
4. 分语言、领域和来源审计，避免系统性偏差。
5. 用小模型训练或下游评估验证分类器效果。

### 23.4 怎么处理 PII？

回答要点：

1. 用规则识别结构化 PII，如邮箱、电话、证件号。
2. 用 NER 和上下文模型识别非结构化个人信息。
3. 对高风险样本删除、脱敏或隔离。
4. 记录处理策略和数据 lineage。
5. 通过抽样审计估计漏检和误检。

### 23.5 毒性内容是不是全部删除？

回答要点：

1. 不能简单全部删除，因为模型需要识别风险语义。
2. 明显高风险内容应删除或隔离。
3. 新闻、研究、教育和安全语境要做上下文判断。
4. 安全专用数据可进入 SFT 或安全训练池，而不是普通混入预训练池。
5. 需要结合分类器、规则和人工审核。

### 23.6 如何判断清洗策略是否有效？

回答要点：

1. 看保留率、重复率、质量分、PII 命中率、安全风险分布。
2. 分语言、领域、来源做统计，防止分布坍缩。
3. 人工抽样审计误删和漏删。
4. 做小模型 ablation，比对 loss、benchmark、安全评估和生成样例。
5. 记录版本，保证可复现。

## 24. 常见误区

误区一：数据越干净越好。

更准确地说，数据要适合目标。过度清洗会删除真实世界多样性。

误区二：规则过滤足够了。

规则能处理明显坏数据，但难以识别语义低质、SEO 垃圾和复杂安全语境。

误区三：质量分类器分数高就一定好。

分类器反映训练标签和来源偏好，可能压制少数语言、口语和非主流风格。

误区四：毒性内容全部删除最安全。

模型需要学会识别和安全处理风险内容，关键是分层、隔离和受控使用。

误区五：PII 用正则就能解决。

正则只能处理结构化模式，非结构化和组合型隐私需要上下文判断和审计。

误区六：清洗完成后就不用管了。

数据清洗是迭代系统，训练结果、评估反馈和安全事件都会反向修正过滤策略。

## 25. 本章小结

清洗过滤与质量评分是大模型数据工程的核心。它决定哪些信息进入训练分布，哪些风险被移除或隔离，哪些能力被强化，哪些能力被削弱。

本章要记住几句话：

1. 大模型不会自动忽略脏数据，它会拟合训练分布。
2. 清洗过滤不是删脏数据这么简单，而是在塑造模型能力和行为。
3. 规则过滤、模型评分、安全过滤、PII 处理和人工审计需要组合使用。
4. 质量不是绝对概念，而是相对于模型目标定义的。
5. 过滤策略必须评估误删和漏删，不能只看删除比例。
6. 成熟数据工程必须可复现、可审计、可版本化。

如果面试中被问到数据清洗，最好的回答不是列几个规则，而是讲清楚完整闭环：数据 schema、解析、基础过滤、质量评分、安全隐私、去重污染、人工审计、小模型验证和版本治理。
