# 第五章：矩阵分解与低秩

矩阵分解和低秩近似，是理解 LoRA、模型压缩、embedding 几何、PCA 和表示分析的重要数学基础。大模型里很多巨大的权重矩阵和表示矩阵，看起来维度极高，但它们的有效变化可能集中在少数方向上。低秩方法的核心直觉就是：不一定需要完整高维空间，很多任务相关的信息可以用更低维的子空间表示。

这一章不追求严格数学证明，而是围绕面试和大模型实践讲清几个问题：SVD 是什么，PCA 为什么能降维，低秩近似为什么有用，LoRA 为什么可以用两个小矩阵表示权重更新，rank 如何影响能力和成本，低秩方法有什么边界。

本章重点：SVD、PCA、低秩近似、rank、LoRA、模型压缩、embedding 几何和面试表达。

## 5.1 为什么大模型需要低秩直觉

大模型有大量矩阵。

例如一个 hidden size 为 4096 的线性层，权重矩阵可能是：

```text
W: [4096, 4096]
```

参数量是：

```text
4096 * 4096 = 16777216
```

如果全参微调，每个这样的大矩阵都要更新，显存、优化器状态和通信成本都很高。

低秩方法的核心问题是：是否真的需要更新整个大矩阵？

LoRA 的答案是：很多任务适配可能只需要在一个低维子空间中调整权重，因此可以学习低秩更新。

这背后不是魔法，而是矩阵分解和低秩近似的直觉。

## 5.2 Rank：矩阵真正有多少独立方向

矩阵的 rank 可以理解为它包含多少个线性独立方向。

如果一个矩阵的很多行或列可以由其他行列组合出来，那么它的有效自由度就低，rank 就低。

直觉例子：

```text
row_3 = row_1 + row_2
```

这说明 `row_3` 没有提供新的独立方向。

在机器学习中，rank 代表表达能力和自由度。高 rank 更灵活，但参数多；低 rank 更受限，但更省参数，也可能有正则化效果。

面试回答：

```text
Rank 可以理解为矩阵中独立方向的数量。低秩矩阵说明它的变化主要发生在少数方向上。LoRA 利用这个直觉，假设微调所需的权重更新不需要完整高秩矩阵，而可以用低秩矩阵近似。
```

## 5.3 SVD：把矩阵拆成重要方向

SVD，也就是奇异值分解，可以把一个矩阵分解成：

```text
W = U S V^T
```

直觉上：

1. `V` 给出输入空间中的方向。
2. `S` 表示每个方向的重要程度。
3. `U` 给出输出空间中的方向。

`S` 中的奇异值通常按从大到小排列。奇异值越大，说明对应方向越重要。

如果只保留最大的 `r` 个奇异值，就得到 rank-r 近似：

```text
W_approx = U_r S_r V_r^T
```

这就是低秩近似：用少数重要方向近似原矩阵。

面试回答：

```text
SVD 可以把矩阵分解成一组从重要到不重要的方向。奇异值越大，说明对应方向贡献越大。如果只保留最大的几个奇异值，就能得到低秩近似，用更少自由度保留矩阵主要信息。
```

## 5.4 为什么低秩近似可行

低秩近似可行的前提是：矩阵信息并不是均匀分散在所有方向上，而是集中在少数重要方向上。

在很多数据和模型中，这个假设经常近似成立：

1. 自然数据有结构，不是完全随机噪声。
2. 任务相关变化可能只发生在少数方向。
3. 高维表示中存在冗余。
4. 微调常常只需要调整部分行为，而不是重写全部知识。

但低秩近似不是万能的。如果任务需要复杂、大范围的参数变化，rank 太小会限制表达能力。

所以低秩方法的核心 trade-off 是：

```text
更低 rank -> 更省参数，但表达能力更受限
更高 rank -> 表达更强，但成本更高，也可能过拟合
```

## 5.5 PCA：寻找数据中最重要的方向

PCA，也就是主成分分析，可以理解为寻找数据中方差最大的方向，并把数据投影到这些方向上。

直觉：

1. 数据在某些方向上变化大，说明这些方向包含更多区分信息。
2. 数据在某些方向上变化小，可能是噪声或次要信息。
3. 保留前几个主方向，就能降维并保留主要结构。

PCA 和 SVD 关系紧密。实际中，PCA 可以通过对中心化数据矩阵做 SVD 得到。

在大模型中，PCA 的直觉常用于：

1. 可视化 embedding。
2. 分析 hidden states。
3. 理解表示空间的主要变化方向。
4. 压缩或去噪。
5. 研究模型行为方向。

面试回答：

```text
PCA 寻找数据中方差最大的方向，并把数据投影到这些主方向上。它的作用是用更低维表示保留主要变化结构。在大模型里，可以用 PCA 直觉分析 embedding 或 hidden states 的主要变化方向，但它只是线性分析工具，不能完整解释模型行为。
```

## 5.6 LoRA：低秩更新而不是低秩权重

LoRA 的关键不是把原始权重 `W` 变成低秩，而是把微调时的权重增量 `Delta W` 表示成低秩。

全参微调：

```text
W_new = W + Delta W
```

LoRA：

```text
Delta W = A B
```

其中：

```text
A: [d_in, r]
B: [r, d_out]
```

`r` 是 LoRA rank，通常远小于 `d_in` 和 `d_out`。

这意味着 LoRA 假设：任务适配所需的更新位于一个低维子空间中。

面试中要特别注意：LoRA 不是说原模型权重本身低秩，而是说微调增量可以低秩建模。

面试回答：

```text
LoRA 的核心是低秩参数化权重更新。它冻结原始权重 W，只学习一个低秩增量 Delta W = A B。这样不是把原权重变成低秩，而是假设下游任务所需的参数更新可以在低秩子空间中表达。
```

## 5.7 LoRA 参数量怎么算

假设原始权重矩阵：

```text
W: [d_in, d_out]
```

全量更新参数量：

```text
d_in * d_out
```

LoRA 参数量：

```text
r * d_in + r * d_out = r * (d_in + d_out)
```

例如：

```text
d_in = 4096
d_out = 4096
r = 8
```

全量参数：

```text
4096 * 4096 = 16777216
```

LoRA 参数：

```text
8 * (4096 + 4096) = 65536
```

参数量大幅减少。

这也是 LoRA 能降低训练显存的原因之一。更少可训练参数意味着更少梯度和 optimizer state。

但注意：前向计算仍然要经过原模型，LoRA 不等于推理成本自动大幅下降。推理时如果不 merge LoRA，可能还会有额外计算；如果 merge 到权重里，则推理结构更简单。

## 5.8 LoRA rank 如何选择

Rank 是 LoRA 最重要的超参之一。

Rank 太小：

1. 表达能力不足。
2. 复杂任务适配不够。
3. 学不到足够行为变化。

Rank 太大：

1. 参数量增加。
2. 显存和 optimizer state 增加。
3. 小数据场景更容易过拟合。
4. 低秩正则化效果减弱。

实际选择 rank 要看：

1. 任务复杂度。
2. 数据量。
3. 模型大小。
4. 目标模块。
5. 训练预算。
6. 是否需要多 adapter。

面试回答：

```text
LoRA rank 控制低秩更新的表达能力。Rank 越大，更新空间越灵活，但参数、显存和过拟合风险也增加。Rank 越小越省，但可能表达不足。实际会通过验证集、人评、能力回归和训练成本做 sweep，而不是固定认为某个 rank 最好。
```

## 5.9 LoRA 应该加在哪些模块

LoRA 常加在 Transformer 的线性层上，例如：

1. `W_Q`。
2. `W_K`。
3. `W_V`。
4. Attention output projection。
5. MLP up projection。
6. MLP down projection。
7. Gate projection。

不同模块影响不同能力。

例如：

1. Attention 相关模块影响信息选择和上下文交互。
2. MLP 相关模块影响特征变换和知识表达。
3. 只调部分模块更省，但能力变化有限。
4. 调更多模块更强，但成本更高。

面试中不要只说“加 LoRA”。要能说明加在哪里、为什么、如何评估。

## 5.10 QLoRA 和量化的关系

QLoRA 的核心是把 base model 权重量化存储，同时用 LoRA 训练低秩 adapter。

直觉：

1. Base model 很大，量化后显存下降。
2. LoRA adapter 很小，保持可训练。
3. 反向传播主要更新 adapter。

QLoRA 适合显存受限的微调场景。

但要注意：

1. 量化可能影响质量。
2. 训练速度不一定总是更快。
3. 需要关注量化格式、计算 dtype 和内存峰值。
4. 对格式稳定性、代码、数学和安全任务要额外评估。

面试回答：

```text
QLoRA 可以理解为量化 base model 加低秩 adapter 训练。它通过量化降低基础模型显存，同时只训练小的 LoRA 参数，因此适合资源受限微调。它的风险是量化误差和训练/推理 kernel 复杂度，需要和 FP16/BF16 baseline 做质量和稳定性对比。
```

## 5.11 低秩和模型压缩

低秩近似也可以用于模型压缩。

如果一个权重矩阵可以近似写成两个低秩矩阵乘积：

```text
W ~= A B
```

原本一次大矩阵乘法：

```text
X W
```

可以变成：

```text
(X A) B
```

如果 rank 很小，参数量和计算量可能下降。

但模型压缩和 LoRA 不同：

1. LoRA 学的是增量 `Delta W`。
2. 压缩是近似原始权重 `W`。
3. 压缩可能直接损伤原模型能力。
4. 压缩后需要评估质量回归。

面试回答：

```text
低秩压缩是用两个小矩阵近似原始大矩阵，从而减少参数和计算。它和 LoRA 不同，LoRA 是低秩表示微调增量，通常保留原权重不动；低秩压缩直接近似原权重，可能带来能力损失，需要做回归评估。
```

## 5.12 Embedding 几何和低维可视化

Embedding 是高维向量。为了观察结构，常用 PCA、t-SNE、UMAP 等方法降维到 2D 或 3D。

这些可视化可以帮助理解：

1. 相似样本是否聚在一起。
2. 不同类别是否可分。
3. 数据是否有异常点。
4. 模型微调前后表示是否发生变化。

但要注意：低维可视化容易误导。

1. 高维距离不一定在二维图中保真。
2. 不同降维方法有不同偏差。
3. 图上聚类不等于模型真的理解。
4. 可视化适合探索，不适合单独作为严肃结论。

面试回答：

```text
PCA 或其他降维方法可以帮助观察 embedding 空间结构，比如类别聚类、异常点和微调前后的表示变化。但二维可视化会损失高维信息，不能单独证明模型能力，只能作为探索性分析，需要配合定量评估。
```

## 5.13 低秩方法的优点和边界

优点：

1. 参数量少。
2. 训练显存低。
3. 多任务 adapter 易管理。
4. 小数据场景可能有正则化效果。
5. 适合快速实验和个性化微调。

边界：

1. Rank 太小表达不足。
2. 复杂任务可能需要更大更新空间。
3. 多个 LoRA adapter 合并可能冲突。
4. LoRA 不一定降低基础前向计算成本。
5. 数据质量差时，低秩也会学坏。
6. 对安全、数学、代码、格式稳定性要单独评估。

面试中要避免把 LoRA 说成万能微调方法。

## 5.14 一个最小 PyTorch 例子：低秩线性层

下面用 PyTorch 演示 LoRA 风格的低秩增量。

```python
import torch
import torch.nn as nn


class LoRALinear(nn.Module):
    def __init__(self, d_in, d_out, rank):
        super().__init__()
        self.weight = nn.Parameter(torch.randn(d_in, d_out), requires_grad=False)
        self.A = nn.Parameter(torch.randn(d_in, rank) * 0.01)
        self.B = nn.Parameter(torch.zeros(rank, d_out))

    def forward(self, x):
        base = x @ self.weight
        delta = x @ self.A @ self.B
        return base + delta


x = torch.randn(2, 4, 16)
layer = LoRALinear(d_in=16, d_out=32, rank=4)
y = layer(x)

print(y.shape)  # [2, 4, 32]
```

这个例子说明：

1. 原始权重被冻结。
2. 可训练参数是 `A` 和 `B`。
3. 输出等于 base output 加低秩增量。
4. Rank 控制 adapter 的表达能力和参数量。

## 5.15 面试高频问题

### 5.15.1 LoRA 为什么省参数

回答：

```text
LoRA 不学习完整的 Delta W，而是把 Delta W 表示成两个低秩矩阵 A 和 B 的乘积。如果原矩阵参数量是 d_in * d_out，LoRA rank 为 r 时参数量是 r * (d_in + d_out)。当 r 远小于原始维度时，可训练参数大幅减少。
```

### 5.15.2 LoRA 是不是让原权重低秩

回答：

```text
不是。LoRA 通常冻结原始权重 W，只让微调增量 Delta W 采用低秩参数化。它的假设是任务适配所需的更新可以在低秩子空间中表达，而不是原模型权重本身低秩。
```

### 5.15.3 Rank 越大越好吗

回答：

```text
不一定。Rank 越大表达能力越强，但参数量、显存、训练成本和过拟合风险也更高。Rank 太小可能学不动复杂任务。实际要根据任务复杂度、数据量、目标模块和验证集表现调。
```

### 5.15.4 PCA 对大模型有什么用

回答：

```text
PCA 可以用来分析 embedding 或 hidden states 的主要变化方向，也可以做降维可视化和初步表示分析。它帮助我们理解高维表示的主要结构，但只是线性工具，不能单独证明模型理解或能力。
```

## 5.16 常见误区

1. 以为 LoRA 压缩的是原模型权重。
2. 只会说低秩省参数，不会算参数量。
3. 认为 rank 越大一定越好。
4. 忽略 LoRA 加在哪些模块。
5. 以为 LoRA 一定降低推理成本。
6. 用二维 embedding 可视化直接证明模型能力。
7. 把 PCA、SVD、LoRA 当成互不相关的概念。

## 5.18 本章总结

矩阵分解和低秩近似是理解参数高效微调的重要数学基础。

本章最重要的结论：

1. Rank 表示矩阵中的独立方向数量。
2. SVD 可以把矩阵拆成从重要到不重要的方向。
3. 低秩近似用少数方向保留主要信息。
4. PCA 寻找数据方差最大的主方向。
5. LoRA 学的是低秩权重增量，而不是把原权重变低秩。
6. LoRA 参数量是 `r * (d_in + d_out)`。
7. Rank 控制表达能力、成本和过拟合风险之间的 trade-off。
8. 低秩压缩和 LoRA 不是一回事。
9. Embedding 可视化只能作为探索，不能单独证明模型能力。

面试中讲 LoRA，不要只说“省显存”。要能从低秩矩阵、参数量、rank 选择、模块选择、表达能力和评估边界讲完整。
