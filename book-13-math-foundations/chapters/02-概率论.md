# 第二章：概率论

概率论是理解语言模型训练目标的核心。大模型不是直接输出“正确答案”，而是在每个位置输出一个 token 概率分布；预训练是在最大化真实文本序列的概率；解码是在概率分布上做 greedy、sampling、top-k、top-p；评估里的 perplexity、本质上也来自概率。

如果概率论不清楚，很多大模型概念会变成口号：next-token prediction、最大似然、交叉熵、KL、reward model、DPO、calibration、uncertainty、hallucination 都会只停留在名词层面。

本章目标：用面向 LLM 的方式讲清随机变量、概率分布、条件概率、链式法则、贝叶斯公式、期望、方差、最大似然和语言模型概率分解。

## 2.1 为什么 LLM 离不开概率论

语言模型的输出不是一个 token，而是整个词表上的概率分布。

例如上下文是：

```text
The capital of France is
```

模型可能输出：

```text
Paris      0.72
Lyon       0.04
the        0.03
located    0.02
...
```

训练时，真实下一个 token 是 `Paris`，模型就应该给它更高概率。

推理时，模型可以选择概率最高的 token，也可以按概率采样，或者只在 top-k / top-p 候选集合中采样。

所以大模型的很多问题都是概率问题：

1. 训练：如何提高真实 token 的概率。
2. 推理：如何从概率分布中选择 token。
3. 评估：如何衡量模型给真实文本的概率。
4. 幻觉：为什么高概率文本不等于真实事实。
5. 对齐：如何让概率分布更符合人类偏好。
6. 不确定性：模型什么时候应该表达“不知道”。

## 2.2 随机变量：把不确定结果形式化

随机变量是对不确定结果的数学表示。

在语言模型中，下一个 token 就可以看成一个离散随机变量。

如果词表大小是 `V`，那么下一个 token 可能取 `V` 个值。模型输出一个长度为 `V` 的概率分布：

```text
P(token = v | context)
```

这里的 `context` 是前面的 token 序列。

比如：

```text
P(next_token = Paris | The capital of France is)
```

面试回答：

```text
在语言模型里，可以把下一个 token 看成随机变量。给定上下文，模型输出这个随机变量在整个词表上的条件概率分布。训练目标就是让真实下一个 token 的概率尽可能高。
```

## 2.3 概率分布：模型输出的真正对象

概率分布描述每个可能结果发生的概率。

离散分布需要满足两个条件：

1. 每个概率不小于 0。
2. 所有概率加起来等于 1。

神经网络最后通常输出 logits。Logits 不是概率，可以是任意实数。需要经过 softmax 才变成概率分布：

```text
prob_i = exp(logit_i) / sum_j exp(logit_j)
```

Softmax 的作用是：

1. 把任意实数变成正数。
2. 让所有输出加起来等于 1。
3. 保留相对大小，logit 越大，对应概率越大。

面试回答：

```text
语言模型最后一层输出 logits，logits 本身不是概率。Softmax 把 logits 转成词表上的概率分布，使每个 token 的概率非负且总和为 1。训练时再根据真实 token 的概率计算 loss。
```

## 2.4 联合概率、条件概率和边缘概率

联合概率表示多个事件同时发生的概率。

例如：

```text
P(X = x, Y = y)
```

条件概率表示在一个事件已经发生的条件下，另一个事件发生的概率：

```text
P(Y = y | X = x)
```

条件概率的定义是：

```text
P(Y | X) = P(X, Y) / P(X)
```

在语言模型中，最重要的是条件概率：

```text
P(next token | previous tokens)
```

例如：

```text
P(Paris | The capital of France is)
```

边缘概率可以理解为把不关心的变量加总掉。例如：

```text
P(X) = sum_y P(X, Y = y)
```

面试中不需要把边缘概率讲得很抽象。更重要的是理解：语言模型不是只学单个 token 的无条件概率，而是学上下文条件下的 token 分布。

## 2.5 概率链式法则：自回归语言模型的基础

任意 token 序列的联合概率都可以用链式法则分解。

给定序列：

```text
x_1, x_2, ..., x_T
```

联合概率可以写成：

```text
P(x_1, x_2, ..., x_T)
= P(x_1) * P(x_2 | x_1) * ... * P(x_T | x_1, ..., x_{T-1})
```

这就是 GPT 类自回归语言模型的概率基础。

模型不需要一次直接建模整个长序列概率，而是每一步建模：

```text
P(x_t | x_1, ..., x_{t-1})
```

面试回答：

```text
自回归语言模型的基础是概率链式法则。任意序列的联合概率都可以分解成一系列条件概率的乘积。GPT 选择从左到右建模，每一步根据前面的 token 预测下一个 token，所以训练目标就是提高真实下一个 token 的条件概率。
```

## 2.6 为什么 next-token prediction 是概率建模

Next-token prediction 不是简单分类任务，而是在每个上下文位置建模一个条件概率分布。

训练样本可以写成：

```text
input:  x_1, ..., x_{t-1}
label:  x_t
```

模型输出：

```text
P(v | x_1, ..., x_{t-1}) for every token v in vocabulary
```

训练希望：

```text
P(x_t | x_1, ..., x_{t-1})
```

尽可能大。

这解释了为什么语言模型可以通过大量文本训练：每个 token 位置都提供一个监督信号。

一篇长度为 `T` 的文本，几乎可以提供 `T-1` 个 next-token prediction 训练样本。

## 2.7 最大似然估计：让训练数据更可能出现

最大似然估计，也就是 MLE，目标是找到一组参数，让训练数据出现的概率尽可能大。

假设模型参数是 `theta`，训练集有很多 token 序列。MLE 的目标可以直观写成：

```text
choose theta to maximize P(training data | theta)
```

对语言模型来说，就是最大化每个真实 token 的条件概率：

```text
maximize product_t P_theta(x_t | x_1, ..., x_{t-1})
```

很多概率相乘会非常小，所以实际优化 log probability：

```text
maximize sum_t log P_theta(x_t | x_1, ..., x_{t-1})
```

训练 loss 通常是负 log likelihood：

```text
loss = - sum_t log P_theta(x_t | x_1, ..., x_{t-1})
```

面试回答：

```text
最大似然估计的目标是让训练数据在模型下的概率尽可能高。对自回归语言模型来说，就是让每个真实下一个 token 的条件概率尽可能高。由于概率乘积很小，实际会最大化 log probability，等价于最小化 negative log likelihood，也就是常见的 cross entropy loss。
```

## 2.8 期望：平均意义下的目标

期望可以理解为随机变量的长期平均值。

离散情况下：

```text
E[X] = sum_x P(X = x) * x
```

机器学习里的很多目标都可以看成期望。

例如训练 loss 本质上是数据分布上的平均损失：

```text
expected loss = E_data[loss(model, sample)]
```

实际训练时，我们拿不到完整数据分布，只能用 mini-batch 近似这个期望：

```text
batch loss ~= expected loss
```

这就是为什么 batch size 会影响梯度噪声。batch 越小，对真实期望的估计噪声越大；batch 越大，估计更稳定，但计算和显存成本更高。

面试回答：

```text
训练目标可以看成数据分布上的期望损失。实际训练用 mini-batch loss 近似这个期望，因此 batch size 会影响估计噪声。小 batch 梯度噪声更大，大 batch 更稳定但成本更高，也可能影响泛化和优化动态。
```

## 2.9 方差：不确定性和噪声大小

方差衡量随机变量围绕均值波动的程度。

直观上：

1. 方差小，结果集中，稳定。
2. 方差大，结果分散，不稳定。

在大模型训练和评估中，方差很常见：

1. Mini-batch 梯度有方差。
2. 不同 random seed 训练结果有方差。
3. LLM-as-a-judge 评分有方差。
4. 人工评估样本太少时，估计方差大。
5. Sampling 解码会带来输出方差。

这解释了为什么一次实验结果不能轻易下结论。

如果一个方法在 100 条样本上提升 1 个百分点，但样本方差很大，这个提升可能没有统计意义。

面试回答：

```text
方差衡量随机结果的波动程度。训练中 mini-batch 梯度是对全量梯度的估计，会有方差；评估中样本量太小也会导致指标不稳定。所以判断模型是否真的变好，不能只看单次结果，要看样本量、置信区间、随机种子和误差分析。
```

## 2.10 贝叶斯公式：根据证据更新信念

贝叶斯公式描述如何根据证据更新概率：

```text
P(H | E) = P(E | H) * P(H) / P(E)
```

其中：

1. `H` 是假设。
2. `E` 是证据。
3. `P(H)` 是先验概率。
4. `P(H | E)` 是看到证据后的后验概率。

贝叶斯公式在大模型中不一定每天直接写，但它的思想很重要：根据新证据更新判断。

例如 RAG 中，模型原本可能根据语言先验猜答案；加入检索证据后，应该把回答约束到 evidence 上。

再例如评估模型质量时，单个 benchmark 分数只是证据之一。你还要结合人评、bad case、线上反馈和污染检查，更新对模型能力的判断。

面试回答：

```text
贝叶斯公式的核心思想是用证据更新信念。先验表示看到证据前的判断，后验表示结合证据后的判断。在 LLM 场景中，RAG、工具验证、不确定性判断和实验复盘都可以用这种思想理解：不要只依赖模型先验，要根据外部证据修正输出或判断。
```

## 2.11 独立性和条件独立

如果两个事件互不影响，可以说它们独立：

```text
P(A, B) = P(A) * P(B)
```

但真实语言中，token 几乎不是独立的。

例如在句子：

```text
The capital of France is Paris
```

`Paris` 的概率强烈依赖前面的上下文。语言模型的核心就是建模这种依赖关系。

早期一些简单模型会做独立性或 Markov 假设，例如只看前一个词或前几个词。Transformer 的优势是可以在上下文窗口内建模更长距离依赖，而不是只依赖固定短窗口。

面试回答：

```text
语言中的 token 通常不是独立的，下一个 token 强依赖前面的上下文。自回归语言模型显式建模 P(next token | previous tokens)。相比只看局部上下文的 n-gram 模型，Transformer 可以通过 self-attention 使用更长范围的上下文信息。
```

## 2.12 采样：从概率分布生成文本

推理时，模型输出概率分布后，需要选择下一个 token。

常见策略：

1. Greedy decoding：选概率最高的 token。
2. Sampling：按概率随机采样。
3. Temperature：调整分布尖锐程度。
4. Top-k：只从概率最高的 k 个 token 中采样。
5. Top-p：从累计概率达到 p 的候选集合中采样。

Temperature 的直觉：

1. 温度低，分布更尖锐，输出更确定。
2. 温度高，分布更平，输出更多样但更不稳定。

面试回答：

```text
语言模型每一步输出概率分布，解码策略决定如何从分布中选 token。Greedy 每次选最高概率，稳定但可能重复；sampling 增加多样性；temperature 控制分布尖锐程度；top-k 和 top-p 限制候选集合，减少低概率 token 带来的不稳定。
```

## 2.13 概率高不等于事实真

这是理解 hallucination 的关键。

语言模型学的是：在当前上下文和训练分布下，什么 token 更可能出现。它不是直接优化事实真伪。

一个回答可能语言上很高概率，但事实是错的。原因包括：

1. 训练数据里有错误信息。
2. 模型缺少某个事实。
3. 上下文证据不足。
4. SFT/RLHF 奖励了流畅和完整回答。
5. 解码策略让模型继续生成看似合理的文本。

面试回答：

```text
语言模型输出的是高概率文本，不等于经过事实验证的真命题。Next-token prediction 优化的是训练分布上的条件概率，而不是 truthfulness。因此模型可能生成流畅但错误的回答。降低幻觉需要结合数据质量、RAG、工具验证、引用约束、不确定性表达和评估治理。
```

## 2.14 校准：概率能否反映真实可信度

Calibration 关注模型的置信度是否可靠。

如果模型对一批问题都给出 80% 置信度，那么理想情况下，这些回答中大约 80% 应该正确。

大模型常常存在校准问题：

1. 错了也很自信。
2. 对简单问题过度保守。
3. 对分布外问题不知道自己不知道。
4. 经过 SFT/RLHF 后语言更自信，但概率未必更可信。

在高风险场景中，校准非常重要。模型不确定时应该检索、调用工具、请求澄清或拒答，而不是自信编造。

面试回答：

```text
校准衡量模型概率或置信度是否和真实正确率匹配。LLM 可能语言上很自信但事实错误，因此高风险场景不能只相信模型表达的置信。需要通过评估、RAG、工具验证、abstention 和人工审核来管理不确定性。
```

## 2.15 一个最小代码例子：bigram 概率模型

下面用纯 Python 写一个最小 bigram 模型，帮助理解条件概率。

```python
from collections import Counter, defaultdict

texts = [
    "I love machine learning",
    "I love deep learning",
    "machine learning is useful",
    "deep learning is powerful",
]

counts = defaultdict(Counter)

for text in texts:
    tokens = text.split()
    for current_token, next_token in zip(tokens, tokens[1:]):
        counts[current_token][next_token] += 1


def next_token_distribution(current_token):
    next_counts = counts[current_token]
    total = sum(next_counts.values())
    if total == 0:
        return {}
    return {
        token: count / total
        for token, count in next_counts.items()
    }


print(next_token_distribution("I"))
print(next_token_distribution("learning"))
```

这个例子非常简单，只根据当前词预测下一个词。GPT 类模型的本质更强：它不是只看一个词，而是看完整上下文，并用 Transformer 学习复杂条件概率。

## 2.16 面试高频问题

### 2.16.1 为什么语言模型可以只预测下一个 token

回答：

```text
根据概率链式法则，任意 token 序列的联合概率都可以分解成一系列条件概率的乘积。自回归语言模型选择从左到右建模这些条件概率，因此只要每一步能预测下一个 token，就可以建模整个序列概率。
```

### 2.16.2 最大似然和交叉熵是什么关系

回答：

```text
最大似然是让训练数据在模型下的概率尽可能高。对分类或语言模型来说，最大化真实标签的 log probability，等价于最小化 negative log likelihood。在 one-hot 标签下，这和最小化交叉熵是等价的。
```

### 2.16.3 为什么 batch size 会影响训练稳定性

回答：

```text
训练目标是数据分布上的期望损失，但实际用 mini-batch 估计。batch size 小时，估计噪声和梯度方差更大；batch size 大时，梯度更稳定但计算成本更高，也可能影响泛化和优化动态。因此 batch size 通常要和 learning rate、warmup、gradient accumulation 一起调。
```

### 2.16.4 为什么概率高不代表事实正确

回答：

```text
语言模型优化的是训练分布上的 token 条件概率，而不是事实验证。一个错误说法如果在训练分布或当前上下文中看起来语言上合理，也可能获得较高概率。因此事实性需要额外机制，比如检索、工具验证、引用约束、事实性评估和不确定性表达。
```

## 2.17 常见误区

1. 把概率最高理解成事实正确。
2. 只会说 next-token prediction，不会讲链式法则。
3. 知道最大似然，但讲不清为什么取 log。
4. 把 batch loss 当成真实总体 loss。
5. 忽略评估结果的方差和样本量。
6. 认为 temperature 可以解决幻觉。
7. 把模型输出的自信语气当成可靠置信度。

## 2.19 本章总结

概率论是语言模型训练和生成的底层语言。

本章最重要的结论：

1. 下一个 token 可以看成随机变量。
2. 模型输出的是词表上的条件概率分布。
3. 自回归建模来自概率链式法则。
4. 最大似然让训练数据在模型下更可能出现。
5. 训练 loss 是对数据分布期望损失的 mini-batch 近似。
6. 方差解释了训练和评估为什么不稳定。
7. 解码是在概率分布上选择 token。
8. 概率高不等于事实真，幻觉需要额外治理。

面试中讲概率论，不要只背公式。要主动把随机变量、条件概率、链式法则、MLE、期望方差和 LLM 的训练、解码、评估、幻觉联系起来。
