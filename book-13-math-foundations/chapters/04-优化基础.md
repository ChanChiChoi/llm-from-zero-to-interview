# 第四章：优化基础

优化是大模型训练的发动机。概率论和信息论告诉我们 loss 是什么，优化告诉我们如何通过梯度一步步降低 loss。大模型训练中的很多高频问题，本质上都是优化问题：learning rate 为什么重要，warmup 为什么必要，AdamW 为什么常用，gradient clipping 解决什么，loss spike 怎么排查，batch size 为什么影响稳定性。

很多候选人会背“用 AdamW、warmup、cosine decay、gradient clipping”，但讲不清它们为什么需要、解决什么问题、有什么副作用。面试官一旦追问“为什么 warmup 能缓解训练早期不稳定”“AdamW 和 Adam 的 weight decay 有什么区别”“batch size 变大 learning rate 怎么调”，回答就容易停在经验层。

本章目标：讲清梯度、Hessian、SGD、Momentum、Adam、AdamW、学习率、warmup、gradient clipping、凸优化和非凸优化直觉，并把它们连接到 LLM 训练稳定性。

## 4.1 优化在大模型训练中解决什么问题

训练大模型时，我们有一个参数量巨大的模型和一个 loss function。

目标是找到一组参数，使训练数据上的 loss 尽可能低，同时希望模型能泛化到未见数据。

可以直观写成：

```text
find parameters that minimize loss(parameters)
```

这个问题很难，因为：

1. 参数量巨大，可能是几十亿到万亿级。
2. Loss surface 是非凸的。
3. 数据是 mini-batch 随机采样，有噪声。
4. 混合精度会带来数值问题。
5. 分布式训练会带来同步和恢复问题。
6. 训练成本极高，不能随便试错。

所以优化不是“调用 optimizer.step()”这么简单，而是要理解训练动态、稳定性和工程约束。

## 4.2 梯度：loss 下降最快的局部方向

梯度表示 loss 对参数的变化敏感度。

如果参数是 `w`，loss 是 `L(w)`，梯度可以理解为：

```text
gradient = dL / dw
```

梯度告诉我们：如果参数往某个方向移动，loss 会如何变化。

梯度下降的基本更新是：

```text
w_new = w - learning_rate * gradient
```

为什么是减号？因为梯度指向 loss 增长最快的方向，想降低 loss，就往反方向走。

面试回答：

```text
梯度表示 loss 对参数的局部变化方向。梯度下降沿负梯度方向更新参数，因为梯度方向是 loss 增长最快的方向。Learning rate 控制每一步走多远，太大可能发散，太小收敛慢。
```

## 4.3 一个最小梯度下降例子

考虑一个简单函数：

```text
L(w) = (w - 5)^2
```

它的最小值在 `w = 5`。

梯度是：

```text
dL / dw = 2 * (w - 5)
```

如果 `w = 0`，梯度是 `-10`。按照梯度下降：

```text
w_new = w - lr * gradient
```

如果 `lr = 0.1`，则：

```text
w_new = 0 - 0.1 * (-10) = 1
```

参数从 0 朝 5 移动。

这个例子虽然简单，但说明了训练的核心：反复计算梯度，沿降低 loss 的方向更新参数。

## 4.4 Learning rate：优化中最敏感的旋钮

Learning rate 决定每次参数更新幅度。

如果 learning rate 太小：

1. 收敛慢。
2. 训练成本高。
3. 可能卡在不好的区域很久。

如果 learning rate 太大：

1. Loss 震荡。
2. Loss spike。
3. NaN/Inf。
4. 模型发散。

大模型训练中，learning rate 不能孤立选择，它和以下因素有关：

1. 模型规模。
2. Batch size。
3. Optimizer。
4. 数据质量。
5. Warmup steps。
6. Precision。
7. Gradient clipping。

面试回答：

```text
Learning rate 控制每一步参数更新幅度。它太小会收敛慢，太大可能导致 loss spike 或发散。大模型训练中 learning rate 要和 batch size、optimizer、warmup、gradient clipping 和混合精度一起看，不能单独调。
```

## 4.5 SGD：随机估计全量梯度

全量梯度需要遍历整个训练集，成本太高。SGD 使用一个 mini-batch 估计梯度。

直觉：

```text
mini-batch gradient ~= full-data gradient
```

这种估计有噪声。噪声既有坏处，也有好处。

坏处：

1. Loss 曲线抖动。
2. 小 batch 不稳定。
3. 梯度方向可能偏离真实下降方向。

好处：

1. 每步成本低。
2. 噪声可能帮助逃离某些不好的局部区域。
3. 有时泛化更好。

Batch size 越大，梯度估计越稳定，但计算和显存成本越高。

面试回答：

```text
SGD 用 mini-batch 梯度近似全量数据梯度。Mini-batch 带来噪声，小 batch 噪声大但便宜，大 batch 更稳定但成本高，也可能影响泛化。LLM 训练中通常看 global batch size，它由 micro batch、gradient accumulation 和 data parallel size 共同决定。
```

## 4.6 Momentum：给梯度下降加惯性

普通 SGD 每一步只看当前梯度。Momentum 会维护一个历史梯度的移动平均，让更新方向更平滑。

直觉上，Momentum 像给优化过程加惯性：

1. 如果多个 step 梯度方向一致，就加速前进。
2. 如果梯度来回震荡，就相互抵消一部分。

可以直观写成：

```text
velocity = beta * velocity + gradient
w = w - lr * velocity
```

Momentum 对峡谷形 loss surface 很有用：一个方向很陡，一个方向很平。没有 momentum 时，参数会在陡峭方向来回震荡；有 momentum 后，更新更稳定。

## 4.7 Adam：自适应学习率

Adam 可以理解为 Momentum + 自适应缩放。

它维护两个统计量：

1. 一阶矩：梯度的移动平均，类似 momentum。
2. 二阶矩：梯度平方的移动平均，表示梯度尺度。

直觉上：

1. 一阶矩告诉我们往哪个方向走。
2. 二阶矩告诉我们这个方向梯度通常有多大。
3. 梯度大的参数方向，步子会被相对缩小。
4. 梯度小的参数方向，步子会被相对放大。

这让 Adam 在高维、稀疏、非平稳优化中很实用。

面试回答：

```text
Adam 维护梯度的一阶矩和二阶矩。一阶矩类似 momentum，表示平均更新方向；二阶矩表示梯度平方的平均大小，用来做自适应缩放。它比普通 SGD 更适合高维和梯度尺度差异大的深度网络优化。
```

## 4.8 AdamW：解耦 weight decay

AdamW 是大模型训练中非常常见的优化器。

要理解 AdamW，先理解 weight decay。

Weight decay 的目标是限制权重过大，起到正则化作用。普通写法类似：

```text
w = w - lr * gradient - lr * weight_decay * w
```

在 Adam 中，如果把 weight decay 直接加到 gradient 里，会受到 Adam 自适应缩放影响，导致正则化效果和预期不一致。

AdamW 的核心是 decoupled weight decay：把 weight decay 从梯度更新中解耦出来，单独对参数做衰减。

面试回答：

```text
AdamW 和 Adam 的关键区别是 weight decay 的处理。Adam 如果把 L2 正则项直接加进 gradient，会被 Adam 的自适应学习率缩放影响。AdamW 把 weight decay 从梯度更新中解耦，直接对权重做衰减，因此正则化行为更符合预期，也更常用于大模型训练。
```

注意：并不是所有参数都适合 weight decay。大模型训练中常常不对 bias、LayerNorm 或 RMSNorm 参数做 weight decay，因为这些参数主要控制平移或尺度，衰减它们可能损害训练稳定性。

## 4.9 Warmup：训练早期不要走太猛

大模型训练常用 warmup：学习率从很小逐渐升到峰值。

为什么需要 warmup？

训练刚开始时：

1. 参数还没有形成稳定表示。
2. 激活和梯度分布不稳定。
3. Adam 的一阶矩和二阶矩估计还不可靠。
4. 大学习率容易导致 loss spike 或 NaN。

Warmup 让模型先用小步子进入稳定区域，再逐渐加大学习率。

面试回答：

```text
Warmup 的作用是缓解训练早期不稳定。刚开始模型参数、激活和 Adam 的动量统计都不稳定，如果直接用 peak learning rate，容易 loss spike 或发散。Warmup 逐步增加学习率，让优化过程先稳定下来。
```

## 4.10 Learning rate decay：后期小步收敛

Warmup 之后，learning rate 通常会逐渐下降。常见 schedule 包括 cosine decay、linear decay、constant with decay 等。

为什么要 decay？

1. 训练前期需要较大步子快速下降。
2. 训练后期需要较小步子细致收敛。
3. 大学习率持续太久可能导致 loss 震荡。

Cosine decay 的直觉是：平滑地把学习率从峰值降到较小值。

面试中不必背所有 schedule 公式，但要能讲 trade-off：schedule 不是独立超参，它和总训练 token、batch size、模型规模、数据质量一起决定训练动态。

## 4.11 Gradient clipping：限制异常更新

Gradient clipping 用于限制梯度范数过大。

常见做法是：如果 gradient norm 超过阈值，就按比例缩小所有梯度，使总 norm 不超过阈值。

它解决的问题：

1. 梯度爆炸。
2. 异常 batch 导致超大更新。
3. 混合精度下的数值不稳定。
4. Loss spike 风险。

但 clipping 不是万能的。如果 loss spike 来自数据错、mask 错、学习率过大或 checkpoint 恢复错误，clipping 只能缓解，不能代替根因排查。

面试回答：

```text
Gradient clipping 用来限制梯度范数，防止某一步更新过大导致训练不稳定。它对梯度爆炸和异常 batch 有缓解作用，但不能替代根因排查。如果 loss spike 来自数据、mask、learning rate 或 resume 状态错误，仍然要定位根因。
```

## 4.12 Hessian 和曲率直觉

梯度是一阶信息，告诉我们往哪里走。Hessian 是二阶信息，描述 loss surface 的曲率。

直觉上：

1. 曲率大，loss 对参数变化很敏感。
2. 曲率小，loss 变化平缓。
3. 不同方向曲率不同，会导致优化困难。

深度学习通常不会直接计算完整 Hessian，因为参数太多，成本不可接受。但 Hessian 直觉很重要。

例如：

1. Learning rate 太大，在高曲率方向容易震荡。
2. Adaptive optimizer 可以缓解不同参数方向梯度尺度差异。
3. Sharp minima 和 flat minima 常用来讨论泛化直觉。

面试回答：

```text
Hessian 描述 loss surface 的二阶曲率。大模型不会显式计算完整 Hessian，但曲率直觉有用：高曲率方向对学习率更敏感，容易震荡；低曲率方向更新慢。优化器、学习率和归一化都会影响模型如何在复杂非凸曲面上移动。
```

## 4.13 凸优化和非凸优化直觉

凸优化中，局部最优就是全局最优，理论分析更清楚。

深度网络训练是非凸优化：

1. 有大量鞍点。
2. 有许多局部区域。
3. 参数对称性很多。
4. Loss surface 高维复杂。

但实际中，大模型仍然可以训练成功，原因包括：

1. 过参数化带来很多可行解。
2. SGD 噪声帮助探索。
3. 残差连接、归一化、初始化改善优化。
4. 大规模数据和计算让统计规律更稳定。
5. AdamW、warmup、clipping 等 recipe 已经高度工程化。

面试回答：

```text
大模型训练是非凸优化，不能用简单凸优化理论解释全部现象。但过参数化、残差结构、归一化、合适初始化、AdamW、warmup 和大规模数据让训练在实践中可行。面试中更重要的是理解训练动态和稳定性，而不是声称能找到全局最优。
```

## 4.14 Batch size、gradient accumulation 和优化噪声

大模型训练中要区分 micro batch 和 global batch。

```text
global batch = micro batch * gradient accumulation steps * data parallel size
```

Micro batch 受单卡显存限制。Gradient accumulation 用多个 micro-batch 累积梯度，模拟更大的 effective batch。

Batch size 影响：

1. 梯度噪声。
2. 硬件吞吐。
3. 学习率选择。
4. 更新步数。
5. 泛化和收敛动态。

面试回答：

```text
LLM 训练中要看 global batch size，而不是单卡 micro batch。Global batch 由 micro batch、gradient accumulation 和 data parallel size 决定。它影响梯度噪声、吞吐和学习率设置。更大 batch 梯度更稳定，但更新步数减少，且需要重新调 learning rate 和 warmup。
```

## 4.15 Loss spike 的优化视角

Loss spike 可能来自很多原因，优化相关原因包括：

1. Learning rate 太大。
2. Warmup 太短。
3. Gradient norm 突然变大。
4. Gradient clipping 缺失或阈值不合适。
5. AdamW 状态异常。
6. Checkpoint resume 后 optimizer state 不连续。
7. FP16 overflow 或 loss scaling 问题。

排查时不要只说“调小学习率”。

更好的回答：

```text
我会先定位 spike 的 step，检查 learning rate、grad norm、optimizer state、mixed precision 和 batch 数据。如果发生在 warmup 或 lr peak 附近，重点看 schedule；如果伴随 grad norm 爆炸，检查 clipping 和异常 batch；如果发生在 resume 后，检查 optimizer、scheduler、scaler 和 random state 是否完整恢复。
```

## 4.16 一个最小 PyTorch 训练循环

下面是一个最小训练循环，用来把优化概念串起来。

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

model = nn.Linear(10, 2)
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)

for step in range(100):
    x = torch.randn(32, 10)
    y = torch.randint(0, 2, (32,))

    logits = model(x)
    loss = F.cross_entropy(logits, y)

    optimizer.zero_grad()
    loss.backward()

    grad_norm = torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
    optimizer.step()

    if step % 10 == 0:
        print(step, loss.item(), float(grad_norm))
```

这个循环包含：

1. Forward 计算 logits。
2. Loss 计算。
3. Backward 计算梯度。
4. Gradient clipping。
5. AdamW 更新参数。
6. 监控 loss 和 grad norm。

真实 LLM 训练会更复杂，还会加入 gradient accumulation、mixed precision、distributed training、scheduler、checkpoint 和 evaluation。

## 4.17 面试高频问题

### 4.17.1 为什么大模型训练常用 warmup

回答：

```text
训练初期参数、激活和 Adam 的动量统计都不稳定。如果一开始就用 peak learning rate，容易造成 loss spike 或发散。Warmup 让学习率从小逐步增加，使优化器状态和模型表示先稳定下来。
```

### 4.17.2 Adam 和 AdamW 有什么区别

回答：

```text
AdamW 的关键是 decoupled weight decay。Adam 如果把 L2 正则直接加到 gradient 中，会被自适应学习率缩放影响。AdamW 把 weight decay 从梯度更新中解耦，单独对权重做衰减，因此正则化行为更符合预期。
```

### 4.17.3 为什么要做 gradient clipping

回答：

```text
Gradient clipping 限制梯度范数，防止某一步更新过大。它能缓解梯度爆炸、异常 batch 和训练不稳定，降低 loss spike 风险。但它不能替代根因排查，如果数据、mask、learning rate 或 checkpoint 状态有问题，仍然要定位根因。
```

### 4.17.4 Batch size 和 learning rate 怎么关联

回答：

```text
Batch size 影响梯度噪声和更新步数。更大的 global batch 通常梯度更稳定，可以支持更大的 learning rate，但不是简单越大越好。实际要结合模型规模、warmup、optimizer、训练 token 数和 validation 表现调参。
```

## 4.18 常见误区

1. 把 optimizer 当成黑盒 API。
2. 只会说 AdamW 好，不会解释 decoupled weight decay。
3. Loss spike 只会说调小 learning rate。
4. 不区分 micro batch 和 global batch。
5. 忽略 optimizer state 对 checkpoint resume 的影响。
6. 认为 gradient clipping 可以解决所有训练不稳定。
7. 不监控 grad norm，只看 loss。

## 4.20 本章总结

优化把 loss 变成参数更新，是大模型训练能够发生的核心机制。

本章最重要的结论：

1. 梯度告诉参数如何局部降低 loss。
2. Learning rate 控制每步更新幅度，是最敏感超参之一。
3. SGD 用 mini-batch 估计全量梯度，因此有噪声。
4. Momentum 平滑更新方向。
5. Adam 用一阶矩和二阶矩做自适应更新。
6. AdamW 解耦 weight decay，更适合大模型训练。
7. Warmup 缓解训练早期不稳定。
8. Gradient clipping 限制异常梯度更新。
9. Hessian 和曲率直觉帮助理解学习率敏感性。
10. Loss spike 要结合数据、梯度、优化器、精度和 checkpoint 状态排查。

面试中讲优化，不要只背优化器名字。要能把梯度、学习率、batch size、AdamW、warmup、clipping 和训练稳定性连成一条完整链路。
