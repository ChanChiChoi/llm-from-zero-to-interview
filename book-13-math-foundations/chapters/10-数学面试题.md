## 10.1 面试回答数学题的基本模板

推荐结构：

1. 先给直觉。
2. 再给核心公式。
3. 说明在 LLM 中怎么用。
4. 说一个常见误区或工程细节。

例如问“交叉熵和 KL 的关系是什么”，不要只写公式。更好的回答是：

```text
交叉熵衡量用模型分布 q 去编码真实分布 p 的平均代价。它可以分解成真实分布自身的熵 H(p) 和 KL(p||q)。训练时 p 固定，所以最小化交叉熵等价于最小化 KL(p||q)。在语言模型中，next-token prediction 的 cross entropy loss 就是在让模型分布接近数据分布。
```

### 10.1.1 关键公式与数学面试总复盘

数学面试里，公式不是背诵目标，而是用来支撑判断。下面这组公式覆盖第十三册最常被追问的主线。

信息论和语言模型 loss：

$$
H(p) = -\sum_x p(x)\log p(x)
$$

$$
H(p,q) = -\sum_x p(x)\log q(x)
$$

$$
D_{\mathrm{KL}}(p||q) = \sum_x p(x)\log \frac{p(x)}{q(x)}
$$

$$
H(p,q)=H(p)+D_{\mathrm{KL}}(p||q)
$$

自回归语言模型：

$$
p_\theta(x_{1:T})=\prod_{t=1}^{T}p_\theta(x_t|x_{1:t-1})
$$

$$
L_{\mathrm{NLL}}=-\sum_{t=1}^{T}\log p_\theta(x_t|x_{1:t-1})
$$

$$
\mathrm{PPL}=\exp(L_{\mathrm{avg}})
$$

Softmax 与交叉熵梯度：

$$
p_i=\frac{\exp(z_i)}{\sum_j \exp(z_j)}
$$

$$
\frac{\partial L}{\partial z_i}=p_i-y_i
$$

优化与稳定性：

$$
\theta_{t+1}=\theta_t-\eta g_t
$$

$$
m_t=\beta_1m_{t-1}+(1-\beta_1)g_t,\qquad v_t=\beta_2v_{t-1}+(1-\beta_2)g_t^2
$$

$$
\theta_{t+1}=\theta_t-\eta\frac{\hat m_t}{\sqrt{\hat v_t}+\epsilon}-\eta\lambda\theta_t
$$

$$
g'=\min(1,c/||g||_2)g
$$

低秩与 LoRA：

$$
W=U\Sigma V^T
$$

$$
W_r=U_r\Sigma_rV_r^T
$$

$$
W'=W+\Delta W,\qquad \Delta W=BA
$$

$$
N_{\mathrm{LoRA}}=r(d_{\mathrm{in}}+d_{\mathrm{out}})
$$

Attention：

$$
S=\frac{QK^T}{\sqrt{d_k}}
$$

泛化和正则：

$$
R(f)=E[\ell(f(x),y)],\qquad \hat R_S(f)=\frac{1}{n}\sum_{i=1}^{n}\ell(f(x_i),y_i)
$$

$$
E[(y-\hat f(x))^2]=\mathrm{Bias}^2+\mathrm{Var}+\sigma^2
$$

贝叶斯、MLE 和 MAP：

$$
p(H|E)=\frac{p(E|H)p(H)}{p(E)}
$$

$$
\theta_{\mathrm{MLE}}=\arg\max_\theta p(D|\theta)
$$

$$
\theta_{\mathrm{MAP}}=\arg\max_\theta p(D|\theta)p(\theta)
$$

RLHF / PPO / DPO：

$$
A_t=G_t-V(s_t)
$$

$$
r_t(\theta)=\frac{\pi_\theta(a_t|s_t)}{\pi_{\theta_{\mathrm{old}}}(a_t|s_t)}
$$

$$
L_{\mathrm{clip}}=E[\min(r_tA_t,\mathrm{clip}(r_t,1-\epsilon,1+\epsilon)A_t)]
$$

$$
L_{\mathrm{RM}}=-\log \sigma(r^+-r^-)
$$

$$
\Delta_{\mathrm{DPO}}=\beta[(\log \pi_\theta(y^+|x)-\log \pi_{\mathrm{ref}}(y^+|x))-(\log \pi_\theta(y^-|x)-\log \pi_{\mathrm{ref}}(y^-|x))]
$$

$$
L_{\mathrm{DPO}}=-\log \sigma(\Delta_{\mathrm{DPO}})
$$

评估统计：

$$
\bar x=\frac{1}{n}\sum_i x_i,\qquad SE(\bar x)=\frac{s}{\sqrt n}
$$

$$
CI_{95}\approx \bar x\pm1.96SE(\bar x)
$$

$$
d_i=m_i^{new}-m_i^{old}
$$

$$
\chi^2_{\mathrm{McNemar}}=\frac{(|b-c|-1)^2}{b+c}
$$

## 10.2 交叉熵和 KL 的关系是什么

核心公式：

```text
H(p, q) = H(p) + KL(p || q)
```

其中：

```text
H(p, q) = - sum_x p(x) log q(x)
H(p) = - sum_x p(x) log p(x)
KL(p || q) = sum_x p(x) log(p(x) / q(x))
```

直觉：

1. 熵 H(p) 是真实分布自身的不确定性。
2. 交叉熵 H(p, q) 是用 q 表示 p 的代价。
3. KL 是由于 q 不等于 p 造成的额外代价。

在监督训练中，数据分布 p 固定，所以 H(p) 是常数，最小化交叉熵等价于最小化 KL(p||q)。

LLM 场景：

1. 预训练 next-token loss 是 token 级交叉熵。
2. Perplexity 是交叉熵的指数形式。
3. SFT 也是在示范数据上最小化交叉熵。
4. Distillation 常用 KL 或软标签交叉熵。

面试回答：

```text
交叉熵可以分解为真实分布的熵加 KL 散度。由于训练数据分布固定，真实熵不随模型变化，所以最小化交叉熵等价于最小化 KL(p||q)。在语言模型中，next-token cross entropy 就是在让模型预测分布靠近数据中的下一个 token 分布。
```

常见误区：

1. 把交叉熵和 KL 当成完全一样。
2. 忘记 H(p) 是否为常数取决于优化对象。
3. 不区分 KL(p||q) 和 KL(q||p)。

## 10.3 KL(p||q) 和 KL(q||p) 有什么区别

KL 是不对称的。

```text
KL(p || q) != KL(q || p)
```

KL(p||q) 的惩罚来自 p 高而 q 低的地方。如果真实分布认为某个事件重要，但模型 q 给了很小概率，惩罚会很大。

KL(q||p) 更关心模型 q 放了概率但真实 p 不支持的地方。

直觉上：

1. Forward KL，KL(p||q)，倾向于覆盖 p 的所有模式。
2. Reverse KL，KL(q||p)，倾向于选择一个高概率模式，可能 mode-seeking。

LLM 场景：

1. 最大似然训练对应 KL(data||model)。
2. RLHF 中 KL penalty 常约束新 policy 不要离 reference 太远。
3. DPO 中隐含了 policy 相对 reference 的对数概率差。

面试回答：

```text
KL 是不对称的。KL(p||q) 对 p 有质量但 q 给低概率的位置惩罚很重，倾向于覆盖真实分布；KL(q||p) 对 q 放到 p 不支持区域的概率敏感，更容易 mode-seeking。语言模型 MLE 更接近最小化 KL(data||model)，而 RLHF 常用 KL 约束防止 policy 偏离 reference。
```

## 10.4 Perplexity 如何理解

Perplexity 可以理解为模型平均每一步有多少“困惑度”。

如果 token 级交叉熵是：

```text
CE = - average log p(x_t | x_<t)
```

那么：

```text
PPL = exp(CE)
```

如果使用 log base 2，则：

```text
PPL = 2^CE
```

直觉：PPL 越低，模型给真实下一个 token 的概率越高。

LLM 场景：

1. PPL 适合衡量语言建模能力。
2. PPL 不等于任务能力。
3. 低 PPL 不一定代表 instruction following 更好。
4. 不同 tokenizer 或不同数据集上的 PPL 不能随便横向比较。

面试回答：

```text
Perplexity 是交叉熵的指数形式，可以理解为模型平均每个 token 的困惑度。PPL 越低说明模型给真实 token 的概率越高。但 PPL 主要衡量语言建模，不等价于推理、对齐或工具使用能力，不同 tokenizer 和数据分布下也不能直接比较。
```

## 10.5 Softmax 为什么常和交叉熵一起用

Softmax 把 logits 转成概率分布：

```text
p_i = exp(z_i) / sum_j exp(z_j)
```

交叉熵衡量预测分布和目标分布的差异。

对 one-hot 标签，softmax cross entropy 的梯度非常简洁：

```text
dL/dz_i = p_i - y_i
```

这意味着：

1. 正确类别概率低，就强烈提高它的 logit。
2. 错误类别概率高，就降低它的 logit。
3. 梯度形式稳定且易于优化。

LLM 场景：每个位置输出词表 logits，经 softmax 得到 next-token 概率，再用目标 token 的负对数似然作为 loss。

面试回答：

```text
Softmax 把 logits 变成概率分布，交叉熵衡量预测概率和目标分布的差距。二者结合后，对 logits 的梯度是 p-y，形式简单稳定。语言模型每个 token 位置都会对词表 logits 做 softmax，然后用真实下一个 token 计算 cross entropy loss。
```

## 10.6 为什么训练语言模型等价于最大似然估计

给定序列：

```text
x_1, x_2, ..., x_T
```

自回归语言模型分解为：

```text
p(x_1, ..., x_T) = product_t p(x_t | x_<t)
```

最大化数据似然：

```text
max sum_t log p(x_t | x_<t)
```

等价于最小化负对数似然：

```text
min - sum_t log p(x_t | x_<t)
```

这就是 token 级交叉熵损失。

面试回答：

```text
自回归语言模型把序列概率分解成每个 token 的条件概率乘积。最大似然就是最大化训练语料中真实 token 的条件概率，取负号后就是最小化 next-token cross entropy。因此预训练本质上是对语料分布做最大似然估计。
```

## 10.7 Adam 和 SGD 的区别是什么

SGD 使用当前梯度更新参数：

```text
theta = theta - lr * grad
```

Adam 使用一阶矩和二阶矩自适应调整每个参数的步长：

```text
m_t = beta1 * m_{t-1} + (1 - beta1) * g_t
v_t = beta2 * v_{t-1} + (1 - beta2) * g_t^2
theta = theta - lr * m_hat / (sqrt(v_hat) + eps)
```

直觉：

1. m 是动量，平滑梯度方向。
2. v 是梯度平方的估计，用来缩放学习率。
3. 梯度经常很大的参数步子变小，梯度小的参数相对步子变大。

LLM 场景：Adam/AdamW 是大模型训练和微调中的常用优化器，因为 Transformer 参数规模大、不同参数梯度尺度差异明显。

面试回答：

```text
SGD 基本按当前梯度统一更新，Adam 会维护梯度一阶矩和二阶矩，对不同参数使用自适应步长。一阶矩类似动量，二阶矩用于按梯度尺度归一化更新。大模型训练中 AdamW 更常用，因为它优化更稳定，且把 weight decay 和梯度更新解耦。
```

## 10.8 AdamW 为什么不同于 Adam

Adam 中如果把 L2 正则直接加到 loss，正则项会进入梯度，并被 Adam 的自适应二阶矩缩放。

AdamW 的核心是 decoupled weight decay：

```text
theta = theta - lr * adam_update - lr * weight_decay * theta
```

也就是权重衰减不再混入梯度自适应缩放，而是直接对参数做衰减。

为什么重要：

1. Weight decay 的含义更接近真正的参数收缩。
2. 避免不同参数因为自适应缩放导致正则强度不一致。
3. 大模型训练中通常比 Adam + L2 更稳定。

面试回答：

```text
AdamW 和 Adam 的关键区别是 weight decay 解耦。Adam 里 L2 正则会进入梯度，然后被自适应二阶矩缩放；AdamW 直接对参数做衰减，不参与 Adam 的梯度归一化。因此 AdamW 的正则含义更清晰，通常是 Transformer 和大模型训练的默认选择。
```

## 10.9 学习率 warmup 为什么有用

训练初期参数、梯度统计和 optimizer state 都不稳定。如果一开始使用很大学习率，容易 loss spike 或训练发散。

Warmup 的做法是前若干 step 从小学习率逐步升到目标学习率。

LLM 中 warmup 有几个作用：

1. 让 Adam 的一阶、二阶矩估计先稳定。
2. 避免初期大步更新破坏预训练表示。
3. 在大 batch 和 mixed precision 下提高稳定性。
4. 微调时减少灾难性遗忘风险。

面试回答：

```text
Warmup 是为了提高训练初期稳定性。刚开始 optimizer 的动量和二阶矩估计还不可靠，直接使用大学习率容易 loss spike。Warmup 让学习率逐渐升高，使梯度统计和参数更新更平滑，所以在大模型预训练、SFT 和 RLHF 中都很常见。
```

## 10.10 梯度裁剪为什么能稳定训练

梯度裁剪限制梯度范数：

```text
if ||g|| > c:
    g = c * g / ||g||
```

它不会改变小梯度，只在梯度过大时缩放。

作用：

1. 防止极端 batch 导致参数大幅跳动。
2. 缓解 exploding gradients。
3. 提高 mixed precision 训练稳定性。
4. 在 RLHF 等高噪声阶段尤其有用。

面试回答：

```text
梯度裁剪通过限制梯度范数，避免少数异常 batch 造成过大的参数更新。它不是让模型学得更快，而是减少训练不稳定和 loss spike。在大模型训练、长序列训练和 RLHF 中，梯度噪声更大，因此裁剪常作为稳定性手段。
```

## 10.11 为什么归一化有助于训练

归一化的作用是控制激活或特征尺度，使优化更稳定。

Transformer 中常见 LayerNorm 或 RMSNorm。

归一化带来的好处：

1. 降低不同层激活尺度漂移。
2. 改善梯度传播。
3. 减少学习率敏感性。
4. 让深层网络更容易训练。

Pre-LN Transformer 把 norm 放在子层之前，通常比 Post-LN 更稳定，尤其适合深层大模型。

面试回答：

```text
归一化通过控制激活尺度，让深层网络的优化更稳定。Transformer 中 LayerNorm 或 RMSNorm 能减少层间尺度漂移，改善梯度传播。大模型通常采用 Pre-LN 或 RMSNorm，因为深层训练更稳定，对学习率也不那么敏感。
```

## 10.12 SVD 和低秩近似是什么

SVD 将矩阵分解为：

```text
W = U Sigma V^T
```

如果只保留最大的 r 个奇异值，可以得到低秩近似：

```text
W_r = U_r Sigma_r V_r^T
```

直觉：矩阵中很多变化方向不一样重要，最大奇异值对应最主要的方向。

低秩近似的意义：

1. 压缩参数。
2. 提取主要结构。
3. 降低计算和存储。
4. 解释 LoRA 等参数高效微调方法。

面试回答：

```text
SVD 把矩阵分解成正交方向和奇异值。保留最大的 r 个奇异值，就得到最佳低秩近似之一。直觉是矩阵变化主要集中在少数重要方向上。大模型中低秩思想可以用于压缩、分析权重更新，也可以解释 LoRA 为什么只学习低秩增量。
```

## 10.13 LoRA 为什么是低秩

LoRA 不直接全量更新权重矩阵 W，而是学习一个低秩增量：

```text
W' = W + Delta W
Delta W = B A
```

其中：

```text
B in R^{d_out x r}
A in R^{r x d_in}
r << min(d_in, d_out)
```

所以 Delta W 的 rank 不超过 r。

为什么有效：

1. 下游任务需要的更新可能集中在少数方向。
2. 预训练模型已经有强表达能力，不需要全量改动。
3. 低秩更新减少显存和可训练参数。
4. 多任务适配时可以保存不同 LoRA adapter。

面试回答：

```text
LoRA 低秩是因为它把权重增量写成两个小矩阵的乘积 Delta W=BA，其中中间维度 r 很小，所以增量矩阵的秩最多是 r。它假设微调所需的参数变化位于低维子空间。这样可以冻结原模型，只训练少量 adapter 参数，显著降低显存和存储成本。
```

追问：为什么不是所有任务都适合很小的 r？

回答：

```text
如果任务和预训练分布差异很大，或者需要改变很多能力方向，低秩子空间可能不够表达，r 太小会欠拟合。实际要根据任务复杂度、数据规模、目标层选择和效果成本 trade-off 调参。
```

## 10.14 Attention 中点积为什么要除以 sqrt(d)

Attention score 是：

```text
score = QK^T / sqrt(d_k)
```

如果 q 和 k 的每个维度方差近似为 1，那么点积 q·k 的方差大约随 d_k 增长。

维度越高，点积数值越大，softmax 越容易饱和，导致梯度变小。

除以 sqrt(d_k) 可以让 score 的尺度更稳定。

面试回答：

```text
Q 和 K 的点积会随着维度增加而方差变大。如果不缩放，softmax 输入过大，分布容易变得极端，梯度变小。除以 sqrt(d_k) 是为了保持 attention score 的尺度稳定，使 softmax 不容易饱和，从而训练更稳定。
```

## 10.15 为什么残差连接有助于深层网络

残差连接形式：

```text
y = x + F(x)
```

作用：

1. 提供信息直通路径。
2. 改善梯度传播。
3. 让网络更容易学习增量变换。
4. 缓解深层退化问题。

在 Transformer 中，每个 attention block 和 MLP block 周围都有 residual connection。

面试回答：

```text
残差连接让层学习相对输入的增量，而不是从零学习完整映射。它提供了信息和梯度的直通路径，使深层网络更容易优化。Transformer 能堆到很多层，残差连接和归一化是关键稳定性组件。
```

## 10.16 Bias-variance trade-off 如何理解

Bias 是模型假设和真实规律之间的系统性偏差。

Variance 是模型对训练数据扰动的敏感程度。

直觉：

1. 模型太简单，bias 高，容易欠拟合。
2. 模型太复杂且数据不足，variance 高，容易过拟合。
3. 正则化、更多数据、早停和数据增强可以影响二者平衡。

LLM 场景：

1. 预训练大模型容量很大，通常靠海量数据和正则化控制泛化。
2. 小数据 SFT 容易过拟合风格和格式。
3. RLHF 过强可能导致 reward hacking 或分布变窄。

面试回答：

```text
Bias-variance trade-off 描述欠拟合和过拟合之间的平衡。Bias 高说明模型表达不足，variance 高说明模型对训练样本过敏。大模型容量很强，预训练阶段依赖大规模数据获得泛化；小数据微调时更需要正则化、早停和评估集，避免过拟合特定格式或偏好。
```

## 10.17 为什么正则化能缓解过拟合

正则化限制模型复杂度或参数规模。

常见方式：

1. Weight decay。
2. Dropout。
3. Early stopping。
4. Data augmentation。
5. Label smoothing。
6. KL penalty。

数学直觉：正则化相当于在优化目标中加入约束或先验，使模型不要为了拟合训练集噪声而走到过于复杂的解。

LLM 场景：

1. SFT 数据少时可能背答案或过拟合格式。
2. RLHF 用 KL penalty 约束 policy 不偏离 reference。
3. DPO 中 beta 控制偏好优化强度。

面试回答：

```text
正则化通过限制参数规模、输出变化或策略偏移，降低模型对训练噪声的拟合。它可以看成对解空间加约束，也可以看成引入先验。大模型后训练中，KL penalty、weight decay、early stopping 和数据去重都可以看成控制过拟合或分布漂移的手段。
```

## 10.18 Bayes 公式在大模型中有什么用

Bayes 公式：

```text
P(A | B) = P(B | A) P(A) / P(B)
```

直觉：看到证据 B 后，更新对假设 A 的信念。

LLM 相关场景：

1. 生成模型可以看成根据上下文更新下一个 token 分布。
2. RAG 中根据 query 和检索证据更新答案可信度。
3. 分类任务中 posterior 取决于 likelihood 和 prior。
4. 不确定性估计中关注模型 confidence 是否校准。

面试回答：

```text
Bayes 公式描述如何用新证据更新信念。它由先验、似然和后验组成。大模型中，虽然 Transformer 不一定显式做 Bayes 推断，但很多问题可以用贝叶斯视角理解，比如根据上下文更新 token 概率、根据检索证据更新答案可信度，以及做不确定性和校准分析。
```

## 10.19 MLE 和 MAP 有什么区别

MLE 最大化数据似然：

```text
theta_MLE = argmax_theta P(D | theta)
```

MAP 最大化后验：

```text
theta_MAP = argmax_theta P(D | theta) P(theta)
```

区别：MAP 多了参数先验。

如果取 log：

```text
log P(D | theta) + log P(theta)
```

其中 log P(theta) 常可理解为正则项。

面试回答：

```text
MLE 只最大化数据似然，MAP 最大化后验，相当于在似然之外加入参数先验。取 log 后，先验项常表现为正则化。例如高斯先验对应 L2 正则。语言模型预训练通常从 MLE 角度理解，而微调中的正则、KL 约束也可以从先验或约束角度理解。
```

## 10.20 DPO loss 如何理解

DPO 用偏好数据直接优化 policy，不显式训练 reward model 后再跑 PPO。

对于同一个 prompt，有 preferred response y_w 和 rejected response y_l。

DPO 关注 policy 相对 reference 的 log probability 差：

```text
log pi(y_w|x) - log pi_ref(y_w|x)
```

和：

```text
log pi(y_l|x) - log pi_ref(y_l|x)
```

目标是让 preferred 的相对优势高于 rejected。

简化理解：

```text
increase relative probability of chosen answer
decrease relative probability of rejected answer
stay close to reference through implicit KL control
```

面试回答：

```text
DPO 可以理解为直接从偏好对中学习 policy。它比较 chosen 和 rejected response 在当前 policy 下相对 reference policy 的 log probability 差，希望 chosen 的相对优势更大。相比 PPO，DPO 不需要显式训练 reward model 再做 RL rollout，形式更像一个带 reference 约束的分类损失。
```

追问：DPO 里的 beta 控制什么？

回答：

```text
beta 会缩放 chosen/rejected 的 log-ratio margin，并间接影响 policy 相对 reference 的偏离强度。不同推导里它也常被解释成 KL 约束或温度相关超参，所以不要只说“越大越好”。实际需要结合验证集、KL、输出质量和安全指标调节。
```

## 10.21 PPO 中 advantage 是什么

Advantage 衡量某个动作或轨迹比预期好多少：

```text
A = return - value
```

如果 A > 0，说明比 baseline 好，应提高概率。

如果 A < 0，说明比 baseline 差，应降低概率。

在 RLHF 中：

1. Policy 是语言模型。
2. Action 是 token。
3. Reward 来自 reward model 或规则。
4. Value model 估计期望回报。
5. Advantage 指导 policy 更新方向。

面试回答：

```text
Advantage 表示实际回报相对 value baseline 好多少。它的作用是降低 policy gradient 方差，并告诉模型哪些 token 或回答应该提高概率。在 RLHF/PPO 中，reward model 给回答打分，value model 提供 baseline，advantage 为正的生成路径会被强化，为负的会被抑制。
```

## 10.22 PPO 为什么需要 clipping

PPO 使用概率比值：

```text
r_t = pi_new(a_t|s_t) / pi_old(a_t|s_t)
```

如果 policy 一次更新太大，训练会不稳定。

Clipping 限制 r_t 的范围：

```text
clip(r_t, 1 - eps, 1 + eps)
```

直觉：不要让新 policy 相对旧 policy 改得太猛。

LLM 场景：RLHF 中如果 policy 偏离 reference 或 old policy 太快，可能出现胡说、格式崩坏、reward hacking 或安全退化。

面试回答：

```text
PPO clipping 是为了限制 policy 每次更新幅度。它通过裁剪新旧 policy 概率比，避免某些高 reward 样本导致模型概率突变。在 RLHF 中这很重要，因为语言模型一旦偏离 reference 太远，可能出现 reward hacking、语言质量下降和安全回归。
```

## 10.23 统计显著和实际显著有什么区别

统计显著回答：观察到的差异是否不像随机噪声。

实际显著回答：这个差异是否值得行动。

例如：

```text
win rate +0.2%, p-value < 0.05
latency +30%, cost +25%
```

这可能统计显著，但未必值得上线。

面试回答：

```text
统计显著说明差异不太像随机波动，实际显著说明差异是否有业务和工程价值。大模型上线不能只看 p-value，还要看 effect size、置信区间、成本、延迟、安全指标和分层用户影响。显著但收益很小、护栏指标恶化的模型不应直接上线。
```

## 10.24 p-value 和置信区间怎么解释

p-value：在零假设成立时，观察到当前或更极端结果的概率。

置信区间：指标估计的不确定范围。

常见错误：

1. p-value 不是“新模型无效的概率”。
2. p-value 不是“新模型更好的概率”。
3. 95% CI 不是说某个固定真值有 95% 概率落在这个具体区间。
4. 显著不等于一定能上线。

面试回答：

```text
p-value 表示零假设成立时，观察到当前或更极端结果的概率。置信区间表达效果大小的不确定范围。模型评估时我不会只看 p-value，而会同时报告点估计、置信区间、样本量、effect size 和实验设计。
```

## 10.25 Bootstrap 为什么适合大模型评估

大模型评估指标常常复杂，很难写解析方差。

例如：

1. Pairwise win rate。
2. LLM judge score。
3. RAG faithfulness。
4. Human eval rubric。
5. Safety violation rate。

Bootstrap 通过重复有放回抽样估计指标分布。

面试回答：

```text
Bootstrap 适合大模型评估，因为很多指标不是简单均值，解析方差不好推。它通过对评估样本有放回重采样，重复计算指标，得到指标分布和置信区间。需要注意的是，bootstrap 只能估计抽样不确定性，不能修复评估集偏差或 judge 偏差。
```

## 10.26 为什么 pairwise eval 常比单独打分更稳

Pairwise eval 让两个模型在同一个 prompt 下直接比较。

优点：

1. 控制样本难度。
2. 标注者更容易判断相对好坏。
3. 比绝对分数更稳定。
4. 适合开放生成任务。

缺点：

1. Tie 处理会影响结果。
2. 顺序偏差需要随机化。
3. 需要做显著性分析。
4. Judge 仍可能有 verbosity bias。

面试回答：

```text
Pairwise eval 在同一 prompt 下比较两个回答，能控制样本难度，通常比单独打绝对分更稳定。它适合开放生成任务，因为标注者判断 A/B 谁更好往往比给绝对分容易。但要处理 tie、随机化展示顺序，并报告 win rate 的置信区间。
```

## 10.27 如果模型分数提升 0.5%，你如何判断是否可信

回答框架：

1. 看样本量和置信区间。
2. 做 paired evaluation。
3. 做显著性检验或 bootstrap。
4. 看分层结果。
5. 检查是否多重比较。
6. 做错误分析。
7. 看成本、延迟和安全护栏。
8. 必要时做线上 A/B。

面试回答：

```text
我不会只看 0.5% 的点估计。首先看评估集大小、方差和置信区间，最好用 paired evaluation 或 paired bootstrap 判断差异是否稳定。然后看分层结果和错误类型，确认不是某些样本偶然推动。最后结合成本、延迟、安全护栏和线上 A/B，再决定是否上线。
```

## 10.28 如何解释 embedding 相似度

Embedding 把文本映射到向量空间。相似度常用 cosine similarity：

```text
cos(a, b) = a dot b / (||a|| ||b||)
```

直觉：

1. 方向越接近，语义越相似。
2. Cosine 关注方向，不直接关注向量长度。
3. Dot product 同时受方向和长度影响。

LLM 场景：

1. RAG 检索。
2. 去重。
3. 聚类。
4. 语义匹配。
5. 评估答案相似性。

面试回答：

```text
Embedding 相似度通常用向量夹角衡量，cosine similarity 越高表示方向越接近，语义上往往越相似。它适合 RAG 检索和语义匹配。但 embedding 相似不等于事实正确，尤其在 RAG 中还要结合 reranker、引用验证和答案 faithfulness。
```

## 10.29 为什么高维空间中的距离会失效

高维空间有一些反直觉现象：

1. 点与点之间距离趋于接近。
2. 最近邻和最远邻差距变小。
3. 数据变稀疏。
4. 噪声维度会干扰相似度。

这叫 curse of dimensionality 的一部分。

LLM 场景：embedding 检索不能只依赖粗糙向量距离，通常还需要：

1. 更好的 embedding model。
2. 归一化。
3. reranker。
4. metadata filter。
5. chunk 策略。
6. 人工评估和线上指标。

面试回答：

```text
高维空间中数据稀疏，距离容易集中，最近邻不一定真正语义相关。Embedding 检索虽然有效，但不能迷信向量距离。实际 RAG 系统通常结合归一化、reranker、过滤条件和人工评估，来缓解高维相似度的不可靠性。
```

## 10.31 高频快问快答

### 10.31.1 Cross entropy 为什么能训练分类模型

回答：

```text
Cross entropy 等价于最大化正确标签的 log likelihood。对 softmax logits 的梯度是 p-y，会提高正确类别概率、降低错误类别概率，因此适合分类和 next-token prediction。
```

### 10.31.2 KL 为什么非负

回答：

```text
KL 非负来自 Gibbs inequality。直觉上，用错误分布 q 编码真实分布 p，不会比用 p 自己编码更省。KL 为 0 当且仅当两个分布相同。
```

### 10.31.3 为什么 log probability 要相加

回答：

```text
序列概率是条件概率的乘积，取 log 后乘积变求和，数值更稳定，也方便优化。语言模型训练最大化 sum log p(x_t|x_<t)。
```

### 10.31.4 Temperature 对 softmax 有什么影响

回答：

```text
Temperature 会缩放 logits。温度低时分布更尖锐，模型更确定；温度高时分布更平滑，输出更多样。采样时 temperature 影响多样性和风险，但不会改变模型参数。
```

### 10.31.5 Label smoothing 有什么作用

回答：

```text
Label smoothing 把 one-hot 标签变得更平滑，避免模型对训练标签过度自信。它能改善校准和泛化，但在语言模型中要谨慎，因为过强 smoothing 可能降低对真实 token 的拟合。
```

### 10.31.6 Weight decay 和 L2 正则一样吗

回答：

```text
在普通 SGD 下二者常等价，但在 Adam 这类自适应优化器中不完全等价。AdamW 使用 decoupled weight decay，避免 weight decay 被自适应梯度缩放影响。
```

### 10.31.7 为什么 batch size 会影响训练

回答：

```text
Batch size 影响梯度估计噪声。小 batch 噪声大，可能有正则化效果但不稳定；大 batch 梯度更稳定、吞吐更高，但可能需要调学习率和 warmup。大模型训练常关注 global batch size 和 token batch size。
```

### 10.31.8 为什么要做 learning rate decay

回答：

```text
训练后期需要更小步长来收敛到更稳定的区域。学习率衰减可以减少参数震荡，提高最终 loss 和泛化表现。大模型常用 cosine decay、linear decay 或 constant with cooldown。
```

### 10.31.9 为什么 checkpoint averaging 或 EMA 有时有效

回答：

```text
它们可以看成在参数空间做平滑，减少单个 checkpoint 的随机波动。EMA 对近期参数赋更大权重，可能得到更稳定的模型。但大模型中是否使用要看训练框架、存储成本和实际验证效果。
```

### 10.31.10 为什么 LLM judge 需要校准

回答：

```text
LLM judge 有位置偏差、长度偏差、自偏好和 prompt 敏感性。校准的目标是确认 judge 分数和人类判断一致，并控制偏差。通常要做人工标注对照、顺序随机化、多 judge prompt 和置信区间分析。
```

## 10.32 常见追问陷阱

### 10.32.1 “最小化 CE 等价于最小化 KL”总是对吗

不总是。它依赖 p 是否固定。如果真实分布 p 固定，则 H(p) 是常数，最小化 H(p,q) 等价于最小化 KL(p||q)。如果 p 也在变，就不能这样简单说。

### 10.32.2 “PPL 低就一定模型好”对吗

不对。PPL 衡量预测下一个 token 的能力，不直接等价于推理、对齐、安全、工具使用和用户满意度。

### 10.32.3 “LoRA 参数少所以一定不如全参微调”对吗

不一定。很多下游任务需要的更新可能低秩，LoRA 可以达到接近全参微调的效果，并显著降低成本。但对于分布差异很大或需要深度改变能力的任务，全参或更高 rank 可能更好。

### 10.32.4 “p-value 小就一定能上线”对吗

不对。上线还要看实际收益、成本、延迟、安全、用户分层和长期影响。

### 10.32.5 “DPO 一定比 PPO 好”对吗

不对。DPO 更简单稳定，不需要在线 RL rollout，但也依赖偏好数据质量和 reference。PPO 在某些需要显式 reward 优化和复杂反馈的场景仍有价值。

## 10.35 本章总结

大模型算法岗的数学面试，重点不是炫技推导，而是把数学概念讲到模型行为和工程决策上。

本章最重要的结论：

1. 交叉熵等于真实熵加 KL，训练中常等价于最小化 KL(data||model)。
2. KL 不对称，方向不同会带来不同优化行为。
3. PPL 是交叉熵的指数形式，但不等于综合能力。
4. Softmax cross entropy 的梯度是 p-y，适合分类和 token prediction。
5. AdamW 的关键是 decoupled weight decay。
6. Warmup、梯度裁剪、归一化和残差都是训练稳定性工具。
7. SVD 和低秩近似是理解 LoRA 的数学基础。
8. Attention 除以 sqrt(d) 是为了控制点积尺度。
9. DPO 可以理解为带 reference 约束的偏好学习目标。
10. 模型评估要同时看显著性、置信区间、effect size 和上线风险。

如果只能记住一句话：数学面试的好答案不是“公式正确”，而是“公式、直觉、LLM 场景和工程取舍同时正确”。
