# 第八章：强化学习数学

强化学习数学是理解 RLHF、PPO、reward model、policy、value、advantage 和 KL penalty 的基础。大模型后训练里，很多候选人知道“RLHF 是用人类反馈训练模型”，但讲不清为什么它是强化学习问题、PPO 在更新什么、value model 有什么用、advantage 为什么能降低方差、为什么要加 KL 约束。

本章不追求完整 RL 教材，而是只讲大模型算法岗最需要的强化学习数学直觉：MDP、policy、reward、return、value、advantage、policy gradient、PPO，以及这些概念如何映射到 RLHF。

## 8.1 为什么 RLHF 需要强化学习

SFT 让模型模仿高质量示范回答，但示范回答不一定覆盖所有可能输出。偏好数据告诉我们：同一个 prompt 下，哪个回答更好。

RLHF 的目标是让模型生成更高 reward 的回答。

这很像强化学习：

1. 模型是 policy。
2. Prompt 是初始状态或上下文。
3. 生成 token 是 action。
4. 完整回答形成 trajectory。
5. Reward model 给完整回答打分。
6. PPO 等算法根据 reward 更新 policy。

面试回答：

```text
RLHF 可以看成语言模型上的强化学习。Policy 是当前语言模型，action 是生成 token，trajectory 是完整回答，reward 来自 reward model 或偏好信号。PPO 用 reward 更新 policy，同时用 KL 约束防止模型偏离 reference 太远。
```

## 8.2 MDP 的基本组成

MDP 是 Markov Decision Process，强化学习中描述决策问题的基本框架。

它通常包含：

1. State：状态。
2. Action：动作。
3. Transition：状态转移。
4. Reward：奖励。
5. Policy：策略。
6. Discount：折扣因子。

在游戏或机器人中，状态可能是环境观测，动作可能是移动或控制指令。

在语言模型中，可以粗略映射为：

1. State：当前 prompt 加已经生成的 tokens。
2. Action：下一个 token。
3. Transition：把 token 追加到上下文。
4. Reward：完整回答由 reward model 打分。
5. Policy：语言模型的 token 分布。

注意：LLM 的 RLHF 和传统 RL 不完全一样。它通常没有复杂外部环境交互，reward 多数在完整回答后给出，而且 policy 已经来自预训练和 SFT，不是从零学习。

## 8.3 Policy：模型如何选择动作

Policy 表示在某个状态下选择动作的概率分布。

在 LLM 中，policy 就是：

```text
pi(next token | current context)
```

每一步生成时，模型根据当前上下文输出词表概率分布。

如果 policy 改变，模型的生成行为也会改变。例如 RLHF 希望 policy 更倾向于生成 helpful、honest、harmless 的回答。

面试回答：

```text
Policy 是状态到动作分布的映射。在 LLM 中，状态是当前上下文，动作是下一个 token，policy 就是语言模型输出的 token 概率分布。RLHF 优化的就是这个 policy，让它更倾向于高偏好的回答。
```

## 8.4 Reward 和 Return

Reward 是某一步或某段 trajectory 的奖励。

Return 是从当前开始累计得到的奖励。

传统 RL 中，每一步都可能有 reward。LLM RLHF 中，reward 通常在完整回答后给出，例如 reward model 对 prompt+response 打一个分数。

这带来两个问题：

1. Reward 很稀疏，只有完整回答后才知道好坏。
2. 很难判断每个 token 对最终 reward 的贡献。

这就是为什么需要 value model 和 advantage 来辅助训练。

## 8.5 Value function：估计未来能拿多少 reward

Value function 用来估计某个状态下未来能获得多少 reward。

直觉上：

```text
value(state) = expected future return from this state
```

在 PPO/RLHF 中，value model 可以帮助估计当前生成过程的基线，降低 policy gradient 的方差。

如果没有 value baseline，单次 reward 的噪声会很大，policy 更新不稳定。

面试回答：

```text
Value function 估计某个状态未来能获得的期望回报。在 RLHF 中，reward 往往只在完整回答后给出，噪声较大。Value model 提供 baseline，帮助计算 advantage，从而降低 policy gradient 方差，让 PPO 更新更稳定。
```

## 8.6 Advantage：比预期好多少

Advantage 衡量一个 action 或 trajectory 比当前预期好多少。

直觉：

```text
advantage = actual return - expected value
```

如果 advantage 为正，说明这个动作比预期好，应该提高它的概率。

如果 advantage 为负，说明比预期差，应该降低它的概率。

面试回答：

```text
Advantage 表示实际回报相对 value baseline 好多少。正 advantage 的 token 或行为应该被增强，负 advantage 应该被抑制。相比直接用 reward，advantage 减去 baseline，可以降低梯度估计方差。
```

## 8.7 Policy gradient 的直觉

Policy gradient 的目标是直接优化 policy，让高 reward 的行为概率变大。

直觉更新方向：

```text
increase log probability of actions with positive advantage
decrease log probability of actions with negative advantage
```

在语言模型里，如果某个回答得到高 reward，那么生成这个回答的 token 序列概率会被整体提高。

但如果只这样做，模型可能出现问题：

1. 为了 reward 牺牲语言质量。
2. 学会 reward hacking。
3. 输出变得重复或极端。
4. 偏离 SFT 模型太远。

因此 RLHF 通常加入 KL 约束。

## 8.8 PPO 的核心直觉

PPO 是 Proximal Policy Optimization，核心是不要让 policy 一次更新太大。

它通过比较新旧 policy 对同一 action 的概率比值，限制更新幅度。

直觉：

1. 如果新 policy 和旧 policy 差不多，可以更新。
2. 如果新 policy 变化太大，就 clip 或惩罚。
3. 这样能避免一次更新把模型行为推得太远。

面试回答：

```text
PPO 的核心思想是限制 policy 每次更新幅度。它用新旧 policy 的概率比值衡量变化，如果变化太大就 clip，避免训练不稳定。在 RLHF 中，PPO 用 reward model 给出的奖励更新语言模型，同时配合 KL penalty 防止偏离 reference model。
```

## 8.9 KL penalty 在 RLHF 中的作用

如果只最大化 reward，模型可能利用 reward model 漏洞。

常见风险：

1. Reward hacking。
2. 输出模板化。
3. 语言质量退化。
4. 过度拒答。
5. 偏离原模型能力。

KL penalty 的作用是让 policy 不要偏离 reference model 太远。

直觉目标：

```text
maximize reward - beta * KL(policy || reference)
```

`beta` 控制约束强度。

1. `beta` 太小，模型可能乱跑。
2. `beta` 太大，模型学不动。

面试回答：

```text
RLHF 中 KL penalty 用来约束 policy 接近 reference model。Reward model 只是人类偏好的 proxy，如果无约束最大化 reward，模型可能 reward hacking 或语言退化。KL 约束能保持基础语言能力和行为稳定性，但 beta 太大也会限制学习。
```

## 8.10 RLHF 完整流程的数学视角

RLHF 通常包括：

1. 训练或准备 SFT model。
2. 收集同一 prompt 下多个回答的人类偏好。
3. 训练 reward model，让 chosen 得分高于 rejected。
4. 用 policy 生成回答。
5. Reward model 给回答打分。
6. 加上 KL penalty 得到最终训练信号。
7. 用 PPO 更新 policy。
8. 用人工评估和安全评估检查效果。

数学视角看，RLHF 不是“人类实时给奖励”，而是用偏好数据训练一个 reward proxy，再优化 policy。

这也解释了 RLHF 的风险：如果 reward proxy 错了，policy 会放大这个错误。

## 8.11 Reward hacking

Reward hacking 指模型找到提高 reward 的捷径，但输出不符合真实人类意图。

LLM 中可能表现为：

1. 输出冗长但空泛。
2. 迎合 reward model 喜欢的格式。
3. 看似安全但过度拒答。
4. 编造引用但语气可信。
5. 利用 judge 或 reward model 的盲区。

缓解方法：

1. 高质量偏好数据。
2. Reward model holdout。
3. KL 约束。
4. 人工评估。
5. 安全和能力回归。
6. Red teaming。

面试回答：

```text
Reward hacking 是模型优化 reward proxy 时找到捷径，但真实质量没有提升。RLHF 必须配合 KL 约束、holdout 偏好集、人工评估、安全评估和 red teaming，不能只看 reward model 分数。
```

## 8.12 DPO 和 RLHF 的关系

DPO 不显式训练 reward model，也不跑 PPO，但它和带 KL 约束的偏好优化有理论联系。

RLHF：

```text
preference data -> reward model -> PPO optimize policy
```

DPO：

```text
preference data -> directly optimize policy
```

DPO 直接利用 chosen/rejected 的 log probability 构造 loss，让 policy 相比 reference 更偏好 chosen。

面试回答：

```text
DPO 可以看作把带 KL 约束的偏好优化改写成直接基于 preference pair 的监督式目标。它省掉 reward model 和 PPO loop，训练更简单稳定，但仍依赖高质量偏好数据、reference model 和 beta 设置。
```

## 8.13 RLHF 和普通 RL 的区别

RLHF 和传统 RL 有明显不同：

1. Policy 不是从零开始，而是预训练/SFT 后的语言模型。
2. 环境通常不是复杂物理环境，而是 token 生成过程。
3. Reward 来自 learned reward model，不是真实环境奖励。
4. Reward 通常在完整回答后给出。
5. 需要强 KL 约束保持语言能力。
6. 评估不能只看 reward，还要看人类偏好、安全和能力回归。

面试回答：

```text
RLHF 不是传统意义上从零探索环境。它是在已有 SFT 模型上，用人类偏好训练 reward model，再用 PPO 等方法微调 policy。Reward 是 learned proxy，而且通常很稀疏，所以更需要 KL 约束、人工评估和安全回归。
```

## 8.14 面试高频问题

### 8.14.1 PPO 在 RLHF 中做什么

回答：

```text
PPO 用 reward model 给出的奖励更新 policy model，让模型更倾向于生成高 reward 回答。它通过限制新旧 policy 的变化幅度提高训练稳定性，并通常配合 KL penalty 防止模型偏离 reference 太远。
```

### 8.14.2 Value model 有什么用

回答：

```text
Value model 估计当前状态未来能获得的期望 reward，作为 baseline 帮助计算 advantage。这样可以降低 policy gradient 的方差，使 PPO 更新更稳定。
```

### 8.14.3 Advantage 是什么

回答：

```text
Advantage 表示实际回报比当前 value 预期好多少。正 advantage 表示行为比预期好，应该提高概率；负 advantage 表示比预期差，应该降低概率。它比直接用 reward 更稳定，因为减去了 baseline。
```

### 8.14.4 为什么 RLHF 不能只看 reward score

回答：

```text
Reward model 只是人类偏好的 proxy，可能有偏差和漏洞。Policy 如果过度优化 reward，可能 reward hacking，导致分数升高但真实质量下降。因此必须结合人工评估、安全评估、能力回归和线上反馈。
```

## 8.15 常见误区

1. 认为 RLHF 是人类实时给模型奖励。
2. 认为 PPO 是训练 reward model 的方法。
3. 只说 PPO，不讲 policy、reward、value、advantage。
4. 认为 reward score 越高模型越好。
5. 忽略 KL penalty。
6. 把 DPO 说成完全没有约束。
7. 不知道 RLHF 通常发生在 SFT 之后。

## 8.17 本章总结

强化学习数学帮助我们理解 RLHF 背后的优化结构。

本章最重要的结论：

1. LLM 的 policy 是 token 分布。
2. 生成 token 可以看作 action。
3. 完整回答可以看作 trajectory。
4. Reward model 给回答提供奖励信号。
5. Value model 估计期望回报。
6. Advantage 衡量比预期好多少。
7. Policy gradient 增强高 advantage 行为。
8. PPO 限制 policy 更新幅度。
9. KL penalty 防止 policy 偏离 reference。
10. RLHF 不能只看 reward score，必须防 reward hacking。

面试中讲 RLHF，不要只背流程。要能把 policy、reward、value、advantage、PPO、KL 和 reward hacking 串成一个完整优化故事。
