# 第三章：信息论

信息论是理解大模型 loss、perplexity、KL 约束、偏好优化和评估指标的关键。概率论告诉我们模型输出的是概率分布，信息论进一步告诉我们：这个分布有多不确定，模型预测真实 token 需要付出多少代价，两个分布差异有多大，以及为什么低 loss 不一定等于好模型。

很多面试回答会卡在这里：知道 cross entropy 是分类损失，但讲不清它和最大似然的关系；知道 KL 散度，但讲不清为什么 RLHF 需要 KL penalty；知道 perplexity，但不知道它为什么受 tokenizer 影响；知道 DPO 有 reference model，但讲不清它为什么像一种隐式 KL 约束。

本章目标：讲清熵、交叉熵、KL 散度、互信息、perplexity，以及它们如何服务于 LLM 训练、对齐、解码、评估和安全。

## 3.1 为什么大模型需要信息论

语言模型每一步输出一个概率分布。信息论关心几个问题：

1. 这个分布有多不确定？这是熵。
2. 用模型分布解释真实数据要付出多少代价？这是交叉熵。
3. 模型分布和目标分布差多远？这是 KL 散度。
4. 模型平均每一步有多少有效候选？这是 perplexity 的直觉。
5. 一个变量知道后能减少另一个变量多少不确定性？这是互信息。

这些问题直接出现在大模型中：

1. Pretraining 用 cross entropy loss。
2. Perplexity 衡量语言建模困惑程度。
3. RLHF 用 KL penalty 限制 policy 偏离 reference。
4. DPO 中 reference model 起到类似锚点作用。
5. Mutual information 直觉可以帮助理解 representation 和 modality alignment。

## 3.2 信息量：越意外，信息越多

一个事件的信息量和它发生的概率有关。

直觉是：越罕见的事件，一旦发生，信息量越大。

常见定义是：

```text
information(x) = -log P(x)
```

如果一个 token 的概率很高，那么看到它并不意外，信息量小。如果一个 token 的概率很低，那么看到它很意外，信息量大。

这和语言模型 loss 直接相关。真实 token 的 loss 通常就是：

```text
loss = -log P(real token | context)
```

如果模型给真实 token 的概率高，loss 小；如果给真实 token 的概率低，loss 大。

面试回答：

```text
信息量可以理解为事件的意外程度，通常写成负 log 概率。语言模型训练中，真实 token 的 negative log likelihood 就是模型看到这个 token 的信息代价。真实 token 概率越高，loss 越低；概率越低，loss 越高。
```

## 3.3 熵：分布的不确定性

熵衡量一个概率分布的不确定性。

离散分布的熵可以写成：

```text
H(P) = - sum_x P(x) log P(x)
```

直觉上：

1. 分布越集中，熵越低。
2. 分布越均匀，熵越高。

例如一个模型在某个位置非常确定：

```text
Paris 0.99
other tokens 0.01
```

这个分布熵很低。

如果模型很不确定：

```text
token A 0.20
token B 0.18
token C 0.17
...
```

熵就更高。

注意：高熵不一定坏。开放式创作任务本来就可能有多个合理答案；事实问答中，如果模型应该确定却高熵，可能说明知识不足或上下文不够。

面试回答：

```text
熵衡量概率分布的不确定性。分布越集中，熵越低；分布越平，熵越高。在语言模型里，某些位置应该低熵，比如事实补全；某些开放任务可以高熵，因为有多个合理续写。熵本身不是质量指标，要结合任务解释。
```

## 3.4 交叉熵：用模型分布解释真实分布的代价

交叉熵衡量：如果真实分布是 `P`，但我们用模型分布 `Q` 来编码或预测，平均代价是多少。

可以写成：

```text
H(P, Q) = - sum_x P(x) log Q(x)
```

在监督学习中，真实标签通常是 one-hot。也就是说，真实 token 的概率是 1，其他 token 是 0。

此时交叉熵会简化为：

```text
loss = -log Q(real token)
```

这就是语言模型常见的 token-level loss。

面试回答：

```text
交叉熵可以理解为用模型分布 Q 去解释真实分布 P 的平均代价。在 one-hot 标签下，真实类别概率为 1，所以交叉熵等于真实 token 的 negative log probability。语言模型最小化 cross entropy，本质上是在提高真实下一个 token 的概率。
```

## 3.5 交叉熵和最大似然的关系

最大似然是让训练数据在模型下的概率尽可能高。

对语言模型来说，目标是最大化：

```text
product_t Q(x_t | previous tokens)
```

取 log 后变成：

```text
sum_t log Q(x_t | previous tokens)
```

训练通常最小化负 log probability：

```text
- sum_t log Q(x_t | previous tokens)
```

这和 one-hot 标签下的交叉熵一致。

面试回答：

```text
最大似然是最大化训练数据中真实 token 的概率。取 log 后，最大化 log probability 等价于最小化 negative log likelihood。在 one-hot 标签下，negative log likelihood 和 cross entropy 是同一个形式，所以语言模型训练常说最小化交叉熵。
```

## 3.6 KL 散度：两个分布的差异

KL 散度衡量用一个分布 `Q` 近似另一个分布 `P` 时的额外代价。

常见形式：

```text
KL(P || Q) = sum_x P(x) log(P(x) / Q(x))
```

直觉上，如果 `Q` 和 `P` 很接近，KL 小；如果 `Q` 在 `P` 认为重要的地方给了很低概率，KL 会变大。

KL 有几个重要性质：

1. KL 不小于 0。
2. KL 等于 0 表示两个分布相同。
3. KL 不是对称的，`KL(P || Q)` 和 `KL(Q || P)` 通常不同。
4. KL 不是严格意义上的距离。

为什么 KL 不对称重要？

因为 `KL(P || Q)` 更强调 `Q` 不要漏掉 `P` 的高概率区域；而 `KL(Q || P)` 更强调 `Q` 不要把概率放到 `P` 不支持的区域。不同方向会导致不同优化行为。

面试回答：

```text
KL 散度衡量两个概率分布的差异，可以理解为用 Q 近似 P 时多付出的信息代价。KL 非负，但不对称，所以不是普通距离。在大模型里，KL 常用于约束一个新 policy 不要偏离 reference model 太远。
```

## 3.7 交叉熵、熵和 KL 的关系

交叉熵可以分解为真实分布自身的不确定性，加上模型分布和真实分布的差异：

```text
H(P, Q) = H(P) + KL(P || Q)
```

这个关系非常重要。

如果真实数据分布 `P` 固定，那么 `H(P)` 是常数。最小化交叉熵，就等价于最小化 `KL(P || Q)`，也就是让模型分布 `Q` 接近真实分布 `P`。

面试回答：

```text
交叉熵可以分成两部分：真实分布自身的熵，以及真实分布和模型分布之间的 KL。数据分布固定时，真实熵是常数，所以最小化交叉熵等价于让模型分布更接近真实数据分布。
```

## 3.8 Perplexity：平均每步有多少有效选择

Perplexity 通常是 average negative log likelihood 的指数形式。

如果平均 token loss 是：

```text
loss_avg
```

那么 perplexity 是：

```text
perplexity = exp(loss_avg)
```

直觉上，perplexity 可以理解为模型平均每一步有多少个有效候选。

例如：

1. Perplexity 越低，模型越不困惑。
2. Perplexity 越高，模型越难预测真实文本。

但 perplexity 有很多边界：

1. 不同 tokenizer 下不一定可比。
2. 低 perplexity 不一定代表聊天质量好。
3. 对齐后的模型可能牺牲部分 perplexity，但用户体验更好。
4. 如果验证集被污染，perplexity 会失真。
5. Perplexity 不能直接衡量事实性、安全性和工具调用能力。

面试回答：

```text
Perplexity 是平均 token loss 的指数，可以理解为模型平均每一步的有效候选数。它适合衡量语言建模能力，但不同 tokenizer、数据集和评估设置下不一定可比。低 perplexity 不保证模型更 helpful、更 factual 或更安全。
```

## 3.9 为什么低 loss 不一定代表好模型

低 cross entropy loss 表示模型更擅长预测验证集 token，但这不等于所有能力更强。

原因包括：

1. 验证集不代表真实用户任务。
2. 数据污染让 loss 虚低。
3. 模型可能学到模板和风格，而不是推理能力。
4. SFT/RLHF 后的交互质量不完全由 pretraining loss 决定。
5. 安全、诚实、引用和工具调用不是单纯 next-token loss 能完全衡量。

面试回答：

```text
Loss 低说明模型更好地拟合了评估数据的 token 分布，但不保证真实任务能力更强。如果评估集不代表目标任务、存在污染，或者我们关心的是事实性、安全和工具调用，单纯 loss 就不够。大模型评估需要结合 benchmark、人评、业务任务、安全评估和 bad case 分析。
```

## 3.10 RLHF 中的 KL penalty

RLHF 中常见目标会加入 KL penalty，约束 policy model 不要偏离 reference model 太远。

直觉上，reward model 会给回答打分，但如果只最大化 reward，policy 可能利用 reward model 漏洞，出现 reward hacking、语言退化或过度拒答。

KL penalty 的作用是：

1. 限制 policy 分布偏离 reference。
2. 保持语言质量和基础能力。
3. 降低 reward hacking 风险。
4. 让优化过程更稳定。

可以直观写成：

```text
objective = reward - beta * KL(policy || reference)
```

其中 `beta` 控制约束强度。

1. `beta` 太小，约束弱，模型可能乱跑。
2. `beta` 太大，约束强，模型学不动。

面试回答：

```text
RLHF 中 KL penalty 用来限制当前 policy 不要偏离 reference model 太远。因为 reward model 只是人类偏好的 proxy，如果无约束地最大化 reward，模型可能 reward hacking 或语言质量退化。KL 太弱风险大，KL 太强又会限制学习，所以 beta 需要调节。
```

## 3.11 DPO 和 KL 约束的关系

DPO 不显式训练 reward model，也不跑 PPO，但它和带 KL 约束的偏好优化有理论联系。

DPO 使用 preference pair：

```text
prompt, chosen response, rejected response
```

它比较 policy model 和 reference model 对 chosen/rejected 的相对 log probability 差异。

直觉上，DPO 希望 policy 相比 reference 更偏好 chosen，而不是 rejected。

Reference model 的作用类似锚点：

1. 防止 policy 任意漂移。
2. 提供“偏离原模型多少”的参照。
3. 让偏好优化不只是简单提高 chosen 概率。

面试回答：

```text
DPO 可以看作把带 KL 约束的偏好优化改写成直接基于 chosen/rejected log probability 的目标。它不显式加入 PPO 式 KL penalty，但 reference model 在目标中起到锚点作用，限制 policy 的偏好变化不能脱离原模型太远。因此 DPO 更简单，但仍依赖 reference、beta 和偏好数据质量。
```

## 3.12 互信息：知道一个变量能减少多少不确定性

互信息衡量两个变量之间共享的信息量。

直觉上，如果知道 `X` 能大幅减少对 `Y` 的不确定性，那么 `X` 和 `Y` 的互信息高。

在大模型和多模态中，互信息直觉很有用：

1. 图文对齐希望图像表示和文本表示共享语义信息。
2. RAG 希望检索文档对答案有高信息量。
3. Prompt 中无关上下文和答案互信息低，可能只是噪声。
4. 表示学习希望 hidden state 保留对任务有用的信息。

面试中不一定要求推导互信息公式，更重要的是直觉。

面试回答：

```text
互信息衡量一个变量能减少另一个变量多少不确定性。多模态对齐可以理解为让图像和文本表示共享更多语义信息；RAG 中好的检索结果应该和答案有高互信息，而不是只在词面相似。
```

## 3.13 信息论视角理解 RAG

RAG 可以看成给模型提供额外证据，降低回答不确定性。

一个好检索结果应该：

1. 和问题相关。
2. 包含回答所需证据。
3. 减少模型对答案的不确定性。
4. 不引入冲突或噪声。

如果检索到的 chunk 和问题词面相似，但不包含答案证据，那么它对答案的信息量低，甚至会干扰模型。

这解释了为什么 RAG 不能只看 embedding similarity，还要看：

1. Recall@k。
2. Rerank 后的 evidence quality。
3. Context precision。
4. Citation accuracy。
5. Faithfulness。

## 3.14 信息论视角理解安全和拒答

安全场景中，信息论也有直觉价值。

例如模型面对高风险请求时，不应该提供会显著降低危险操作门槛的信息。安全拒答不是拒绝所有内容，而是控制输出中高风险信息的粒度。

对于资料不足的问题，模型应该表达不确定性，而不是输出看似高信息量但无依据的细节。

面试回答：

```text
安全回答不是简单减少信息量，而是控制危险信息和无依据信息。对正常教育性请求，模型可以提供高层解释；对高风险可操作请求，应该拒绝危险细节并给安全替代；对证据不足的问题，应表达不确定性，避免生成 unsupported claims。
```

## 3.15 一个最小代码例子：cross entropy

下面用 PyTorch 展示交叉熵的直觉。

```python
import torch
import torch.nn.functional as F

logits = torch.tensor([[2.0, 0.5, -1.0]])
target = torch.tensor([0])

loss = F.cross_entropy(logits, target)
probs = torch.softmax(logits, dim=-1)
manual_loss = -torch.log(probs[0, target[0]])

print(probs)
print(loss.item())
print(manual_loss.item())
```

这里要看懂：

1. `logits` 先经过 softmax 变成概率。
2. `target` 指定真实类别。
3. Cross entropy 等于真实类别概率的负 log。

语言模型中，只是把这个过程扩展到每个 token 位置。

## 3.16 面试高频问题

### 3.16.1 交叉熵和 KL 有什么关系

回答：

```text
交叉熵可以分解为真实分布的熵加上真实分布到模型分布的 KL。数据分布固定时，真实熵是常数，所以最小化交叉熵等价于让模型分布接近真实分布。
```

### 3.16.2 Perplexity 是什么

回答：

```text
Perplexity 是平均 token loss 的指数形式，可以直观理解为模型平均每一步有多少有效候选。它越低说明模型越不困惑，但不同 tokenizer 和数据集下不一定可比，也不能直接代表事实性、安全性和用户体验。
```

### 3.16.3 为什么 RLHF 需要 KL penalty

回答：

```text
Reward model 只是人类偏好的 proxy。如果 policy 无约束地最大化 reward，可能出现 reward hacking、语言质量退化或过度拒答。KL penalty 约束 policy 接近 reference model，在提升偏好的同时保持原有语言能力和行为稳定性。
```

### 3.16.4 DPO 为什么需要 reference model

回答：

```text
Reference model 给 DPO 提供偏好变化的锚点。DPO 不只是让 policy 提高 chosen 的概率，而是让 policy 相对 reference 更偏好 chosen 而不是 rejected。这样可以避免 policy 无限制漂移，也保留了类似 KL 约束的作用。
```

## 3.17 常见误区

1. 把交叉熵只理解成分类 API，不理解 negative log probability。
2. 以为 KL 是对称距离。
3. 以为 perplexity 可以直接比较所有模型。
4. 认为低 loss 就一定更 helpful。
5. 认为 KL penalty 越大越安全。
6. 认为 DPO 没有 reward model 就没有约束。
7. 把互信息当成必须背公式，而不是理解“减少不确定性”的直觉。

## 3.19 本章总结

信息论把概率分布和学习目标联系起来。

本章最重要的结论：

1. 信息量是负 log 概率，真实 token 概率越低，loss 越高。
2. 熵衡量分布不确定性。
3. 交叉熵衡量用模型分布解释真实分布的平均代价。
4. One-hot 标签下，交叉熵就是真实 token 的 negative log probability。
5. KL 衡量分布差异，但不是对称距离。
6. Perplexity 是平均 token loss 的指数形式。
7. RLHF 中 KL penalty 用来限制 policy 偏离 reference。
8. DPO 的 reference model 也起到类似锚点作用。
9. 互信息直觉可以帮助理解多模态对齐、RAG 和表示学习。

面试中讲信息论，要主动连接 cross entropy、MLE、perplexity、KL penalty、DPO、RAG 和安全，而不是只背公式。
