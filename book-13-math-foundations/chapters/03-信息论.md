# 第三章：信息论

信息论是理解大模型 loss、perplexity、KL 约束、偏好优化和评估指标的关键。概率论告诉我们模型输出的是概率分布，信息论进一步告诉我们：这个分布有多不确定，模型预测真实 token 需要付出多少代价，两个分布差异有多大，以及为什么低 loss 不一定等于好模型。

很多面试回答会卡在这里：知道 cross entropy 是分类损失，但讲不清它和最大似然的关系；知道 KL 散度，但讲不清为什么 RLHF 需要 KL penalty；知道 perplexity，但不知道它为什么受 tokenizer 影响；知道 DPO 有 reference model，但讲不清它为什么像一种隐式 KL 约束。

本章目标：讲清熵、交叉熵、KL 散度、互信息、perplexity，以及它们如何服务于 LLM 训练、对齐、解码、评估和安全。

## 0. 本讲资料边界与第二轮精修口径

第二轮精修时，本章按 `WRITING_PLAN.md` 先核对信息论基础定义、主流深度学习框架中的 cross entropy 口径、语言模型 perplexity 评估说明、RLHF 中 reference / KL 约束，以及 DPO 与带 KL 约束偏好优化的关系。

本章聚焦大模型算法岗最常用的信息论工具：

1. 单个 token 的信息量、NLL 和平均 token loss。
2. 熵、交叉熵、KL 散度和三者关系。
3. Perplexity 的定义、直觉和 tokenizer / 数据集边界。
4. 互信息在表示学习、多模态对齐、RAG 和数据选择中的直觉。
5. KL penalty、reference model 和偏好优化中的分布约束。

本章不展开完整编码定理、信道容量、率失真理论、信息瓶颈完整推导、变分推断全套数学或 PPO / DPO 的完整证明。面试中更重要的是：能把公式说回 LLM loss、PPL、RLHF、DPO、RAG 证据质量和安全信息粒度，而不是孤立背公式。

## 3.1 为什么大模型需要信息论

语言模型每一步输出一个概率分布。信息论关心几个问题：

1. 这个分布有多不确定？这是熵。
2. 用模型分布解释真实数据要付出多少代价？这是交叉熵。
3. 模型分布和目标分布差多远？这是 KL 散度。
4. 模型平均每一步有多少有效候选？这是 perplexity 的直觉。
5. 一个变量知道后能减少另一个变量多少不确定性？这是互信息。

这些问题直接出现在大模型中：

1. Pretraining 用 cross entropy loss。
2. Perplexity 衡量语言建模困惑程度。
3. RLHF 用 KL penalty 限制 policy 偏离 reference。
4. DPO 中 reference model 起到类似锚点作用。
5. Mutual information 直觉可以帮助理解 representation 和 modality alignment。

### 3.1.1 关键公式与 LLM 信息论速查

下面默认使用自然对数。此时信息量、loss 和熵的单位可以理解为 nat；如果使用 `log_2`，单位是 bit。大多数深度学习框架里的 cross entropy / NLL 默认按自然对数计算，所以语言模型里常见的 perplexity 写成 `exp(平均 token loss)`。

单个事件的信息量：

$$
I(x) = -\log p(x)
$$

如果事件概率越小，`I(x)` 越大。语言模型里真实 token 概率越低，单 token NLL 越大。

离散分布的熵：

$$
H(p) = -\sum_x p(x)\log p(x)
$$

`H(p)` 衡量分布本身的不确定性。分布越均匀，熵越高；分布越集中，熵越低。

真实分布 `p` 和模型分布 `q` 的交叉熵：

$$
H(p,q) = -\sum_x p(x)\log q(x)
$$

如果标签是 one-hot，真实 token 为 `y`，则交叉熵退化成：

$$
L_{\mathrm{token}} = -\log q(y)
$$

训练集上的平均 token loss 常写成：

$$
L_{\mathrm{avg}} = -\frac{1}{N}\sum_{i=1}^{N}\log p_\theta(x_i \mid c_i)
$$

其中 `c_i` 是第 `i` 个预测位置可见的上下文，`x_i` 是真实下一个 token，`N` 是参与 loss 的 token 数。

Perplexity：

$$
\mathrm{PPL} = \exp(L_{\mathrm{avg}})
$$

它可以粗略理解为平均每个位置的有效候选数，但只有在 tokenizer、语料、截断方式和 loss 口径一致时才适合比较。

KL 散度：

$$
D_{\mathrm{KL}}(p \parallel q) = \sum_x p(x)\log \frac{p(x)}{q(x)}
$$

KL 非负、非对称，不是普通距离。`D_KL(p || q)` 强调 `q` 不要漏掉 `p` 的高概率区域；换方向后优化偏好会变。

交叉熵分解：

$$
H(p,q) = H(p) + D_{\mathrm{KL}}(p \parallel q)
$$

当真实数据分布 `p` 固定时，`H(p)` 对模型参数是常数，所以最小化交叉熵等价于把模型分布 `q` 推近 `p`。

互信息：

$$
I(X;Y) = H(X) - H(X \mid Y)
$$

等价地：

$$
I(X;Y) = \sum_{x,y}p(x,y)\log \frac{p(x,y)}{p(x)p(y)}
$$

互信息衡量知道 `Y` 后能减少多少关于 `X` 的不确定性。RAG 里，一个 chunk 和问题词面相似不够；它还要真正减少答案的不确定性。

RLHF 中常见的 KL 约束直觉：

$$
J(\theta) = \mathbb{E}_{y \sim \pi_\theta} [r(y)] - \beta D_{\mathrm{KL}}(\pi_\theta(\cdot \mid x) \parallel \pi_{\mathrm{ref}}(\cdot \mid x))
$$

`beta` 越大，policy 越不容易远离 reference；`beta` 越小，reward hacking 和语言退化风险更高。

DPO 的核心比较量可以写成：

$$
z = \beta[(\log \pi_\theta(y_w \mid x)-\log \pi_{\mathrm{ref}}(y_w \mid x)) - (\log \pi_\theta(y_l \mid x)-\log \pi_{\mathrm{ref}}(y_l \mid x))]
$$

$$
L_{\mathrm{DPO}} = -\log \sigma(z)
$$

这里 `y_w` 是 chosen response，`y_l` 是 rejected response。这个式子展示了 reference model 为什么是偏好变化的锚点，而不只是一个可有可无的 baseline。

## 3.2 信息量：越意外，信息越多

一个事件的信息量和它发生的概率有关。

直觉是：越罕见的事件，一旦发生，信息量越大。

常见定义是：

$$
I(x) = -\log p(x)
$$

其中 `p(x)` 是事件 `x` 发生的概率，`I(x)` 是观察到 `x` 后获得的信息量。

如果一个 token 的概率很高，那么看到它并不意外，信息量小。如果一个 token 的概率很低，那么看到它很意外，信息量大。

这和语言模型 loss 直接相关。真实 token 的 loss 通常就是：

$$
L_{\mathrm{token}} = -\log p_\theta(y \mid c)
$$

其中 `c` 是上下文，`y` 是真实下一个 token，`p_theta(y | c)` 是模型给真实 token 的条件概率。

如果模型给真实 token 的概率高，loss 小；如果给真实 token 的概率低，loss 大。

面试回答：

```text
信息量可以理解为事件的意外程度，通常写成负 log 概率。语言模型训练中，真实 token 的 negative log likelihood 就是模型看到这个 token 的信息代价。真实 token 概率越高，loss 越低；概率越低，loss 越高。
```

## 3.3 熵：分布的不确定性

熵衡量一个概率分布的不确定性。

离散分布的熵可以写成：

$$
H(p) = -\sum_x p(x)\log p(x)
$$

其中求和遍历所有可能事件或词表 token。概率为 0 的项通常按极限理解为 0。

直觉上：

1. 分布越集中，熵越低。
2. 分布越均匀，熵越高。

例如一个模型在某个位置非常确定：

```text
Paris 0.99
other tokens 0.01
```

这个分布熵很低。

如果模型很不确定：

```text
token A 0.20
token B 0.18
token C 0.17
...
```

熵就更高。

注意：高熵不一定坏。开放式创作任务本来就可能有多个合理答案；事实问答中，如果模型应该确定却高熵，可能说明知识不足或上下文不够。

面试回答：

```text
熵衡量概率分布的不确定性。分布越集中，熵越低；分布越平，熵越高。在语言模型里，某些位置应该低熵，比如事实补全；某些开放任务可以高熵，因为有多个合理续写。熵本身不是质量指标，要结合任务解释。
```

## 3.4 交叉熵：用模型分布解释真实分布的代价

交叉熵衡量：如果真实分布是 `P`，但我们用模型分布 `Q` 来编码或预测，平均代价是多少。

可以写成：

$$
H(p,q) = -\sum_x p(x)\log q(x)
$$

其中 `p` 是真实分布，`q` 是模型分布。交叉熵越低，表示模型在真实分布高概率的位置也给了更高概率。

在监督学习中，真实标签通常是 one-hot。也就是说，真实 token 的概率是 1，其他 token 是 0。

此时交叉熵会简化为：

$$
L_{\mathrm{token}} = -\log q(y)
$$

其中 `y` 是真实类别或真实 token。

这就是语言模型常见的 token-level loss。

面试回答：

```text
交叉熵可以理解为用模型分布 Q 去解释真实分布 P 的平均代价。在 one-hot 标签下，真实类别概率为 1，所以交叉熵等于真实 token 的 negative log probability。语言模型最小化 cross entropy，本质上是在提高真实下一个 token 的概率。
```

## 3.5 交叉熵和最大似然的关系

最大似然是让训练数据在模型下的概率尽可能高。

对语言模型来说，目标是最大化：

$$
\prod_{t=1}^{T}p_\theta(x_t \mid x_{1:t-1})
$$

取 log 后变成：

$$
\sum_{t=1}^{T}\log p_\theta(x_t \mid x_{1:t-1})
$$

训练通常最小化负 log probability：

$$
-\sum_{t=1}^{T}\log p_\theta(x_t \mid x_{1:t-1})
$$

如果只对部分 token 计算 loss，例如 chat SFT 中只对 assistant token 计算 loss，就把求和限制在参与训练的 token 集合上。

这和 one-hot 标签下的交叉熵一致。

面试回答：

```text
最大似然是最大化训练数据中真实 token 的概率。取 log 后，最大化 log probability 等价于最小化 negative log likelihood。在 one-hot 标签下，negative log likelihood 和 cross entropy 是同一个形式，所以语言模型训练常说最小化交叉熵。
```

## 3.6 KL 散度：两个分布的差异

KL 散度衡量用一个分布 `Q` 近似另一个分布 `P` 时的额外代价。

常见形式：

$$
D_{\mathrm{KL}}(p \parallel q) = \sum_x p(x)\log \frac{p(x)}{q(x)}
$$

当 `p(x)>0` 但 `q(x)=0` 时，KL 会发散；工程中通常要依赖 softmax、label smoothing、clip 或数值稳定处理避免不可计算。

直觉上，如果 `Q` 和 `P` 很接近，KL 小；如果 `Q` 在 `P` 认为重要的地方给了很低概率，KL 会变大。

KL 有几个重要性质：

1. KL 不小于 0。
2. KL 等于 0 表示两个分布相同。
3. KL 不是对称的，`KL(P || Q)` 和 `KL(Q || P)` 通常不同。
4. KL 不是严格意义上的距离。

为什么 KL 不对称重要？

因为 `KL(P || Q)` 更强调 `Q` 不要漏掉 `P` 的高概率区域；而 `KL(Q || P)` 更强调 `Q` 不要把概率放到 `P` 不支持的区域。不同方向会导致不同优化行为。

面试回答：

```text
KL 散度衡量两个概率分布的差异，可以理解为用 Q 近似 P 时多付出的信息代价。KL 非负，但不对称，所以不是普通距离。在大模型里，KL 常用于约束一个新 policy 不要偏离 reference model 太远。
```

## 3.7 交叉熵、熵和 KL 的关系

交叉熵可以分解为真实分布自身的不确定性，加上模型分布和真实分布的差异：

$$
H(p,q) = H(p) + D_{\mathrm{KL}}(p \parallel q)
$$

这个关系非常重要。

如果真实数据分布 `P` 固定，那么 `H(P)` 是常数。最小化交叉熵，就等价于最小化 `KL(P || Q)`，也就是让模型分布 `Q` 接近真实分布 `P`。

面试回答：

```text
交叉熵可以分成两部分：真实分布自身的熵，以及真实分布和模型分布之间的 KL。数据分布固定时，真实熵是常数，所以最小化交叉熵等价于让模型分布更接近真实数据分布。
```

## 3.8 Perplexity：平均每步有多少有效选择

Perplexity 通常是 average negative log likelihood 的指数形式。

如果平均 token loss 是：

$$
L_{\mathrm{avg}} = \frac{1}{N}\sum_{i=1}^{N}L_i
$$

那么 perplexity 是：

$$
\mathrm{PPL} = \exp(L_{\mathrm{avg}})
$$

直觉上，perplexity 可以理解为模型平均每一步有多少个有效候选。

例如：

1. Perplexity 越低，模型越不困惑。
2. Perplexity 越高，模型越难预测真实文本。

但 perplexity 有很多边界：

1. 不同 tokenizer 下不一定可比。
2. 低 perplexity 不一定代表聊天质量好。
3. 对齐后的模型可能牺牲部分 perplexity，但用户体验更好。
4. 如果验证集被污染，perplexity 会失真。
5. Perplexity 不能直接衡量事实性、安全性和工具调用能力。

面试回答：

```text
Perplexity 是平均 token loss 的指数，可以理解为模型平均每一步的有效候选数。它适合衡量语言建模能力，但不同 tokenizer、数据集和评估设置下不一定可比。低 perplexity 不保证模型更 helpful、更 factual 或更安全。
```

## 3.9 为什么低 loss 不一定代表好模型

低 cross entropy loss 表示模型更擅长预测验证集 token，但这不等于所有能力更强。

原因包括：

1. 验证集不代表真实用户任务。
2. 数据污染让 loss 虚低。
3. 模型可能学到模板和风格，而不是推理能力。
4. SFT/RLHF 后的交互质量不完全由 pretraining loss 决定。
5. 安全、诚实、引用和工具调用不是单纯 next-token loss 能完全衡量。

面试回答：

```text
Loss 低说明模型更好地拟合了评估数据的 token 分布，但不保证真实任务能力更强。如果评估集不代表目标任务、存在污染，或者我们关心的是事实性、安全和工具调用，单纯 loss 就不够。大模型评估需要结合 benchmark、人评、业务任务、安全评估和 bad case 分析。
```

## 3.10 RLHF 中的 KL penalty

RLHF 中常见目标会加入 KL penalty，约束 policy model 不要偏离 reference model 太远。

直觉上，reward model 会给回答打分，但如果只最大化 reward，policy 可能利用 reward model 漏洞，出现 reward hacking、语言退化或过度拒答。

KL penalty 的作用是：

1. 限制 policy 分布偏离 reference。
2. 保持语言质量和基础能力。
3. 降低 reward hacking 风险。
4. 让优化过程更稳定。

可以直观写成：

$$
J = R - \beta D_{\mathrm{KL}}(\pi_\theta \parallel \pi_{\mathrm{ref}})
$$

其中 `beta` 控制约束强度。

1. `beta` 太小，约束弱，模型可能乱跑。
2. `beta` 太大，约束强，模型学不动。

面试回答：

```text
RLHF 中 KL penalty 用来限制当前 policy 不要偏离 reference model 太远。因为 reward model 只是人类偏好的 proxy，如果无约束地最大化 reward，模型可能 reward hacking 或语言质量退化。KL 太弱风险大，KL 太强又会限制学习，所以 beta 需要调节。
```

## 3.11 DPO 和 KL 约束的关系

DPO 不显式训练 reward model，也不跑 PPO，但它和带 KL 约束的偏好优化有理论联系。

DPO 使用 preference pair：

```text
prompt, chosen response, rejected response
```

它比较 policy model 和 reference model 对 chosen/rejected 的相对 log probability 差异。

直觉上，DPO 希望 policy 相比 reference 更偏好 chosen，而不是 rejected。

Reference model 的作用类似锚点：

1. 防止 policy 任意漂移。
2. 提供“偏离原模型多少”的参照。
3. 让偏好优化不只是简单提高 chosen 概率。

面试回答：

```text
DPO 可以看作把带 KL 约束的偏好优化改写成直接基于 chosen/rejected log probability 的目标。它不显式加入 PPO 式 KL penalty，但 reference model 在目标中起到锚点作用，限制 policy 的偏好变化不能脱离原模型太远。因此 DPO 更简单，但仍依赖 reference、beta 和偏好数据质量。
```

## 3.12 互信息：知道一个变量能减少多少不确定性

互信息衡量两个变量之间共享的信息量。

直觉上，如果知道 `X` 能大幅减少对 `Y` 的不确定性，那么 `X` 和 `Y` 的互信息高。

在大模型和多模态中，互信息直觉很有用：

1. 图文对齐希望图像表示和文本表示共享语义信息。
2. RAG 希望检索文档对答案有高信息量。
3. Prompt 中无关上下文和答案互信息低，可能只是噪声。
4. 表示学习希望 hidden state 保留对任务有用的信息。

面试中不一定要求推导互信息公式，更重要的是直觉。

常见公式是：

$$
I(X;Y) = H(X) - H(X \mid Y)
$$

也可以写成联合分布和边缘分布的 KL：

$$
I(X;Y) = \sum_{x,y}p(x,y)\log \frac{p(x,y)}{p(x)p(y)}
$$

所以互信息高，不等于表面相似度高，而是一个变量确实能减少另一个变量的不确定性。

面试回答：

```text
互信息衡量一个变量能减少另一个变量多少不确定性。多模态对齐可以理解为让图像和文本表示共享更多语义信息；RAG 中好的检索结果应该和答案有高互信息，而不是只在词面相似。
```

## 3.13 信息论视角理解 RAG

RAG 可以看成给模型提供额外证据，降低回答不确定性。

一个好检索结果应该：

1. 和问题相关。
2. 包含回答所需证据。
3. 减少模型对答案的不确定性。
4. 不引入冲突或噪声。

如果检索到的 chunk 和问题词面相似，但不包含答案证据，那么它对答案的信息量低，甚至会干扰模型。

这解释了为什么 RAG 不能只看 embedding similarity，还要看：

1. Recall@k。
2. Rerank 后的 evidence quality。
3. Context precision。
4. Citation accuracy。
5. Faithfulness。

## 3.14 信息论视角理解安全和拒答

安全场景中，信息论也有直觉价值。

例如模型面对高风险请求时，不应该提供会显著降低危险操作门槛的信息。安全拒答不是拒绝所有内容，而是控制输出中高风险信息的粒度。

对于资料不足的问题，模型应该表达不确定性，而不是输出看似高信息量但无依据的细节。

面试回答：

```text
安全回答不是简单减少信息量，而是控制危险信息和无依据信息。对正常教育性请求，模型可以提供高层解释；对高风险可操作请求，应该拒绝危险细节并给安全替代；对证据不足的问题，应表达不确定性，避免生成 unsupported claims。
```

## 3.15 最小可运行信息论审计 demo

下面用纯 Python 展示本章几个关键关系：熵比较、交叉熵、KL 分解、one-hot NLL、平均 NLL / PPL、互信息、KL penalty 以及 RAG 证据降低不确定性的直觉。

```python
import math

EPS = 1e-12


def entropy(dist):
    return -sum(p * math.log(p) for p in dist.values() if p > 0)


def cross_entropy(p, q):
    return -sum(px * math.log(max(q.get(x, 0.0), EPS)) for x, px in p.items() if px > 0)


def kl(p, q):
    return sum(px * math.log(px / max(q.get(x, 0.0), EPS)) for x, px in p.items() if px > 0)


def mutual_information(joint):
    px = {}
    py = {}
    for (x, y), p in joint.items():
        px[x] = px.get(x, 0.0) + p
        py[y] = py.get(y, 0.0) + p

    total = 0.0
    for (x, y), p in joint.items():
        if p > 0:
            total += p * math.log(p / (px[x] * py[y]))
    return total


peaked = {"A": 0.92, "B": 0.04, "C": 0.03, "D": 0.01}
uniform = {"A": 0.25, "B": 0.25, "C": 0.25, "D": 0.25}

true_dist = {"Paris": 0.7, "Lyon": 0.2, "Marseille": 0.1}
model_good = {"Paris": 0.6, "Lyon": 0.25, "Marseille": 0.15}
model_bad = {"Paris": 0.2, "Lyon": 0.3, "Marseille": 0.5}

h_true = entropy(true_dist)
ce_good = cross_entropy(true_dist, model_good)
ce_bad = cross_entropy(true_dist, model_bad)
kl_good = kl(true_dist, model_good)

one_hot = {"Paris": 1.0, "Lyon": 0.0, "Marseille": 0.0}
one_hot_nll = cross_entropy(one_hot, model_good)

token_probs = [0.6, 0.75, 0.5, 0.8]
nlls = [-math.log(p) for p in token_probs]
avg_nll = sum(nlls) / len(nlls)
ppl = math.exp(avg_nll)

joint = {
    ("image_cat", "text_cat"): 0.45,
    ("image_cat", "text_dog"): 0.05,
    ("image_dog", "text_cat"): 0.10,
    ("image_dog", "text_dog"): 0.40,
}
mi = mutual_information(joint)

ref = {"helpful": 0.55, "refuse": 0.25, "unsafe_detail": 0.20}
policy = {"helpful": 0.70, "refuse": 0.25, "unsafe_detail": 0.05}
kl_policy_ref = kl(policy, ref)
reward = 1.35
beta = 0.2
objective = reward - beta * kl_policy_ref

without_doc = {"correct": 0.45, "wrong_a": 0.35, "wrong_b": 0.20}
with_doc = {"correct": 0.85, "wrong_a": 0.10, "wrong_b": 0.05}

checks = {
    "entropy_order": entropy(uniform) > entropy(peaked),
    "ce_decomposition": abs(ce_good - (h_true + kl_good)) < 1e-9,
    "better_model_lower_ce": ce_good < ce_bad,
    "one_hot_nll": abs(one_hot_nll + math.log(model_good["Paris"])) < 1e-9,
    "ppl_positive": ppl > 1.0,
    "mutual_information_positive": mi > 0,
    "kl_non_negative": kl_policy_ref >= 0,
    "evidence_reduces_entropy": entropy(with_doc) < entropy(without_doc),
}

print("entropy_peaked=", round(entropy(peaked), 3))
print("entropy_uniform=", round(entropy(uniform), 3))
print("h_true=", round(h_true, 3), "ce_good=", round(ce_good, 3), "kl_good=", round(kl_good, 3))
print("ce_bad=", round(ce_bad, 3), "one_hot_nll=", round(one_hot_nll, 3))
print("avg_nll=", round(avg_nll, 3), "ppl=", round(ppl, 3))
print("mutual_information=", round(mi, 3))
print("kl_policy_ref=", round(kl_policy_ref, 3), "kl_penalized_objective=", round(objective, 3))
print("entropy_without_doc=", round(entropy(without_doc), 3), "entropy_with_doc=", round(entropy(with_doc), 3))
print("checks=", checks)
```

## 3.16 面试高频问题

### 3.16.1 交叉熵和 KL 有什么关系

回答：

```text
交叉熵可以分解为真实分布的熵加上真实分布到模型分布的 KL。数据分布固定时，真实熵是常数，所以最小化交叉熵等价于让模型分布接近真实分布。
```

### 3.16.2 Perplexity 是什么

回答：

```text
Perplexity 是平均 token loss 的指数形式，可以直观理解为模型平均每一步有多少有效候选。它越低说明模型越不困惑，但不同 tokenizer 和数据集下不一定可比，也不能直接代表事实性、安全性和用户体验。
```

### 3.16.3 为什么 RLHF 需要 KL penalty

回答：

```text
Reward model 只是人类偏好的 proxy。如果 policy 无约束地最大化 reward，可能出现 reward hacking、语言质量退化或过度拒答。KL penalty 约束 policy 接近 reference model，在提升偏好的同时保持原有语言能力和行为稳定性。
```

### 3.16.4 DPO 为什么需要 reference model

回答：

```text
Reference model 给 DPO 提供偏好变化的锚点。DPO 不只是让 policy 提高 chosen 的概率，而是让 policy 相对 reference 更偏好 chosen 而不是 rejected。这样可以避免 policy 无限制漂移，也保留了类似 KL 约束的作用。
```

### 3.16.5 为什么 perplexity 不能跨 tokenizer 随便比较

回答：

```text
Perplexity 是平均 token NLL 的指数。Tokenizer 会改变文本被切成多少个 token，也会改变每个 token 的预测难度，所以不同 tokenizer、上下文截断、loss mask 和验证集下的 PPL 不一定可比。比较 PPL 时要先统一评估口径。
```

### 3.16.6 KL 散度为什么不是普通距离

回答：

```text
KL 非负，但不对称，KL(p||q) 和 KL(q||p) 的优化含义不同。如果 p 在某个区域有概率而 q 给 0，KL 还会发散。因此 KL 更适合解释分布近似的额外信息代价，而不是普通几何距离。
```

### 3.16.7 互信息在 RAG 里怎么理解

回答：

```text
互信息衡量一个变量能减少另一个变量多少不确定性。RAG 中，一个 chunk 和 query 词面相似不够，关键是它能否减少答案的不确定性、提供可引用证据并避免引入冲突噪声。
```

## 3.17 常见误区

1. 把交叉熵只理解成分类 API，不理解 negative log probability。
2. 以为 KL 是对称距离。
3. 以为 perplexity 可以直接比较所有模型。
4. 认为低 loss 就一定更 helpful。
5. 认为 KL penalty 越大越安全。
6. 认为 DPO 没有 reward model 就没有约束。
7. 把互信息当成必须背公式，而不是理解“减少不确定性”的直觉。
8. 脱离 tokenizer、数据集和 loss mask 比较 perplexity。

## 3.19 本章总结

信息论把概率分布和学习目标联系起来。

本章最重要的结论：

1. 信息量是负 log 概率，真实 token 概率越低，loss 越高。
2. 熵衡量分布不确定性。
3. 交叉熵衡量用模型分布解释真实分布的平均代价。
4. One-hot 标签下，交叉熵就是真实 token 的 negative log probability。
5. KL 衡量分布差异，但不是对称距离。
6. Perplexity 是平均 token loss 的指数形式。
7. RLHF 中 KL penalty 用来限制 policy 偏离 reference。
8. DPO 的 reference model 也起到类似锚点作用。
9. 互信息直觉可以帮助理解多模态对齐、RAG 和表示学习。

面试中讲信息论，要主动连接 cross entropy、MLE、perplexity、KL penalty、DPO、RAG 和安全，而不是只背公式。
