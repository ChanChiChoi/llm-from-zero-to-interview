# 第一章：线性代数

线性代数是理解大模型的第一层数学语言。Embedding 是向量，Linear layer 是矩阵乘法，Attention 是一组矩阵乘法和归一化，LoRA 是低秩矩阵更新，SVD 是理解表示压缩和低秩近似的基础。

很多人觉得自己“知道 Transformer”，但一问 Q、K、V 的 shape、attention score 为什么是 `[T, T]`、LoRA 为什么省参数、embedding 检索为什么要归一化，就会发现线性代数地基并不稳。

本章目标不是重写一本线性代数教材，而是讲清大模型算法岗最常用的线性代数直觉：向量、矩阵、矩阵乘法、内积、范数、正交、投影、特征值、SVD，以及它们如何出现在 embedding、attention、MLP、LoRA、RAG 和评估里。

## 1.1 为什么大模型需要线性代数

大模型中的绝大多数中间状态都可以看成向量、矩阵或更高维张量。

一段 token 序列进入模型后，常见 hidden states 的 shape 是：

```text
X: [B, T, d]
```

其中：

1. `B` 是 batch size。
2. `T` 是 sequence length。
3. `d` 是 hidden size。

每个 token 是一个 `d` 维向量，一句话是一组 token 向量，一个 batch 是多组句子向量。

Transformer 的主要计算，就是对这些向量反复做：

1. 线性变换。
2. 相似度计算。
3. 加权求和。
4. 归一化。
5. 低秩或高维空间中的表示变换。

如果线性代数理解清楚，很多 LLM 问题会变得自然：

1. Linear layer 参数量为什么是 `d_in * d_out`？
2. Attention 为什么要做 `Q K^T`？
3. 为什么 attention score 的 shape 是 `[B, H, T, T]`？
4. 为什么 embedding 检索常用 cosine similarity？
5. 为什么 LoRA 用两个小矩阵近似一个大矩阵更新？
6. 为什么 gradient norm 能反映训练稳定性？

### 1.1.1 关键公式与 LLM 线性代数速查

token hidden state 通常写成：

$$
h_i \in \mathbb{R}^{d}
$$

其中 `d` 是 hidden size，`h_i` 是第 `i` 个 token 在某一层的上下文表示。

两个向量的内积：

$$
x^\top y=\sum_{j=1}^{d} x_j y_j
$$

L2 范数：

$$
\|x\|_2=\sqrt{\sum_{j=1}^{d} x_j^2}
$$

cosine similarity：

$$
\cos(x,y)=
\frac{x^\top y}{\|x\|_2\|y\|_2}
$$

如果 `x` 和 `y` 都已经 L2 normalize，那么 dot product 和 cosine similarity 等价：

$$
\bar{x}^\top \bar{y}=\cos(x,y)
$$

矩阵乘法 shape：

$$
X\in\mathbb{R}^{B\times d_{in}},\quad
W\in\mathbb{R}^{d_{in}\times d_{out}},\quad
Y=XW+b\in\mathbb{R}^{B\times d_{out}}
$$

多头 attention 的核心 shape：

$$
Q,K,V\in\mathbb{R}^{B\times H\times T\times d_h}
$$

$$
S=\frac{QK^\top}{\sqrt{d_h}}
\in\mathbb{R}^{B\times H\times T\times T}
$$

$$
O=\mathrm{softmax}(S)V
\in\mathbb{R}^{B\times H\times T\times d_h}
$$

这里后两维 `T x T` 表示每个 query token 都和每个 key token 计算一次相似度。

向量投影：

$$
\mathrm{proj}_{u}(x)=
\frac{x^\top u}{u^\top u}u
$$

如果 `u` 是单位向量，则 `proj_u(x)=(x^T u)u`。

特征值和特征向量：

$$
Av=\lambda v
$$

SVD：

$$
W=U\Sigma V^\top
$$

rank-r 近似：

$$
W_r=U_r\Sigma_r V_r^\top
$$

LoRA 低秩更新：

$$
\Delta W=AB,\quad
A\in\mathbb{R}^{d_{in}\times r},\quad
B\in\mathbb{R}^{r\times d_{out}}
$$

LoRA 可训练参数量：

$$
N_{LoRA}=r(d_{in}+d_{out})
$$

相对全量更新的参数比例：

$$
\rho_{LoRA}=
\frac{r(d_{in}+d_{out})}{d_{in}d_{out}}
$$

线性代数面试题的底层能力不是背公式，而是看到任意 LLM 结构后能快速说清：对象是向量还是矩阵，shape 是否能相乘，哪个维度在做相似度，哪个维度在做加权求和，参数量和计算量如何随维度变化。

## 1.2 向量：模型中的基本表示单位

向量可以理解为一组有方向和大小的数字。

在大模型中，一个 token 的 hidden state 可以写成：

```text
h_i in R^d
```

它不是一个人工可解释的标签，而是模型内部对当前 token 在当前上下文中的表示。

例如“苹果”这个词，在“我吃了一个苹果”和“苹果发布了新芯片”中，初始 token embedding 可能相同，但经过 Transformer 层以后，上下文表示会变得不同。因为 self-attention 会让这个 token 从周围 token 中读取信息。

向量有两个重要直觉：

1. 方向：常常对应语义、属性或特征组合。
2. 长度：常常对应尺度、激活强度或模型内部表示大小。

在 embedding 检索中，很多时候我们更关心方向而不是长度。因此常常先把向量归一化，再用 dot product 或 cosine similarity 比较语义相似度。

## 1.3 内积：相似度的基础

两个向量的内积是对应元素相乘再求和：

```text
dot(x, y) = sum_i x_i * y_i
```

内积可以理解为两个向量的匹配程度。

1. 方向相近，内积通常较大。
2. 方向近似垂直，内积接近 0。
3. 方向相反，内积可能为负。

Cosine similarity 会去掉长度影响，只比较方向：

```text
cos(x, y) = dot(x, y) / (norm(x) * norm(y))
```

在 RAG 和 embedding 检索中，query embedding 和 document embedding 的相似度常用 cosine similarity 或归一化后的 dot product。

面试回答：

```text
Dot product 同时受到方向和长度影响，cosine similarity 会除以向量范数，更关注方向相似性。在 embedding 检索里，我们通常希望语义相似的文本向量方向接近，所以常用 cosine similarity，或者先归一化 embedding 再做 dot product。
```

## 1.4 范数：向量和梯度的大小

范数是衡量向量大小的函数。最常见的是 L2 norm：

```text
norm_2(x) = sqrt(sum_i x_i^2)
```

在大模型中，范数经常出现在：

1. Embedding 归一化。
2. Gradient norm 监控。
3. Gradient clipping。
4. Weight decay。
5. LayerNorm 和 RMSNorm。
6. Cosine similarity。

训练时，gradient norm 是非常重要的监控指标。如果 gradient norm 突然变大，可能说明 learning rate 过大、数据 batch 异常、混合精度溢出、loss mask 错误或模型开始不稳定。

面试回答：

```text
范数可以理解为向量的尺度。训练中常看 gradient norm 来判断梯度是否异常，必要时使用 gradient clipping 限制更新幅度。检索中常对 embedding 做归一化，是为了避免向量长度主导相似度，让相似度更关注语义方向。
```

## 1.5 矩阵：线性变换的参数化形式

矩阵可以理解为一组向量，也可以理解为一个线性变换。

一个 Linear layer 通常写成：

```text
Y = X W + b
```

如果：

```text
X: [B, d_in]
W: [d_in, d_out]
b: [d_out]
Y: [B, d_out]
```

那么这个线性层把每个输入向量从 `d_in` 维映射到 `d_out` 维。

Transformer 中的 Q、K、V 就来自三个线性投影：

```text
Q = X W_Q
K = X W_K
V = X W_V
```

这说明矩阵不是抽象符号，而是模型学习到的表示变换。`W_Q` 学会把 hidden state 变成“查询向量”，`W_K` 学会变成“索引向量”，`W_V` 学会变成“被读取的内容向量”。

## 1.6 矩阵乘法：最重要的是 shape

矩阵乘法的 shape 规则是：

```text
A: [m, n]
B: [n, p]
C = A B
C: [m, p]
```

中间维度 `n` 必须匹配。

在大模型面试中，shape 推导非常重要。以多头注意力为例：

```text
Q: [B, H, T, d_h]
K: [B, H, T, d_h]
V: [B, H, T, d_h]
```

计算 attention scores：

```text
scores = Q @ K^T
scores: [B, H, T, T]
```

再加权求和：

```text
context = softmax(scores) @ V
context: [B, H, T, d_h]
```

后两维 `[T, T]` 的含义是：每个 query token 都和每个 key token 计算一个相似度。

面试回答：

```text
Attention 的 shape 可以从矩阵乘法推出来。Q 和 K 的最后一维都是 head_dim，Q 乘 K 的转置后，每个 token 会和所有 token 得到一个匹配分数，所以 score 的后两维是 T by T。softmax 后再乘 V，就得到每个 token 对所有 value 的加权汇总。
```

## 1.7 Attention 中的线性代数

Self-attention 的核心可以写成：

```text
Attention(Q, K, V) = softmax(Q K^T / sqrt(d_h)) V
```

这行式子包含三个线性代数动作。

第一，线性投影。

```text
Q = X W_Q
K = X W_K
V = X W_V
```

第二，内积相似度。

```text
scores = Q K^T
```

第三，加权求和。

```text
output = softmax(scores) V
```

为什么要除以 `sqrt(d_h)`？

因为维度变大时，随机向量内积的尺度会变大，softmax 容易变得过于尖锐，导致梯度不稳定。除以 `sqrt(d_h)` 是为了稳定 attention score 的尺度。

## 1.8 正交和投影

两个向量正交，表示它们内积为 0：

```text
dot(x, y) = 0
```

直观上，它们代表互不重叠的方向。

投影可以理解为：一个向量在某个方向上的分量。如果 `u` 是单位向量，那么 `x` 在 `u` 方向上的投影长度是：

```text
dot(x, u)
```

投影在机器学习里很常见。PCA 可以看成寻找一组重要方向，把数据投影到这些方向上，尽量保留最大方差。Embedding 分析、表示压缩和低秩近似也都和投影直觉有关。

面试中不一定会直接问投影公式，但会问类似问题：为什么低维表示可以保留主要信息？为什么 PCA 能降维？为什么某些表示方向能对应模型行为？这些背后都需要投影直觉。

## 1.9 特征值和特征向量

如果一个矩阵 `A` 作用在向量 `v` 上，只改变它的长度，不改变它的方向，就有：

```text
A v = lambda v
```

这里 `v` 是特征向量，`lambda` 是特征值。

直觉上，特征向量是这个线性变换的“稳定方向”，特征值表示这个方向被放大或缩小多少。

在大模型面试中，特征值不一定高频直接出现，但它支撑很多重要概念：

1. PCA 的主方向。
2. Hessian 和优化曲率。
3. 矩阵稳定性。
4. 表示空间中的主要变化方向。

面试回答可以保持直觉：

```text
特征向量是矩阵变换后方向不变的向量，特征值表示这个方向被缩放的倍数。它帮助我们理解一个线性变换在哪些方向上放大、压缩或保持信息。PCA、优化曲率和稳定性分析都和这个直觉有关。
```

## 1.10 SVD：低秩近似的基础

SVD，也就是奇异值分解，可以把一个矩阵分解为：

```text
W = U S V^T
```

直觉上，SVD 把一个矩阵拆成一组从重要到不重要的方向。`S` 中的奇异值越大，说明对应方向越重要。

如果只保留最大的 `r` 个奇异值，就得到一个 rank-r 近似：

```text
W_approx = U_r S_r V_r^T
```

这说明很多矩阵的信息可能集中在少数重要方向上，可以用低秩矩阵近似。

这和大模型里的 LoRA 有直接联系。LoRA 假设微调时权重更新 `Delta W` 可以用低秩矩阵表示：

```text
Delta W = A B
```

其中：

```text
A: [d_in, r]
B: [r, d_out]
r << min(d_in, d_out)
```

原本更新一个大矩阵需要 `d_in * d_out` 个参数，而 LoRA 只需要：

```text
r * (d_in + d_out)
```

当 `r` 很小，参数量和显存都会显著下降。

面试回答：

```text
SVD 说明一个矩阵可以分解为若干重要方向的组合，如果主要信息集中在少数方向上，就可以用低秩近似。LoRA 的直觉类似：微调不一定需要更新完整权重矩阵，而是假设任务相关的权重增量位于一个低秩子空间中，因此用两个小矩阵表示 Delta W，从而大幅减少可训练参数。
```

## 1.11 LoRA 为什么是线性代数问题

普通全参微调会直接更新权重：

```text
W_new = W + Delta W
```

LoRA 不直接学习完整的 `Delta W`，而是学习低秩分解：

```text
Delta W = A B
```

假设 `W` 的 shape 是：

```text
W: [4096, 4096]
```

全量更新参数量约为：

```text
4096 * 4096 = 16777216
```

如果 LoRA rank 是 8，那么参数量约为：

```text
4096 * 8 + 8 * 4096 = 65536
```

这就是 LoRA 省参数的线性代数原因。

但低秩也有边界。rank 太小，表达能力可能不足；rank 太大，参数和显存节省变少，也更容易过拟合。面试中要体现这个 trade-off。

## 1.12 Embedding 检索中的线性代数

RAG 中的向量检索，本质上也是线性代数。

离线阶段：

```text
document -> embedding vector
```

在线阶段：

```text
query -> embedding vector
```

然后比较 query 向量和 document 向量的相似度。

常见相似度：

1. Dot product。
2. Cosine similarity。
3. L2 distance。

如果 embedding 已经归一化，dot product 和 cosine similarity 排序通常等价。

面试回答：

```text
Embedding 检索把文本映射到向量空间，语义相似的文本应该方向接近。检索时用 dot product、cosine similarity 或 L2 distance 找最近邻。归一化后，dot product 更接近 cosine similarity，常用于高效向量检索。
```

## 1.13 最小可运行线性代数审计 demo

下面这个 demo 不依赖 NumPy 或 PyTorch，直接用 Python 标准库检查向量相似度、矩阵乘法 shape、scaled dot-product attention、SVD 能量直觉和 LoRA 参数量。

```python
from math import exp, sqrt


def shape(matrix):
    return (len(matrix), len(matrix[0]) if matrix else 0)


def dot(x, y):
    return sum(a * b for a, b in zip(x, y))


def l2_norm(x):
    return sqrt(dot(x, x))


def cosine(x, y):
    return dot(x, y) / (l2_norm(x) * l2_norm(y))


def normalize(x):
    n = l2_norm(x)
    return [v / n for v in x]


def matmul(a, b):
    rows, inner = shape(a)
    inner_b, cols = shape(b)
    assert inner == inner_b, (shape(a), shape(b))
    return [[sum(a[i][k] * b[k][j] for k in range(inner)) for j in range(cols)] for i in range(rows)]


def transpose(a):
    rows, cols = shape(a)
    return [[a[i][j] for i in range(rows)] for j in range(cols)]


def softmax(row):
    m = max(row)
    values = [exp(v - m) for v in row]
    total = sum(values)
    return [v / total for v in values]


def scaled_dot_product_attention(q, k, v):
    head_dim = len(q[0])
    scores = matmul(q, transpose(k))
    scaled = [[s / sqrt(head_dim) for s in row] for row in scores]
    attn = [softmax(row) for row in scaled]
    return scaled, attn, matmul(attn, v)


def project(x, u):
    scale = dot(x, u) / dot(u, u)
    return [round(scale * value, 3) for value in u]


def lora_params(d_in, d_out, rank):
    full = d_in * d_out
    lora = rank * (d_in + d_out)
    return full, lora, round(lora / full, 6)


x = [1.0, 2.0, 2.0]
y = [2.0, 0.0, 1.0]
x_bar = normalize(x)
y_bar = normalize(y)
l2_sq_normalized = sum((a - b) ** 2 for a, b in zip(x_bar, y_bar))

q = [[1.0, 0.0], [0.0, 1.0], [1.0, 1.0]]
k = [[1.0, 0.0], [0.0, 1.0], [1.0, -1.0]]
v = [[1.0, 2.0], [3.0, 0.0], [0.0, 1.0]]
scores, attn, context = scaled_dot_product_attention(q, k, v)

singular_values = [5.0, 2.0, 0.5]
energy_top2 = sum(s * s for s in singular_values[:2]) / sum(s * s for s in singular_values)
full_params, rank8_params, ratio = lora_params(4096, 4096, 8)

checks = {
    "normalized_dot_equals_cosine": round(dot(x_bar, y_bar), 3) == round(cosine(x, y), 3),
    "normalized_l2_relation": round(l2_sq_normalized, 3) == round(2 - 2 * cosine(x, y), 3),
    "attention_rows_sum_to_one": all(abs(sum(row) - 1.0) < 1e-9 for row in attn),
    "context_shape_ok": shape(context) == (3, 2),
    "lora_saves_params": rank8_params < full_params,
}

print("dot=", round(dot(x, y), 3))
print("norms=", (round(l2_norm(x), 3), round(l2_norm(y), 3)))
print("cosine=", round(cosine(x, y), 3))
print("normalized_l2_sq=", round(l2_sq_normalized, 3))
print("projection_on_x_axis=", project(x, [1.0, 0.0, 0.0]))
print("score_shape=", shape(scores))
print("attention_row_sums=", [round(sum(row), 3) for row in attn])
print("context_shape=", shape(context))
print("first_context=", [round(v, 3) for v in context[0]])
print("svd_energy_top2=", round(energy_top2, 3))
print("lora_params=", {"full": full_params, "rank8": rank8_params, "ratio": ratio})
print("checks=", checks)
```

运行后可以看到：

```text
dot= 4.0
norms= (3.0, 2.236)
cosine= 0.596
normalized_l2_sq= 0.807
projection_on_x_axis= [1.0, 0.0, 0.0]
score_shape= (3, 3)
attention_row_sums= [1.0, 1.0, 1.0]
context_shape= (3, 2)
first_context= [0.994, 1.203]
svd_energy_top2= 0.991
lora_params= {'full': 16777216, 'rank8': 65536, 'ratio': 0.003906}
checks= {'normalized_dot_equals_cosine': True, 'normalized_l2_relation': True, 'attention_rows_sum_to_one': True, 'context_shape_ok': True, 'lora_saves_params': True}
```

这个 demo 要看懂五点：

1. dot product、cosine similarity 和归一化后的 L2 距离是同一组向量几何关系的不同表达。
2. attention score 的 shape 来自 `Q @ K^T`，不是凭空记忆。
3. softmax 后每一行 attention weight 都应接近 1，表示一个 query token 对所有 value token 的加权分布。
4. top singular values 的平方和占比可以帮助直觉理解低秩近似保留了多少主要能量。
5. LoRA 参数量从 `d_in * d_out` 降到 `r * (d_in + d_out)`，rank 越小越省参数，但表达能力也越受限。

## 1.14 面试高频问题

### 1.14.1 为什么 attention score 是 T by T

回答：

```text
因为 self-attention 中每个 query token 都要和每个 key token 计算相似度。序列长度是 T，所以一共有 T 个 query 和 T 个 key，得到的 score matrix 后两维就是 T by T。
```

### 1.14.2 Dot product 和 cosine similarity 有什么区别

回答：

```text
Dot product 同时受方向和向量长度影响。Cosine similarity 会除以两个向量的范数，只看方向相似性。Embedding 检索中如果向量长度不希望影响语义相似度，通常使用 cosine similarity 或先归一化再做 dot product。
```

### 1.14.3 LoRA 为什么省参数

回答：

```text
LoRA 不直接学习完整的权重更新矩阵 Delta W，而是把它写成两个低秩矩阵 A 和 B 的乘积。如果原矩阵是 d_in by d_out，全量更新需要 d_in * d_out 个参数；LoRA rank 为 r 时只需要 r * (d_in + d_out) 个参数。当 r 远小于原维度时，参数量显著降低。
```

### 1.14.4 为什么要关注 gradient norm

回答：

```text
Gradient norm 反映梯度整体尺度。训练中如果 gradient norm 突然升高，可能说明学习率过大、数据异常、数值溢出或 loss mask 有问题。监控 gradient norm 能帮助定位 loss spike，gradient clipping 也常用于限制异常更新。
```

## 1.15 常见误区

1. 只记公式，不会推 shape。
2. 把向量长度和方向混为一谈。
3. 以为 cosine similarity 和 dot product 总是一样。
4. 只说 LoRA 省参数，不会算参数量。
5. 认为 attention 是黑盒，不知道它就是相似度矩阵和加权求和。
6. 不知道范数和训练稳定性、检索相似度都有关系。

## 1.17 本章总结

线性代数不是大模型之外的数学基础，而是大模型内部计算的基本语言。

本章最重要的结论：

1. Token hidden state 是向量。
2. Linear layer 是矩阵乘法。
3. Attention 是相似度矩阵加加权求和。
4. Shape 推导是理解 Transformer 的关键能力。
5. 范数连接训练稳定性和向量检索。
6. SVD 和低秩近似帮助理解 LoRA。
7. RAG embedding 检索本质是向量空间最近邻搜索。

面试中讲线性代数，不要停留在公式定义，要主动连接到 attention、embedding、LoRA、训练稳定性和系统评估。
