# 第七章：贝叶斯与不确定性

贝叶斯思想和不确定性，是理解大模型可靠性的关键。LLM 很会生成流畅文本，但流畅不等于正确，自信不等于可信。很多真实问题不是“模型能不能答”，而是“模型什么时候应该答、什么时候应该检索、什么时候应该承认不知道、什么时候应该交给人工或工具验证”。

概率论告诉我们如何表示不确定性，贝叶斯思想告诉我们如何根据证据更新判断。大模型系统里的 RAG、工具调用、事实核验、置信度校准、安全拒答、人审抽检和上线门禁，都可以用这套直觉理解。

本章重点：先验、似然、后验、MAP、校准、不确定性估计、模型置信度、幻觉、高风险场景决策。

## 7.1 为什么大模型需要不确定性意识

大模型常见失败不是完全不会，而是“不知道自己不知道”。

典型表现：

1. 对不存在的论文编造引用。
2. 对资料不足的问题给出确定答案。
3. 对高风险医疗、法律、金融建议过度自信。
4. RAG 没检索到证据仍然强行回答。
5. 工具调用失败后编造工具结果。
6. LLM judge 给出分数但缺少校准。

如果模型或系统缺少不确定性意识，就容易把“语言上合理”误当成“事实上可靠”。

面试回答：

```text
LLM 的输出流畅性不等于可靠性。很多高风险错误来自模型无法正确表达不确定性。可靠系统需要让模型在证据不足时检索、调用工具、澄清问题、拒答或交给人工，而不是自信编造。
```

### 7.1.1 关键公式与不确定性速查

Bayes rule：

$$
p(H\mid E)=\frac{p(E\mid H)p(H)}{p(E)}
$$

其中 `H` 是假设，`E` 是证据，`p(H)` 是 prior，`p(E|H)` 是 likelihood，`p(H|E)` 是 posterior，`p(E)` 是 evidence / normalizer。

两个互斥假设下，后验可以直接按归一化写：

$$
p(H\mid E)=\frac{p(E\mid H)p(H)}{p(E\mid H)p(H)+p(E\mid \neg H)p(\neg H)}
$$

MLE：

$$
\theta_{\mathrm{MLE}}=\arg\max_\theta p(D\mid \theta)
$$

MAP：

$$
\theta_{\mathrm{MAP}}=\arg\max_\theta p(\theta\mid D)
=\arg\max_\theta p(D\mid \theta)p(\theta)
$$

取负对数后：

$$
\theta_{\mathrm{MAP}}
=\arg\min_\theta -\log p(D\mid \theta)-\log p(\theta)
$$

如果先验偏好较小参数，例如零均值高斯先验：

$$
p(\theta)\propto \exp(-\lambda \|\theta\|_2^2)
$$

则 MAP 会出现类似 L2 正则的目标：

$$
L_{\mathrm{MAP}}(\theta)=L_{\mathrm{NLL}}(\theta)+\lambda\|\theta\|_2^2
$$

Beta-Bernoulli 更新常用于理解“模型在某类任务上的可靠率”：

$$
\alpha'=\alpha+s,\quad \beta'=\beta+f
$$

$$
\mathbb{E}[q\mid D]=\frac{\alpha'}{\alpha'+\beta'}
$$

其中 `s` 是成功次数，`f` 是失败次数，`q` 可以理解为某个切片上的正确率或可靠率。

预测熵：

$$
H(p)=-\sum_i p_i\log p_i
$$

熵越高，分布越分散；但低熵也不保证事实正确。

置信度：

$$
c(x)=\max_y p_\theta(y\mid x)
$$

校准的理想性质：

$$
p(Y=\hat Y\mid c(X)=r)=r
$$

Expected Calibration Error：

$$
\mathrm{ECE}=\sum_{b=1}^{B}\frac{|S_b|}{n}\left|\mathrm{acc}(S_b)-\mathrm{conf}(S_b)\right|
$$

Brier score：

$$
\mathrm{Brier}=\frac{1}{n}\sum_{i=1}^{n}(p_i-y_i)^2
$$

其中 `p_i` 是预测为正类或正确的概率，`y_i` 是 0/1 标签。

Abstention / selective prediction 可以写成：

$$
a(x)=
\begin{cases}
\mathrm{answer}, & c(x)\ge \tau \land r(x)<r_{\mathrm{high}} \\
\mathrm{verify}, & c(x)\ge \tau \land r(x)\ge r_{\mathrm{high}} \\
\mathrm{abstain}, & c(x)<\tau
\end{cases}
$$

这里 `r(x)` 表示风险等级。高风险任务即使置信度高，也可能需要工具验证或人工复核。

## 7.2 先验、似然和后验

贝叶斯公式的核心是：根据证据更新信念。

可以直观写成：

```text
posterior is proportional to likelihood times prior
```

也就是：

```text
P(H | E) = P(E | H) * P(H) / P(E)
```

其中：

1. `H` 是假设。
2. `E` 是证据。
3. `P(H)` 是先验。
4. `P(E | H)` 是似然。
5. `P(H | E)` 是后验。

在大模型系统里，可以这样理解：

1. 模型参数和语言模式提供先验。
2. 用户问题、RAG 文档、工具结果提供证据。
3. 系统最终回答应该基于证据更新后的判断。

如果没有外部证据，模型很容易依赖语言先验猜答案。

## 7.3 语言先验和证据

LLM 有很强的语言先验。它知道“某类问题通常应该怎么回答”，但这不代表它知道当前问题的真实答案。

例如用户问：

```text
某公司 2024 年第三季度的内部故障报告结论是什么？
```

如果模型没有内部文档，它可能根据类似报告的写法编造一个很像真的答案。

这就是语言先验压过证据的问题。

RAG 的作用可以理解为给模型提供证据，让回答从“凭先验续写”变成“基于证据回答”。

面试回答：

```text
LLM 的语言先验很强，能生成看起来合理的答案，但不一定有事实依据。RAG、工具调用和引用约束的作用，就是把外部证据引入上下文，让模型不要只靠先验猜测，而是基于证据更新回答。
```

## 7.4 MAP：先验和数据的折中

MAP 是 maximum a posteriori，也就是最大后验估计。

直觉上，MLE 只看数据似然；MAP 同时考虑数据和先验。

可以直观理解为：

```text
MAP = fit the data while respecting the prior
```

在机器学习中，很多正则化都可以从 MAP 角度理解。

例如 weight decay 可以理解为一种先验：我们不希望参数过大。训练时既要拟合数据，也要避免参数变得过大。

面试回答：

```text
MLE 只最大化数据似然，MAP 会同时考虑先验和数据。很多正则化可以从 MAP 角度理解，例如 weight decay 相当于偏好较小参数的先验。它提醒我们模型训练不是只拟合数据，还要通过先验或正则化控制解的形态。
```

## 7.5 不确定性的两类来源

机器学习中常把不确定性分成两类。

第一类是 aleatoric uncertainty，也就是数据本身的随机性或噪声。

例如：

1. 用户问题含糊。
2. 标注本身有歧义。
3. 同一个 prompt 有多个合理答案。
4. 图像或语音输入质量差。

第二类是 epistemic uncertainty，也就是模型知识不足带来的不确定性。

例如：

1. 模型没见过某领域知识。
2. 当前问题分布外。
3. RAG 没有检索到证据。
4. 模型能力不足以完成推理。

两者处理方式不同。

1. 数据噪声类不确定性，可能需要澄清问题、输出多个可能答案或表达条件。
2. 知识不足类不确定性，可能需要检索、工具、人工或拒答。

面试回答：

```text
不确定性可以分为数据本身噪声和模型知识不足。前者来自输入歧义或任务本身多解，后者来自模型不了解或分布外。LLM 系统要区分这两类：歧义问题可以澄清，知识不足要检索、调用工具或承认不知道。
```

## 7.6 校准：置信度是否可信

Calibration 关注模型的置信度是否和真实正确率匹配。

如果模型对一批回答都说自己 80% 确定，理想情况下这些回答大约 80% 应该正确。

LLM 常见校准问题：

1. 错了也很自信。
2. 对简单问题过度保守。
3. 对分布外问题不会表达不知道。
4. SFT/RLHF 让语言更自信，但不一定让事实更可靠。
5. Judge model 给分稳定，但和人工不一致。

校准不是只看模型自己说“我有 90% 把握”。语言表达的置信度和真实概率未必一致。

面试回答：

```text
校准衡量模型置信度是否能反映真实正确率。LLM 可能语气自信但事实错误，所以不能直接把自然语言中的自信表达当作可靠概率。高风险场景需要用验证集、人工 gold set、工具结果或 RAG 证据来校准模型和 judge。
```

## 7.7 如何估计 LLM 的不确定性

LLM 不确定性估计没有一个万能方法。常见信号包括：

1. Token probability 或 log probability。
2. 输出 entropy。
3. 多次采样答案是否一致。
4. Self-consistency。
5. RAG 是否检索到足够证据。
6. 引用是否支持结论。
7. Verifier 或工具检查是否通过。
8. LLM judge 和人工评审是否一致。

这些信号都不是完美的。

例如 token probability 高，不代表事实正确；多次采样一致，也可能一致地错；RAG 检索到文档，也可能文档不支持答案。

成熟做法是多信号结合，并按风险等级决定是否自动回答。

## 7.8 幻觉和不确定性

幻觉往往发生在模型应该不确定但仍然生成确定答案的时候。

常见原因：

1. Next-token prediction 奖励流畅续写。
2. SFT 数据常包含完整答案，模型学会总要回答。
3. RLHF 可能偏好自信、完整、友好的回答。
4. 模型缺少外部事实验证。
5. Prompt 没有明确要求资料不足时拒答。

降低幻觉的一条主线是：让模型在证据不足时不要假装知道。

方法包括：

1. RAG 和引用约束。
2. 工具验证。
3. 训练不知道和澄清样本。
4. Abstention eval。
5. Unsupported claim 检测。
6. 高风险场景人工复核。

## 7.9 RAG 中的不确定性处理

RAG 系统中，不确定性可能来自多个环节：

1. 文档是否入库。
2. 检索是否召回正确证据。
3. Reranker 是否排序正确。
4. Context 是否被截断或污染。
5. LLM 是否基于证据回答。
6. 引用是否真的支持结论。

如果证据不足，系统应该允许模型回答“不知道”或“资料中未找到”。

面试回答：

```text
RAG 不是只要检索到文本就回答。系统要判断检索证据是否足够支持答案。如果召回为空、证据冲突或 citation 不支持结论，模型应该表达不确定、要求补充资料或拒绝编造。这需要检索评估、引用检查和资料不足样本训练。
```

## 7.10 高风险场景如何决策

不确定性在高风险场景尤其重要。

例如：

1. 医疗建议。
2. 法律判断。
3. 金融决策。
4. 安全和危险操作。
5. 企业内部权限数据。

高风险场景不能只看模型是否会回答，还要看：

1. 是否有可靠证据。
2. 是否需要专业人士确认。
3. 是否应该降低细节级别。
4. 是否需要拒答。
5. 是否需要记录审计日志。

面试回答：

```text
高风险场景中，不确定时不应该强答。系统应该按风险等级处理：低风险可以给一般信息，高风险需要引用证据、工具验证、人工复核或拒答。模型的目标不是始终回答，而是在安全边界内提供可靠帮助。
```

## 7.11 LLM-as-a-Judge 的校准

LLM judge 也是模型，也有不确定性。

常见偏差：

1. 偏好长答案。
2. 偏好某种格式。
3. 位置偏差。
4. 自我偏好。
5. 对事实错误不敏感。
6. 对高风险样本判断不稳。

校准方法：

1. 用人工 gold set 对齐。
2. 做 pairwise 且随机交换顺序。
3. 要求结构化 rubric。
4. 抽样人工复核。
5. 报告置信区间。
6. 分任务、分风险切片分析。

面试回答：

```text
LLM-as-a-judge 可以规模化评估开放式输出，但它必须校准。要用人工 gold set 检查它和人类一致性，控制位置偏差、长度偏差和格式偏差。上线决策不能只看 judge 总分，还要看切片、置信区间和人工复核。
```

## 7.12 最小可运行贝叶斯与校准审计 demo

下面这个 demo 不调用模型，也不依赖第三方库。它用 toy 数据演示：Bayes 更新、Beta-Bernoulli 可靠率估计、MAP 相比 MLE 的先验收缩、entropy / self-consistency、ECE / Brier 校准指标，以及按风险等级做 answer / verify / abstain 决策。

```python
import math
from collections import Counter


def bayes_binary(prior_h, likelihood_e_given_h, likelihood_e_given_not_h):
    numerator = likelihood_e_given_h * prior_h
    denominator = numerator + likelihood_e_given_not_h * (1 - prior_h)
    return numerator / denominator


def beta_update(alpha, beta, successes, failures):
    post_alpha = alpha + successes
    post_beta = beta + failures
    mean = post_alpha / (post_alpha + post_beta)
    map_estimate = (post_alpha - 1) / (post_alpha + post_beta - 2)
    return {"alpha": post_alpha, "beta": post_beta, "mean": mean, "map": map_estimate}


def gaussian_map_estimate(observations, noise_var, prior_mean, prior_var):
    sample_mean = sum(observations) / len(observations)
    mle = sample_mean
    precision_data = len(observations) / noise_var
    precision_prior = 1 / prior_var
    numerator = precision_data * sample_mean + precision_prior * prior_mean
    denominator = precision_data + precision_prior
    return {"mle": mle, "map": numerator / denominator}


def entropy(probs):
    return -sum(p * math.log(p) for p in probs if p > 0)


def calibration_report(records, num_bins=5):
    bins = [[] for _ in range(num_bins)]
    for conf, correct in records:
        index = min(int(conf * num_bins), num_bins - 1)
        bins[index].append((conf, correct))

    total = len(records)
    ece = 0.0
    summaries = []
    for index, bucket in enumerate(bins):
        if not bucket:
            continue
        avg_conf = sum(conf for conf, _ in bucket) / len(bucket)
        acc = sum(correct for _, correct in bucket) / len(bucket)
        gap = abs(avg_conf - acc)
        ece += len(bucket) / total * gap
        summaries.append({
            "bin": f"{index / num_bins:.1f}-{(index + 1) / num_bins:.1f}",
            "n": len(bucket),
            "avg_conf": round(avg_conf, 3),
            "acc": round(acc, 3),
            "gap": round(gap, 3),
        })

    brier = sum((conf - correct) ** 2 for conf, correct in records) / total
    return {"ece": ece, "brier": brier, "bins": summaries}


def self_consistency(answers):
    counts = Counter(answers)
    total = len(answers)
    probs = [count / total for count in counts.values()]
    top_answer, top_count = counts.most_common(1)[0]
    return {
        "top_answer": top_answer,
        "top_share": top_count / total,
        "entropy": entropy(probs),
    }


def route_decision(posterior_reliable, risk):
    if posterior_reliable < 0.60:
        return "abstain"
    if risk == "high":
        return "verify"
    return "answer"


prior_reliable = 0.70
posterior_strong = bayes_binary(prior_reliable, 0.90, 0.20)
posterior_weak = bayes_binary(prior_reliable, 0.20, 0.70)

beta_posterior = beta_update(alpha=2, beta=2, successes=8, failures=2)
point_estimates = gaussian_map_estimate(
    observations=[2.4, 2.0, 1.6],
    noise_var=0.25,
    prior_mean=0.0,
    prior_var=1.0,
)

token_distribution = [0.70, 0.20, 0.10]
sampled_answers = ["answer_a", "answer_a", "answer_b", "answer_a", "answer_c"]
uncertainty = {
    "token_entropy": entropy(token_distribution),
    "self_consistency": self_consistency(sampled_answers),
}

calibration_records = [
    (0.95, 1), (0.90, 1), (0.85, 0), (0.80, 1),
    (0.75, 1), (0.70, 0), (0.65, 1), (0.60, 0),
    (0.55, 0), (0.40, 0),
]
calibration = calibration_report(calibration_records)

decision_cases = {
    "low_risk_strong_evidence": route_decision(posterior_strong, "low"),
    "high_risk_strong_evidence": route_decision(posterior_strong, "high"),
    "low_risk_weak_evidence": route_decision(posterior_weak, "low"),
}

checks = {
    "strong_evidence_raises_posterior": posterior_strong > prior_reliable,
    "weak_evidence_lowers_posterior": posterior_weak < prior_reliable,
    "map_shrinks_mle_toward_prior": point_estimates["map"] < point_estimates["mle"],
    "calibration_detects_gap": calibration["ece"] > 0.10,
    "high_risk_requires_verification": decision_cases["high_risk_strong_evidence"] == "verify",
    "weak_evidence_abstains": decision_cases["low_risk_weak_evidence"] == "abstain",
}

print("bayes_update=", {"strong_evidence": round(posterior_strong, 3), "weak_evidence": round(posterior_weak, 3)})
print("beta_posterior=", {k: round(v, 3) for k, v in beta_posterior.items()})
print("mle_map=", {k: round(v, 3) for k, v in point_estimates.items()})
print("uncertainty=", {
    "token_entropy": round(uncertainty["token_entropy"], 3),
    "self_consistency": {
        "top_answer": uncertainty["self_consistency"]["top_answer"],
        "top_share": round(uncertainty["self_consistency"]["top_share"], 3),
        "entropy": round(uncertainty["self_consistency"]["entropy"], 3),
    },
})
print("calibration=", {
    "ece": round(calibration["ece"], 3),
    "brier": round(calibration["brier"], 3),
    "bins": calibration["bins"],
})
print("decision_cases=", decision_cases)
print("checks=", checks)
```


这个 demo 对应真实 LLM 系统中的几个面试重点：

1. RAG 证据、工具结果和引用检查都可以看作更新可靠性后验的证据。
2. MAP 不是“比 MLE 一定更准”，而是把先验约束显式纳入估计。
3. Token 熵和 self-consistency 只是信号，不是事实正确性的证明。
4. ECE 和 Brier 可以发现“模型概率看起来高，但真实正确率跟不上”的校准问题。
5. 高风险场景不应只按置信度自动回答，还要引入验证、拒答或人工复核。

## 7.13 面试高频问题

### 7.13.1 贝叶斯公式的直觉是什么

回答：

```text
贝叶斯公式描述如何根据证据更新信念。先验是看到证据前的判断，似然表示某个假设下看到证据的可能性，后验是结合证据后的判断。在 LLM 系统中，RAG 文档、工具结果和人审都可以看作更新模型先验的证据。
```

### 7.13.2 LLM 为什么会自信地错

回答：

```text
LLM 优化的是 token 条件概率，不是事实验证。SFT/RLHF 还可能强化流畅、完整和自信的回答风格。如果模型缺少证据或处于分布外，它仍可能根据语言先验生成看似合理但错误的答案。
```

### 7.13.3 如何让模型知道自己不知道

回答：

```text
需要训练、评估和系统共同处理。训练上加入资料不足、需要澄清和拒答样本；系统上接入 RAG、工具验证和引用检查；评估上看 abstention accuracy、unsupported claim rate 和高风险人工复核。不能只靠 prompt 让模型说不知道。
```

### 7.13.4 如何校准 LLM judge

回答：

```text
用人工 gold set 校准 judge，比较它和人工标注的一致性；做位置交换和长度偏差检查；固定 judge prompt 和版本；输出结构化 rubric；对低置信、高风险和分歧样本做人审复核。
```

## 7.14 常见误区

1. 把模型语气自信当成真实置信度。
2. 认为 log probability 高就事实正确。
3. 认为 RAG 检索到文档就一定可靠。
4. 只靠 prompt 解决不知道的问题。
5. 把 LLM judge 当作绝对真值。
6. 不区分输入歧义和模型知识不足。
7. 高风险场景仍然追求“尽量回答”。

## 7.16 本章总结

贝叶斯和不确定性帮助我们从“模型会回答”升级到“模型是否应该回答”。

本章最重要的结论：

1. 贝叶斯思想是根据证据更新信念。
2. LLM 有强语言先验，但先验不等于事实证据。
3. MAP 可以理解为数据和先验的折中。
4. 不确定性既可能来自数据歧义，也可能来自模型知识不足。
5. Calibration 衡量置信度是否可信。
6. ECE 和 Brier score 可以把校准问题变成可审计指标。
7. 幻觉常来自模型在不确定时仍自信生成。
8. RAG、工具、引用和人审都是管理不确定性的系统手段。
9. 高风险场景需要 answer / verify / abstain 决策，而不是一律自动回答。
10. LLM judge 也需要校准和人工复核。

面试中讲不确定性，不要只说“让模型说不知道”。要能讲清先验、证据、校准、评估、系统防护和高风险场景决策。
