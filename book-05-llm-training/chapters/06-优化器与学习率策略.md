# 第六章：优化器与学习率策略

## 本章定位

优化器是大模型训练中最容易被低估的部分。

很多人只知道：

```text
训练大模型一般用 AdamW。
```

但真实训练中，优化器选择会影响：

1. 收敛速度。
2. 训练稳定性。
3. 显存占用。
4. 通信开销。
5. batch size scaling。
6. loss spike 和 NaN 风险。
7. 最终模型泛化能力。

近几年除了 Adam、AdamW，也出现了 Lion、Adafactor、Sophia、Muon 等优化器或优化思路。尤其是 Muon 这类方法重新强调矩阵参数更新方向的几何结构，在一些大模型训练讨论中受到关注。

本章目标不是把所有优化器公式背下来，而是建立一个面试和工程都可用的优化器谱系。

## 本章目标

学完本章，你需要掌握：

1. SGD、Momentum、Adam、AdamW 的关系。
2. Adam 和 AdamW 的核心区别。
3. 为什么大模型训练长期偏爱 AdamW。
4. Adafactor、Lion、Sophia、Muon 等方法想解决什么问题。
5. Muon 的基本思想和适用边界。
6. warmup、cosine decay、weight decay、gradient clipping 如何配合优化器。
7. loss spike、NaN、训练不稳定时如何从优化器和学习率角度排查。

## 1. 优化器在训练中解决什么问题

神经网络训练的目标是最小化 loss：

```text
min_theta L(theta)
```

最简单的更新是梯度下降：

```text
theta = theta - lr * grad
```

但大模型训练里，简单 SGD 往往不够。

原因是：

1. 参数规模巨大。
2. 不同参数梯度尺度差异很大。
3. 训练数据噪声大。
4. batch size 很大。
5. mixed precision 容易带来数值问题。
6. loss landscape 复杂。

优化器本质上是在回答：

```text
拿到当前梯度后，应该朝哪个方向走，走多大一步？
```

## 2. 优化器谱系总览

| 优化器 | 核心思想 | 优点 | 主要问题 | 常见场景 |
|---|---|---|---|---|
| SGD | 直接沿负梯度更新 | 简单、泛化好 | 收敛慢、调参敏感 | CV、小模型、研究基线 |
| Momentum SGD | 累积历史梯度方向 | 减少震荡、加速 | 仍需仔细调 lr | CNN、传统深度学习 |
| Adam | 一阶动量 + 二阶自适应 | 收敛快、鲁棒 | weight decay 耦合不理想 | NLP、Transformer |
| AdamW | Adam + decoupled weight decay | 大模型默认强基线 | optimizer state 显存大 | LLM 预训练、SFT |
| Adafactor | factored second moment | 显存低 | 调参和稳定性更复杂 | T5、大模型省显存训练 |
| Lion | sign-based momentum update | 状态少、可能更省 | 需要重新调 lr/wd | 部分视觉和语言训练实验 |
| Sophia | 利用 Hessian 近似 | 提高样本效率 | 实现和超参复杂 | 研究和特定训练场景 |
| Muon | 对矩阵更新做正交化/几何约束 | 改善矩阵参数更新方向 | 新方法，适用边界仍需验证 | 新一代大模型训练探索 |

面试中最常被问的是 AdamW，但如果能补充 Muon、Adafactor、Lion 等方法，会显得你关注当前训练进展。

## 3. SGD 与 Momentum

SGD 更新公式：

```text
theta_t = theta_{t-1} - lr * g_t
```

其中 `g_t` 是当前 mini-batch 梯度。

SGD 的问题是梯度噪声大，容易震荡。

Momentum 引入历史方向：

```text
v_t = beta * v_{t-1} + g_t
theta_t = theta_{t-1} - lr * v_t
```

直觉是：

```text
如果连续很多步都朝同一个方向，就加速；如果方向来回变，就抵消震荡。
```

Momentum 是理解 Adam 的前置概念。

## 4. Adam

Adam 同时维护一阶动量和二阶动量。

```text
m_t = beta1 * m_{t-1} + (1 - beta1) * g_t
v_t = beta2 * v_{t-1} + (1 - beta2) * g_t^2
```

更新时用：

```text
theta = theta - lr * m_hat / (sqrt(v_hat) + eps)
```

其中：

1. `m_t` 表示梯度方向的滑动平均。
2. `v_t` 表示梯度平方的滑动平均。
3. 分母让不同参数有自适应学习率。

Adam 的优点：

1. 收敛快。
2. 对学习率相对不那么敏感。
3. 适合 Transformer 这种梯度尺度差异大的模型。

Adam 的问题：

1. optimizer state 大，需要保存 `m` 和 `v`。
2. weight decay 和梯度更新耦合，正则效果不够理想。
3. 大规模训练中仍然需要仔细调 warmup、lr 和 clipping。

## 5. AdamW：大模型训练默认强基线

AdamW 的核心是 decoupled weight decay。

普通 Adam 中，L2 正则会进入梯度项。

AdamW 把 weight decay 从梯度更新里解耦出来：

```text
theta = theta - lr * adam_update
theta = theta - lr * weight_decay * theta
```

直觉是：

```text
Adam 负责根据梯度找方向，weight decay 单独负责让参数不要无限变大。
```

为什么大模型常用 AdamW？

1. Transformer 上经验稳定。
2. 收敛速度快。
3. 和 warmup + cosine decay 配合成熟。
4. 工具链支持完善。
5. 训练 recipe 和经验很多。

常见超参起点：

```text
beta1 = 0.9
beta2 = 0.95 或 0.999
eps = 1e-8
weight_decay = 0.01 到 0.1
```

注意：不同模型规模、batch size、数据质量和训练阶段会改变最优超参。

## 6. Adafactor：省显存的 Adam 替代

AdamW 的显存开销很大。

对每个参数，要保存：

```text
parameter
gradient
first moment
second moment
```

如果是 mixed precision，还可能有 master weights。

Adafactor 的思路是对二阶矩估计做 factored approximation。

对于一个矩阵参数，不完整保存每个元素的二阶矩，而是保存行统计和列统计。

优点：

1. 明显降低 optimizer state 显存。
2. 适合超大模型或显存紧张场景。
3. 曾在 T5 等模型训练中使用。

缺点：

1. 调参更复杂。
2. 稳定性依赖具体 recipe。
3. 不一定总能替代 AdamW。

面试表达：

```text
Adafactor 主要解决 Adam 二阶状态显存过大的问题，通过对矩阵二阶矩做分解近似来节省显存，但它的训练稳定性和超参选择需要更谨慎。
```

## 7. Lion：更少状态的符号更新

Lion 是一种使用 momentum 和 sign update 的优化器。

它不像 Adam 那样维护完整二阶矩。

直觉上：

```text
用动量估计方向，再用 sign 决定更新方向。
```

优点：

1. optimizer state 更少。
2. 更新形式简单。
3. 在部分视觉和语言任务上有不错结果。

缺点：

1. 学习率和 weight decay 通常不能直接沿用 AdamW。
2. 对大模型预训练的通用性需要结合具体实验判断。
3. sign update 可能带来不同的稳定性特征。

面试中可以把 Lion 作为“AdamW 之外的轻量优化器探索”来讲，不要夸成无条件替代 AdamW。

## 8. Sophia：利用二阶信息的高效近似

Sophia 这类方法关注的是：Adam 只用梯度一阶和二阶矩，能不能利用 Hessian 相关信息提高训练效率？

它试图用可承受的方式近似二阶信息。

潜在优点：

1. 提高样本效率。
2. 更好适配 loss landscape 曲率。
3. 可能减少训练 steps。

挑战：

1. 实现复杂。
2. 二阶估计成本和噪声要控制。
3. 大规模 LLM recipe 不如 AdamW 成熟。

## 9. Muon：矩阵参数更新的几何约束

Muon 是近年较受关注的新优化器思路之一。

它的核心动机是：神经网络里很多重要参数是矩阵，例如：

```text
attention projection matrix
FFN up/down/gate matrix
embedding projection
```

普通 AdamW 把矩阵参数看成很多独立元素分别做自适应更新。

Muon 更强调矩阵整体的几何结构，常见理解是对矩阵梯度或更新方向做正交化/规范化，让更新方向更接近“结构良好”的矩阵变换。

可以用直觉理解：

```text
AdamW：每个参数元素单独自适应缩放。
Muon：把二维权重矩阵作为整体，约束或改善它的更新方向。
```

为什么这可能有用？

大模型中，矩阵乘法是主要计算。

如果矩阵更新方向病态，可能导致：

1. 某些方向更新过强。
2. 表示空间塌缩或过度拉伸。
3. 训练不稳定。
4. 有效 rank 或谱结构不理想。

Muon 类方法希望通过更好的矩阵更新几何，改善训练效率和稳定性。

### 9.1 Muon 和 AdamW 的关系

Muon 不一定要替代所有参数上的 AdamW。

实际讨论中常见做法是：

```text
对二维矩阵参数使用 Muon
对 embedding、bias、norm 参数等使用 AdamW 或其他规则
```

原因是 Muon 的核心优势针对矩阵结构。

对于一维参数，例如 LayerNorm/RMSNorm 的 scale，未必适合做同样的矩阵正交化。

### 9.2 Muon 的优势

潜在优势包括：

1. 对矩阵参数更新更有结构感。
2. 可能提高训练速度或最终效果。
3. 可能降低对部分 AdamW 超参的依赖。
4. 与大模型中大量矩阵乘法结构匹配。

### 9.3 Muon 的风险和边界

需要谨慎看待：

1. 它是相对新的训练优化方向。
2. 不同模型规模和任务上的稳定结论还需要更多验证。
3. 工具链成熟度不如 AdamW。
4. 可能需要重新调学习率、weight decay、warmup 和参数分组。
5. 分布式训练中要考虑额外计算和实现成本。

面试中可以这样说：

```text
AdamW 把每个参数元素独立做自适应更新，而 Muon 更关注矩阵参数作为整体的更新几何，常通过正交化或规范化改善矩阵更新方向。它适合讨论大模型中 attention/MLP 权重矩阵的优化效率，但目前仍属于较新的优化器路线，是否替代 AdamW 要看规模、任务和训练 recipe。
```

## 10. 参数分组策略

大模型训练通常不会对所有参数使用完全相同的优化规则。

常见参数分组：

```text
1. weight matrix：使用 AdamW/Muon，启用 weight decay。
2. bias：通常不使用 weight decay。
3. norm 参数：通常不使用 weight decay。
4. embedding：可单独设置 lr 或 weight decay。
5. LoRA 参数：微调时常设置更大学习率。
```

为什么 norm 和 bias 通常不 decay？

因为它们不是普通权重矩阵，过度衰减可能影响尺度控制和训练稳定性。

## 11. Learning Rate Warmup

大模型训练几乎都会使用 warmup。

原因是训练初期参数还不稳定，直接使用大学习率容易导致 loss spike 或 NaN。

warmup 做法：

```text
lr 从 0 或很小值逐步增加到 peak lr
```

常见设置：

```text
warmup_steps = 总 steps 的 1% 到 5%
```

如果 warmup 太短：

```text
初期容易不稳定。
```

如果 warmup 太长：

```text
训练前期学习太慢，浪费计算。
```

## 12. Cosine Decay

warmup 后常用 cosine decay：

```text
lr 从 peak lr 平滑下降到 min lr
```

优点：

1. 下降平滑。
2. 后期有利于收敛。
3. 大模型 recipe 成熟。

常见组合：

```text
linear warmup + cosine decay
```

## 13. Weight Decay

weight decay 控制参数规模，帮助泛化和稳定训练。

大模型里要注意：

1. AdamW 中 weight decay 是 decoupled。
2. norm 和 bias 通常不 decay。
3. weight decay 过大可能欠拟合。
4. weight decay 过小可能参数范数变大，泛化变差。

对于 LoRA/SFT，weight decay 常常要重新调，不要照搬预训练设置。

## 14. Gradient Clipping

Gradient clipping 用于防止梯度爆炸。

常见做法是 global norm clipping：

```text
if ||g|| > threshold:
    g = g * threshold / ||g||
```

大模型训练中常见阈值：

```text
clip_grad_norm = 1.0
```

但这不是固定答案。

如果经常触发 clipping，说明可能存在：

1. 学习率过大。
2. batch 数据异常。
3. loss scale 问题。
4. 模型初始化或数值精度问题。

## 15. Batch Size Scaling

batch size 和学习率强相关。

batch size 增大后，梯度噪声降低，可以适当增大学习率。

但大模型训练不是简单线性关系。

需要关注：

1. global batch size。
2. gradient accumulation。
3. token batch size，而不是 sample 数。
4. sequence length 变化。
5. loss spike 风险。

对于 LLM，常说 batch size 时更应该说：

```text
每步训练多少 token
```

而不是只说多少条样本。

## 16. 常见优化器配置示例

### 16.1 预训练 AdamW 配置

```text
optimizer: AdamW
beta1: 0.9
beta2: 0.95
eps: 1e-8
weight_decay: 0.1
lr_schedule: linear warmup + cosine decay
grad_clip: 1.0
```

### 16.2 SFT 配置

```text
optimizer: AdamW
lr: 通常小于预训练或按数据规模调节
weight_decay: 0.0 到 0.1
warmup_ratio: 0.03
grad_clip: 1.0
```

### 16.3 LoRA 微调配置

```text
optimizer: AdamW
lr: 通常比全参微调更大
weight_decay: 常较小
trainable_params: LoRA A/B matrix
```

### 16.4 Muon 探索配置

```text
matrix_params: Muon
norm/bias/embedding_params: AdamW 或单独规则
lr: 需要重新搜索
warmup: 建议保守设置
eval: 必须和 AdamW baseline 对比
```

## 17. 训练不稳定时如何排查

如果出现 loss spike、NaN 或训练发散，可以按下面顺序排查。

### 17.1 学习率是否过大

现象：

```text
loss 突然升高
grad norm 爆炸
参数范数异常增大
```

处理：

```text
降低 peak lr
延长 warmup
增大 grad clipping
回滚 checkpoint
```

### 17.2 warmup 是否太短

训练初期 spike 常常和 warmup 太短有关。

处理：

```text
warmup_steps 增加 2 到 5 倍做对比实验
```

### 17.3 beta2 是否不合适

AdamW 的 `beta2` 控制二阶矩平滑。

`beta2` 太大，二阶统计响应慢。

`beta2` 太小，更新可能更噪。

大模型预训练中常见 `0.95` 或 `0.999`，需要结合 batch size 和训练稳定性调。

### 17.4 weight decay 是否过强

weight decay 过大可能导致欠拟合或能力变差。

尤其是 SFT、RLHF 和 LoRA 阶段，不要无脑沿用预训练 decay。

### 17.5 参数分组是否错误

常见错误：

```text
对 norm 参数使用 weight decay
对 frozen 参数传入 optimizer
LoRA 参数学习率过大
embedding 使用不合适 decay
```

### 17.6 新优化器是否没有重新调参

从 AdamW 换成 Muon、Lion、Sophia 时，不能照搬 AdamW 超参。

必须重新搜索：

```text
lr
weight_decay
warmup
grad_clip
参数分组
```

## 18. 面试高频问法

### 问法 1：Adam 和 AdamW 有什么区别？

可以这样答：

```text
Adam 使用一阶动量和二阶动量做自适应更新，但普通 L2 正则会和 Adam 的梯度归一化耦合。AdamW 把 weight decay 从梯度更新里解耦出来，先按 Adam 方向更新，再单独对参数做衰减，因此正则行为更清晰，也成为 Transformer 和 LLM 训练中的常用默认优化器。
```

### 问法 2：为什么大模型训练常用 AdamW？

可以这样答：

```text
因为 Transformer 参数多、梯度尺度差异大，AdamW 的自适应学习率和动量机制能提供较快且稳定的收敛。同时 decoupled weight decay 更适合控制参数范数。它和 warmup、cosine decay、gradient clipping 的训练 recipe 非常成熟，所以是大模型预训练和后训练的强基线。
```

### 问法 3：Muon 和 AdamW 有什么不同？

可以这样答：

```text
AdamW 基本把参数元素独立看待，维护一阶和二阶矩做自适应更新。Muon 更关注矩阵参数整体的更新几何，通常对矩阵梯度或更新方向做正交化或规范化，让 attention、MLP 这类二维权重矩阵的更新更有结构。它是比较新的优化器方向，潜力在于提升矩阵参数训练效率，但是否稳定优于 AdamW 需要具体规模和 recipe 验证。
```

### 问法 4：Adafactor 解决什么问题？

可以这样答：

```text
Adafactor 主要解决 Adam/AdamW optimizer state 显存过大的问题。它对矩阵参数的二阶矩估计做分解近似，只保存行列统计，从而减少显存开销。代价是训练稳定性和超参选择更复杂，不一定能无脑替代 AdamW。
```

### 问法 5：训练 loss spike 时如何从优化器角度排查？

可以这样答：

```text
我会先看学习率是否过大、warmup 是否太短、grad norm 是否爆炸、gradient clipping 是否生效。然后检查 AdamW 的 beta2、eps、weight decay 和参数分组，确认 norm/bias 是否错误 decay，LoRA 或新参数是否学习率过大。如果使用 Muon/Lion 等新优化器，还要确认没有照搬 AdamW 超参，并和 AdamW baseline 对比。
```

## 19. 本章小结

本章系统介绍了大模型训练中的优化器和学习率策略。

核心结论如下：

1. 优化器决定拿到梯度后如何更新参数。
2. SGD 简单但大模型训练中通常不如 Adam 系方法稳定高效。
3. Adam 使用一阶和二阶动量做自适应更新。
4. AdamW 通过 decoupled weight decay 成为 LLM 训练强基线。
5. Adafactor 主要解决 optimizer state 显存问题。
6. Lion 用更少状态和 sign update 探索轻量优化。
7. Sophia 代表利用二阶信息近似提高训练效率的方向。
8. Muon 关注矩阵参数整体更新几何，是当前值得关注的新优化器路线。
9. 新优化器不能无脑替换 AdamW，必须重新调 lr、weight decay、warmup 和参数分组。
10. warmup、cosine decay、gradient clipping、batch size scaling 和优化器必须一起看。
11. 训练不稳定时，优化器和学习率策略是最先排查的方向之一。
