# 第十二章：多模态训练

## 本章目标

理解图文、语音、视频等多模态模型的训练流程。

## 核心议题

1. CLIP 图文对比学习。
2. VLM 的 vision encoder、connector、LLM 联合训练。
3. 多模态 instruction tuning。
4. Diffusion 模型训练。
5. Whisper 类 ASR 训练。
6. 视频生成训练的数据和算力挑战。

## 面试重点

多模态训练要关注模态对齐、数据质量、时序一致性、评估难度和安全风险。

## 本章资料边界

本章第二轮精修时对照了 CLIP、BLIP-2、LLaVA、Flamingo、Whisper、DDPM、Latent Diffusion 和视频 diffusion 相关论文。这里不把多模态训练扩展成完整多模态专题，而是聚焦训练全流程面试中必须能讲清的几件事：

1. 图文对比学习为什么能得到共享语义空间。
2. VLM 为什么通常需要 vision encoder、connector 和 LLM 分阶段对齐。
3. 多模态 instruction tuning 中 image token、chat template 和 label mask 的边界。
4. 图像、音频、视频和 diffusion 训练目标如何不同。
5. 多模态数据审计、token 成本和评估门禁如何设计。

## 为什么多模态训练和纯文本训练不一样

纯文本训练只处理 token 序列。

多模态训练要处理：

1. 图像。
2. 音频。
3. 视频。
4. 文本。
5. 甚至结构化元数据和工具信号。

不同模态之间的单位、长度、分辨率和噪声特性都不一样，所以训练流程也不一样。

例如：

```text
文本是离散 token 序列
图像是像素网格
音频是时间序列波形或 codec token
视频是时间上的图像序列
```

多模态训练的核心不是把不同模态简单拼一起，而是让不同模态在同一个任务目标下对齐。

面试表达：多模态训练的关键不是“模型能不能看见图片”，而是不同模态能不能在同一个表示空间和任务目标下有效对齐。

## 1. 多模态训练的几类主任务

多模态训练常见任务包括：

| 任务 | 输入 | 输出 | 目标 |
| --- | --- | --- | --- |
| 图文对比学习 | 图像 + 文本 | 相似度 | 学对齐表示 |
| 图文生成 | 图像 + prompt | 文本 | 学会描述和问答 |
| OCR / 文档理解 | 图像或文档图 | 文本答案 | 学会读图中文字和布局 |
| 图表理解 | 图像 + 问题 | 文本答案 | 学会数值和结构关系 |
| 语音识别 | 音频 | 文本 | 学会转写 |
| 语音问答 | 音频 + 问题 | 文本答案 | 学会听懂并回答 |
| 视频理解 / 生成 | 视频 + 文本 | 文本或视频 | 学会时序信息 |

不同任务的训练目标不同，但都围绕模态对齐、表示压缩和生成能力。

## 2. CLIP 风格图文对齐

CLIP 是多模态训练里最经典的入门方法之一。

它的目标很简单：

```text
匹配的图文对更近，不匹配的更远。
```

### 2.1 训练形式

给定一个 batch 的图文对：

```text
image_i <-> text_i
```

计算图像 embedding 和文本 embedding 的相似度矩阵，然后做对比学习。

更具体地说，图像编码器输出 `u_i`，文本编码器输出 `v_i`，先做 L2 normalize：

```math
\hat{u}_i=\frac{u_i}{\|u_i\|_2},\qquad
\hat{v}_i=\frac{v_i}{\|v_i\|_2}
```

batch 内相似度矩阵可以写成：

```math
S_{ij}=\tau\cdot \hat{u}_i^\top \hat{v}_j
```

其中 `tau` 是可学习或固定的 logit scale。第 `i` 张图的正确文本是第 `i` 个文本，所以 image-to-text loss 是：

```math
L_{\mathrm{i2t}}
=-\frac{1}{B}\sum_{i=1}^{B}
\log
\frac{\exp(S_{ii})}
{\sum_{j=1}^{B}\exp(S_{ij})}
```

text-to-image loss 对相似度矩阵按列做同样的交叉熵：

```math
L_{\mathrm{t2i}}
=-\frac{1}{B}\sum_{i=1}^{B}
\log
\frac{\exp(S_{ii})}
{\sum_{j=1}^{B}\exp(S_{ji})}
```

最终常写成双向 loss：

```math
L_{\mathrm{CLIP}}=\frac{1}{2}(L_{\mathrm{i2t}}+L_{\mathrm{t2i}})
```

直觉上，这和普通分类交叉熵很像：每张图要在一个 batch 的所有文本里选中自己的 caption，每段文本也要在一个 batch 的所有图里选中自己的 image。

### 2.2 为什么重要

CLIP 学到的是共享语义空间。

这个空间可以用于：

1. 检索。
2. zero-shot 分类。
3. 作为后续 VLM 的图文对齐底座。

### 2.3 训练难点

1. 正负样本构造。
2. batch 内负样本质量。
3. 图文噪声对齐。
4. 中文/多语言效果。
5. 细粒度和 OCR 场景不足。

面试表达：CLIP 的核心价值是把图像和文本映射到同一个语义空间，为后续检索、VLM 和多模态理解打下基础。

## 3. VLM：Vision Encoder + Connector + LLM

视觉语言模型通常不是直接把图像像素喂给 LLM，而是通过三段式结构：

```text
vision encoder -> connector / projector -> LLM
```

### 3.1 Vision encoder

Vision encoder 负责把图像变成视觉表示。

常见实现包括：

1. ViT。
2. ConvNet + transformer。
3. CLIP vision tower。

### 3.2 Connector / projector

因为视觉表示和语言 token 的维度、分布不同，需要一个连接器把视觉特征映射到 LLM 可消费的表示空间。

常见 connector：

1. linear projector。
2. MLP projector。
3. Q-Former。
4. Perceiver Resampler。

### 3.3 LLM

LLM 负责把图像表示和文本 prompt 结合起来生成答案。

训练时通常有几种策略：

1. 冻结 vision encoder，只训练 connector 和 LLM。
2. 冻结 LLM，只训练 connector。
3. 联合微调部分或全部参数。

如果视觉编码器输出：

```math
V\in\mathbb{R}^{B\times N_v\times d_v}
```

其中 `B` 是 batch size，`N_v` 是视觉 token 数，`d_v` 是视觉特征维度。connector 的目标是把它投影到 LLM hidden size `d`：

```math
Z=g(V),\qquad Z\in\mathbb{R}^{B\times N_v\times d}
```

文本 token 经过 embedding 得到：

```math
E\in\mathbb{R}^{B\times T\times d}
```

最简单的拼接式 VLM 会把视觉 token 当成一段前缀：

```math
H_0=[Z;E],\qquad H_0\in\mathbb{R}^{B\times (N_v+T)\times d}
```

这里的关键不是公式本身，而是 shape 必须对齐：connector 输出的最后一维必须等于 LLM hidden size；视觉 token 数会进入上下文长度，直接影响 attention 成本和 KV Cache 显存。

面试表达：VLM 的关键不是“把图接到 LLM 上”，而是如何设计视觉到语言的桥接模块，并让两个模态在训练中对齐。

## 4. 多模态 instruction tuning

多模态 instruction tuning 是把图像、文本、OCR、图表、空间关系等任务统一成对话格式。

例如：

```text
<image>
请描述这张图。
```

或者：

```text
<image>
图中的价格是多少？
```

### 4.1 目标

1. 学会看图回答。
2. 学会 OCR。
3. 学会图表理解。
4. 学会多轮多模态对话。
5. 学会 grounding 和空间关系描述。

### 4.2 关键问题

1. image token 怎么插入。
2. label mask 怎么做。
3. 哪些 token 参与 loss。
4. 图像和文本的对齐是否一致。
5. 多轮对话中如何保留图像上下文。

多模态 SFT 的 loss 和文本 SFT 很像，但必须明确哪些位置参与监督。设序列中第 `t` 个 token 的 label 是 `y_t`，mask 是 `m_t`，集合 `A` 表示 assistant 文本答案所在的位置：

```math
m_t=
\begin{cases}
1, & t\in A\\
0, & t\notin A
\end{cases}
```

则 assistant-only loss 可以写成：

```math
L_{\mathrm{mm\_sft}}
=-
\frac{\sum_{t=1}^{T}m_t\log p_\theta(y_t\mid I,x_{1:t-1})}
{\sum_{t=1}^{T}m_t}
```

其中 `I` 表示图像或其他非文本模态输入。工程上常把 `m_t=0` 的位置对应到 `labels=-100`，让交叉熵忽略它们。最容易出错的是把 image placeholder、user prompt 或 padding token 也算进 loss，导致模型学会复读输入格式，而不是学习回答。

面试表达：多模态 instruction tuning 的本质是把视觉信息包装成可供语言模型学习的对话样本，同时保证图文对齐和 label mask 正确。

## 5. 图像 caption 和 VQA

图像 caption 任务让模型描述图片。

VQA 任务让模型回答关于图片的问题。

这两者看似相近，但训练重点不同：

| 任务 | 强调 |
| --- | --- |
| Caption | 全局语义、描述流畅性 |
| VQA | 局部细节、问答准确性 |

### 5.1 Caption 训练

输入是图像，输出是描述文本。

模型要学会：

1. 识别主要物体。
2. 把视觉内容转成自然语言。
3. 避免无中生有。

### 5.2 VQA 训练

输入是图像加问题，输出是答案。

模型要学会：

1. 根据问题关注局部区域。
2. 结合视觉和文本推理。
3. 回答简洁准确。

### 5.3 常见问题

1. 幻觉。
2. OCR 错误。
3. 计数错误。
4. 空间关系错误。
5. 对小字、遮挡和图表不敏感。

## 6. OCR 和文档理解

OCR 是多模态训练中的高频任务。

文档理解比普通图像理解更复杂，因为它不仅要识别文字，还要理解版式和结构。

### 6.1 训练数据

1. 票据。
2. 表格。
3. 截图。
4. 表单。
5. PDF 页面。
6. 带布局的扫描文档。

### 6.2 训练目标

1. 文字识别。
2. 结构抽取。
3. 布局理解。
4. 表格填充。
5. 关键信息抽取。

### 6.3 难点

1. 小字。
2. 倾斜。
3. 模糊。
4. 多栏布局。
5. 表格跨行跨列。
6. 文本和视觉证据对齐。

面试表达：OCR 和文档理解训练的关键，不只是识别文字，还要把文字、布局和语义一起建模。

## 7. 语音训练

语音模型常见训练包括 ASR、TTS 和语音问答。

### 7.1 ASR

ASR 的目标是把语音转成文本。

训练时需要处理：

1. 音频特征提取。
2. 对齐。
3. 长音频切分。
4. 噪声和口音。

如果音频采样率是 `f_s`，时长是 `L` 秒，特征提取的 frame shift 是 `h` 个采样点，那么帧数大致是：

```math
N_{\mathrm{frame}}\approx \left\lceil \frac{L f_s}{h}\right\rceil
```

例如 16 kHz 音频、20 ms frame shift 时，`h=320`，3.2 秒音频约有 160 帧。帧数决定 encoder 侧序列长度，也会影响流式 ASR 的 chunk size、首包延迟和长音频切分策略。

### 7.2 语音问答

语音问答通常是：

```text
音频 -> 转写 -> 文本理解 -> 生成答案
```

或者端到端直接从音频到文本答案。

### 7.3 难点

1. 噪声。
2. 口音。
3. 方言。
4. 数字和专有名词。
5. 长音频。
6. 流式输出和低延迟。

## 8. 视频训练

视频比图像更难，因为它包含空间和时间两种结构。

训练时要处理：

1. 帧采样。
2. 时间建模。
3. 长序列显存。
4. 动作和事件理解。
5. 时序一致性。

如果每帧图像大小是 `H*W`，patch size 是 `P`，采样 `F` 帧，那么视觉 patch token 数近似为：

```math
N_{\mathrm{video}}
=F\cdot
\left\lceil\frac{H}{P}\right\rceil
\cdot
\left\lceil\frac{W}{P}\right\rceil
```

标准 full attention 的 score cell 数量随序列长度近似二次增长：

```math
C_{\mathrm{attn}}\propto N_{\mathrm{video}}^2
```

所以视频训练常常需要帧采样、时空下采样、latent space 训练、局部/稀疏时序建模或专门的视频 diffusion 架构，否则 token 数和显存会很快失控。

视频任务包括：

1. 视频描述。
2. 视频问答。
3. 事件理解。
4. 视频生成。
5. world model 相关任务。

面试表达：视频训练的难点不仅是看懂帧内容，更是理解时序关系和长上下文中的事件演化。

## 9. Diffusion 训练

Diffusion 模型是图像和视频生成的重要路线。

### 9.1 训练目标

核心是学习去噪：

```text
给定加噪后的样本，预测噪声或还原原始样本
```

### 9.2 训练流程

1. 从真实图像采样。
2. 随机加噪到某个时间步。
3. 让模型预测噪声或 denoised 表示。
4. 用重建损失训练。

DDPM 风格训练常用的 forward noising 可以写成：

```math
x_t=\sqrt{\bar{\alpha}_t}x_0+\sqrt{1-\bar{\alpha}_t}\epsilon,
\qquad \epsilon\sim\mathcal{N}(0,I)
```

模型学习从 `x_t`、时间步 `t` 和条件 `c` 中预测噪声：

```math
L_{\mathrm{diff}}
=
\mathbb{E}_{x_0,t,\epsilon}
\left[
\left\|\epsilon-\epsilon_\theta(x_t,t,c)\right\|_2^2
\right]
```

其中 `c` 可以是文本 prompt、类别、图像条件或其他控制信号。Latent Diffusion 的核心工程动机是先把图像压到 latent space，再在 latent 上做 diffusion，从而降低高分辨率图像生成的训练和采样成本。

### 9.3 难点

1. 训练慢。
2. 采样慢。
3. 对噪声 schedule 敏感。
4. 数据和 prompt 对齐要求高。

面试表达：Diffusion 训练的核心是学会逐步去噪，而不是像语言模型那样做 next-token prediction。

## 10. 多模态数据组织

多模态训练最难的是数据格式。

常见样本可能包括：

```json
{
  "id": "sample_001",
  "image": "...",
  "audio": "...",
  "video": "...",
  "text": "...",
  "task": "vqa",
  "answer": "...",
  "metadata": {
    "domain": "ocr",
    "difficulty": "medium"
  }
}
```

需要特别注意：

1. 模态文件路径或二进制存储。
2. 对齐关系。
3. 采样率和分辨率。
4. 截断策略。
5. 元数据和标签。

可以把多模态样本抽象成：

```math
z_i=(m_i,x_i,y_i,a_i,c_i)
```

其中 `m_i` 是模态集合，例如 image/audio/video/text；`x_i` 是输入内容或文件引用；`y_i` 是答案、caption 或转写；`a_i` 是对齐信息，例如图文是否匹配、音频时间戳、OCR box 或视频事件区间；`c_i` 是任务和元数据。数据审计的目标不是只看字段非空，而是检查模态、文本、答案和证据是否真的对齐。

## 11. 多模态训练中的对齐问题

多模态训练不是把所有模态独立训好就行，而是要让它们对齐。

典型对齐问题：

1. 图像和文本语义对齐。
2. 音频和文本转写对齐。
3. 视频帧和文本事件对齐。
4. OCR 文本和视觉区域对齐。
5. 问题和答案引用同一证据对齐。

如果对齐不好，模型会出现：

1. 幻觉。
2. 答非所问。
3. 只看文本不看图。
4. 只会泛泛描述，不会精确抽取。

## 12. 多模态训练的显存和算力

多模态训练通常比纯文本更贵。

原因包括：

1. 图像和视频输入更大。
2. 视觉 encoder 额外计算。
3. 长序列更长。
4. batch 更难做大。
5. 时序建模更耗显存。

常用优化包括：

1. 冻结部分模块。
2. mixed precision。
3. activation checkpointing。
4. 更小的视觉分辨率。
5. 帧采样。
6. connector 轻量化。

### 12.1 最小可运行多模态训练审计 demo

下面这个 0 依赖 demo 同时演示五件事：

1. CLIP 双向对比 loss 如何从相似度矩阵计算。
2. VLM connector 后视觉 token 和文本 token 拼接后的 shape。
3. 多模态 SFT label mask 为什么必须忽略 image token 和 prompt token。
4. Diffusion forward noising、ASR frame 数和视频 patch token 成本的直觉。
5. 多模态样本审计如何发现空答案和图文不对齐。

输入是 toy embedding、toy label mask、toy 音视频参数和 3 条多模态样本；输出是 loss、shape、token 成本和数据审计结果。

```python
import math
from collections import Counter


def dot(a, b):
    return sum(x * y for x, y in zip(a, b))


def norm(a):
    return math.sqrt(dot(a, a))


def normalize(a):
    n = norm(a)
    return [x / n for x in a]


def softmax(row):
    m = max(row)
    exps = [math.exp(x - m) for x in row]
    z = sum(exps)
    return [x / z for x in exps]


def cross_entropy(logits, target):
    probs = softmax(logits)
    return -math.log(probs[target])


# 1. CLIP-style bidirectional contrastive loss.
image_emb = [
    normalize([0.9, 0.1, 0.0]),
    normalize([0.1, 0.8, 0.2]),
    normalize([0.0, 0.2, 0.9]),
]
text_emb = [
    normalize([0.8, 0.2, 0.0]),
    normalize([0.0, 0.9, 0.1]),
    normalize([0.1, 0.1, 0.9]),
]
logit_scale = 8.0
sim = [[logit_scale * dot(img, txt) for txt in text_emb] for img in image_emb]
i2t_loss = sum(cross_entropy(row, i) for i, row in enumerate(sim)) / len(sim)
t2i_loss = sum(
    cross_entropy([sim[i][j] for i in range(len(sim))], j)
    for j in range(len(sim))
) / len(sim)
clip_loss = 0.5 * (i2t_loss + t2i_loss)
top1 = [max(range(len(row)), key=lambda j: row[j]) for row in sim]
clip_top1_acc = sum(pred == i for i, pred in enumerate(top1)) / len(top1)


# 2. VLM connector shape and multimodal SFT label mask.
batch_size = 2
vision_tokens = 4
llm_dim = 6
text_lengths = [5, 3]
combined_shape = (batch_size, vision_tokens + max(text_lengths), llm_dim)
labels = [
    [-100, -100, -100, -100, -100, 31, 32, 33, 2],
    [-100, -100, -100, -100, -100, 41, 2, -100, -100],
]
valid_label_count = sum(y != -100 for row in labels for y in row)
image_tokens_ignored = all(
    row[t] == -100 for row in labels for t in range(vision_tokens)
)
prompt_tokens_ignored = labels[0][vision_tokens] == -100 and labels[1][vision_tokens] == -100
mask_ok = image_tokens_ignored and prompt_tokens_ignored and valid_label_count == 6


# 3. Diffusion forward noising.
x0 = 0.8
eps = -0.4
alpha_bars = [0.9, 0.5, 0.1]
noised = [
    round(math.sqrt(a) * x0 + math.sqrt(1 - a) * eps, 4)
    for a in alpha_bars
]


# 4. ASR frame count and video patch token cost.
frame_shift_ms = 20
audio_seconds = [3.2, 18.0]
```

## 13. 多模态安全和评估

多模态模型的评估比纯文本更难。

要关注：

1. OCR 准确性。
2. 图表理解。
3. 幻觉率。
4. 计数和空间关系。
5. 语音转写错误传播。
6. 视频时序理解。
7. 安全和隐私风险。

比如语音识别错一个否定词，后面整个问答就会偏掉。

因此多模态训练要带着评估思维，而不是只看 loss。

## 14. 常见训练策略

### 14.1 冻结视觉编码器

常用于数据不够大或训练预算有限的场景。

### 14.2 只训练 connector

适合先建立模态对齐桥接。

### 14.3 联合微调

适合数据较多、预算较高的场景。

### 14.4 分阶段训练

常见顺序是：

```text
对齐预训练 -> VLM instruction tuning -> 任务特化微调 -> 安全和回归评估
```

## 15. 面试官会怎么问

### 问法 1：VLM 是怎么训练的？

可以这样答：

```text
通常是先用 vision encoder 提取视觉特征，再通过 connector 把视觉表示映射到 LLM 可接受的空间，然后用图文对齐、caption、VQA 或多模态 instruction tuning 训练。训练时可以冻结部分模块，也可以联合微调，关键是处理好模态对齐、label mask 和数据质量。
```

### 问法 2：CLIP 和 VLM 有什么区别？

可以这样答：

```text
CLIP 主要是图文对比学习，目标是学到共享语义空间，擅长检索和 zero-shot 分类。VLM 则是在对齐基础上进一步让视觉信息进入 LLM，支持生成式问答、OCR、图表理解和多轮多模态对话。CLIP 更像表示学习，VLM 更像生成式多模态助手。
```

### 问法 3：多模态 instruction tuning 的重点是什么？

可以这样答：

```text
重点是把图像、文本、OCR、图表等任务统一成对话格式，并确保 image token、对话模板和 label mask 一致。训练的目标不仅是能回答，还要能对齐视觉证据、减少幻觉、处理多轮多模态上下文。
```

### 问法 4：Diffusion 训练和语言模型训练有什么不同？

可以这样答：

```text
语言模型通常做 next-token prediction，而 Diffusion 训练的核心是加噪后学习去噪，或者预测噪声。它更依赖噪声 schedule、采样和重建过程，训练和推理都更重。
```

### 问法 5：多模态训练最容易出什么问题？

可以这样答：

```text
最容易出的是数据对齐问题、模态噪声传播和评估困难。比如图文没对齐、OCR 错位、语音转写错误传到下游、视频时序信息丢失，都会导致模型看起来 loss 正常但实际回答很差，所以必须结合分任务评估、bad case 分析和安全检查。
```

## 16. 本章小结

本章核心结论：

1. 多模态训练和纯文本训练的输入结构不同，难点在模态对齐。
2. CLIP 通过图文对比学习建立共享语义空间。
3. VLM 通常由 vision encoder、connector 和 LLM 组成。
4. 多模态 instruction tuning 把图像和对话统一成生成式任务。
5. OCR、文档理解、语音、视频和 Diffusion 都有各自不同的训练目标。
6. 多模态数据的组织、对齐和元数据管理很关键。
7. 多模态训练通常更贵，显存和算力压力更大。
8. 多模态评估要关注幻觉、OCR、图表、时序和安全风险。
9. 训练策略常是分阶段、分模块、分任务进行。
10. 面试中要强调模态对齐、数据质量、训练成本和评估难度。
