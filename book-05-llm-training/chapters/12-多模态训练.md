# 第十二章：多模态训练

## 本章目标

理解图文、语音、视频等多模态模型的训练流程。

## 核心议题

1. CLIP 图文对比学习。
2. VLM 的 vision encoder、connector、LLM 联合训练。
3. 多模态 instruction tuning。
4. Diffusion 模型训练。
5. Whisper 类 ASR 训练。
6. 视频生成训练的数据和算力挑战。

## 面试重点

多模态训练要关注模态对齐、数据质量、时序一致性、评估难度和安全风险。

## 为什么多模态训练和纯文本训练不一样

纯文本训练只处理 token 序列。

多模态训练要处理：

1. 图像。
2. 音频。
3. 视频。
4. 文本。
5. 甚至结构化元数据和工具信号。

不同模态之间的单位、长度、分辨率和噪声特性都不一样，所以训练流程也不一样。

例如：

```text
文本是离散 token 序列
图像是像素网格
音频是时间序列波形或 codec token
视频是时间上的图像序列
```

多模态训练的核心不是把不同模态简单拼一起，而是让不同模态在同一个任务目标下对齐。

面试表达：多模态训练的关键不是“模型能不能看见图片”，而是不同模态能不能在同一个表示空间和任务目标下有效对齐。

## 1. 多模态训练的几类主任务

多模态训练常见任务包括：

| 任务 | 输入 | 输出 | 目标 |
| --- | --- | --- | --- |
| 图文对比学习 | 图像 + 文本 | 相似度 | 学对齐表示 |
| 图文生成 | 图像 + prompt | 文本 | 学会描述和问答 |
| OCR / 文档理解 | 图像或文档图 | 文本答案 | 学会读图中文字和布局 |
| 图表理解 | 图像 + 问题 | 文本答案 | 学会数值和结构关系 |
| 语音识别 | 音频 | 文本 | 学会转写 |
| 语音问答 | 音频 + 问题 | 文本答案 | 学会听懂并回答 |
| 视频理解 / 生成 | 视频 + 文本 | 文本或视频 | 学会时序信息 |

不同任务的训练目标不同，但都围绕模态对齐、表示压缩和生成能力。

## 2. CLIP 风格图文对齐

CLIP 是多模态训练里最经典的入门方法之一。

它的目标很简单：

```text
匹配的图文对更近，不匹配的更远。
```

### 2.1 训练形式

给定一个 batch 的图文对：

```text
image_i <-> text_i
```

计算图像 embedding 和文本 embedding 的相似度矩阵，然后做对比学习。

### 2.2 为什么重要

CLIP 学到的是共享语义空间。

这个空间可以用于：

1. 检索。
2. zero-shot 分类。
3. 作为后续 VLM 的图文对齐底座。

### 2.3 训练难点

1. 正负样本构造。
2. batch 内负样本质量。
3. 图文噪声对齐。
4. 中文/多语言效果。
5. 细粒度和 OCR 场景不足。

面试表达：CLIP 的核心价值是把图像和文本映射到同一个语义空间，为后续检索、VLM 和多模态理解打下基础。

## 3. VLM：Vision Encoder + Connector + LLM

视觉语言模型通常不是直接把图像像素喂给 LLM，而是通过三段式结构：

```text
vision encoder -> connector / projector -> LLM
```

### 3.1 Vision encoder

Vision encoder 负责把图像变成视觉表示。

常见实现包括：

1. ViT。
2. ConvNet + transformer。
3. CLIP vision tower。

### 3.2 Connector / projector

因为视觉表示和语言 token 的维度、分布不同，需要一个连接器把视觉特征映射到 LLM 可消费的表示空间。

常见 connector：

1. linear projector。
2. MLP projector。
3. Q-Former。
4. Perceiver Resampler。

### 3.3 LLM

LLM 负责把图像表示和文本 prompt 结合起来生成答案。

训练时通常有几种策略：

1. 冻结 vision encoder，只训练 connector 和 LLM。
2. 冻结 LLM，只训练 connector。
3. 联合微调部分或全部参数。

面试表达：VLM 的关键不是“把图接到 LLM 上”，而是如何设计视觉到语言的桥接模块，并让两个模态在训练中对齐。

## 4. 多模态 instruction tuning

多模态 instruction tuning 是把图像、文本、OCR、图表、空间关系等任务统一成对话格式。

例如：

```text
<image>
请描述这张图。
```

或者：

```text
<image>
图中的价格是多少？
```

### 4.1 目标

1. 学会看图回答。
2. 学会 OCR。
3. 学会图表理解。
4. 学会多轮多模态对话。
5. 学会 grounding 和空间关系描述。

### 4.2 关键问题

1. image token 怎么插入。
2. label mask 怎么做。
3. 哪些 token 参与 loss。
4. 图像和文本的对齐是否一致。
5. 多轮对话中如何保留图像上下文。

面试表达：多模态 instruction tuning 的本质是把视觉信息包装成可供语言模型学习的对话样本，同时保证图文对齐和 label mask 正确。

## 5. 图像 caption 和 VQA

图像 caption 任务让模型描述图片。

VQA 任务让模型回答关于图片的问题。

这两者看似相近，但训练重点不同：

| 任务 | 强调 |
| --- | --- |
| Caption | 全局语义、描述流畅性 |
| VQA | 局部细节、问答准确性 |

### 5.1 Caption 训练

输入是图像，输出是描述文本。

模型要学会：

1. 识别主要物体。
2. 把视觉内容转成自然语言。
3. 避免无中生有。

### 5.2 VQA 训练

输入是图像加问题，输出是答案。

模型要学会：

1. 根据问题关注局部区域。
2. 结合视觉和文本推理。
3. 回答简洁准确。

### 5.3 常见问题

1. 幻觉。
2. OCR 错误。
3. 计数错误。
4. 空间关系错误。
5. 对小字、遮挡和图表不敏感。

## 6. OCR 和文档理解

OCR 是多模态训练中的高频任务。

文档理解比普通图像理解更复杂，因为它不仅要识别文字，还要理解版式和结构。

### 6.1 训练数据

1. 票据。
2. 表格。
3. 截图。
4. 表单。
5. PDF 页面。
6. 带布局的扫描文档。

### 6.2 训练目标

1. 文字识别。
2. 结构抽取。
3. 布局理解。
4. 表格填充。
5. 关键信息抽取。

### 6.3 难点

1. 小字。
2. 倾斜。
3. 模糊。
4. 多栏布局。
5. 表格跨行跨列。
6. 文本和视觉证据对齐。

面试表达：OCR 和文档理解训练的关键，不只是识别文字，还要把文字、布局和语义一起建模。

## 7. 语音训练

语音模型常见训练包括 ASR、TTS 和语音问答。

### 7.1 ASR

ASR 的目标是把语音转成文本。

训练时需要处理：

1. 音频特征提取。
2. 对齐。
3. 长音频切分。
4. 噪声和口音。

### 7.2 语音问答

语音问答通常是：

```text
音频 -> 转写 -> 文本理解 -> 生成答案
```

或者端到端直接从音频到文本答案。

### 7.3 难点

1. 噪声。
2. 口音。
3. 方言。
4. 数字和专有名词。
5. 长音频。
6. 流式输出和低延迟。

## 8. 视频训练

视频比图像更难，因为它包含空间和时间两种结构。

训练时要处理：

1. 帧采样。
2. 时间建模。
3. 长序列显存。
4. 动作和事件理解。
5. 时序一致性。

视频任务包括：

1. 视频描述。
2. 视频问答。
3. 事件理解。
4. 视频生成。
5. world model 相关任务。

面试表达：视频训练的难点不仅是看懂帧内容，更是理解时序关系和长上下文中的事件演化。

## 9. Diffusion 训练

Diffusion 模型是图像和视频生成的重要路线。

### 9.1 训练目标

核心是学习去噪：

```text
给定加噪后的样本，预测噪声或还原原始样本
```

### 9.2 训练流程

1. 从真实图像采样。
2. 随机加噪到某个时间步。
3. 让模型预测噪声或 denoised 表示。
4. 用重建损失训练。

### 9.3 难点

1. 训练慢。
2. 采样慢。
3. 对噪声 schedule 敏感。
4. 数据和 prompt 对齐要求高。

面试表达：Diffusion 训练的核心是学会逐步去噪，而不是像语言模型那样做 next-token prediction。

## 10. 多模态数据组织

多模态训练最难的是数据格式。

常见样本可能包括：

```json
{
  "id": "sample_001",
  "image": "...",
  "audio": "...",
  "video": "...",
  "text": "...",
  "task": "vqa",
  "answer": "...",
  "metadata": {
    "domain": "ocr",
    "difficulty": "medium"
  }
}
```

需要特别注意：

1. 模态文件路径或二进制存储。
2. 对齐关系。
3. 采样率和分辨率。
4. 截断策略。
5. 元数据和标签。

## 11. 多模态训练中的对齐问题

多模态训练不是把所有模态独立训好就行，而是要让它们对齐。

典型对齐问题：

1. 图像和文本语义对齐。
2. 音频和文本转写对齐。
3. 视频帧和文本事件对齐。
4. OCR 文本和视觉区域对齐。
5. 问题和答案引用同一证据对齐。

如果对齐不好，模型会出现：

1. 幻觉。
2. 答非所问。
3. 只看文本不看图。
4. 只会泛泛描述，不会精确抽取。

## 12. 多模态训练的显存和算力

多模态训练通常比纯文本更贵。

原因包括：

1. 图像和视频输入更大。
2. 视觉 encoder 额外计算。
3. 长序列更长。
4. batch 更难做大。
5. 时序建模更耗显存。

常用优化包括：

1. 冻结部分模块。
2. mixed precision。
3. activation checkpointing。
4. 更小的视觉分辨率。
5. 帧采样。
6. connector 轻量化。

## 13. 多模态安全和评估

多模态模型的评估比纯文本更难。

要关注：

1. OCR 准确性。
2. 图表理解。
3. 幻觉率。
4. 计数和空间关系。
5. 语音转写错误传播。
6. 视频时序理解。
7. 安全和隐私风险。

比如语音识别错一个否定词，后面整个问答就会偏掉。

因此多模态训练要带着评估思维，而不是只看 loss。

## 14. 常见训练策略

### 14.1 冻结视觉编码器

常用于数据不够大或训练预算有限的场景。

### 14.2 只训练 connector

适合先建立模态对齐桥接。

### 14.3 联合微调

适合数据较多、预算较高的场景。

### 14.4 分阶段训练

常见顺序是：

```text
对齐预训练 -> VLM instruction tuning -> 任务特化微调 -> 安全和回归评估
```

## 15. 面试官会怎么问

### 问法 1：VLM 是怎么训练的？

可以这样答：

```text
通常是先用 vision encoder 提取视觉特征，再通过 connector 把视觉表示映射到 LLM 可接受的空间，然后用图文对齐、caption、VQA 或多模态 instruction tuning 训练。训练时可以冻结部分模块，也可以联合微调，关键是处理好模态对齐、label mask 和数据质量。
```

### 问法 2：CLIP 和 VLM 有什么区别？

可以这样答：

```text
CLIP 主要是图文对比学习，目标是学到共享语义空间，擅长检索和 zero-shot 分类。VLM 则是在对齐基础上进一步让视觉信息进入 LLM，支持生成式问答、OCR、图表理解和多轮多模态对话。CLIP 更像表示学习，VLM 更像生成式多模态助手。
```

### 问法 3：多模态 instruction tuning 的重点是什么？

可以这样答：

```text
重点是把图像、文本、OCR、图表等任务统一成对话格式，并确保 image token、对话模板和 label mask 一致。训练的目标不仅是能回答，还要能对齐视觉证据、减少幻觉、处理多轮多模态上下文。
```

### 问法 4：Diffusion 训练和语言模型训练有什么不同？

可以这样答：

```text
语言模型通常做 next-token prediction，而 Diffusion 训练的核心是加噪后学习去噪，或者预测噪声。它更依赖噪声 schedule、采样和重建过程，训练和推理都更重。
```

### 问法 5：多模态训练最容易出什么问题？

可以这样答：

```text
最容易出的是数据对齐问题、模态噪声传播和评估困难。比如图文没对齐、OCR 错位、语音转写错误传到下游、视频时序信息丢失，都会导致模型看起来 loss 正常但实际回答很差，所以必须结合分任务评估、bad case 分析和安全检查。
```

## 16. 本章小结

本章核心结论：

1. 多模态训练和纯文本训练的输入结构不同，难点在模态对齐。
2. CLIP 通过图文对比学习建立共享语义空间。
3. VLM 通常由 vision encoder、connector 和 LLM 组成。
4. 多模态 instruction tuning 把图像和对话统一成生成式任务。
5. OCR、文档理解、语音、视频和 Diffusion 都有各自不同的训练目标。
6. 多模态数据的组织、对齐和元数据管理很关键。
7. 多模态训练通常更贵，显存和算力压力更大。
8. 多模态评估要关注幻觉、OCR、图表、时序和安全风险。
9. 训练策略常是分阶段、分模块、分任务进行。
10. 面试中要强调模态对齐、数据质量、训练成本和评估难度。
