# 第五章：预训练目标与训练循环

## 本章目标

掌握 next-token prediction 的训练目标和标准训练循环。

## 核心议题

1. 自回归语言模型的概率分解。
2. cross entropy loss 与最大似然。
3. packing、sequence length、attention mask。
4. gradient accumulation。
5. 训练 loop、validation loop、日志记录。

## 面试重点

能从数据 batch 到 loss 计算再到 optimizer step 讲清楚完整训练过程。

## 为什么预训练目标看起来简单但很关键

GPT 类大语言模型的预训练目标非常简单：根据前面的 token 预测下一个 token。

```text
input:  x_1, x_2, ..., x_{t-1}
target: x_t
```

但这个简单目标一旦放到海量数据和大模型容量上，就会让模型学习语言结构、事实知识、代码模式、数学模式、推理模板和世界常识。

预训练阶段得到的是 base model。它通常擅长续写，但不一定会稳定遵循指令。后续 SFT、DPO、RLHF 或 reasoning RL，都是在这个底座上继续塑造行为。

面试表达：预训练不是让模型“背数据”，而是通过大规模 next-token prediction 学到可泛化的语言和知识表示。

### 预训练目标与训练循环的资料边界

按照 `WRITING_PLAN.md` 的要求，本章第二轮精修前核对了 Hugging Face causal language modeling 文档、Transformers `Trainer` 的 gradient accumulation 说明、Hugging Face Accelerate 梯度累积说明、PyTorch `CrossEntropyLoss(ignore_index)` 文档，以及 PyTorch AMP / `GradScaler` 文档。

## 1. 自回归语言模型的概率分解

给定一个 token 序列：

```text
x_1, x_2, ..., x_T
```

自回归语言模型把联合概率分解为：

```math
p_\theta(x_1,\ldots,x_T)=
\prod_{t=1}^{T}
p_\theta(x_t\mid x_{1:t-1})
```

也就是说，每个位置只根据前面的 token 预测当前 token。

在 causal LM 中，模型不能看到未来 token，所以需要 causal mask。

```text
位置 t 只能看 <= t 的上下文
不能看 t+1, t+2, ...
```

这个约束保证训练目标和推理过程一致：推理时模型也是从左到右逐 token 生成。

## 2. 输入和标签如何错位

语言模型训练里，输入 `x` 和标签 `y` 通常是同一段 token 序列错开一位。

例如 token 序列是：

```text
[10, 20, 30, 40, 50]
```

如果 block size 是 4：

```text
x = [10, 20, 30, 40]
y = [20, 30, 40, 50]
```

模型在每个位置预测下一个 token：

```text
看到 10       -> 预测 20
看到 10,20    -> 预测 30
看到 10,20,30 -> 预测 40
```

实现时常见 shape：

```text
input_ids: [B, T]
labels:    [B, T]
logits:    [B, T, V]
```

其中：

1. `B` 是 batch size。
2. `T` 是 sequence length。
3. `V` 是 vocabulary size。

如果原始训练块是：

```math
z=(z_1,\ldots,z_T)
```

那么输入和标签通常是：

```math
x=(z_1,\ldots,z_{T-1})
```

```math
y=(z_2,\ldots,z_T)
```

模型输出 logits：

```math
\ell_\theta\in \mathbb{R}^{B\times (T-1)\times V}
```

这里最容易错的是 off-by-one：logits 的第 `t` 个位置预测的是 `z_{t+1}`，不是 `z_t` 本身。

## 3. Cross entropy loss 与最大似然

模型输出 logits 后，会对词表做 softmax，得到每个 token 的概率分布。

训练目标是让真实下一个 token 的概率尽可能高。

对单个位置来说，cross entropy loss 可以写成：

```math
\ell_{b,t}=
-\log p_\theta(y_{b,t}\mid x_{b,\le t})
```

用 logits 展开就是：

```math
\ell_{b,t}=
-a_{b,t,y_{b,t}}
+\log\sum_{v=1}^{V}\exp(a_{b,t,v})
```

其中 `a` 是模型输出 logits，`v` 遍历词表。实际实现通常使用数值稳定的 `cross_entropy`，不要先手写 softmax 再取 log。

整段序列的 loss 通常是所有有效位置的平均：

```math
L=
\frac{
\sum_{b,t}m_{b,t}\ell_{b,t}
}{
\max\left(\sum_{b,t}m_{b,t},1\right)
}
```

其中 `m_{b,t}=1` 表示该位置参与 loss，`m_{b,t}=0` 表示 padding 或其他忽略位置。PyTorch / Hugging Face 中常见做法是把忽略位置的 label 设为 `-100`。

从概率角度看，最小化 cross entropy 等价于最大化训练数据的似然。

面试表达：next-token prediction 的训练本质是最大似然估计，cross entropy 是对应的负对数似然损失。

## 4. Attention mask 和 causal mask

训练 decoder-only 模型时必须使用 causal mask。

一个长度为 4 的序列，允许看的位置可以表示为：

```text
1 0 0 0
1 1 0 0
1 1 1 0
1 1 1 1
```

第 3 个位置可以看前 3 个 token，但不能看第 4 个 token。

如果没有 causal mask，模型训练时会偷看答案，loss 会异常好看，但推理时完全不成立。

causal mask 可以写成：

```math
C_{i,j}=\mathbf{1}[j\le i]
```

scaled dot-product attention 加 mask 后可以理解为：

```math
A=
\mathrm{softmax}
\left(
\frac{QK^\top}{\sqrt{d_h}}+M
\right)
```

其中未来位置的 `M_i,j` 取一个很大的负数，使 softmax 后概率接近 0。

### Padding mask

如果 batch 中有 padding，还需要 padding mask，避免模型关注 PAD token，也避免 PAD 位置参与 loss。

预训练常用 packing 减少 padding，但 SFT 或变长样本中 padding mask 依然很重要。

padding attention mask 可以写成：

```math
a_{b,t}=\mathbf{1}[x_{b,t}\ne z_{\mathrm{pad}}]
```

label mask 则通常写成：

```math
m_{b,t}=\mathbf{1}[y_{b,t}\ne -100]
```

注意 attention mask 和 label mask 不是同一个东西：前者控制模型能不能看某个输入位置，后者控制某个 label 位置是否贡献 loss。

## 5. Packing 与 sequence length

预训练数据通常来自很多长短不同的文档。如果每篇文档单独 padding 到固定长度，会浪费大量计算。

Packing 的做法是把多个文档拼接成固定长度 block。

```text
doc1 <eos> doc2 <eos> doc3 <eos> ... -> chunks of length T
```

这样可以减少 padding，提高 GPU 利用率。

### Packing 的好处

1. batch shape 稳定。
2. 减少 padding 浪费。
3. 提高 tokens/sec。
4. 更适合大规模流式数据加载。

### Packing 的风险

1. 文档边界处理不当。
2. EOS 插入不一致。
3. 不相关文档被错误看成连续上下文。
4. 短文档过多导致上下文结构不自然。

对普通预训练，跨文档拼接通常可以接受，但需要 EOS 或边界设计让模型知道文档结束。

设每个文档编码后追加 EOS：

```math
s_i=T(d_i)\oplus [z_{\mathrm{eos}}]
```

拼成总 token 流：

```math
S=s_1\oplus s_2\oplus \cdots \oplus s_n
```

固定 sequence length 为 `T_seq` 时，第 `k` 个 block 是：

```math
B_k=S_{kT_{\mathrm{seq}}:(k+1)T_{\mathrm{seq}}-1}
```

如果最后一个 block 不足长度，再用 PAD 补齐，并把对应 labels 设为 ignore index。

## 6. 一个最小训练 batch

一个 batch 至少包含：

```text
input_ids
labels
attention_mask
```

例如：

```python
batch = {
    "input_ids": input_ids,        # [B, T]
    "labels": labels,              # [B, T]
    "attention_mask": attention_mask,  # [B, T]
}
```

对于纯 causal LM，有些框架会在模型内部自动构造 causal mask，但 padding mask 和 label ignore 仍要正确处理。

常见 ignore index 是：

```python
IGNORE_INDEX = -100
```

PyTorch 的 cross entropy 默认会忽略 label 为 `-100` 的位置。

## 7. 标准训练循环

一个简化训练循环如下：

```python
for step, batch in enumerate(train_loader):
    input_ids = batch["input_ids"].to(device)
    labels = batch["labels"].to(device)
    attention_mask = batch["attention_mask"].to(device)

    outputs = model(
        input_ids=input_ids,
        attention_mask=attention_mask,
        labels=labels,
    )
    loss = outputs.loss

    loss.backward()
    optimizer.step()
    scheduler.step()
    optimizer.zero_grad()
```

真实大模型训练会更复杂，还包括：

1. mixed precision。
2. gradient accumulation。
3. gradient clipping。
4. distributed data parallel。
5. checkpoint 保存。
6. 日志记录。
7. validation。
8. 异常检测。

但核心顺序不变：forward、loss、backward、optimizer step。

## 8. Gradient accumulation

大模型训练常常希望使用很大的 global batch size，但单张 GPU 显存放不下。

Gradient accumulation 的做法是把一个大 batch 拆成多个 micro-batch，累积梯度后再更新一次参数。

```text
global_batch = micro_batch * accumulation_steps * data_parallel_size
```

如果 micro batch token 数是 `B_micro`，序列长度是 `T_seq`，累积步数是 `A`，数据并行规模是 `P_dp`，那么每次 optimizer step 对应的 token 数近似是：

```math
B_{\mathrm{tok}}=
B_{\mathrm{micro}}T_{\mathrm{seq}}AP_{\mathrm{dp}}
```

对等大小 micro-batch，梯度累积的平均梯度可以写成：

```math
g=
\frac{1}{A}
\sum_{a=1}^{A}
\nabla_\theta L_a
```

如果每个 micro-batch 有效 token 数不同，更稳妥的口径是按有效 token 总数归一化：

```math
g=
\frac{
\sum_{a=1}^{A}
\sum_{t\in M_a}
\nabla_\theta \ell_t
}{
\sum_{a=1}^{A}|M_a|
}
```

其中 `M_a` 是第 `a` 个 micro-batch 中参与 loss 的 token 集合。

伪代码：

```python
optimizer.zero_grad()

for micro_step, batch in enumerate(loader):
    loss = model(**batch).loss
    loss = loss / accumulation_steps
    loss.backward()

    if (micro_step + 1) % accumulation_steps == 0:
        optimizer.step()
        scheduler.step()
        optimizer.zero_grad()
```

注意 loss 要除以 `accumulation_steps`，否则等价于把梯度放大了。

面试表达：gradient accumulation 用时间换显存，让多个 micro-batch 的梯度累积成一个更大的 effective batch。

## 9. Mixed precision 与 loss scaling

大模型训练通常使用 FP16、BF16 或混合精度来提升吞吐和降低显存。

BF16 动态范围更大，训练大模型时通常比 FP16 更稳。

FP16 训练中可能需要 loss scaling，避免梯度 underflow。

loss scaling 的核心可以写成：

```math
\tilde L=sL
```

先对放大后的 loss 反向传播，再把梯度除回去：

```math
g=
\frac{1}{s}
\nabla_\theta \tilde L
```

如果检测到 Inf 或 NaN，动态 loss scaling 会跳过本次 optimizer step 并降低 scale。

常见数值风险：

1. loss 变成 NaN。
2. gradient norm 爆炸。
3. logits 过大。
4. optimizer state 出现 Inf。

这些问题不一定是模型结构错，也可能来自学习率、数据异常、精度设置或实现 bug。

## 10. Validation loop

训练过程中必须定期做 validation。

Validation loop 和 training loop 类似，但不反向传播，也不更新参数。

```python
model.eval()
val_losses = []

with torch.no_grad():
    for batch in val_loader:
        outputs = model(**batch)
        val_losses.append(outputs.loss.item())

model.train()
val_loss = sum(val_losses) / len(val_losses)
```

Validation 的作用：

1. 判断模型是否泛化。
2. 发现过拟合或数据问题。
3. 辅助 checkpoint 选择。
4. 观察不同 domain 的 loss。

不要只看 train loss。train loss 下降不代表模型能力一定提升，尤其当数据重复、污染或评估维度不足时。

validation loss 可以按有效 token 平均：

```math
L_{\mathrm{val}}=
\frac{
\sum_{b,t}m_{b,t}\ell_{b,t}
}{
\sum_{b,t}m_{b,t}
}
```

困惑度是：

```math
\mathrm{PPL}=\exp(L_{\mathrm{val}})
```

PPL 对 base model 有参考价值，但不能替代下游能力评估、安全评估和污染检查。

## 11. 日志应该记录什么

预训练日志至少要记录四类指标。

| 类别 | 指标 |
| --- | --- |
| 优化指标 | train loss、val loss、learning rate、grad norm |
| 系统指标 | tokens/sec、GPU utilization、MFU、dataloader time |
| 数值指标 | NaN、Inf、loss spike、gradient clipping ratio |
| 训练状态 | step、epoch/token count、checkpoint、数据版本 |

如果只记录 loss，出了问题很难定位。

例如：

1. loss 不变，可能是学习率太小，也可能是数据重复或 label 错位。
2. tokens/sec 下降，可能是数据加载慢、通信慢或 checkpoint 卡住。
3. val loss 上升，可能是过拟合，也可能是训练数据和验证数据分布差异。

面试表达：训练日志要同时覆盖优化健康、系统健康、数值健康和数据健康。

## 12. Checkpoint 保存与恢复

大模型训练很贵，必须定期保存 checkpoint。

一个完整 checkpoint 通常包括：

1. model weights。
2. optimizer state。
3. scheduler state。
4. random seed / RNG state。
5. 当前 step。
6. tokenizer 和 config。
7. 数据迭代位置或数据版本。

只保存模型权重可以用于推理，但不一定能无缝恢复训练。

恢复训练时，如果 optimizer state、learning rate schedule 或数据位置丢失，训练曲线可能发生跳变。

一个训练 checkpoint 可以抽象为：

```math
C_s=
(\theta_s,o_s,r_s,u_s,d_s,c_s)
```

其中 `theta_s` 是模型权重，`o_s` 是 optimizer state，`r_s` 是随机数状态，`u_s` 是 scheduler state，`d_s` 是数据迭代位置或数据版本，`c_s` 是模型 config 和 tokenizer。缺任意关键项，都可能无法严格复现继续训练。

## 13. Checkpoint 选择不是只看最后一步

训练最后一步不一定最好。

选择 checkpoint 要看：

1. validation loss。
2. per-domain loss。
3. 下游 benchmark。
4. 安全和毒性评估。
5. 训练稳定性。
6. 推理成本。

对于 base model，可以重点看 validation loss 和基础能力评估。对于后训练模型，还要看指令遵循、人类偏好、安全和回归测试。

一个简化的 checkpoint 选择规则可以写成：

```math
s^\*=
\arg\min_s
\left[
L_{\mathrm{val}}(s)
+\lambda R_{\mathrm{risk}}(s)
+\mu C_{\mathrm{serve}}(s)
\right]
```

其中 `R_risk` 表示安全、污染或回归风险，`C_serve` 表示服务成本。这个公式不是说一定要真的线性加权，而是提醒你 checkpoint 选择不是只看最后一步或单一 loss。

下面这个 demo 用标准库实现一个极小 causal LM 训练步。它不训练真实 Transformer，只用一个 token 到 logits 的表模拟 next-token prediction，目的是检查 label shift、`-100` ignore、causal mask、packing、gradient accumulation 缩放和一次 SGD step 后 loss 是否下降。

```python
import math

PAD = 0
EOS = 1
IGNORE_INDEX = -100
vocab = {"<pad>": PAD, "<eos>": EOS, "the": 2, "cat": 3, "sat": 4, "dog": 5, "ran": 6}
id_to_token = {idx: tok for tok, idx in vocab.items()}

docs = [
    [vocab["the"], vocab["cat"], vocab["sat"]],
    [vocab["the"], vocab["dog"], vocab["ran"]],
]
block_size = 5


def pack_documents(documents, block_size):
    stream = []
    for doc in documents:
        stream.extend(doc + [EOS])
    blocks = []
    for start in range(0, len(stream), block_size):
        block = stream[start : start + block_size]
        block += [PAD] * (block_size - len(block))
        blocks.append(block)
    return blocks


def build_batch(blocks):
    input_ids = [block[:-1] for block in blocks]
    labels = []
    attention_mask = []
    for block in blocks:
        shifted = block[1:]
        labels.append([tok if tok != PAD else IGNORE_INDEX for tok in shifted])
        attention_mask.append([1 if tok != PAD else 0 for tok in block[:-1]])
    return input_ids, labels, attention_mask


def causal_mask(seq_len):
    return [[1 if j <= i else 0 for j in range(seq_len)] for i in range(seq_len)]


def valid_pairs(input_ids, labels):
    pairs = []
    for row_x, row_y in zip(input_ids, labels):
        for x, y in zip(row_x, row_y):
            if y != IGNORE_INDEX:
                pairs.append((x, y))
    return pairs


def init_logits(vocab_size):
    return [[0.03 * (i - j) for j in range(vocab_size)] for i in range(vocab_size)]


def softmax(logits):
    m = max(logits)
    exps = [math.exp(x - m) for x in logits]
    total = sum(exps)
    return [x / total for x in exps]


def loss_and_grad(table, pairs):
    vocab_size = len(table)
    grad = [[0.0] * vocab_size for _ in range(vocab_size)]
    loss_sum = 0.0
    for prev, target in pairs:
        probs = softmax(table[prev])
        loss_sum += -math.log(probs[target])
        for j, p in enumerate(probs):
            grad[prev][j] += p
        grad[prev][target] -= 1.0
    denom = max(len(pairs), 1)
    return loss_sum / denom, [[g / denom for g in row] for row in grad]


def add_grads(left, right):
    return [
        [a + b for a, b in zip(row_a, row_b)]
        for row_a, row_b in zip(left, right)
    ]


def scale_grad(grad, scale):
    return [[g * scale for g in row] for row in grad]


def grad_norm(grad):
    return math.sqrt(sum(g * g for row in grad for g in row))


def apply_sgd(table, grad, lr):
    return [
        [w - lr * g for w, g in zip(row_w, row_g)]
        for row_w, row_g in zip(table, grad)
    ]


blocks = pack_documents(docs, block_size)
input_ids, labels, attention_mask = build_batch(blocks)
pairs = valid_pairs(input_ids, labels)
logits = init_logits(len(vocab))
full_loss, full_grad = loss_and_grad(logits, pairs)

micro_batches = [pairs[:3], pairs[3:]]
accum_grad = [[0.0] * len(vocab) for _ in range(len(vocab))]
bad_grad = [[0.0] * len(vocab) for _ in range(len(vocab))]
for micro in micro_batches:
    _, micro_grad = loss_and_grad(logits, micro)
    accum_grad = add_grads(accum_grad, scale_grad(micro_grad, 1 / len(micro_batches)))
    bad_grad = add_grads(bad_grad, micro_grad)

updated_logits = apply_sgd(logits, accum_grad, lr=0.5)
after_loss, _ = loss_and_grad(updated_logits, pairs)
max_grad_diff = max(
    abs(a - b)
    for row_a, row_b in zip(full_grad, accum_grad)
    for a, b in zip(row_a, row_b)
)

print("blocks=", [[id_to_token[t] for t in block] for block in blocks])
print("input_ids=", input_ids)
print("labels=", labels)
print("attention_mask=", attention_mask)
print("causal_mask=", causal_mask(seq_len=4))
print("valid_pairs=", [(id_to_token[x], id_to_token[y]) for x, y in pairs])
print("valid_token_count=", len(pairs))
print("initial_loss=", round(full_loss, 4))
print("after_one_step_loss=", round(after_loss, 4))
print("loss_decreased=", after_loss < full_loss)
print("max_grad_diff=", round(max_grad_diff, 10))
print("full_grad_norm=", round(grad_norm(full_grad), 6))
print("accum_grad_norm=", round(grad_norm(accum_grad), 6))
print("bad_unscaled_grad_norm=", round(grad_norm(bad_grad), 6))
print("bad_grad_ratio=", round(grad_norm(bad_grad) / grad_norm(full_grad), 2))

checks = {
    "labels_shifted": labels[0][:3] == blocks[0][1:4],
    "pad_ignored": labels[1][-1] == IGNORE_INDEX,
    "causal_lower_triangular": causal_mask(4)[2] == [1, 1, 1, 0],
    "accum_matches_full": max_grad_diff < 1e-12,
    "unscaled_accum_too_large": grad_norm(bad_grad) > grad_norm(full_grad),
    "loss_decreased": after_loss < full_loss,
}
print("checks=", checks)
print("gate_pass=", all(checks.values()))
```

运行后应看到：

```text
blocks= [['the', 'cat', 'sat', '<eos>', 'the'], ['dog', 'ran', '<eos>', '<pad>', '<pad>']]
```

## 14. 常见训练 bug

### 14.1 输入标签没有错位

如果 `x` 和 `y` 没有 shift，模型可能学习复制当前 token，而不是预测下一个 token。

### 14.2 causal mask 错误

如果模型看到未来 token，训练 loss 会虚假变好。

### 14.3 PAD 参与 loss

模型会学习预测 PAD，浪费容量并影响 loss。

### 14.4 accumulation loss 没有缩放

梯度会被放大，等价于学习率异常变大。

### 14.5 train/eval 模式没切换

dropout、norm 或其他层行为可能不一致。

### 14.6 数据重复或污染

train loss 很好，但真实评估不可靠。

## 15. 一个面试级回答模板

如果面试官问：

```text
请讲一下大模型预训练的训练循环。
```

可以这样答：

```text
预训练通常使用 next-token prediction。数据先经过 tokenizer 变成 token ids，再通过 packing 构造固定长度序列。训练时 input_ids 和 labels 通常错开一位，模型在 causal mask 下只能看到当前位置之前的 token，输出每个位置对词表的 logits，然后用 cross entropy 计算真实下一个 token 的负对数似然。

训练循环上，每个 batch 先 forward 得到 loss，再 backward 计算梯度，必要时做 gradient accumulation 和 gradient clipping，然后 optimizer step、scheduler step、zero grad。过程中会使用 mixed precision 和分布式训练提升效率。训练还要定期跑 validation，记录 train loss、val loss、learning rate、grad norm、tokens/sec、NaN/Inf、checkpoint 等指标，并根据 validation 和下游评估选择 checkpoint。
```

## 16. 本章小结

本章核心结论：

1. 预训练目标通常是 next-token prediction。
2. 自回归语言模型把联合概率分解为逐 token 条件概率。
3. Cross entropy 等价于最大似然的负对数形式。
4. 输入和标签通常是同一序列错开一位。
5. Causal mask 防止模型偷看未来 token。
6. Packing 可以提高训练效率，但要处理好文档边界和 EOS。
7. Gradient accumulation 用多个 micro-batch 模拟更大的 global batch。
8. Validation loop、日志和 checkpoint 是训练工程不可缺少的部分。
9. 训练 bug 常来自 label shift、mask、padding、accumulation 和数据问题。
10. 面试中要能从 batch 到 loss、backward、optimizer step、validation 和 checkpoint 讲完整闭环。
