# O. 面试高频对比表

条目：Adam vs AdamW、BatchNorm vs LayerNorm、BERT vs GPT、MHA vs MQA vs GQA、SFT vs RLHF vs DPO、Top-k vs Top-p、LoRA vs Full Fine-Tuning、Quantization vs Distillation。

## 如何回答对比题

一句话定义：面试中的对比题不是背两个定义，而是说明它们解决的问题、核心机制、优缺点、适用场景和工程取舍。

推荐回答结构：先给一句话区分，再讲核心机制差异，再讲优缺点和适用场景，最后给工程判断。

通用模板：A 和 B 都是为了解决某类问题，但 A 更偏向某个目标，B 更偏向另一个目标。它们的关键差异在于机制、成本、稳定性和适用场景。实际使用时要根据数据规模、模型大小、延迟、显存、质量和安全要求选择。

常见误区：只说“A 更好”或“B 更先进”，不说明约束条件；只背英文缩写，不讲为什么；只讲理论，不讲工程成本。

面试表达：对比题的核心是 trade-off。没有绝对最优，只有在具体约束下更合适。

## Adam vs AdamW

一句话区分：AdamW 把 weight decay 从 Adam 的梯度更新中解耦出来，是大模型训练中更常用的优化器变体。

Adam：使用一阶矩和二阶矩做自适应学习率更新。如果直接把 L2 正则加到梯度里，会和 Adam 的自适应缩放耦合。

AdamW：将 weight decay 作为参数衰减单独施加，不混入梯度自适应项，更符合权重衰减本意。

优点对比：Adam 收敛快、鲁棒性好；AdamW 泛化通常更好，训练 Transformer 和 LLM 时更常用。

工程场景：预训练、微调、视觉 Transformer、语言模型训练中通常优先用 AdamW。

面试表达：AdamW 的关键不是多了 weight decay，而是 decoupled weight decay。它避免 L2 正则被 Adam 的自适应学习率扭曲。

## SGD vs Adam

一句话区分：SGD 更新简单、泛化常被认为较好；Adam 自适应调整每个参数学习率，收敛更快、更稳定。

SGD：按全局学习率沿梯度方向更新，可配 momentum。优点是简单、显存开销低，缺点是调参敏感、在稀疏梯度或复杂模型中收敛慢。

Adam：维护一阶矩和二阶矩，对不同参数做自适应缩放。优点是收敛快、对学习率不那么敏感，缺点是 optimizer state 显存大。

大模型场景：LLM 训练通常用 AdamW 而不是原始 SGD，因为模型规模大、梯度分布复杂、训练稳定性要求高。

面试表达：SGD 更像统一步长的梯度下降，Adam 更像给每个参数配了自适应学习率。大模型里 AdamW 是默认强基线。

## BatchNorm vs LayerNorm

一句话区分：BatchNorm 沿 batch 统计归一化，LayerNorm 沿单个样本的 hidden 维度归一化。

BatchNorm：依赖 batch 统计量，在 CNN 中常用。batch 大且分布稳定时效果好，但小 batch、变长序列和自回归生成场景不方便。

LayerNorm：对每个 token 或样本内部 hidden 维度做归一化，不依赖 batch 统计，适合 Transformer 和序列模型。

为什么 LLM 用 LayerNorm：语言模型推理时通常一个 token 一个 token 生成，batch 和序列长度动态变化，LayerNorm 更稳定、更自然。

面试表达：BatchNorm 依赖 batch 维统计，LayerNorm 不依赖 batch，所以 Transformer 和 LLM 更常用 LayerNorm 或 RMSNorm。

## LayerNorm vs RMSNorm

一句话区分：LayerNorm 同时减均值和除标准差，RMSNorm 只按均方根缩放，不做均值中心化。

LayerNorm：计算 hidden 维度均值和方差，再归一化，有 scale 和 bias 参数。

RMSNorm：只用 root mean square 做缩放，计算更简单，常用于现代 LLM。

优点对比：LayerNorm 更标准；RMSNorm 省计算、实现简单，在大模型中效果通常足够好。

面试表达：RMSNorm 可以看作简化版 LayerNorm，去掉均值中心化，保留尺度归一化，在 LLaMA 等模型中很常见。

## Pre-Norm vs Post-Norm

一句话区分：Pre-Norm 在子层前做归一化，Post-Norm 在残差相加后做归一化。

Post-Norm：原始 Transformer 常用。浅层模型可以工作，但深层训练容易梯度不稳定。

Pre-Norm：先归一化再进入 attention 或 FFN，残差路径更直接，深层 Transformer 训练更稳定。

缺点：Pre-Norm 有时最终表达能力或收敛性质需要配合额外技巧调整，但现代 LLM 普遍采用类似 Pre-Norm 结构。

面试表达：Pre-Norm 的关键优势是让梯度可以更顺畅地沿残差路径传播，因此更适合训练很深的大模型。

## BERT vs GPT

一句话区分：BERT 是 encoder-only、双向理解模型；GPT 是 decoder-only、自回归生成模型。

BERT：使用 masked language modeling，能同时看左右上下文，适合分类、抽取、匹配、序列标注等理解类任务。

GPT：使用 causal language modeling，只看左侧上下文逐 token 预测，适合生成、对话、代码、推理和工具调用。

训练目标差异：BERT 预测被 mask 的 token；GPT 预测下一个 token。

工程选择：如果任务是离线文本理解和 embedding，可考虑 encoder；如果任务是开放式生成和助手交互，decoder-only 更主流。

面试表达：BERT 强在双向表示，GPT 强在自回归生成。现代通用大模型多采用 GPT 式 decoder-only，因为生成接口统一、可扩展性强。

## Encoder-Decoder vs Decoder-Only

一句话区分：encoder-decoder 把输入编码和输出生成分开，decoder-only 用同一个自回归堆栈处理上下文和生成。

Encoder-decoder：适合翻译、摘要、文本到文本任务。encoder 双向理解输入，decoder 条件生成输出。

Decoder-only：结构更简单，训练目标统一，适合大规模 causal LM 和多任务指令模型。

大模型趋势：通用 LLM 多采用 decoder-only，因为架构统一、扩展简单、上下文学习自然。

面试表达：encoder-decoder 在条件生成上结构清晰，decoder-only 在大规模统一建模和开放式生成上更主流。

## MHA vs MQA vs GQA

一句话区分：MHA 每个 query head 有独立 K/V，MQA 所有 query head 共享一组 K/V，GQA 多组 query head 共享 K/V。

MHA：表达能力强，但 KV cache 大，长上下文推理显存和带宽压力高。

MQA：所有 query heads 共享 K/V，极大减少 KV cache，但可能损失部分质量。

GQA：把 query heads 分组，每组共享 K/V，在质量和效率之间折中。

工程场景：推理部署时 KV cache 是核心瓶颈之一，GQA/MQA 能显著降低显存和带宽成本。

面试表达：MHA、MQA、GQA 的核心差异是 K/V head 数量。它们主要是在注意力表达能力和推理缓存成本之间做 trade-off。

## Absolute Position vs RoPE vs ALiBi

一句话区分：绝对位置编码给 token 固定位置向量，RoPE 用旋转方式编码相对位置信息，ALiBi 用 attention bias 引入距离惩罚。

Absolute Position：实现简单，但外推能力有限。

RoPE：现代 LLM 常用，能在 Q/K 空间注入相对位置信息，适合 decoder-only 模型。

ALiBi：不显式学习位置向量，而是在 attention score 中加入距离相关 bias，对长度外推友好。

面试表达：位置编码的核心问题是让 Transformer 获得顺序感。长上下文场景下，外推能力和训练长度匹配非常关键。

## ReLU vs GELU vs SwiGLU

一句话区分：ReLU 是硬截断激活，GELU 是平滑概率式激活，SwiGLU 是门控 FFN 激活，现代 LLM 常用 SwiGLU 变体。

ReLU：简单高效，但负半轴直接置零。

GELU：BERT、GPT 等早期 Transformer 常用，平滑性更好。

SwiGLU：引入门控机制，提升 FFN 表达能力，在 LLaMA 等现代模型中常见。

面试表达：LLM 中 FFN 不只是两层 MLP，激活函数和门控结构会明显影响模型容量和训练效果。

## SFT vs RLHF vs DPO

一句话区分：SFT 学会模仿高质量回答，RLHF 用 reward model 和强化学习优化偏好，DPO 直接用偏好对优化模型。

SFT：输入是指令和理想回答，目标是监督学习。优点是稳定、简单，缺点是只能模仿数据分布。

RLHF：先训练 reward model，再用 PPO 等方法优化模型。优点是能优化偏好目标，缺点是复杂、不稳定、成本高。

DPO：使用 chosen/rejected 偏好对直接优化 policy。优点是简单稳定，缺点是依赖离线偏好数据质量。

工程顺序：常见 pipeline 是先 SFT，再做偏好优化。DPO 可作为比 RLHF 更轻量的偏好优化方案。

面试表达：SFT 解决“会不会按指令回答”，RLHF/DPO 解决“哪个回答更符合人类偏好”。DPO 简化了 RLHF，但不等于在所有场景都替代 RL。

## RLHF vs DPO

一句话区分：RLHF 显式训练 reward model 并用 RL 优化，DPO 不训练显式 reward model，而是直接从偏好对优化策略。

RLHF：更复杂，更适合在线探索和更通用奖励建模，但训练成本高、调参难。

DPO：更接近监督学习，训练稳定，易复现，适合离线偏好数据充分的场景。

关键差异：RLHF 是 reward modeling 加 policy optimization，DPO 是 preference objective 直接优化。

面试表达：DPO 的优势是工程简单和稳定，但它的上限仍受偏好数据覆盖和质量限制。

## RLHF vs RLAIF

一句话区分：RLHF 使用人类反馈，RLAIF 使用 AI 反馈或 AI 辅助反馈。

RLHF：反馈更接近人类偏好，但成本高、速度慢、一致性有限。

RLAIF：可规模化、成本低，适合安全原则和大规模偏好生成，但会继承 judge 模型偏差。

工程选择：关键任务仍需要人工校准，AI feedback 可用于扩展数据规模和初筛。

面试表达：RLAIF 不是完全替代人类反馈，而是在成本和规模上补充 RLHF。

## Top-k vs Top-p

一句话区分：Top-k 保留概率最高的固定 k 个 token，Top-p 保留累计概率达到 p 的最小 token 集合。

Top-k：候选集大小固定，简单可控，但不同分布下灵活性不足。

Top-p：候选集大小随概率分布变化，更自适应；分布尖锐时候选少，分布平坦时候选多。

适用场景：需要稳定输出可降低 temperature、k、p；需要创意和多样性可适当提高。

面试表达：Top-k 控制候选数量，Top-p 控制候选概率质量。两者都是在质量和多样性之间调节采样空间。

## Greedy vs Beam Search vs Sampling

一句话区分：greedy 每步选最大概率 token，beam search 保留多个高概率序列，sampling 按概率随机采样。

Greedy：确定性强、速度快，但容易保守和陷入局部最优。

Beam Search：适合翻译等目标较确定的任务，但开放式生成容易重复、僵硬。

Sampling：多样性强，适合对话、创意和多候选推理，但稳定性较差。

面试表达：解码策略要匹配任务。开放式对话不一定适合 beam search，代码和结构化任务通常更偏确定性解码。

## Temperature vs Top-p

一句话区分：temperature 改变概率分布的尖锐程度，top-p 截断候选 token 集合。

Temperature：低温让高概率 token 更突出，输出更稳定；高温让分布更平，输出更多样。

Top-p：只在累计概率阈值内采样，过滤低概率长尾 token。

组合使用：temperature 控制分布形状，top-p 控制候选范围。两者常一起调。

面试表达：temperature 是调概率分布，top-p 是裁剪候选集合。生产中要按任务质量、稳定性和多样性要求调参。

## LoRA vs Full Fine-Tuning

一句话区分：full fine-tuning 更新所有参数，LoRA 冻结基座模型，只训练低秩 adapter。

Full fine-tuning：表达能力强，适合数据充分、算力充足、需要深度改变模型行为的任务。缺点是显存高、存储大、容易遗忘。

LoRA：训练参数少、成本低、易保存多个任务版本，适合领域适配和中小规模微调。

关键取舍：质量上限、训练成本、部署复杂度、多任务管理和灾难性遗忘风险。

面试表达：LoRA 不是比全量微调绝对更好，而是在资源受限和多任务部署时更实用。

## LoRA vs Adapter vs Prefix Tuning

一句话区分：LoRA 修改权重更新的低秩子空间，Adapter 插入小模块，Prefix Tuning 在输入侧学习可训练前缀。

LoRA：常用于 attention/MLP 权重，部署时可合并权重，工程友好。

Adapter：在层间插入小网络，模块化强，但推理可能增加结构开销。

Prefix Tuning：学习虚拟 token 或 KV 前缀，参数少，但表达能力和稳定性依任务而定。

面试表达：三者都是 PEFT 方法，区别在于改哪里：LoRA 改权重更新，Adapter 加模块，Prefix 改上下文或 KV 表示。

## Quantization vs Distillation

一句话区分：quantization 降低数值精度，主要减少显存和带宽；distillation 用教师模型训练学生模型，主要压缩能力到更小模型。

Quantization：把 FP16/BF16 权重或激活压到 INT8、INT4 等，推理更省显存、更快，但可能损失精度。

Distillation：让小模型学习大模型输出、logits、偏好或推理过程，得到结构更小的模型。

工程选择：如果模型结构不变但部署成本高，优先考虑量化；如果需要更小架构和更低延迟，可考虑蒸馏。

面试表达：量化压的是数值表示，蒸馏压的是模型能力和结构。

## PTQ vs QAT

一句话区分：PTQ 是训练后量化，QAT 是量化感知训练。

PTQ：不重新训练或只做少量校准，成本低，适合快速部署。

QAT：训练时模拟量化误差，让模型适应低精度，效果通常更好但成本更高。

LLM 场景：权重量化常用于降低显存，KV cache 量化和激活量化需要更谨慎评估。

面试表达：PTQ 更便宜，QAT 更稳。选择取决于可接受的精度损失和训练成本。

## FP16 vs BF16

一句话区分：FP16 精度位更多但指数范围小，BF16 指数范围接近 FP32，训练更不容易溢出。

FP16：需要 loss scaling 来缓解梯度下溢和溢出问题。

BF16：动态范围更大，训练稳定性更好，现代硬件支持后大模型训练常用。

面试表达：BF16 牺牲部分尾数精度换更大动态范围，因此大模型训练中通常比 FP16 更省心。

## FlashAttention vs Sparse Attention

一句话区分：FlashAttention 不改变 attention 数学结果，是 exact attention 的 IO-aware 实现；Sparse Attention 改变 attention 连接模式，只计算部分 token pair。

FlashAttention：通过分块和 online softmax 减少显存读写和中间矩阵存储。

Sparse Attention：通过局部窗口、全局 token 或稀疏模式降低计算量，但结果是近似或结构受限的 attention。

面试表达：FlashAttention 优化的是实现和内存访问，Sparse Attention 改变的是 attention 计算图。

## PagedAttention vs FlashAttention

一句话区分：FlashAttention 优化 attention kernel，PagedAttention 优化 KV cache 管理。

FlashAttention：主要解决单次 attention 计算中的显存访问效率。

PagedAttention：把 KV cache 分页管理，减少连续显存分配和碎片，提高 serving 场景并发能力。

面试表达：FlashAttention 更偏 kernel 计算优化，PagedAttention 更偏推理服务内存管理。

## KV Cache vs Activation Checkpointing

一句话区分：KV cache 用于推理时缓存历史 K/V，activation checkpointing 用于训练时减少激活显存。

KV Cache：自回归推理中避免重复计算历史 token 的 K/V，加速生成，但会占用显存且随序列长度增长。

Activation Checkpointing：训练时不保存全部中间激活，反向传播时重算，节省显存但增加计算。

面试表达：KV cache 是推理优化，checkpointing 是训练省显存。一个缓存避免重算，一个主动重算换显存。

## Data Parallel vs Tensor Parallel vs Pipeline Parallel

一句话区分：data parallel 切 batch，tensor parallel 切层内矩阵，pipeline parallel 切模型层。

Data Parallel：每卡一份完整模型，处理不同数据，同步梯度。简单成熟，但不解决单卡放不下模型的问题。

Tensor Parallel：把矩阵计算拆到多卡，适合单层很大，但通信频繁。

Pipeline Parallel：把不同层放到不同设备，适合模型很深，但有 pipeline bubble 和调度复杂度。

面试表达：三种并行分别从样本维、张量维和层维拆训练负载，真实大模型训练常组合使用。

## DDP vs ZeRO vs FSDP

一句话区分：DDP 复制完整模型并同步梯度，ZeRO/FSDP 对参数、梯度和 optimizer state 做分片来省显存。

DDP：实现简单，适合单卡能放下模型的场景。

ZeRO：分阶段切 optimizer state、gradient、parameter，降低数据并行冗余。

FSDP：按模块分片参数，需要时 all-gather，用完释放，和 ZeRO-3 思路相近。

面试表达：DDP 提升吞吐但不省模型状态显存，ZeRO/FSDP 用通信换显存容量。

## RAG vs Fine-Tuning

一句话区分：RAG 把知识放在外部检索库中，fine-tuning 把行为或知识部分写入模型参数。

RAG：适合知识更新频繁、私有文档、需要引用和可追溯的场景。

Fine-tuning：适合改变输出风格、任务格式、领域术语和行为模式。

关键判断：如果问题是“模型不知道最新事实”，优先 RAG；如果问题是“模型不会按这种方式回答”，考虑微调。

面试表达：RAG 解决知识访问，微调解决行为适配。不要用微调硬记频繁变化的知识库。

## Dense Retrieval vs Sparse Retrieval

一句话区分：dense retrieval 用向量语义相似度检索，sparse retrieval 用关键词和倒排索引检索。

Dense：擅长语义改写和同义表达，但对数字、代码、专有名词可能不稳。

Sparse：擅长精确匹配和可解释检索，但对语义改写能力有限。

工程方案：企业 RAG 常用 hybrid retrieval，再用 reranker 排序。

面试表达：向量检索不是万能的。真实 RAG 往往需要 dense、sparse 和 reranker 组合。

## Retriever vs Reranker

一句话区分：retriever 负责快速召回候选，reranker 负责精排候选。

Retriever：追求快和召回率，通常从大规模库中取 top-k。

Reranker：用更强模型判断 query-document 相关性，提升进入 prompt 的证据质量。

关键限制：如果 retriever 没召回正确文档，reranker 无法凭空补救。

面试表达：retriever 决定候选池，reranker 决定最终证据顺序。RAG 排障时要区分召回问题和排序问题。

## Long Context vs RAG

一句话区分：long context 让模型一次看更多内容，RAG 先从外部库中筛选相关内容再给模型看。

Long Context：适合必须完整审阅固定材料的任务，但成本高、噪声多、有效利用不一定稳定。

RAG：适合知识库很大、查询相关内容稀疏、需要动态更新和引用的场景。

组合方式：先 RAG 召回，再用较长上下文容纳证据、历史和工具结果。

面试表达：长上下文解决容量问题，RAG 解决选择问题。窗口大不代表应该把所有材料都塞进去。

## Agent vs Workflow

一句话区分：agent 由模型动态决定步骤和工具，workflow 按预定义流程执行。

Agent：灵活，适合开放任务，但不稳定、难评估、成本高。

Workflow：可控、可审计、稳定，适合标准业务流程，但灵活性差。

工程选择：生产系统常把关键路径工作流化，把开放决策点交给模型。

面试表达：Agent 不是越自由越好。高风险业务应优先可控 workflow，再在局部引入模型智能。

## Chain-of-Thought vs Tool Use

一句话区分：CoT 让模型在文本中推理，tool use 让模型调用外部系统获得计算、检索或执行结果。

CoT：适合模型内部可完成的逻辑推理和问题拆解。

Tool Use：适合计算、实时信息、数据库查询、代码运行和业务操作。

面试表达：能用工具验证的问题，不要只依赖模型自说自话的推理链。

## Human Evaluation vs LLM-as-a-Judge

一句话区分：human evaluation 用人评估，LLM-as-a-Judge 用强模型评估。

Human Evaluation：更接近真实用户和专家判断，但成本高、慢、一致性难保证。

LLM-as-a-Judge：便宜、快、可规模化，但有长度偏好、位置偏好、事实判断不可靠等风险。

工程做法：用 LLM judge 做大规模初筛，用人工做校准、仲裁和高风险样本评估。

面试表达：LLM judge 是辅助评估工具，不是真值来源。关键评估仍要有人类或可执行验证闭环。

## Accuracy vs Pass@k

一句话区分：accuracy 看单次答案是否正确，pass@k 看 k 个候选中是否至少一个正确。

Accuracy：适合分类、选择题和单答案任务。

Pass@k：适合代码、数学和可验证任务，反映多次采样后找到正确解的概率。

面试表达：pass@k 高不代表生产体验一定好，还要看生成成本、延迟、测试覆盖率和选择器质量。

## Factuality vs Grounding

一句话区分：factuality 关注输出是否真实，grounding 关注输出是否被给定证据支持。

Factuality：答案可能真实，但不一定来自当前证据。

Grounding：答案必须能由上下文或引用片段支持，即使外部世界另有事实，也不能胡乱引入。

RAG 场景：两者都重要，尤其要检查引用是否真正支持结论。

面试表达：事实正确不等于证据支持，引用存在也不等于 grounding 成立。

## Robustness vs Generalization

一句话区分：generalization 关注模型在未见样本上的泛化能力，robustness 关注面对扰动、噪声和攻击时是否稳定。

Generalization：更多是训练分布到测试分布的能力迁移。

Robustness：强调输入改写、无关噪声、对抗提示、格式变化和分布外样本下的稳定性。

面试表达：一个模型 benchmark 分数高，不代表 robustness 好。真实线上输入往往更脏、更长、更对抗。

## Over-Refusal vs Under-Refusal

一句话区分：over-refusal 是该回答却拒绝，under-refusal 是该拒绝却回答。

Over-refusal：损害 helpfulness，常见于安全策略过保守。

Under-refusal：突破安全底线，可能提供危险或违法帮助。

安全评估：必须同时看两者，否则只优化一个会牺牲另一个。

面试表达：安全模型不是拒绝越多越好，而是要在正确边界上拒绝和帮助。

## Jailbreak vs Prompt Injection

一句话区分：jailbreak 通常是用户直接诱导模型绕过安全策略，prompt injection 通常是外部内容间接注入恶意指令。

Jailbreak：例如角色扮演、忽略规则、多轮诱导、编码规避。

Prompt Injection：常见于 RAG、网页浏览、工具调用和 agent，恶意指令藏在文档或工具返回中。

防护重点：jailbreak 依赖安全对齐和输入审核；prompt injection 还需要区分可信指令和不可信数据，限制工具权限。

面试表达：prompt injection 是 LLM 应用安全问题，不只是模型安全问题；尤其要关注 RAG 和 Agent 链路。

## Helpful vs Harmless

一句话区分：helpful 要尽量帮助用户，harmless 要避免造成伤害。

冲突场景：用户请求具有双重用途，例如网络安全、化学、生物、医疗、金融和法律建议。

工程做法：低风险请求充分帮助，高风险请求拒绝危险细节并提供安全替代，模糊请求先澄清。

面试表达：对齐不是无条件听话，而是在安全边界内最大化帮助。

## Dense Model vs MoE

一句话区分：dense model 每个 token 使用全部参数，MoE 每个 token 只激活部分专家。

Dense：结构简单、训练和部署更直接，但参数和计算一起增长。

MoE：总参数量大、每 token 计算相对可控，但有路由、负载均衡、通信和部署复杂度。

面试表达：MoE 的优势是 sparse activation，用更大参数容量提升能力；代价是系统复杂度和路由稳定性。

## 本章小结

本章整理了大模型算法岗最常见的高频对比题。

核心结论如下：

1. 对比题要围绕问题、机制、优缺点、场景和 trade-off 回答。
2. AdamW、LayerNorm/RMSNorm、Pre-Norm 是现代 Transformer 训练中的高频基础。
3. BERT vs GPT、encoder-decoder vs decoder-only 是架构路线核心对比。
4. MHA、MQA、GQA 主要比较 KV cache 成本和表达能力。
5. SFT、RLHF、DPO 对应监督模仿、偏好强化和直接偏好优化。
6. 解码策略要按任务选择，不能机械说 beam search 或 sampling 更好。
7. LoRA、量化、蒸馏、FlashAttention、PagedAttention 都是在不同层面降低训练或推理成本。
8. RAG、长上下文和 Agent 分别解决知识获取、信息容量和多步行动问题。
9. 评估和安全对比题要同时关注正确性、证据、鲁棒性、过拒和漏拒。
10. 面试回答不要追求绝对结论，要体现约束条件和工程判断。
