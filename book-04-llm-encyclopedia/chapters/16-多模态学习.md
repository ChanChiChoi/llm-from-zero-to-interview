## 本章速览

一句话定义：多模态学习是让模型同时处理文本、图像、音频、视频、传感器等不同模态信息，并学习它们之间的语义对应关系。

大模型语境下，最常见的多模态模型是视觉语言模型，也就是让语言模型理解图片、回答图像问题、描述图片内容、进行 OCR、定位物体、解释图表，甚至调用工具完成复杂视觉任务。

核心问题：不同模态的数据格式不同、统计分布不同、信息密度不同。文本是离散 token，图像是连续像素或 patch，音频是时序信号，视频同时有空间和时间维度。多模态学习要解决的不是简单把它们拼在一起，而是让模型在统一语义空间里对齐、融合和推理。

典型路线：先用专门 encoder 提取非文本模态特征，再通过投影层、查询模块或 cross-attention 将这些特征接入语言模型，最后通过指令微调让模型学会用自然语言完成多模态任务。

面试表达：多模态模型的关键不是“能输入图片”，而是如何把视觉特征转成语言模型能理解的 token，以及如何保证跨模态对齐、推理可靠性和幻觉可控。

## Multimodal Learning

Multimodal Learning 指利用多个模态进行表示学习、对齐学习、融合学习和任务学习。

常见模态包括文本、图像、音频、视频、表格、网页结构、点云、传感器数据等。不同业务会关注不同组合，例如图文检索关注 image-text，语音助手关注 speech-text，自动驾驶关注 camera-lidar-text，短视频理解关注 video-audio-text。

多模态学习常见任务如下：

| 任务 | 输入 | 输出 | 典型目标 |
| --- | --- | --- | --- |
| 图文检索 | 图像和文本 | 相似度排序 | 找到与图片匹配的文本，或与文本匹配的图片 |
| Image Captioning | 图像 | 文本描述 | 描述图片内容 |
| VQA | 图像和问题 | 文本答案 | 回答视觉问题 |
| OCR | 图像 | 文本 | 识别图片中文字 |
| Grounding | 图像和文本 | 区域或框 | 将语言短语定位到图像区域 |
| 多模态对话 | 图像和对话历史 | 回复 | 围绕图片进行多轮问答 |
| 图表理解 | 图表图片和问题 | 答案或解释 | 读取图表结构并推理 |
| 视频理解 | 视频和问题 | 文本答案 | 理解事件、动作和时间关系 |

多模态学习的三个关键词：对齐、融合、生成。

对齐是让不同模态的语义表示可比较，例如“狗的图片”和“a dog”在向量空间中接近。融合是让模型联合利用多个模态的信息，例如同时看图片和问题。生成是把多模态输入转成文本、图像、音频或动作等输出。

常见误区：把多模态等同于图像分类。现代多模态大模型更关注开放式理解、指令跟随、跨模态推理和生成式交互。

## Modality Gap

Modality Gap 指不同模态之间存在天然表示差异。

文本 token 是离散符号，序列长度通常较短，但语义密度高。图像像素是连续信号，空间分辨率高，局部细节多。音频是长时序信号，对时间对齐敏感。视频同时包含空间、时间、声音和事件关系。

模态差异带来的问题包括：

| 问题 | 含义 | 影响 |
| --- | --- | --- |
| 表示空间不同 | 图像向量和文本向量分布不同 | 难以直接相似度比较 |
| 信息粒度不同 | 图像有局部区域，文本有词和句子 | 难以精确对齐 |
| 序列长度不同 | 图像 patch 或视频帧很多 | 上下文成本高 |
| 噪声形式不同 | 图片模糊、文字遮挡、语音噪声 | 鲁棒性难保证 |
| 标注方式不同 | 文本标注便宜，区域标注昂贵 | 高质量数据稀缺 |

面试表达：多模态模型的工程难点之一是 modality gap。模型需要通过对比学习、投影映射、cross-attention 或指令微调，把不同模态映射到可协同推理的空间中。

## Visual Token Budget

Visual Token Budget 指图像或视频进入 LLM 前占用的视觉 token 数量预算。它通常由分辨率、patch size、采样帧数、token 压缩方式和是否加入额外全局 token 决定。

图像 patch token 的粗略估算如下：

```math
N_{\mathrm{img}}=
\left\lceil \frac{H}{P}\right\rceil
\cdot
\left\lceil \frac{W}{P}\right\rceil
```

其中 `H` 和 `W` 是输入分辨率，`P` 是 patch size。分辨率翻倍时，patch token 数通常按面积增长，这会直接影响 VLM 的上下文长度、显存和延迟。

视频 token 还要乘以采样后的帧数：

```math
N_{\mathrm{vid}}=F' N_{\mathrm{img}},\qquad
F'=\left\lceil \frac{F}{s}\right\rceil
```

面试表达：视觉 token budget 是多模态工程的第一张成本表。OCR、图表、截图和细粒度 grounding 往往需要更多 token，但 token 越多，推理成本和延迟越高。

## Multimodal Context Budget

Multimodal Context Budget 指文本、图像、音频、视频和额外控制 token 拼接后共同占用的上下文预算。

```math
T_{\mathrm{total}}=
T_{\mathrm{text}}+
N_{\mathrm{img}}+
N_{\mathrm{vid}}+
N_{\mathrm{aud}}+
N_{\mathrm{extra}}
```

当这些 token 进入普通全 attention LLM 时，主要 attention 成本会随 `T_total` 近似二次增长。因此一个输入是否可上线，不只取决于模型能否接受图片，还取决于 `T_total` 是否小于模型上下文、显存和延迟预算。

面试表达：多模态上下文预算能把“看图能力”落到工程约束上。高分辨率、多帧视频和长音频都可能把系统推过上下文上限，即使单个样例看起来能跑。

## Multimodal Cost Audit

Multimodal Cost Audit 指上线或实验前，对多模态输入的 token 数、显存、延迟、吞吐、评估门禁和安全风险进行统一审计。

常见审计字段包括：

| 字段 | 含义 | 典型风险 |
| --- | --- | --- |
| image tokens | 图像 patch 或压缩后的视觉 token | 分辨率过高导致上下文爆炸 |
| video tokens | 采样帧乘以每帧 token | 长视频延迟高、显存高 |
| audio frames | 语音特征帧或音频 token | 实时语音首包延迟高 |
| connector shape | 视觉 hidden size 到 LLM hidden size | 维度不匹配或 token 压缩过强 |
| hallucination rate | 答案不被媒体证据支持的比例 | 流畅但不可靠 |
| injection rate | 不可信媒体文字触发错误指令的比例 | 截图或网页内指令污染 |

面试表达：多模态成本审计不是单纯算钱，而是把 token 预算、质量、延迟、幻觉和安全门禁放在同一张表里，决定某个方案能否进入真实产品。

## Multimodal Product

一句话定义：multimodal product 是把图像、文档、语音、视频或生成能力做成可评估、可控成本、可处理隐私版权、可人审、可进入真实工作流的产品系统。

和模型 demo 区别：demo 往往展示一张清晰图片、一段干净语音或一条高质量生成样例；产品要面对真实上传质量、低光照、噪声、遮挡、小字、多人说话、长视频、授权边界、安全审核、延迟成本和用户反馈。

面试表达：多模态产品不是“能看图 / 能说话 / 能生成视频”就结束，而是要证明输入质量、证据支持、切片指标、安全隐私版权、人工复核、成本延迟和业务指标都过线。

## Input Quality Pass Rate

一句话定义：input quality pass rate 是多模态输入中满足处理要求的比例，例如图片清晰、文档可读、音频信噪比足够、视频关键帧完整、文件格式和权限有效。

产品意义：多模态系统很多失败来自输入分布，而不是模型本身。输入不过线时，产品应引导重新拍摄、裁剪、重新上传、澄清或转人工。

## Multimodal Evidence Support Rate

一句话定义：multimodal evidence support rate 是输出关键断言中，被图片、OCR、ASR、视频帧、文档页、表格或引用证据支持的比例。

常见误区：图像问答、文档问答和医疗 / 工业辅助场景不能只看回答流畅度。答案碰巧正确但没有媒体证据支持，在产品审计中仍然是风险样本。

## OCR Quality Score

一句话定义：OCR quality score 是用 `1 - CER` 或字段抽取准确率等指标描述图片文字、文档小字、表格和截图文本识别质量。

面试表达：OCR 切片要单独评估。平均任务成功率可能掩盖小字、低分辨率、票据、表格、代码截图和多语言文档的失败。

## ASR Quality Score

一句话定义：ASR quality score 是用 `1 - WER`、专有名词召回、时间戳误差和噪声切片指标描述语音识别质量。

产品意义：语音助手的 ASR 错误会继续传给 LLM 和 TTS，形成级联错误。实时语音还要同时看首字延迟、打断和说话人分离。

## Generation Adoption Rate

一句话定义：generation adoption rate 是图像、音频或视频生成结果被保存、下载、进入编辑链路或用于业务草稿的比例。

面试表达：生成产品的真实价值不只是主观好看，而是采纳、可控编辑、品牌一致、授权清楚和节省创作时间。

## Media Safety Pass Rate

一句话定义：media safety pass rate 是多模态输入和输出通过安全审核的比例，覆盖暴力、色情、未成年人、深伪、冒充、危险内容和不可信媒体指令。

产品要求：多模态安全审核要覆盖输入和输出。图片、音频、视频、文档和截图中的指令都只能作为数据，不能覆盖系统或开发者指令。

## Privacy Pass Rate

一句话定义：privacy pass rate 是人脸、身份信息、会议录音、家庭环境、医疗影像和企业机密等隐私风险被正确脱敏、阻断、最小化保存或授权处理的比例。

## Copyright Pass Rate

一句话定义：copyright pass rate 是输入素材授权、参考图使用边界、生成输出使用范围、品牌素材和内容溯源通过审查的比例。

## Multimodal Product Gate

一句话定义：multimodal product gate 是多模态产品上线前的综合门禁，覆盖任务成功、输入质量、证据支持、OCR / ASR、生成采纳、安全、隐私、版权、人审、上下文预算、P95 延迟、单位成本、评估集、反馈闭环和业务指标。

面试表达：多模态产品门禁比纯文本更复杂，因为输入质量和输出风险都更不可控。少数高质量演示样例不能替代真实输入分布和安全版权审计。

## CLIP

CLIP 是 Contrastive Language-Image Pre-training 的代表方法，用大规模图文对训练图像 encoder 和文本 encoder，使匹配的图文向量相近，不匹配的图文向量远离。

CLIP 的基本结构：一个 image encoder 负责把图片编码成向量，一个 text encoder 负责把文本编码成向量。训练时，对一个 batch 中的 N 对图文样本计算 N x N 相似度矩阵，正样本是同一对图文，负样本是 batch 内其他图文组合。

简化目标如下：

```text
image_i -> image_embedding_i
text_i  -> text_embedding_i
```

$$
s_{ij}=\frac{\mathrm{cosine}(v_i,t_j)}{\tau}
$$

$$
L_{\mathrm{CLIP}}=\frac{L_{I2T}+L_{T2I}}{2}
$$

CLIP 的价值：它把图像和文本对齐到同一个语义空间，使模型可以做零样本分类、图文检索、相似度排序和视觉特征初始化。

零样本分类方式：把类别名写成 prompt，例如 “a photo of a cat”、“a photo of a dog”，再比较图片向量和各类别文本向量的相似度。相似度最高的类别作为预测结果。

优点：训练目标简单、可扩展性强、零样本能力好、迁移能力强。

缺点：对细粒度空间关系、计数、OCR、复杂推理不够强；依赖图文对质量；容易学习数据中的偏见；对 prompt 表述敏感。

面试表达：CLIP 的核心是图文对比学习。它不直接生成文本，而是学习图像和文本的共享语义空间，是很多视觉语言模型的视觉对齐基础。

## CLIP Loss

CLIP Loss 是图文双塔模型常用的双向对比学习目标。

设 batch 中有 `N` 对图文，归一化图像向量为 `v_i`，归一化文本向量为 `t_j`，相似度矩阵为：

```math
S_{ij}=\frac{v_i^\top t_j}{\tau}
```

image-to-text 方向把每张图对应的正确文本当作类别：

```math
L_{I2T}=
-\frac{1}{N}
\sum_{i=1}^{N}
\log
\frac{\exp(S_{ii})}
{\sum_{j=1}^{N}\exp(S_{ij})}
```

text-to-image 方向对 `S` 转置做同样的 cross entropy：

```math
L_{T2I}=
-\frac{1}{N}
\sum_{i=1}^{N}
\log
\frac{\exp(S_{ii})}
{\sum_{j=1}^{N}\exp(S_{ji})}
```

最终取平均：

```math
L_{\mathrm{CLIP}}=\frac{L_{I2T}+L_{T2I}}{2}
```

关键实现点：先 L2 normalize，再做矩阵乘法；`logit_scale` 通常对应 `1 / tau`，控制 softmax 分布尖锐程度；label 是 `[0,1,...,N-1]`，因为 batch 中第 `i` 张图和第 `i` 段文本匹配。

面试表达：CLIP loss 本质是两个方向的 batch 内分类题。图片检索文本看行 softmax，文本检索图片看列 softmax，对角线是正样本。

## Zero-Shot Classification

Zero-Shot Classification 指不为新类别训练分类头，而是把类别名称写成文本 prompt，再用图文相似度完成分类。

流程：

1. 为每个类别构造 prompt，例如 `a photo of a dog`。
2. 用 text encoder 编码类别 prompt。
3. 用 image encoder 编码待分类图片。
4. 选择相似度最高的类别。

如果每个类别有多个 prompt 模板，可以做 prompt ensemble：先平均同一类别的多个文本向量，再归一化为类别向量。

面试表达：CLIP 的 zero-shot 分类把“分类头权重”换成了自然语言类别 prompt 的 embedding，因此可以用语言描述新类别，但效果会受 prompt 模板、训练分布和类别粒度影响。

## Image-Text Retrieval

Image-Text Retrieval 指用图像检索文本，或用文本检索图像。

CLIP 适合检索的原因是双塔结构：图像库和文本库可以离线编码成向量，线上只需编码 query 并做相似度排序。

常见指标：

| 指标 | 含义 | 适用 |
| --- | --- | --- |
| Recall@K | 正确匹配是否出现在 Top-K | 检索召回 |
| MRR | 正确匹配排名倒数的平均值 | 关注排名靠前程度 |
| Median Rank | 正确匹配的中位排名 | 大规模检索稳定性 |
| Top-1 Accuracy | Top-1 是否正确 | 小规模配对评估 |

面试表达：CLIP 检索项目不能只展示 Top-K 样例，还要报告 Recall@K、MRR、bad case 类型和数据噪声，例如 OCR、细粒度关系、中文 prompt、近重复图和错误 caption。

## Contrastive Learning

Contrastive Learning 的目标是拉近正样本，推远负样本。

在图文场景中，正样本通常是真实配对的图片和描述，负样本通常是 batch 内其他图片或文本。模型通过对比目标学习“哪些图片和哪些文本语义匹配”。

常见公式是 InfoNCE。对 image-to-text 方向，可以写成：

$$
L_i=-\log\frac{\exp(\mathrm{sim}(I_i,T_i)/\tau)}{\sum_j \exp(\mathrm{sim}(I_i,T_j)/\tau)}
$$

其中 `I_i` 是第 i 张图片向量，`T_i` 是对应文本向量，`sim` 是相似度函数，`tau` 是 temperature。text-to-image 方向类似，通常两个方向相加。

关键点：

| 概念 | 说明 |
| --- | --- |
| 正样本 | 真实匹配的图文对 |
| 负样本 | 不匹配图文对，常来自同 batch |
| temperature | 控制相似度分布尖锐程度 |
| batch size | batch 越大，负样本越多，对比学习通常越有效 |
| hard negative | 语义相近但不匹配的负样本，对模型更有挑战 |

优点：无需复杂人工标签，只需图文配对数据；表示学习效果好；适合检索和零样本分类。

缺点：需要大 batch 或高效负样本构造；可能把真实语义相关但非配对样本当成负样本；难以直接学习复杂生成和多步推理。

面试表达：对比学习学的是判别式对齐，不是生成式理解。它能告诉你图片和文本是否匹配，但不天然具备长文本回答能力。

## Image Encoder

Image Encoder 是把图片转成向量或 token 序列的模块。

常见 image encoder 包括 CNN、ViT、Swin Transformer、ConvNeXt、CLIP ViT 等。在视觉语言模型中，最常见的是使用预训练好的 CLIP ViT 或其他视觉 backbone 提取 patch-level 特征。

输出形式有两类：

| 输出 | 含义 | 用途 |
| --- | --- | --- |
| 全局向量 | 一张图一个 embedding | 图文检索、分类、相似度排序 |
| patch token | 一张图多个视觉 token | VQA、caption、多模态对话、grounding |

如果只做图文检索，通常全局向量足够。如果要让语言模型回答图像细节问题，需要保留 patch token，因为局部区域信息很重要。

工程坑：视觉 token 数量会显著影响上下文长度。例如 336 x 336 图片切成 14 x 14 patch，会有 576 个 patch token。如果多图输入或视频输入，视觉 token 会快速膨胀，影响显存、延迟和长上下文预算。

面试表达：image encoder 决定模型能看到什么视觉信息，connector 决定这些视觉信息如何进入 LLM，LLM 决定如何用语言进行推理和回答。

## Patch Embedding

Patch Embedding 是 ViT 和很多 VLM 视觉塔的第一步：把图像切成固定大小 patch，并把每个 patch 映射成 hidden vector。

设输入图像为 `X: [B,C,H,W]`，patch size 为 `P`，则 patch token 数近似为：

```math
N_{\mathrm{patch}}=
\left\lceil\frac{H}{P}\right\rceil
\cdot
\left\lceil\frac{W}{P}\right\rceil
```

每个 patch 展平后的维度是：

```math
d_{\mathrm{patch}}=C P^2
```

实现上常用 `Conv2d(C, d_v, kernel_size=P, stride=P)`，权重 shape 等价于 `[d_v,C,P,P]`，输出再展平为 `[B,N,d_v]`。

面试表达：patch embedding 可以理解为把图片翻译成视觉 token。patch 越小，token 越多，细节越多，但 attention 和上下文成本越高。

## CLS Token

CLS Token 是 ViT 中常见的全局汇聚 token。它会和 patch tokens 一起进入 Transformer，最后常用 CLS 位置的输出表示整张图。

在 CLIP 的 ViT vision tower 中，常见流程是：

```text
image -> patch embedding -> add CLS token -> add position embedding -> Transformer -> CLS output -> projection
```

CLS token 适合图像分类、图文检索和全局 image embedding；patch tokens 更适合 VLM 问答、OCR、grounding、计数和图表理解。

面试表达：CLS token 不是“看到所有细节”的保证，它是全局汇聚表示。需要细粒度视觉证据时，通常要保留 patch tokens 或多层特征。

## Vision Encoder Shape Audit

Vision Encoder Shape Audit 指上线或训练前检查 vision encoder 的输入输出 shape、patch grid、position embedding、视觉 token 数、projector shape 和成本。

常见检查表：

| 检查项 | 典型 shape / 指标 | 风险 |
| --- | --- | --- |
| 输入图像 | `[B,C,H,W]` | 预处理分辨率或通道顺序错误 |
| patch grid | `ceil(H/P) x ceil(W/P)` | token 数和 position embedding 不匹配 |
| patch tokens | `[B,N,d_v]` | 高分辨率导致 token 爆炸 |
| CLS / position | `(N+1,d_v)` | 加不加 CLS 与 checkpoint 不一致 |
| projector | `[B,N,d_v] -> [B,N,d_l]` | 视觉维度和 LLM hidden size 不一致 |
| attention cells | `(N+1)^2` | 显存和延迟超预算 |

面试表达：vision encoder shape audit 能把“视觉塔能不能用”落到可检查证据上。很多 VLM bug 不是模型不会推理，而是图像预处理、patch 数、位置 embedding 或 projector shape 一开始就错了。

## Text Encoder

Text Encoder 是把文本编码为向量或 token 表示的模块。

在 CLIP 类双塔模型中，text encoder 通常和 image encoder 对称存在，输出文本 embedding，用于和图像 embedding 计算相似度。它更偏表示学习，不负责开放式生成。

在多模态大模型中，语言模型通常是 decoder-only LLM，不再只是 text encoder。它既处理文本 prompt，也接收视觉 token，并自回归生成答案。

两类文本模块的差异：

| 模块 | 目标 | 输出 | 典型场景 |
| --- | --- | --- | --- |
| Text Encoder | 编码文本语义 | embedding | CLIP、检索、分类 |
| Decoder-only LLM | 条件生成 | token 序列 | 对话、VQA、推理、工具调用 |

面试表达：CLIP 中的 text encoder 主要用于图文对齐；LLaVA 这类 VLM 中的 LLM 负责把视觉信息和指令结合起来生成自然语言答案。

## ViT

ViT 是 Vision Transformer，把图片切成 patch，再像文本 token 一样送入 Transformer。

基本流程：图片被切成固定大小 patch，每个 patch 展平成向量并通过线性层映射到 hidden dimension，加上位置编码后进入 Transformer encoder。

例如 224 x 224 图片，patch size 为 16，则得到 14 x 14 = 196 个 patch token。再加上 cls token 或 pooling，得到全局表示。

ViT 的优点：结构与 Transformer 统一，适合大规模预训练，和语言模型 connector 更容易衔接。

ViT 的缺点：对数据规模敏感；高分辨率输入会增加 token 数；局部归纳偏置弱于 CNN，需要更多数据或改进结构。

在多模态大模型中，ViT 常作为视觉前端。面试中如果问“图片怎么变成 token”，可以回答：通常先用 ViT 将图像切成 patch token，再通过投影层或 resampler 转成 LLM hidden size 对应的视觉 token。

## SigLIP

SigLIP 是 CLIP 类图文对齐方法的一个重要变体，使用 sigmoid loss 替代 softmax contrastive loss。

CLIP 的 softmax 对比学习通常在 batch 内做归一化，一个图片对应一个正文本，其他文本作为负样本。SigLIP 将图文匹配建模成独立的二分类问题，对每个 image-text pair 使用 sigmoid binary loss。

简化理解：CLIP 更像“在这一批文本中选出与图片最匹配的那个”，SigLIP 更像“判断每个图文对是否匹配”。

SigLIP 的特点：

| 方面 | CLIP | SigLIP |
| --- | --- | --- |
| 损失形式 | softmax contrastive loss | sigmoid binary loss |
| 归一化范围 | batch 内归一化 | pair-level 判断 |
| 通信依赖 | 大 batch 训练时跨设备通信重要 | 对大规模分布式更友好 |
| 目标理解 | 多分类式匹配 | 多标签式匹配 |

优点：在大规模训练时更容易扩展，减少对全局 batch softmax 的依赖，实际视觉语言模型中常作为强视觉 encoder 或对齐 backbone。

面试表达：SigLIP 和 CLIP 都做图文对齐，但 SigLIP 用 sigmoid loss，把图文匹配看成 pair-wise 二分类，在大规模训练和分布式场景下更友好。

## Vision-Language Model

Vision-Language Model，简称 VLM，是能同时处理视觉和语言的模型。

VLM 可以分为三类：

| 类型 | 代表 | 特点 |
| --- | --- | --- |
| 双塔对齐模型 | CLIP、SigLIP | 擅长检索和相似度，通常不生成长答案 |
| 编码器-解码器模型 | BLIP、OFA | 适合 caption、VQA 等任务 |
| LLM 接视觉模型 | Flamingo、LLaVA、MiniGPT-4 | 用 LLM 进行开放式对话和推理 |

现代多模态大模型通常采用“vision encoder + connector + LLM”的结构。

```text
image -> vision encoder -> visual tokens -> connector -> LLM hidden space
text  -> tokenizer      -> text tokens   -> embedding -> LLM hidden space

LLM attends over visual tokens and text tokens -> generates answer
```

核心模块：

| 模块 | 作用 |
| --- | --- |
| Vision Encoder | 提取视觉特征 |
| Connector | 把视觉特征映射到 LLM 可用空间 |
| LLM | 进行指令理解、语言推理和生成 |
| Instruction Data | 教模型如何按用户指令使用图片信息 |

面试表达：VLM 的能力上限不只由 LLM 决定，还受视觉 encoder 分辨率、视觉 token 压缩方式、多模态数据质量和对齐训练影响。

## Multimodal Alignment

Multimodal Alignment 指让不同模态在语义层面对齐。

对齐分为粗粒度和细粒度。粗粒度对齐是整张图和整段文本匹配，例如 CLIP。细粒度对齐是图像区域和文本短语匹配，例如“红色杯子”对应图片中某个区域。

常见对齐方式：

| 方法 | 说明 | 适用场景 |
| --- | --- | --- |
| 对比学习 | 拉近匹配图文，推远不匹配图文 | 检索、零样本分类 |
| Caption 生成 | 根据图像生成描述 | 图像描述、多模态预训练 |
| Matching loss | 判断图文是否匹配 | 图文匹配、rerank |
| Masked modeling | 遮挡部分模态并预测 | 表示学习 |
| Instruction tuning | 用图文指令数据训练回答 | 多模态对话 |
| Grounding supervision | 用区域框或 mask 对齐短语 | 定位和细粒度理解 |

对齐不足会导致模型看图不准、答非所问、忽略图片、凭语言先验猜答案或产生多模态幻觉。

面试表达：多模态对齐不是一次性完成的。预训练对齐解决基础语义空间问题，指令微调解决交互方式问题，细粒度 grounding 解决区域和实体对齐问题。

## Connector

Connector 是连接视觉 encoder 和 LLM 的模块，也常被称为 projector、adapter、resampler 或 bridge。

为什么需要 connector：视觉 encoder 的输出维度、分布和 token 数量通常与 LLM 不一致。LLM 原本只见过文本 embedding，如果直接把视觉特征塞进去，模型很难理解。因此需要 connector 将视觉 token 映射到 LLM hidden space。

常见 connector：

| Connector | 机制 | 特点 |
| --- | --- | --- |
| Linear Projection | 线性层映射维度 | 简单、便宜，LLaVA 早期常用 |
| MLP Projector | 多层 MLP 映射 | 表达能力更强，成本仍较低 |
| Q-Former | learnable queries 通过 cross-attention 查询视觉特征 | 可压缩视觉 token，BLIP-2 常用 |
| Perceiver Resampler | 固定数量 latent query 汇聚视觉特征 | 适合压缩大量视觉 token，Flamingo 常用 |
| Cross-Attention | LLM 层中插入视觉 cross-attention | 融合能力强，但改造和训练成本高 |

选择 connector 的关键 trade-off：线性投影简单高效，但压缩和对齐能力有限；Q-Former 和 resampler 更强，但结构更复杂、训练更麻烦；cross-attention 融合深，但会改变 LLM 架构。

面试表达：connector 的本质是把视觉 token 翻译成 LLM 能消费的表示。它决定视觉信息是完整保留、压缩后保留，还是在 LLM 内部通过 cross-attention 融合。

## VLM Connector Audit

VLM Connector Audit 指检查视觉特征接入 LLM 的 shape、占位符、上下文预算、token 压缩和 label mask 是否一致。

最小检查链路如下：

```math
Z_v\in\mathbb{R}^{B\times N_v\times d_v},
\qquad
H_v=Z_vW_p+b_p,
\qquad
H_v\in\mathbb{R}^{B\times N_v\times d_l}
```

如果直接拼接视觉 token，上下文长度近似为：

```math
T_{\mathrm{total}}=T_{\mathrm{text}}+I N_v+N_{\mathrm{special}}
```

上线或训练前至少检查：图片数量是否等于 image placeholder 数量、projector 输出是否等于 LLM hidden size、多图是否超过上下文上限、assistant-only loss mask 是否只覆盖回答 token、OCR / 图表等高风险样本是否需要更高分辨率或外部工具。

面试表达：VLM connector audit 能把“图像接入 LLM”拆成可验证证据。很多多模态 bug 不是模型能力问题，而是 placeholder、shape、token budget 或 labels 对齐一开始就错了。

## Image Placeholder

Image Placeholder 是多模态 prompt 中表示图片插入位置的特殊标记，例如 `<image>`、`<image_1>` 或模型自定义 image token。

它有两层含义：文本模板里告诉模型这里有图片，batch collator 中告诉系统把对应 visual tokens 插入到哪个位置。

常见风险：

| 风险 | 现象 |
| --- | --- |
| placeholder 少于图片数 | 有图片被忽略或插入位置不确定 |
| placeholder 多于图片数 | 模型以为空缺位置也有视觉证据 |
| 多轮引用混乱 | 回答把不同图片内容混在一起 |
| labels 未同步 | visual tokens 插入后 loss mask 错位 |

面试表达：image placeholder 不是普通文本装饰，它是 prompt、视觉 token 和 label mask 对齐的锚点。

## Visual Token Compression

Visual Token Compression 指把大量 patch-level visual tokens 压缩为较少的 query / latent tokens，常见于 Q-Former、Perceiver Resampler、pooling、token pruning 或区域裁剪。

压缩比例可以粗略写为：

```math
\rho=\frac{Q}{N_v}
```

其中 `Q` 是压缩后的 token 数，`N_v` 是原始视觉 token 数。压缩能降低上下文长度和 attention 成本，但会带来信息瓶颈，尤其影响 OCR、小目标、计数、空间关系和图表读数。

面试表达：视觉 token 压缩不是越强越好。它是在成本和可见证据之间做取舍，任务越依赖局部细节，越要谨慎压缩。

## Assistant-Only Multimodal Loss

Assistant-Only Multimodal Loss 指多模态 SFT 中只对 assistant 回答 token 计算 next-token loss，用户问题、system prompt、image placeholder 和 visual tokens 通常不参与 loss。

```math
L_{\mathrm{sft}}=
-\frac{1}{\sum_{t=1}^{T}m_t}
\sum_{t=1}^{T}
m_t\log p_\theta(y_t\mid y_{1:t-1},H_v,x)
```

其中 `m_t=1` 表示第 `t` 个 token 是 assistant answer。这个设计避免模型学习复述用户 prompt 或把 `<image>` 等控制 token 当成输出目标。

面试表达：多模态 SFT 的 loss 仍是语言模型 loss，但条件里多了视觉表示；关键工程点是 label mask 必须只覆盖 assistant 回答。

## Q-Former

Q-Former 是 BLIP-2 中常见的视觉到语言桥接模块。

核心思想：不用把所有视觉 patch token 都直接送给 LLM，而是引入一组可学习 query token，让这些 query 通过 cross-attention 从视觉 encoder 输出中读取信息。最终得到固定数量的视觉 query 表示，再接入语言模型。

简化流程：

```text
image -> frozen vision encoder -> many visual features
learnable queries -> Q-Former cross-attention -> compact visual tokens
compact visual tokens -> projection -> LLM
```

优点：可以压缩视觉 token 数量，降低 LLM 上下文成本；query 可以学习选择对语言任务有用的视觉信息；适合在冻结大型视觉 encoder 和 LLM 的情况下训练桥接模块。

缺点：结构比简单 MLP projector 复杂；query 数量是瓶颈，压缩过强可能丢细节；训练目标和数据设计会显著影响效果。

面试表达：Q-Former 可以理解为用一组可学习查询从图像特征中“问出”语言模型需要的信息，是视觉特征压缩和跨模态对齐的桥。

## Perceiver Resampler

Perceiver Resampler 是 Flamingo 等模型中使用的视觉 token 压缩模块。

它使用固定数量的 latent tokens，通过 attention 从大量视觉特征中汇聚信息，将可变长度或大量视觉输入压缩成固定数量的 token。

为什么重要：图片高分辨率或视频多帧输入会产生大量视觉 token。如果全部送入 LLM，计算和显存成本很高。resampler 可以把大量视觉特征压缩成较少 token，方便后续语言模型处理。

和 Q-Former 的共同点：都使用 learnable query/latent token 从视觉特征中抽取信息，都能压缩视觉 token。

差异：Q-Former 更强调作为 BLIP-2 式视觉语言桥接模块，通常配合特定预训练目标；Perceiver Resampler 更像通用的 token 汇聚器，常用于处理多图、多帧和可变长度视觉输入。

面试表达：Perceiver Resampler 解决的是视觉 token 太多的问题，它用固定数量 latent token 汇聚视觉信息，使多图和视频输入更可控。

## Flamingo

Flamingo 是 DeepMind 提出的视觉语言模型路线，重点是让大语言模型通过少量多模态示例进行 in-context learning。

核心设计包括：使用视觉 encoder 提取图像特征；使用 Perceiver Resampler 将视觉特征压缩；在冻结或部分冻结的语言模型中插入 gated cross-attention，让语言模型在生成时访问视觉信息。

Flamingo 的特点：

| 方面 | 说明 |
| --- | --- |
| 模型结构 | vision encoder + resampler + LLM + cross-attention |
| 训练方式 | 大规模图文和交错图文数据训练 |
| 能力重点 | 多模态 few-shot 和 in-context learning |
| 工程代价 | 架构改造较大，训练复杂 |

Flamingo 的启发：多模态模型不一定只做单图问答，也可以处理交错的图文上下文，例如“图片 A + 描述 + 图片 B + 问题”。这对文档理解、网页理解和多轮视觉对话很重要。

面试表达：Flamingo 的关键在于通过 resampler 压缩视觉输入，并用 gated cross-attention 把视觉信息注入 LLM，使语言模型具备多模态 in-context learning 能力。

## BLIP 与 BLIP-2

BLIP 是 Bootstrapping Language-Image Pre-training，代表了一类结合图文理解和生成的视觉语言预训练方法。BLIP 通过 captioning、filtering 和多任务目标提升图文数据质量与模型能力。

BLIP-2 的核心贡献是用 Q-Former 连接冻结的视觉 encoder 和冻结的 LLM，从而降低训练成本。它不从头训练完整多模态大模型，而是学习一个桥接模块，让强视觉模型和强语言模型协同工作。

BLIP-2 常见两阶段思想：先让 Q-Former 学会从图像中抽取和文本相关的信息，再把 Q-Former 输出接到 LLM 上进行视觉到语言生成。

优点：复用已有强模型，训练成本低；Q-Former 能压缩视觉信息；适合构建图像描述和 VQA 模型。

缺点：桥接模块可能成为瓶颈；冻结模型限制端到端适配；复杂场景下仍需要高质量指令数据。

面试表达：BLIP-2 的重点不是单纯“用了大模型”，而是用 Q-Former 低成本连接冻结 vision encoder 和冻结 LLM，是多模态对齐中的经典桥接方案。

## LLaVA

LLaVA 是 Large Language and Vision Assistant 的代表路线，特点是用视觉 encoder 加投影层连接 LLM，并通过多模态指令微调获得视觉对话能力。

典型结构：CLIP ViT 作为视觉 encoder，MLP projector 将视觉特征映射到 LLM hidden size，LLM 负责根据视觉 token 和文本指令生成答案。

训练通常包括两个阶段：

| 阶段 | 目标 | 常见做法 |
| --- | --- | --- |
| 预对齐 | 让视觉特征进入 LLM 空间 | 冻结 vision encoder 和 LLM，只训练 projector |
| 指令微调 | 学会多模态对话和问答 | 用图文指令数据微调 projector 和部分或全部 LLM |

LLaVA 路线的优势：结构简单，复现成本较低，容易接入开源 LLM，指令能力强。

局限：视觉细节能力依赖 vision encoder 和分辨率；简单 projector 可能对复杂空间关系和细粒度 grounding 不足；指令数据质量对输出风格和幻觉影响很大。

面试表达：LLaVA 是把视觉 token 通过 projector 接到 LLM 上，再用多模态指令数据教模型按人类指令看图回答。它代表了低成本构建 VLM 的主流工程路线。

## Image Captioning

Image Captioning 是给定图片生成自然语言描述。

它是早期视觉语言任务，也是多模态大模型的重要数据来源和基础能力。caption 可以是简短描述，也可以是细粒度描述，例如主体、背景、动作、关系、文字和风格。

常见训练方式：输入图像，模型自回归生成 caption，以交叉熵作为训练目标。现代模型也会用更复杂的指令形式，例如“请详细描述这张图片”。

评估指标包括 BLEU、METEOR、ROUGE、CIDEr、SPICE 等，但这些指标和人类感知不完全一致。大模型时代更常用人工评测或 LLM-as-judge 辅助评估事实准确性、详细程度和可读性。

工程注意：caption 数据质量很关键。过短 caption 会导致模型只学到粗粒度对象，过度想象的 caption 会诱导幻觉，模板化 caption 会让输出单调。

面试表达：Image Captioning 看似简单，但它决定模型能否把视觉内容转成语言表达。高质量 caption 数据对多模态指令模型非常重要。

## Visual Question Answering

VQA 是给定图片和问题，输出答案。

VQA 比 caption 更接近真实交互，因为用户通常不是要求模型完整描述图片，而是问具体问题。例如“图中有几个人？”、“红色车在什么位置？”、“这张图表说明了什么趋势？”。

VQA 难点：

| 难点 | 示例 |
| --- | --- |
| 细粒度识别 | 识别小物体、文字、图标 |
| 空间关系 | 左右、上下、遮挡、包含 |
| 计数 | 准确数人数、物体数量 |
| OCR | 读取图片中文字 |
| 常识推理 | 根据场景推断行为和意图 |
| 图表推理 | 读取坐标轴、趋势和数值 |

常见问题：模型可能不看图，仅凭语言先验回答。例如问“香蕉是什么颜色”，即使图中没有香蕉，模型也可能回答黄色。这就是多模态幻觉和语言先验偏置。

面试表达：VQA 的关键不只是生成流畅答案，而是答案必须被图像证据支持。评估时要区分语言能力、视觉识别能力和跨模态推理能力。

## OCR

OCR 是 Optical Character Recognition，即从图像中识别文字。

在多模态大模型中，OCR 是非常重要的基础能力。很多真实业务图片包含文字，例如截图、票据、表格、PPT、网页、菜单、合同和图表。如果 OCR 不准，后续问答和推理都会出错。

OCR 对 VLM 的挑战：

| 挑战 | 说明 |
| --- | --- |
| 小字识别 | 分辨率不足会丢字 |
| 复杂排版 | 表格、双栏、海报、截图结构复杂 |
| 多语言 | 中英文、数字、符号混排 |
| 旋转和遮挡 | 文本角度、模糊、反光影响识别 |
| 结构理解 | 不仅要识别文字，还要理解行列关系 |

提升 OCR 能力的方式：提高输入分辨率；使用更强视觉 encoder；加入 OCR 专项数据；用文档理解数据训练；必要时接外部 OCR 工具，把识别结果作为文本上下文输入 LLM。

面试表达：OCR 是多模态模型落地的硬指标。很多模型会“看懂大物体但看不清小字”，这在文档问答、图表分析和企业知识库场景中会直接影响可用性。

## Grounding

Grounding 指把语言表达与图像中的具体区域、框、点或 mask 对齐。

例如用户说“左边穿红衣服的人”，模型不仅要理解这句话，还要知道它对应图片中的哪个区域。Grounding 是多模态模型从“描述图片”走向“精确操作”的关键能力。

Grounding 相关任务：

| 任务 | 输入 | 输出 |
| --- | --- | --- |
| Referring Expression Comprehension | 图片和短语 | 目标框 |
| Phrase Grounding | 图片和多个短语 | 短语对应区域 |
| Detection with Text Prompt | 图片和类别文本 | 目标框 |
| Segmentation with Text Prompt | 图片和文本 | mask |
| GUI Grounding | 截图和操作指令 | 点击坐标或区域 |

Grounding 难点：需要细粒度视觉表示、空间关系理解和区域级监督。普通图文 caption 数据通常只提供全局描述，不足以训练强 grounding 能力。

面试表达：没有 grounding 的 VLM 可能会说得很像懂图，但不能指出证据在哪里。对于机器人、GUI agent、工业质检和医疗影像，grounding 比流畅回答更重要。

## Multimodal Instruction Tuning

Multimodal Instruction Tuning 指用图像或其他模态加自然语言指令的数据训练模型，使模型学会按照用户指令进行多模态问答、解释和推理。

数据形式通常如下：

```json
{
  "image": "example.jpg",
  "conversations": [
    {"from": "human", "value": "请描述这张图片中的主要内容。"},
    {"from": "assistant", "value": "图片中有一只狗坐在草地上，旁边有一个蓝色飞盘。"}
  ]
}
```

指令微调数据来源包括人工标注、已有 VQA 数据集改写、caption 数据改写、GPT-4V 等强模型生成、业务日志清洗、合成图表问答、OCR 问答和文档问答。

质量控制要点：

| 维度 | 要求 |
| --- | --- |
| 事实一致 | 答案必须被图片支持 |
| 指令多样 | 覆盖描述、问答、推理、拒答、格式化输出 |
| 难度分层 | 包含简单识别、关系理解、计数、OCR、图表推理 |
| 风格稳定 | 回答清晰，不乱编细节 |
| 安全合规 | 人脸、隐私、医疗、金融等场景谨慎处理 |

常见问题：用强模型生成数据时，生成器可能会凭空补充图片不存在的细节。如果不做过滤，学生模型会学习幻觉。

面试表达：多模态指令微调解决的是“模型如何和用户交互”的问题。它不能替代底层视觉识别能力，如果 vision encoder 看不到细节，指令微调也只能让模型更会表达，不能凭空恢复信息。

## Multimodal SFT Data Audit

Multimodal SFT Data Audit 指在多模态指令微调前，对图文对话样本的格式、任务覆盖、证据支持、拒答边界和 loss mask 做训练前审计。

一条样本可以抽象为：

```math
s_i=(M_i,U_i,A_i,T_i,Q_i,R_i)
```

其中 `M_i` 是媒体集合，`U_i` 是用户指令或多轮上下文，`A_i` 是 assistant 目标回答，`T_i` 是任务类型，`Q_i` 是质量标签，`R_i` 是风险或拒答标签。

最小审计项包括：

| 检查项 | 目的 |
| --- | --- |
| image placeholder 数量 | 保证图片和 prompt 对齐 |
| assistant-only labels | 保证 loss 只覆盖回答 |
| total tokens | 避免视觉 token 和文本 token 超上下文 |
| task coverage | 防止只会 caption 或普通 VQA |
| evidence support | 控制视觉幻觉 |
| refusal boundary | 控制看不清、资料不足和高风险场景 |

面试表达：多模态 SFT 数据不能只看 JSON 格式能否解析，还要证明回答被视觉证据支持、该拒答时会拒答、任务类型覆盖真实使用场景。

## Chat Template

Chat Template 是把 system、user、assistant、image placeholder 和分隔符组织成模型训练输入的模板。

在 VLM 中，chat template 还承担 image placeholder 对齐作用。训练和推理模板不一致时，模型可能找不到图片位置、角色边界混乱，或者把 assistant 起始标记当成普通文本。

常见检查包括：`<image>` 是否在 user 消息中、placeholder 数量是否等于图片数、assistant 起始标记是否一致、多轮分隔符是否稳定、labels 是否跟随模板正确右移。

面试表达：chat template 是数据格式的一部分，不是前端展示文本。训练、评估和推理必须使用一致模板，否则同一个 VLM 可能表现大幅波动。

## Multimodal Task Coverage

Multimodal Task Coverage 衡量多模态 SFT 数据是否覆盖目标任务集合，例如 caption、VQA、OCR、chart、grounding、多轮对话和 refusal。

```math
C_{\mathrm{task}}=
\frac{
\left|\mathcal{T}_{\mathrm{train}}\cap\mathcal{T}_{\mathrm{req}}\right|
}{
\left|\mathcal{T}_{\mathrm{req}}\right|
}
```

如果训练集主要由 caption 组成，模型可能描述图片还可以，但 OCR、计数、空间关系、图表和资料不足拒答都很弱。

面试表达：task coverage 是多模态 SFT 的能力地图。覆盖率不够时，训练 loss 再低也不能说明模型满足业务需求。

## Evidence Support Rate

Evidence Support Rate 衡量回答是否被图片、截图、文档或图表中的可见证据支持。

```math
S_{\mathrm{support}}=
\frac{1}{N}
\sum_{i=1}^{N}
\mathbf{1}[A_i\ \mathrm{supported}]
```

低 evidence support 往往对应多模态幻觉：模型回答流畅，但答案不是从图像证据中来的。OCR、图表、医疗影像、质检和安全审核尤其需要严格检查。

面试表达：VLM 回答不能只看语言是否自然，还要看每个 claim 是否能回到图片证据。证据支持率是比通用流畅度更接近落地可靠性的指标。

## Missing Refusal Rate

Missing Refusal Rate 指样本应该拒答或表达不确定，但目标回答仍然硬答的比例。

```math
R_{\mathrm{miss\_refuse}}=
\frac{
\sum_i \mathbf{1}[r_i=1\land \hat r_i=0]
}{
\sum_i \mathbf{1}[r_i=1]
}
```

在多模态场景中，资料不足包括图片模糊、关键区域被遮挡、分辨率不足、文字不可读、图表单位不清或用户要求图片中不存在的信息。

面试表达：拒答数据不是让模型少回答，而是训练模型在视觉证据不足时别编。missing refusal rate 高，通常会带来严重视觉幻觉。

## Multimodal Hallucination

Multimodal Hallucination 指模型生成了与视觉输入不一致的内容。

典型表现：图片中没有某个物体，模型说有；物体颜色、数量、位置说错；图片文字识别错误；把常识当视觉证据；用户问不确定问题时模型强行回答。

产生原因：

| 原因 | 说明 |
| --- | --- |
| 语言先验过强 | LLM 根据常见搭配猜答案 |
| 视觉对齐不足 | visual token 没有被正确利用 |
| 分辨率不足 | 小物体和文字看不清 |
| 指令数据有幻觉 | 训练答案包含编造内容 |
| 评估偏流畅性 | 模型学会说得好听而非说得准确 |
| 解码过发散 | 高 temperature 增加不受证据支持的细节 |

缓解方法：

| 方法 | 说明 |
| --- | --- |
| 提升视觉输入质量 | 高分辨率、裁图、多尺度输入 |
| 加强细粒度数据 | OCR、grounding、计数、区域描述 |
| 训练拒答能力 | 看不清或证据不足时说明不确定 |
| 引入 grounding | 要求答案引用区域证据 |
| 使用偏好优化 | 惩罚编造细节，奖励忠实答案 |
| 工具辅助 | 接 OCR、检测、分割、检索工具 |
| 评估分项化 | 单独评估存在性、属性、数量、位置和文字 |

面试表达：多模态幻觉不是单纯语言模型幻觉，而是视觉证据没有正确约束生成。解决它要从数据、结构、分辨率、训练目标和评估体系一起入手。

## Diffusion Model

Diffusion Model 是一类通过“加噪”和“去噪”建模数据分布的生成模型。

训练时，先把真实样本 `x0` 按噪声日程加到某个时间步 `t`，得到 noisy sample `x_t`；模型输入 `x_t`、`t` 和条件 `c`，学习预测噪声或去噪方向。生成时，从随机噪声开始，按 scheduler 多步去噪，逐渐得到图片、音频、视频或 latent 表示。

面试表达：diffusion 的关键不是一次生成图片，而是学习一条从噪声到数据的反向去噪路径。

## DDPM

DDPM 是 Denoising Diffusion Probabilistic Models 的代表方法，定义了固定的 forward diffusion 和可学习的 reverse denoising。

Forward 过程：

```math
q(x_t\mid x_{t-1})=
\mathcal{N}\left(
x_t;
\sqrt{\alpha_t}x_{t-1},
\beta_t I
\right)
```

闭式加噪：

```math
x_t=
\sqrt{\bar{\alpha}_t}x_0+
\sqrt{1-\bar{\alpha}_t}\epsilon,
\qquad
\epsilon\sim\mathcal{N}(0,I)
```

其中 `beta_t` 是噪声强度，`alpha_t=1-beta_t`，`alpha_bar_t` 是累计信号保留比例。

面试表达：DDPM 训练时可以直接从任意时间步构造 `x_t`，不需要真的一步一步加噪，这让训练非常方便。

## Forward Diffusion

Forward Diffusion 指人为定义的加噪过程，从真实样本逐步走向近似标准高斯噪声。

关键性质：时间步越大，`alpha_bar_t` 越小，样本中的真实信号越少，噪声越强。

面试表达：forward 过程不是模型学出来的，它是训练者设定的噪声日程；模型真正学习的是反向去噪。

## Reverse Denoising

Reverse Denoising 指模型从 `x_t` 预测 `x_{t-1}` 或预测去噪方向的过程。

DDPM 中常用预测噪声来构造反向均值：

```math
\mu_\theta(x_t,t)=
\frac{1}{\sqrt{\alpha_t}}
\left(
x_t-
\frac{\beta_t}{\sqrt{1-\bar{\alpha}_t}}
\epsilon_\theta(x_t,t,c)
\right)
```

面试表达：采样时的每一步都不是直接生成完整图片，而是把当前 noisy sample 往更干净的分布方向推一点。

## Noise Scheduler

Noise Scheduler 定义 diffusion 中每个时间步的噪声强度和采样更新规则。

它影响三件事：训练时如何从 `x0` 构造 `x_t`，推理时如何从 `x_t` 更新到 `x_{t-1}`，以及多少采样步能在质量和速度之间取得平衡。

面试表达：scheduler 是 diffusion 的“时间表”。模型权重相同，换 scheduler 或采样步数，也可能显著影响质量、速度和稳定性。

## Noise Prediction Loss

Noise Prediction Loss 是 diffusion 入门中最常见的训练目标：模型预测加入 `x0` 的噪声 `epsilon`。

```math
L_{\mathrm{simple}}=
\mathbb{E}_{x_0,t,\epsilon}
\left[
\left\|
\epsilon-\epsilon_\theta(x_t,t,c)
\right\|_2^2
\right]
```

预测噪声之后，可以估计干净样本：

```math
\hat{x}_0=
\frac{
x_t-\sqrt{1-\bar{\alpha}_t}\epsilon_\theta(x_t,t,c)
}{
\sqrt{\bar{\alpha}_t}
}
```

面试表达：噪声预测好学，是因为训练时真实噪声已知；预测噪声也能转化为去噪和 `x0` 估计。

## Classifier-Free Guidance

Classifier-Free Guidance，简称 CFG，是文生图中常见的条件增强方法。

模型同时学习有条件去噪和无条件去噪。采样时把两者相减得到条件方向，再用 guidance scale 放大：

```math
\hat{\epsilon}_{\mathrm{cfg}}=
\epsilon_\theta(x_t,t,\varnothing)
+
w
\left(
\epsilon_\theta(x_t,t,c)
-
\epsilon_\theta(x_t,t,\varnothing)
\right)
```

`w` 越大，越强调条件，但过大可能导致过饱和、失真、多样性下降或 prompt 过拟合。

面试表达：CFG 是 prompt 遵循和图像质量之间的旋钮，不是越大越好。

## Latent Diffusion

Latent Diffusion 指先用 autoencoder 把图片压缩到 latent space，再在 latent space 中做 diffusion。

直觉流程：

```text
image -> encoder -> latent -> diffusion denoising -> decoder -> image
```

如果空间分辨率被压缩 `f` 倍，spatial cell 数大约降到：

```math
R_{\mathrm{latent}}\approx\frac{1}{f^2}
```

面试表达：latent diffusion 的价值是把昂贵的像素空间去噪搬到更小的 latent 空间，使高分辨率生成更可行。

## Stable Diffusion

Stable Diffusion 是 latent diffusion 的代表系统，把文生图拆成 text encoder、latent denoiser、scheduler 和 VAE decoder 等模块。

典型推理流程：

```text
prompt -> text encoder -> text embeddings
random latent -> denoiser + scheduler -> clean latent
clean latent -> VAE decoder -> image
```

面试表达：Stable Diffusion 的关键不是一个单独网络，而是一个 pipeline。VAE 负责压缩和解码，text encoder 负责 prompt 条件，U-Net 或 DiT 负责在 latent space 中去噪，scheduler 负责采样路径。

## VAE Compression Ratio

VAE Compression Ratio 指图片进入 latent diffusion 前被 autoencoder 压缩后的空间或元素比例。

空间比例：

```math
R_{\mathrm{spatial}}=\frac{1}{f^2}
```

如果把 latent channel 也算进去，元素比例可以粗略写成：

```math
R_{\mathrm{elem}}=
\frac{C_z(H/f)(W/f)}{3HW}
```

面试表达：`1/f^2` 解释空间网格变小，元素比例还要考虑 latent channel；两者都说明 latent diffusion 为什么比像素空间去噪更省。

## Negative Prompt

Negative Prompt 是文生图中描述“不希望出现什么”的条件，例如 blurry、low quality、extra fingers。

在 CFG 口径下，negative prompt 通常作为负条件参与方向计算：

```math
\epsilon_{\mathrm{guided}}=
\epsilon_\theta(z_t,t,c_{\mathrm{neg}})
w
\left(
\epsilon_\theta(z_t,t,c_{\mathrm{pos}})
-
\epsilon_\theta(z_t,t,c_{\mathrm{neg}})
\right)
```

面试表达：negative prompt 不是硬约束，它只是改变 CFG 的负条件方向。它能缓解常见质量问题，但不能保证严格消除错误结构或风险内容。

## Text-to-Image Pipeline Audit

Text-to-Image Pipeline Audit 是文生图上线或实验前的形状、成本和参数审计。

常见检查项：

1. 图片分辨率、VAE scale、latent shape 是否一致。
2. prompt / negative prompt token 数是否超过 text encoder 上限。
3. CFG 是否导致每步需要正负两次 denoiser 调用。
4. 采样步数、scheduler 和 seed 是否可复现。
5. ControlNet、inpainting 或 image-to-image 的条件 shape 是否和图像对齐。
6. 安全过滤、水印、版权和高风险类别是否覆盖。

面试表达：文生图 pipeline 审计的核心是把“效果不好”拆成可检查的 shape、成本、条件、采样和安全门禁，而不是只调 prompt。

## ControlNet

ControlNet 给 diffusion 模型加入边缘、深度、姿态、分割图或草图等结构条件。

直觉公式：

```math
h_l^{\mathrm{out}}=
h_l^{\mathrm{base}}+
r_l(C)
```

其中 `C` 是结构条件，`r_l(C)` 是条件分支产生的残差。

面试表达：prompt 控制语义，ControlNet 控制结构。它解决的是纯文本 prompt 难以精确指定姿态、布局和边缘的问题。

## Image-to-Image

Image-to-Image 指以输入图片作为起点，把图片编码到 latent 后加噪，再按 prompt 去噪生成新图片。

关键参数是 denoising strength：强度低时保留原图更多，强度高时生成自由度更大，也更容易偏离原图。

面试表达：image-to-image 不是纯文本从零生成，而是在原图 latent 附近重新采样，所以它天然适合风格转换、局部重绘前置和构图保留。

## Inpainting

Inpainting 是局部重绘：输入原图、mask 和 prompt，只重绘或主要改变 mask 区域。

常见难点包括边界融合、光照一致、结构连续和语义不冲突。

面试表达：inpainting 的重点是“哪里能改”和“哪里要保留”的约束，比普通文生图更依赖 mask 对齐和边界质量。

## DALL-E

DALL-E 是 OpenAI 文生图系列。早期 DALL-E 把文本和离散图像 token 统一放进自回归 Transformer，DALL-E 2 论文强调从文本生成 CLIP image latent，再由 decoder 生成图像。

面试表达：DALL-E 系列体现了文生图路线从离散图像 token 自回归生成，到更强的图文语义表示、扩散 decoder、编辑和安全机制的演进。

## Autoregressive Image Tokens

Autoregressive Image Tokens 指先把图像离散化为 token，再像语言模型一样逐个预测图像 token。

```math
p(y_1,\ldots,y_M\mid u)=
\prod_{m=1}^{M}
p(y_m\mid u,y_{1:m-1})
```

面试表达：自回归图像 token 的优点是和 next-token prediction 统一，缺点是高分辨率下 token 序列很长，生成成本和 tokenizer 质量压力很大。

## Video Generation

Video Generation 是生成连续帧序列的多模态生成任务。它不只是生成多张图片，还要保持时间维度上的身份、纹理、几何、动作、相机运动和物理关系连续。

面试表达：视频生成的核心难点是时间一致性。单帧看起来好，不代表视频好；闪烁、身份漂移和动作跳变会直接破坏结果。

## Spatiotemporal Patch

Spatiotemporal Patch 是把视频按时间和空间一起切成 token 的方法。若时间 patch size 为 `P_t`，空间 patch size 为 `P`，token 数约为：

```math
N_{\mathrm{st}}=
\left\lceil\frac{T}{P_t}\right\rceil
\left\lceil\frac{H}{P}\right\rceil
\left\lceil\frac{W}{P}\right\rceil
```

面试表达：时空 patch 用更少 token 表示视频片段，能降低成本，但太大也会损失快速运动、小目标和细节。

## Video Diffusion

Video Diffusion 是在视频像素或视频 latent 上做扩散建模。训练时给视频 latent 加噪，模型在时间步和条件下预测噪声；推理时从随机视频噪声逐步去噪。

常见目标：

```math
L_{\mathrm{video}}=
\mathbb{E}_{z_0,t,\epsilon,c}
\left[
\left\|
\epsilon-
\epsilon_\theta(z_t,t,c)
\right\|_2^2
\right]
```

面试表达：video diffusion 和 image diffusion 的 loss 直觉类似，难点在于样本多了时间维度，denoiser 要同时建模空间和运动。

## Temporal Consistency

Temporal Consistency 指视频相邻帧之间在身份、纹理、颜色、几何和运动上的连续性。

常见审计维度：

1. flicker 是否过大。
2. 主体 identity embedding 是否稳定。
3. 运动轨迹是否平滑。
4. 遮挡后物体是否仍保持存在。
5. 镜头和背景是否连续。

面试表达：时序一致性是视频生成和图像生成最大的分水岭。很多失败不是一帧画坏了，而是帧与帧之间不连贯。

## Identity Drift

Identity Drift 指视频中的主体身份随时间漂移，例如同一个人脸、衣服、车辆或角色逐渐变成另一个样子。

面试表达：identity drift 往往说明模型没有稳定维护对象状态，或条件控制、时序建模和训练数据不足。

## Flickering

Flickering 指视频帧间亮度、颜色、纹理或局部细节快速抖动。

一个简单代理指标是相邻帧统计量变化：

```math
F_{\mathrm{flicker}}=
\frac{1}{T-1}
\sum_{t=2}^{T}
\left|
b_t-b_{t-1}
\right|
```

面试表达：flicker 会让单帧质量看起来不错的视频变得不可用，尤其在产品演示、商业短片、影视和虚拟人场景中非常明显。

## World Model

World Model 是模型对世界状态和动态规律的内部建模。最小抽象是状态转移：

```math
s_{t+1}\sim p_\theta(s_{t+1}\mid s_t,a_t)
```

面试表达：视频生成模型可能学到部分物体持久性、运动和遮挡规律，但能生成合理视频不等于具备可交互、可规划、可验证的完整 world model。

## Physics Consistency

Physics Consistency 指视频中的物体运动、接触、遮挡、碰撞、重力、光照和透视关系符合基本物理直觉。

常见失败包括脚不接地、物体穿模、球滚动方向异常、遮挡后形状变化、液体和烟雾运动不自然。

面试表达：物理一致性往往需要时间、三维结构和因果关系判断，不能只靠单帧图像指标衡量。

## Video Generation Evaluation

Video Generation Evaluation 需要同时评估单帧画质、文本一致性、时序一致性、运动自然度、身份保持、物理合理性、多样性和安全性。

面试表达：视频评估通常要结合自动指标、任务型评估和人工偏好。单一分数很难覆盖 prompt 遵循、时序质量和安全风险。

## FVD

FVD 是 Fréchet Video Distance，常用于比较真实视频和生成视频在视频特征空间中的分布差异。

```math
\mathrm{FVD}=
\|\mu_r-\mu_g\|_2^2+
\mathrm{Tr}
\left(
\Sigma_r+
\Sigma_g-
2(\Sigma_r\Sigma_g)^{1/2}
\right)
```

面试表达：FVD 是分布级指标，不能单独证明 prompt 遵循、物理正确或业务可用。

## Video Token Audit

Video Token Audit 是视频生成或视频理解系统上线前的 token、latent shape、采样成本和时序质量审计。

常见检查项：

1. 帧数、分辨率、patch size 和时空 patch token 数。
2. 视频 latent shape 和元素量比例。
3. 采样步数、CFG 带来的 denoiser 调用次数。
4. flicker、identity stability、motion smoothness 和 object permanence。
5. world model rollout 误差、物理失败和安全门禁。

面试表达：视频 token audit 能把“视频太贵、太闪、太漂”拆成可检查的数字，而不是只凭主观观感调 prompt。

## Audio Generation

Audio Generation 指生成语音、音乐、音效或环境声等时间信号的任务。语音系统常见链路包括 ASR、TTS、speech-to-speech、voice cloning 和 audio codec 建模。

面试表达：音频生成和图像生成不同，输出是时间信号，既要内容正确，也要音色、韵律、节奏、自然度和延迟可控。

## Waveform

Waveform 是原始音频采样序列。若采样率为 `r`，时长为 `D` 秒，采样点数约为：

```math
N=rD
```

面试表达：waveform 保留最原始信息，但序列很长，很多 ASR / TTS 系统会先转成频谱或 codec tokens。

## Log-Mel Spectrogram

Log-Mel Spectrogram 是常见语音特征，把 waveform 转成时间-频率能量表示。

```math
M\in\mathbb{R}^{F\times T_{\mathrm{mel}}}
```

其中 `F` 是 mel bins 数，`T_mel` 是声学帧数。

面试表达：log-mel 特征比原始 waveform 更接近人耳感知，也更适合 encoder 建模，是 Whisper 类 ASR 模型的常见输入。

## ASR

ASR 是 Automatic Speech Recognition，把语音转成文本。

```math
p(y_1,\ldots,y_U\mid M)=
\prod_{u=1}^{U}
p(y_u\mid y_{1:u-1},M)
```

面试表达：ASR 的核心是根据音频特征生成 transcript，真实系统要处理口音、噪声、多说话人、低质量录音、标点和专有名词。

## Whisper

Whisper 是 OpenAI 提出的 encoder-decoder 语音识别模型。典型流程是：

```text
audio -> log-mel spectrogram -> audio encoder -> text decoder -> transcript
```

面试表达：Whisper 的关键是大规模多语言弱监督训练和 encoder-decoder 结构，使它具备较强 ASR、翻译和噪声鲁棒性。

## TTS

TTS 是 Text-to-Speech，把文本合成为语音。

常见系统需要决定内容、音色、语速、停顿、情感和最终 waveform。现代路线可以用 acoustic model + vocoder，也可以用 codec language model 生成 audio tokens。

面试表达：TTS 不只是读出文字，还要让声音自然、清晰、可控，并满足说话人授权和安全要求。

## Vocoder

Vocoder 负责把中间声学表示变成 waveform，例如 `mel spectrogram -> waveform` 或 `codec tokens -> waveform`。

面试表达：vocoder 直接影响音质、噪声、音色和自然度。TTS 文本内容对了，但 vocoder 差，听感仍可能不可用。

## Voice Cloning

Voice Cloning 是根据参考语音生成相同或相似音色的新语音。

面试表达：voice cloning 的技术点是说话人音色保持、内容准确和韵律控制；安全点是说话人授权、水印、深伪检测和高风险拒绝。

## Audio Codec

Audio Codec 把连续音频压缩成离散 tokens，并能解码回 waveform。若 codec 帧率为 `q` Hz、时长为 `D` 秒、codebooks 数为 `K`，token 数约为：

```math
N_{\mathrm{codec}}=
\left\lceil Dq\right\rceil K
```

面试表达：audio codec 是把连续音频接入语言模型的重要桥梁，但 token rate 高、码本多、质量和语义纠缠都是工程难点。

## Speech Token

Speech Token 是从语音或音频 codec 中得到的离散 token，可以像文本 token 一样被 Transformer 建模。

面试表达：speech token 让 TTS、speech-to-speech 和统一多模态模型更容易使用 next-token prediction，但序列长度和实时解码成本很高。

## Codec Language Model

Codec Language Model 把 audio codec tokens 当成语言模型序列来生成。

```math
p(a_1,\ldots,a_M\mid u)=
\prod_{m=1}^{M}
p(a_m\mid u,a_{1:m-1})
```

面试表达：codec LM 的优点是目标统一，缺点是 audio token 序列长、推理速度压力大，韵律和情感控制也更复杂。

## Speech-to-Speech

Speech-to-Speech 指语音输入、语音输出的交互形式。

两条常见路线：

1. Cascade：`speech -> ASR -> LLM -> TTS -> speech`。
2. End-to-end：`speech tokens -> model -> speech tokens`。

面试表达：cascade 清晰可靠、易调试；端到端更自然、可能低延迟，但数据、可控性、安全评估和 debug 难度更高。

## VAD

VAD 是 Voice Activity Detection，用于判断音频中什么时候有人说话。

面试表达：实时语音助手离不开 VAD。VAD 影响首包延迟、打断、静音截断和误触发。

## WER

WER 是 Word Error Rate，ASR 常用词错误率：

```math
\mathrm{WER}=
\frac{S+D+I}{N_{\mathrm{ref}}}
```

面试表达：WER 统计替换、删除、插入错误，适合评估英文等空格分词明显的转写任务。

## CER

CER 是 Character Error Rate，字符错误率：

```math
\mathrm{CER}=
\frac{S_c+D_c+I_c}{N_{\mathrm{char}}}
```

面试表达：CER 对中文、短命令、拼写错误和细粒度转写分析常更直观。

## MOS

MOS 是 Mean Opinion Score，常用于 TTS 或语音质量人工主观评分。

面试表达：MOS 能反映自然度和听感，但成本高、主观性强，最好和内容一致性、说话人相似度、延迟和安全指标一起看。

## Audio Token Audit

Audio Token Audit 是语音系统上线前的采样率、声学帧、codec token、WER / CER、延迟和安全门禁审计。

常见检查项：

1. 采样率、时长、采样点数和 log-mel shape。
2. ASR WER / CER、噪声场景和多语言覆盖。
3. codec token rate、码本数、压缩比和上下文长度。
4. cascade latency、streaming 首包延迟和 barge-in。
5. voice cloning 授权、深伪检测、水印和隐私保护。

面试表达：audio token audit 能把语音系统从“听起来还行”拆成可检查的准确率、延迟、token 成本、音质和安全门禁。

## Unified Multimodal Model

Unified Multimodal Model 指用统一接口、共享表示或统一架构处理文本、图像、音频、视频等多种模态的模型或系统。

统一可以发生在三层：

1. 接口统一：用户看到一个助手，内部是多个模块级联。
2. 表示统一：不同模态映射到共享 embedding 空间。
3. 架构 / 目标统一：不同模态 token 化后进入统一 Transformer 或统一训练目标。

面试表达：统一多模态不是简单串模型，而是要同时解决 token 成本、跨模态对齐、生成解码、路由、训练目标和安全门禁。

## Any-to-Any Multimodal Model

Any-to-Any Multimodal Model 指任意输入模态到任意输出模态，例如 text -> image、speech -> speech、video -> text、image + audio -> text。

```math
p(Y_{M_{\mathrm{out}}}\mid X_{M_{\mathrm{in}}})
```

面试表达：Any-to-Any 是理想目标，但真实系统常混合使用专用 encoder、中心 LLM、生成模型、TTS 和工具，而不是所有模态都由单一 decoder 完成。

## Unified Tokenization

Unified Tokenization 指把文本、图像、音频和视频都变成可被统一模型处理的 token 序列。

总输入 token 数可写成：

```math
N_{\mathrm{in}}=
N_t+N_i+N_a+N_v+N_s
```

面试表达：统一 token 化最大的工程问题是 token rate。图像、长音频、多帧视频的 token 数远高于短文本，会直接推高上下文、显存和延迟。

## Early-Fusion Multimodal Transformer

Early-Fusion Multimodal Transformer 指较早把不同模态 tokens 混合进同一序列，让 Transformer 在统一上下文中建模跨模态关系。

面试表达：early fusion 信息交互更直接，但序列长、训练成本高、模态 tokenizer 质量和数据配比都会成为瓶颈。

## Center-LLM Multimodal System

Center-LLM Multimodal System 指以 LLM 为推理和调度中心，周围连接 vision encoder、audio encoder、OCR、retriever、image generator、TTS 和工具。

面试表达：中心 LLM 路线工程可控，能继承语言推理和工具调用能力；缺点是感知信息可能被压缩，端到端优化不如统一模型自然。

## Multimodal Router

Multimodal Router 根据输入模态、任务和安全要求决定调用 OCR、ASR、video sampler、retriever、image generator、TTS 或 tool executor。

```math
r=\mathrm{Route}(S)
```

面试表达：统一助手一定需要路由。没有路由，系统要么成本过高，要么错过 OCR / ASR / 视频采样 / 安全审核等必要模块。

## Multimodal Loss Mixture

Multimodal Loss Mixture 指统一多模态训练中多个目标的加权组合：

```math
L=
\sum_{k=1}^{K}
\lambda_k L_k,
\qquad
\sum_{k=1}^{K}\lambda_k=1
```

面试表达：loss mixture 决定不同能力的训练权重。配比不当会导致某些模态退化、生成能力压过理解能力，或安全拒答能力不足。

## Modality Conflict

Modality Conflict 指不同模态证据互相冲突，例如图片显示猫但文字说狗、视频和字幕不一致、OCR 文本包含不可信指令。

面试表达：多模态模型不能盲信任意模态。冲突时应说明证据来源、不确定性和优先级，并隔离不可信媒体中的指令性内容。

## Unified Multimodal Audit

Unified Multimodal Audit 是统一多模态系统上线前的 token、路由、loss mixture、输出预算和安全门禁审计。

常见检查项：

1. 文本、图像、音频、视频和 special token 总数是否超上下文。
2. 输出文本、音频、图像、工具调用 token 是否超预算。
3. OCR、ASR、video sampler、TTS、image generator 和工具路由是否匹配任务。
4. 训练 loss mixture 是否覆盖目标能力。
5. 不可信媒体指令、未授权 voice cloning、未审核生成输出和工具越权是否被阻断。

面试表达：统一多模态 audit 能把“系统很强”拆成可验证的 token 成本、路由正确性、训练覆盖和安全边界。

## Multimodal Evaluation

Multimodal Evaluation 指同时评估多模态系统的感知、跨模态对齐、推理、生成质量、忠实性和安全性。

常见评估面包括：

1. VQA：物体、属性、计数、空间关系和常识推理。
2. OCR / ASR：图片文字和语音内容是否识别正确。
3. Chart QA：坐标轴、图例、单位、趋势和数值计算。
4. Grounding：语言答案是否能定位到区域、框、点或 mask。
5. Faithfulness：回答中的 claim 是否被图片、音频或视频证据支持。
6. Generation：图像 / 视频 / 语音生成的质量、一致性、多样性和安全性。
7. Safety：不可信媒体指令、隐私、身份、deepfake、工具越权和 provenance。

面试表达：多模态评估不能只看一个平均 benchmark 分数。真实系统要把任务成功率、证据支持、幻觉率、过度拒答、安全拦截、延迟和成本放在同一张表里看。

## Multimodal Incident

Multimodal Incident 指多模态系统因为输入预处理、OCR、ASR、抽帧、grounding、跨模态冲突、安全过滤、确认流程或预算控制失败，导致答案错误、安全漏检、误操作或用户体验事故。

排查重点：要保存原始媒体、预处理后媒体、中间转换结果、模型最终可见证据、回答 claim、引用区域 / 时间戳、安全判断和预算字段。只看最终回答无法定位错误来自哪条模态边界。

## Input Fidelity Rate

Input Fidelity Rate 指原始媒体经过 resize、crop、压缩、抽帧、降噪、切片或转码后，关键证据仍被保留的比例。

典型失败：长截图底部被裁掉，小字被压缩不可读，视频关键瞬间没抽到，音频降噪把专有名词削弱。输入保真失败时，后面的 VLM、OCR、ASR 和 LLM 很难补救。

## Multimodal Evidence Recall

Multimodal Evidence Recall 指标准视觉、文字、语音或视频证据中，有多少真正进入系统中间表示和最终上下文。

产品意义：多模态系统不是“看过文件”就算有证据。证据必须被 OCR / ASR / 抽帧 / 检索 / grounding 链路保留下来，并能被最终回答引用。

## OCR Critical Field Accuracy

OCR Critical Field Accuracy 指金额、日期、姓名、地址、错误码、表格列名、单位等业务关键字段的识别准确率。

产品意义：普通 OCR 字符准确率较高也可能掩盖关键字段错误。财务、合同、医疗、客服截图和表格任务要单独统计关键字段。

## ASR Critical Field Accuracy

ASR Critical Field Accuracy 指语音转写中的金额、日期、姓名、地址、订单号、操作对象等关键字段准确率。

产品意义：ASR 错一个数字或专有名词，后续 LLM 可能生成错误工具参数。写操作前应复述确认关键字段。

## Temporal Evidence Recall

Temporal Evidence Recall 指视频、音频或多模态时间序列中，关键事件或片段被抽帧、转写、检索和上下文构造覆盖的比例。

典型失败：固定间隔抽帧漏掉短暂事件，字幕和画面不同步，长视频摘要只覆盖开头和结尾。视频理解要引用时间范围，而不是只给笼统结论。

## Multimodal Incident Gate

Multimodal Incident Gate 是面向多模态项目事故复盘和回归测试的门禁，覆盖 input fidelity、evidence recall、OCR / ASR critical fields、temporal evidence recall、grounding IoU、evidence support、hallucination rate、safety block、confirmation coverage、budget overrun 和 P95 latency。

面试表达：多模态项目事故复盘要说清楚错误来自输入保真、模态转换、时间采样、grounding、生成、安全还是预算。不能把所有错误都归因成 VLM 不准。

## VQA Accuracy

VQA Accuracy 是视觉问答任务常用分数。开放式 VQA 通常有多个标注答案，模型答案与越多人工答案一致，分数越高。

简化口径：

```math
s_i^{\mathrm{vqa}}=
\min\left(\frac{n_i}{3},1\right)
```

其中 `n_i` 是模型答案与人工参考答案匹配的数量。

面试表达：VQA accuracy 适合看开放问答平均能力，但要配合答案标准化、分题型切片和错误归因；它不能替代 OCR、grounding 和幻觉评估。

## Chart Relaxed Accuracy

Chart Relaxed Accuracy 指图表问答中的数值容忍评估。因为图表读数可能有小误差，评估时常允许一定相对误差。

```math
\delta_i=
\frac{|p_i-g_i|}{\max(|g_i|,\epsilon)}
```

```math
s_i^{\mathrm{chart}}=
\mathbb{1}[\delta_i\le \tau]
```

其中 `p_i` 是预测值，`g_i` 是参考值，`\tau` 是容忍阈值。

面试表达：图表评估要同时看数值、单位、图例、趋势和计算步骤。relaxed accuracy 只能处理小数值误差，不能证明模型真正理解图表结构。

## Grounding IoU

Grounding IoU 用于评估语言表达对应的图像区域是否定位正确。

```math
\mathrm{IoU}(B_p,B_g)=
\frac{|B_p\cap B_g|}
{|B_p\cup B_g|}
```

常见做法是统计 IoU 超过阈值的比例：

```math
\mathrm{Prec}@\tau=
\frac{1}{N}\sum_{i=1}^{N}
\mathbb{1}[\mathrm{IoU}(B_i^p,B_i^g)\ge \tau]
```

面试表达：Grounding IoU 能把“模型说看见了什么”落到视觉证据上。它对 GUI agent、文档理解、视觉幻觉治理和高风险审核都很关键。

## Multimodal Hallucination Rate

Multimodal Hallucination Rate 指回答中不被媒体证据支持的 claim 占比。

```math
R_{\mathrm{hall}}=
\frac{\sum_i\sum_j u_{ij}}
{\sum_i M_i}
```

其中 `u_ij=1` 表示第 `i` 个样本的第 `j` 条 claim 不被图片、音频或视频证据支持。

面试表达：多模态幻觉不是“答错”这么简单，而是模型把不存在、看不清、听不清或证据不足的信息说成确定事实。治理时要做 claim-level evidence check 和拒答边界评估。

## Multimodal Prompt Injection

Multimodal Prompt Injection 指恶意或不可信指令隐藏在图片、文档截图、网页、音频或视频中，诱导模型把媒体内容当成更高优先级指令。

典型例子包括 OCR 读到网页截图里的越权指令、文档中要求模型忽略系统规则、音频中夹带工具调用命令等。

面试表达：防护原则是外部媒体内容只能作为数据，不应覆盖 system / developer 指令。工具调用前还要做权限检查、参数校验、二次确认和 audit trace。

## Biometric Identity Safety

Biometric Identity Safety 指多模态系统在处理人脸、声音、步态、虹膜等身份相关信号时的安全边界。

高风险问题包括未授权人脸识别、根据图片识别真实人物、声音克隆、名人或普通人 deepfake、隐私属性推断和身份欺诈。

面试表达：多模态安全不能只看有害文本。人脸和声音属于高度敏感信号，系统应区分一般描述、身份识别、授权验证、未授权生成和高风险欺诈场景。

## Content Provenance

Content Provenance 指记录内容来源、生成过程、编辑历史和可信凭证，用于判断图片、音频或视频是否为 AI 生成、是否被编辑以及是否可追溯。

常见机制包括水印、内容凭证、签名 metadata、生成日志、模型版本记录和审核链路。

面试表达：provenance 不能单独解决所有 deepfake 问题，但它能为内容分发、审核、溯源和责任界定提供证据链。它应和检测模型、用户授权和人工审核一起使用。

## Multimodal Safety Audit

Multimodal Safety Audit 是多模态系统上线前的安全审计，覆盖不可信媒体指令、隐私、身份、deepfake、生成内容审核、工具权限、过度拒答和事故回溯。

一个简化门禁可以写成：

```math
G_{\mathrm{mm\_safe}}=
\mathbb{1}[
A_{\mathrm{clean}}\ge \alpha
\land
R_{\mathrm{hall}}\le \beta
\land
R_{\mathrm{inj}}\le \gamma
\land
R_{\mathrm{priv}}\le \eta
\land
R_{\mathrm{over}}\le \rho
]
```

面试表达：多模态安全 audit 要同时证明两件事：正常任务还能完成，高风险媒体输入、身份请求和越权工具调用会被正确拦截。

## Multimodal Interview Readiness

Multimodal Interview Readiness 指候选人是否能把多模态知识从术语背诵提升到可解释、可手算、可写 demo、可讲项目和可讨论安全边界的面试状态。

常见检查项：

1. 能否讲清 CLIP / VLM / diffusion / video / audio / unified multimodal 的主线。
2. 能否写出关键公式和 shape，例如 CLIP loss、视觉 token、connector shape、DDPM、CFG、WER、IoU 和安全门禁。
3. 是否做过 0 依赖 demo 或最小可运行审计脚本。
4. 能否用 bad case 分析幻觉、OCR 错误、grounding 错误、时序不一致和 prompt injection。
5. 能否讲清成本、质量、安全、模块化和端到端训练之间的 trade-off。

面试表达：多模态面试准备度不是“看过多少模型名”，而是能把输入输出、结构、loss、评估、失败模式和项目证据串成完整回答。

## Multimodal Interview Rubric

Multimodal Interview Rubric 是多模态面试自评规约，用于给每道题打分并定位短板。

评分维度可以包括：

| 维度 | 高分回答特征 |
| --- | --- |
| 机制 | 能讲清输入、输出、结构和训练目标 |
| 公式 | 能写出核心公式或 shape |
| demo | 能说出自己写过的最小代码或审计脚本 |
| trade-off | 能比较成本、质量、延迟和安全 |
| 风险 | 能覆盖幻觉、隐私、身份和不可信媒体指令 |
| 项目 | 能把项目讲成问题、方案、指标、失败和改进 |

面试表达：rubric 的价值是把“感觉掌握了”变成可复盘分数，避免只复习自己熟悉的题。

## Multimodal Formula Coverage

Multimodal Formula Coverage 指多模态高频面试公式的覆盖率。

```math
C_{\mathrm{formula}}=
\frac{|F_{\mathrm{covered}}\cap F_{\mathrm{required}}|}
{|F_{\mathrm{required}}|}
```

必备公式通常包括 CLIP 相似度 / loss、视觉 token 数、connector shape、assistant-only loss、DDPM forward、CFG、latent ratio、video tokens、WER、unified tokens、VQA accuracy 和 safety gate。

面试表达：公式覆盖不是为了背复杂推导，而是让回答能落到可检查的数量关系、shape 和门禁上。

## Multimodal Demo Coverage

Multimodal Demo Coverage 指候选人是否为核心多模态主题准备了可运行或可手算 demo。

```math
C_{\mathrm{demo}}=
\frac{|D_{\mathrm{covered}}\cap D_{\mathrm{required}}|}
{|D_{\mathrm{required}}|}
```

推荐 demo 包括 CLIP 检索、VLM shape audit、多模态 SFT data audit、diffusion audit、video / audio audit、unified multimodal audit 和 evaluation / safety audit。

面试表达：demo coverage 能证明你不是只会说概念，而是能把概念转成输入、输出、指标和检查项。

## Weak Multimodal Question

Weak Multimodal Question 指 mock interview 中得分低、追问容易断、缺少公式或缺少项目证据的多模态问题。

常见弱项包括：audio codec、voice cloning 安全、统一多模态 loss mixture、FVD / FID 边界、图表评估、grounding 证据、项目 bad case 复盘。

面试表达：弱题不是失败记录，而是下一轮复习的索引。每个弱题都要绑定一个公式、一个 demo 和一个 3 分钟回答模板。

## Multimodal Revision Plan

Multimodal Revision Plan 是多模态面试复盘后的补齐计划。

一个有效计划至少包含：

1. 缺失公式清单。
2. 缺失 demo 清单。
3. 低分题清单。
4. 项目讲述改进点。
5. 下一次 mock interview 的验证标准。

面试表达：复盘计划要能被验证。例如“补 VQA accuracy 和 connector shape 公式”“补 eval safety audit demo”“重练 audio generation 和 multimodal safety 两道题”，比“继续复习多模态”更可执行。

## 多模态评估

多模态评估要同时看感知、对齐、推理和生成质量。

常见评估维度：

| 维度 | 关注点 | 示例 |
| --- | --- | --- |
| 物体识别 | 是否识别主要实体 | 图中有什么 |
| 属性识别 | 颜色、材质、状态 | 杯子是什么颜色 |
| 空间关系 | 左右、上下、远近 | 猫在桌子上还是下 |
| 计数能力 | 数量是否准确 | 有几辆车 |
| OCR | 文字读取是否正确 | 招牌上写了什么 |
| 图表理解 | 坐标轴、趋势、数值 | 哪一年增长最快 |
| Grounding | 语言是否能定位区域 | 指出红色按钮 |
| 推理 | 结合视觉和常识推断 | 为什么这个人可能在等车 |
| 忠实性 | 是否编造不存在内容 | 图片里没有就不能说有 |

自动 benchmark 有帮助，但不能完全替代人工评测。很多多模态答案开放性强，单一准确率无法反映是否忠实、是否详细、是否符合业务要求。

工程评估建议：为目标场景构建小而精的 golden set，包含真实图片、边界样本、低质量图片、OCR 样本、多图样本和拒答样本。每次模型升级都固定评测，避免只看公开榜单。

面试表达：多模态评估不能只问“模型答对了吗”，还要问“答案是否被视觉证据支持”。尤其在业务落地中，幻觉率和拒答质量往往比平均分更重要。

## 多模态工程取舍

多模态模型落地时，常见取舍如下：

| 取舍 | 选择 A | 选择 B | 判断标准 |
| --- | --- | --- | --- |
| 分辨率 | 低分辨率 | 高分辨率 | 延迟成本 vs 细节/OCR |
| 视觉 token | 多 token | 压缩 token | 细节保留 vs 上下文成本 |
| 训练方式 | 冻结 encoder/LLM | 端到端微调 | 成本稳定性 vs 适配上限 |
| 数据来源 | 公开数据 | 业务数据 | 泛化能力 vs 场景效果 |
| 结构 | projector | Q-Former/resampler/cross-attention | 简单高效 vs 融合能力 |
| 推理 | 单模型回答 | 工具增强 | 简洁部署 vs 可靠性 |
| 评估 | 公开榜单 | 场景 golden set | 横向比较 vs 真实可用性 |

常见落地路线：如果只是做图片问答 demo，可以选开源 VLM 直接微调；如果做企业文档理解，需要重点优化 OCR、版面结构和图表推理；如果做 GUI agent，需要 grounding 和坐标输出；如果做内容审核，需要安全分类、证据定位和低幻觉。

面试表达：多模态工程不是模型越大越好。要根据任务判断瓶颈是视觉识别、OCR、grounding、语言推理、延迟还是数据质量，然后选择对应优化方向。

## 高频面试题

### 1. CLIP 为什么能做零样本分类？

CLIP 学到了图像和文本的共享语义空间。分类时可以把类别名写成文本 prompt，编码成文本向量，再和图片向量计算相似度。相似度最高的类别就是预测结果。

关键点是 CLIP 训练时见过大量图文对，学会了图片语义和自然语言描述之间的对应关系，所以不需要针对每个分类任务重新训练分类头。

### 2. CLIP 和 LLaVA 的区别是什么？

CLIP 是双塔对齐模型，主要输出图像和文本 embedding，适合检索、分类和相似度匹配，不直接生成长回答。

LLaVA 是视觉语言助手，通常用 CLIP ViT 提取视觉特征，再通过 projector 接入 LLM，通过多模态指令微调生成自然语言答案。

一句话：CLIP 解决图文表示对齐，LLaVA 解决看图对话和指令跟随。

### 3. 为什么 VLM 需要 connector？

因为视觉 encoder 输出的特征维度、分布和 token 数量与 LLM 的文本 embedding 不一致。connector 负责把视觉特征映射到 LLM hidden space，并在必要时压缩视觉 token。

如果没有 connector，LLM 很难直接消费视觉特征。不同 connector 的差异主要体现在表达能力、压缩能力、训练成本和是否改造 LLM 架构。

### 4. Q-Former 和简单 MLP projector 有什么区别？

MLP projector 直接把视觉 token 映射到 LLM hidden size，结构简单、训练便宜，但对视觉信息选择和压缩能力有限。

Q-Former 使用可学习 query 通过 cross-attention 从视觉特征中抽取信息，可以把大量视觉特征压缩成固定数量 query token，更适合低成本桥接强 vision encoder 和强 LLM。

### 5. 多模态幻觉怎么解决？

先定位幻觉类型：是物体存在性错误、属性错误、数量错误、OCR 错误、空间关系错误，还是推理编造。

再针对原因处理：如果看不清，提升分辨率或裁图；如果对齐差，改进 connector 和多模态训练；如果数据有幻觉，清洗指令数据；如果语言先验太强，加入拒答和忠实性偏好数据；如果需要精确定位，引入 grounding 或工具。

面试中要强调：不能只靠 prompt 解决系统性幻觉，必须结合数据、模型结构、训练目标和评估闭环。

### 6. 多模态模型为什么 OCR 经常不好？

原因包括输入分辨率不足、小字信息在 patch 化和压缩中丢失、训练数据 OCR 样本不足、模型更偏自然图像语义而非文档结构，以及视觉 token 数量受限。

解决方式包括高分辨率输入、局部裁剪、多尺度视觉特征、加入文档和 OCR 数据、接外部 OCR 工具、使用专门文档理解模型。

### 7. 视觉 token 太多怎么办？

可以降低分辨率、增大 patch size、使用 pooling、Q-Former、Perceiver Resampler、token pruning、区域裁剪、多阶段粗到细推理，或者只把任务相关区域送入模型。

取舍是：压缩越强，成本越低，但细节越容易丢。OCR、计数、grounding 等任务通常不能过度压缩。

### 8. 多模态指令微调数据怎么构造？

需要覆盖描述、问答、OCR、计数、空间关系、图表理解、拒答、安全和多轮对话。数据答案必须忠实于图像，不能凭空扩写。可以用人工标注、已有数据集改写、强模型生成再过滤、业务日志清洗和合成数据混合构造。

高质量数据比盲目扩大数据量更重要，因为带幻觉的数据会直接教坏模型。

## 本章小结

多模态学习的核心是让不同模态在统一语义空间中对齐、融合和生成。

CLIP 和 SigLIP 代表图文对齐路线，擅长检索和零样本分类。ViT 是常见视觉 encoder，将图片转成 patch token。Q-Former、Perceiver Resampler 和 MLP projector 是常见 connector，用于把视觉特征接入语言模型。Flamingo、BLIP-2 和 LLaVA 代表不同视觉语言模型路线。

落地时，最重要的问题通常不是模型会不会说话，而是视觉证据是否真的被看见、对齐和忠实使用。OCR、grounding、计数、图表理解和多模态幻觉，是面试和工程中最容易暴露模型真实水平的地方。

面试最终表达：多模态大模型可以理解为“视觉 encoder 负责看，connector 负责翻译，LLM 负责推理和表达，数据与评估负责约束它不要乱说”。
