# B. 深度学习组件

条目：MLP、Linear Layer、Activation Function、ReLU、GeLU、SiLU、GLU、SwiGLU、Dropout、BatchNorm、LayerNorm、RMSNorm、Residual Connection、Skip Connection、Optimizer、SGD、Momentum、Adam、AdamW、Weight Decay、Learning Rate Warmup、Cosine Decay、Gradient Clipping、Initialization、Transformer Components、Token Embedding、LM Head Weight Tying、Causal Mask、Scaled Dot-Product Attention、Multi-Head Self-Attention、KV Cache、Transformer Component Audit、PyTorch Engineering Interview Readiness。

## MLP Multi-Layer Perceptron

一句话定义：MLP 是由线性层和非线性激活函数堆叠而成的前馈神经网络模块。

核心形式：`MLP(x) = W2 * act(W1 * x + b1) + b2`。

在 Transformer 中：MLP 通常位于 attention 子层之后，也常被称为 FFN 或 feed-forward network。

为什么需要它：attention 负责 token 之间的信息交互，MLP 负责对每个 token 的 hidden state 做非线性特征变换。

典型维度：Transformer MLP 的中间维度通常大于 hidden size，例如 `4 * hidden_size`，SwiGLU 结构中也常使用特定扩展比例。

常见误区：MLP 不是“可有可无的简单层”。在大模型中，MLP 参数量通常占比很高，是模型记忆、变换和组合特征的重要位置。

面试表达：Transformer block 可以粗略理解为 attention 负责跨 token 混合，MLP 负责逐 token 非线性变换，二者共同构成表达能力。

## Feed-Forward Network / FFN

一句话定义：FFN 是 Transformer 中对每个 token 独立应用的前馈网络，通常就是 MLP 模块。

核心特点：同一个 FFN 参数会作用在序列中的每个位置，但不同位置之间不在 FFN 内直接通信。

与 attention 的区别：attention 的输出依赖其他 token，FFN 的输出只依赖当前位置经过 attention 后的表示。

在大模型中的作用：FFN 提升模型的非线性表达能力，并在参数规模上承担大量知识存储。

面试追问：为什么 Transformer 需要 attention 后再接 FFN？因为 token 混合之后，还需要对混合后的表示做非线性投影和特征重组。

## Linear Layer

一句话定义：线性层对输入向量做仿射变换，是神经网络中最基础的参数化模块。

核心公式：`y = xW^T + b`。

在大模型中：embedding 投影、QKV 投影、attention 输出投影、MLP 上投影和下投影都依赖线性层。

参数量来源：一个 `d_in -> d_out` 的线性层参数量约为 `d_in * d_out`，如果有 bias 还要加 `d_out`。

常见误区：线性层本身表达能力有限，但多层线性层中间如果没有非线性激活，整体仍等价于一个线性变换。

面试表达：大模型的很多“结构复杂性”最终都会落到若干大矩阵乘法上，因此推理和训练优化常围绕 GEMM、显存访问和并行切分展开。

## Activation Function

一句话定义：激活函数为神经网络引入非线性，使多层网络能够表达复杂函数。

为什么需要它：如果没有激活函数，多层线性层堆叠仍然等价于单层线性层，模型表达能力会严重受限。

常见类型：ReLU、GeLU、SiLU、Swish、GLU、SwiGLU。

在大模型中：现代 LLM 更常见 GeLU、SiLU 或 SwiGLU，而不是最朴素的 ReLU。

常见误区：激活函数不是只影响精度，也会影响数值稳定性、梯度传播、计算效率和硬件实现。

面试追问：为什么很多新模型使用 SwiGLU？因为门控结构能增强特征选择能力，在相近计算量下通常有更好的效果。

## ReLU Rectified Linear Unit

一句话定义：ReLU 是将负数截断为 0、正数保持不变的激活函数。

核心公式：`ReLU(x) = max(0, x)`。

优点：计算简单，正区间梯度稳定，曾广泛用于 CNN 和早期深度网络。

局限：负区间梯度为 0，可能出现 dead neuron；输出均值偏正，也可能影响深层网络稳定性。

在大模型中：ReLU 在现代主流 LLM 中不是最常见选择，更多被 GeLU、SiLU 或门控激活替代。

面试表达：ReLU 的优势是简单高效，但 Transformer 大模型更关注平滑梯度、表达能力和训练稳定性，因此常采用更平滑或带门控的激活。

## GeLU Gaussian Error Linear Unit

一句话定义：GeLU 是一种平滑激活函数，可以理解为按输入值大小对特征进行概率式门控。

核心直觉：输入越大越容易通过，输入越小越容易被抑制，但不是像 ReLU 那样硬截断。

近似公式：`GeLU(x) = x * Φ(x)`，其中 `Φ` 是标准正态分布的累积分布函数。

在 Transformer 中：BERT、GPT 系列早期模型常使用 GeLU。

优点：相比 ReLU 更平滑，负区间也保留一定梯度，有利于稳定训练。

局限：计算比 ReLU 复杂，不过现代框架通常有高效近似实现。

面试追问：GeLU 和 ReLU 的核心区别是什么？ReLU 是硬门控，GeLU 是平滑门控；GeLU 对接近 0 的输入处理更柔和。

## SiLU / Swish

一句话定义：SiLU 是 `x * sigmoid(x)` 形式的平滑激活函数，也常被称为 Swish。

核心公式：`SiLU(x) = x * sigmoid(x)`。

核心直觉：它用 sigmoid 作为软门控，控制输入特征通过的程度。

在大模型中：SiLU 常作为 SwiGLU 的基础组件之一。

优点：平滑、非单调，在很多深层网络中效果较好。

常见误区：SiLU 不等于 sigmoid。sigmoid 输出在 0 到 1 之间，而 SiLU 会保留输入幅度并进行门控。

面试表达：SiLU 可以看作输入值乘以一个由自身决定的软门，既保留幅度信息，又引入非线性选择。

## GLU Gated Linear Unit

一句话定义：GLU 是一种带门控机制的前馈结构，用一个分支控制另一个分支的信息通过。

核心形式：`GLU(x) = (xW_a) * sigmoid(xW_b)`。

核心直觉：一个线性分支产生内容，另一个分支产生门控，二者逐元素相乘。

为什么重要：门控结构可以动态选择哪些特征需要保留，增强 FFN 的表达能力。

与普通 MLP 的区别：普通 MLP 主要依赖激活函数做非线性变换，GLU 额外引入显式乘性门控。

面试追问：门控为什么有用？因为它允许模型根据输入内容动态调节特征通路，而不是对所有输入使用固定变换强度。

## SwiGLU

一句话定义：SwiGLU 是把 GLU 中的 sigmoid 门控替换或扩展为 Swish/SiLU 风格门控的 FFN 变体。

常见形式：`SwiGLU(x) = (xW_gate) * SiLU(xW_up)`，再经过 `W_down` 投回 hidden size。

在 LLM 中：LLaMA 等模型使用 SwiGLU 作为 MLP/FFN 结构的重要组成部分。

优点：相比普通 GeLU FFN，SwiGLU 通常在相近计算量下有更好的建模效果。

代价：结构中通常有 gate、up、down 三个投影矩阵，参数量和实现细节需要配合中间维度设计。

常见误区：SwiGLU 不是单纯换一个激活函数，而是改变了 FFN 的门控结构。

面试表达：SwiGLU 的关键是乘性门控，它让 FFN 能根据输入动态筛选特征，这也是现代 LLM 常用它的重要原因。

## Dropout

一句话定义：Dropout 在训练时随机置零部分神经元输出，用来减少过拟合。

核心直觉：让模型不能过度依赖某些单一特征或路径，从而提升泛化能力。

训练与推理差异：训练时随机丢弃，推理时通常不丢弃，并通过缩放保持期望一致。

在大模型中：预训练阶段是否使用 dropout 取决于规模、数据量和训练策略；大数据大模型中 dropout 可能较小甚至不用。

局限：dropout 会引入噪声，可能影响收敛速度；在超大规模预训练中，数据规模本身已经提供了强正则效果。

常见误区：dropout 不是越大越好。过大的 dropout 会损害有效容量，导致欠拟合。

面试表达：dropout 是一种随机正则化手段，但在 LLM 中要结合数据量、模型规模、训练稳定性和目标任务判断是否使用。

## BatchNorm Batch Normalization

一句话定义：BatchNorm 使用 batch 维度上的均值和方差对激活进行归一化。

核心公式：`y = γ * (x - mean_batch) / sqrt(var_batch + eps) + β`。

优点：在 CNN 中常用于稳定训练、加速收敛，并带有一定正则化效果。

局限：依赖 batch 统计，训练和推理行为不同；小 batch、变长序列、自回归生成场景中容易带来问题。

为什么 LLM 少用：语言模型训练常有变长序列、分布式切分和自回归推理，BatchNorm 的 batch 统计依赖不够自然。

常见误区：BatchNorm 不是所有深度网络的默认最优选择。Transformer 中更常见 LayerNorm 或 RMSNorm。

面试表达：BatchNorm 适合许多视觉模型，但自回归语言模型更偏好不依赖 batch 统计、训练推理一致的归一化方法。

## LayerNorm Layer Normalization

一句话定义：LayerNorm 对单个样本的 hidden dimension 做归一化，不依赖 batch 统计。

核心公式：`LN(x) = γ * (x - mean_hidden) / sqrt(var_hidden + eps) + β`。

为什么适合 Transformer：它对每个 token 的 hidden state 独立归一化，训练和推理行为一致，也不依赖 batch size。

在大模型中：LayerNorm 可用于稳定激活尺度，改善梯度传播，是 Transformer 稳定训练的核心组件之一。

常见位置：Pre-LN 结构中放在 attention 和 MLP 子层之前；Post-LN 结构中放在残差之后。

局限：LayerNorm 需要计算均值和方差，相比 RMSNorm 稍复杂。

面试追问：LayerNorm 和 BatchNorm 的核心区别是什么？BatchNorm 沿 batch 统计，LayerNorm 沿 hidden dimension 统计；前者依赖 batch，后者更适合序列模型。

## RMSNorm Root Mean Square Layer Normalization

一句话定义：RMSNorm 使用均方根归一化 hidden state 的尺度，但不减去均值。

核心公式：`RMSNorm(x) = γ * x / sqrt(mean(x^2) + eps)`。

与 LayerNorm 的区别：LayerNorm 做去均值和除标准差，RMSNorm 只做尺度归一化。

在 LLM 中：LLaMA 等模型采用 RMSNorm，以减少计算并保持训练稳定。

优点：计算更简单，实践中效果稳定，适合大规模模型。

潜在局限：不显式中心化特征，理论表达和数值行为与 LayerNorm 不完全一致。

面试表达：RMSNorm 可以看作 LayerNorm 的简化版，它保留了控制激活尺度的关键作用，省去了均值中心化。

## Pre-LN vs Post-LN

一句话定义：Pre-LN 和 Post-LN 描述 Transformer 中 normalization 放在子层之前还是残差之后。

Pre-LN 形式：`x = x + sublayer(Norm(x))`。

Post-LN 形式：`x = Norm(x + sublayer(x))`。

核心差异：Pre-LN 通常更利于深层模型梯度传播和训练稳定，Post-LN 在某些设置下可能最终表达更强但训练更难。

为什么 Pre-LN 稳定：残差主路径更接近恒等映射，梯度可以更直接地穿过深层网络。

在大模型中：现代 LLM 通常更偏向 Pre-LN 或其变体。

常见误区：Pre-LN 不是简单“更先进”。它主要解决深层训练稳定性问题，具体效果仍与初始化、学习率、残差缩放和训练策略有关。

面试表达：如果面试官问深层 Transformer 为什么常用 Pre-LN，可以回答梯度路径更顺畅，训练更稳定，尤其适合大规模深层模型。

## Residual Connection

一句话定义：残差连接把输入直接加到子层输出上，使网络学习相对于输入的增量变化。

核心公式：`y = x + F(x)`。

为什么需要它：深层网络如果每层都必须重新学习完整映射，训练很难；残差让模型可以在恒等映射附近学习小修正。

在 Transformer 中：attention 子层和 MLP 子层外通常都有残差连接。

优点：缓解梯度消失，改善深层网络可训练性，使模型更容易扩展到很多层。

常见误区：残差连接不是简单“防止信息丢失”，更关键的是提供稳定的优化路径和梯度通路。

面试表达：残差连接是深层 Transformer 可训练的基础组件之一，它让每个 block 学习增量而不是重建全部表示。

## Skip Connection

一句话定义：Skip connection 是跨层传递信息的连接方式，residual connection 是其中最常见的一类。

与 residual connection 的关系：所有 residual connection 都是 skip connection，但 skip connection 不一定都通过加法残差实现。

在大模型中：常见形式是同维度张量相加，保证主干路径稳定。

工程意义：skip connection 会影响激活保存、梯度流、并行切分和数值尺度。

面试追问：为什么残差相加要求维度一致？因为 `x + F(x)` 是逐元素加法，两个张量形状必须兼容；若维度不同，需要投影对齐。

## Optimizer

一句话定义：优化器根据梯度和内部状态更新模型参数。

为什么需要它：反向传播只计算梯度，真正改变参数的是优化器。

核心组成：学习率、梯度、参数状态、动量或矩估计、weight decay、梯度裁剪等。

在大模型中：优化器不仅影响收敛速度，还影响显存占用、训练稳定性、最终泛化和可扩展性。

常见误区：优化器不是训练脚本中的普通配置项。AdamW 的状态显存可能是参数本身的数倍，是大模型训练成本的重要来源。

面试表达：选择优化器时要同时考虑优化效果、数值稳定性、显存成本和分布式训练实现。

## SGD Stochastic Gradient Descent

一句话定义：SGD 使用 mini-batch 梯度直接更新参数。

核心公式：`θ_new = θ_old - η * g_t`。

优点：简单、内存开销低、状态少，在一些任务上泛化表现好。

局限：对学习率敏感，收敛速度可能慢，在复杂非凸深度网络中调参成本较高。

在大模型中：纯 SGD 不是主流 LLM 预训练优化器，更多使用 AdamW 类自适应优化器。

常见误区：SGD 中的 stochastic 不是指每次只用一个样本，而是通常使用 mini-batch 对全量梯度做随机近似。

面试表达：SGD 的核心是用 mini-batch 梯度近似全量梯度，以更低计算成本进行迭代优化。

## Momentum

一句话定义：Momentum 在当前梯度之外引入历史更新方向，减少震荡并加速稳定方向上的下降。

核心形式：`v_t = βv_{t-1} + g_t`，`θ = θ - ηv_t`。

核心直觉：像物理中的惯性，连续一致的梯度方向会被累积，来回震荡的方向会相互抵消。

优点：加快收敛，缓解狭长峡谷地形中的震荡。

与 Adam 的关系：Adam 可以看作同时使用一阶矩和二阶矩的自适应方法，一阶矩部分与 momentum 思想相关。

面试追问：Momentum 为什么能减少震荡？因为相反方向的梯度在动量累积中会抵消，而一致方向会增强。

## Adam Adaptive Moment Estimation

一句话定义：Adam 使用梯度的一阶矩和二阶矩估计，为不同参数自适应调整更新幅度。

核心形式：一阶矩 `m_t` 估计梯度均值，二阶矩 `v_t` 估计梯度平方均值，更新时用 `m_t / sqrt(v_t)`。

优点：收敛快，对学习率相对不那么敏感，适合稀疏梯度和大规模深度网络。

代价：需要保存额外优化器状态，通常包括参数、梯度、一阶矩、二阶矩，显存开销大。

在 LLM 中：Adam 或 AdamW 是常见预训练和微调优化器基础。

常见误区：Adam 自适应学习率不代表不需要调学习率。base learning rate、warmup、decay、batch size 仍然非常关键。

面试表达：Adam 的核心价值是对不同参数按历史梯度尺度自适应归一化更新，提升复杂深度网络训练的稳定性和效率。

## AdamW

一句话定义：AdamW 是将 weight decay 从 Adam 梯度更新中解耦的优化器。

核心区别：Adam 中的 L2 正则会进入梯度并被自适应缩放；AdamW 单独对参数执行 decay，不把它混入梯度矩估计。

为什么重要：解耦后 weight decay 的含义更清晰，调参更稳定，也更适合 Transformer 等大模型训练。

在大模型中：AdamW 是预训练、SFT 和很多后训练流程中的常见默认选择。

工程注意：bias、LayerNorm、RMSNorm 等参数通常不做 weight decay。

常见误区：AdamW 不是 Adam 加一个配置名，它改变了 weight decay 的应用方式。

面试追问：为什么 decoupled weight decay 对 Transformer 训练重要？因为自适应缩放会改变 L2 正则的实际效果，解耦后正则强度更可控。

## Weight Decay

一句话定义：Weight decay 是训练时让参数逐步向 0 衰减的正则化方法。

核心形式：`θ = θ - η * λ * θ`，其中 `λ` 是 decay 系数。

作用：控制参数规模，缓解过拟合，并可能提升训练稳定性。

与 L2 正则关系：在 SGD 中，L2 正则和 weight decay 常等价；在 Adam 这类自适应优化器中，二者不完全等价。

在大模型中：常对矩阵权重使用 weight decay，但通常排除 bias 和归一化层参数。

常见误区：weight decay 不是越大越好。过强会压制模型容量，导致欠拟合或能力下降。

面试表达：谈 AdamW 时要强调 decoupled weight decay，而不是简单说“AdamW 就是 Adam 加 L2 正则”。

## Learning Rate

一句话定义：学习率控制每次参数更新的步长。

核心公式：`θ_new = θ_old - η * update_direction`。

为什么重要：学习率过大可能 loss spike、发散或 NaN；学习率过小会收敛慢，甚至陷入低效训练。

在大模型中：学习率通常配合 warmup、decay、batch size、梯度裁剪和精度策略共同设计。

常见误区：学习率不是一个孤立超参数。改变 batch size、训练 token 数、模型规模后，最优学习率也可能变化。

面试表达：学习率决定优化步长，是训练稳定性和收敛质量最敏感的超参数之一。

## Learning Rate Warmup

一句话定义：warmup 是训练初期逐步增大学习率的策略。

为什么需要它：训练初期参数、激活、梯度尺度和 Adam 矩估计都不稳定，直接使用大学习率容易造成 loss spike 或 NaN。

常见形式：线性 warmup，从 0 或很小学习率逐步增加到目标学习率。

在大模型中：warmup 是预训练和微调常见稳定化手段，尤其在大 batch、混合精度和深层 Transformer 中重要。

局限：warmup 只能缓解初期不稳定，不能根治错误数据、过大学习率、异常 loss 或数值溢出问题。

面试追问：warmup 为什么对 Adam 有帮助？因为 Adam 的一阶和二阶矩估计在训练初期还不可靠，逐步放大学习率更稳。

## Cosine Decay

一句话定义：Cosine decay 是按余弦曲线逐渐降低学习率的调度策略。

核心直觉：早期保持较大学习率快速探索，后期降低学习率稳定收敛。

常见组合：linear warmup + cosine decay。

在大模型中：常用于预训练和微调，使训练后期参数更新更细致。

与 step decay 区别：step decay 是分段突降，cosine decay 更平滑。

常见误区：cosine decay 不能替代选择合适的 base learning rate；如果峰值学习率过大，warmup 后仍可能不稳定。

面试表达：cosine decay 的优势是平滑退火，工程上常和 warmup 结合形成完整学习率 schedule。

## Gradient Clipping

一句话定义：Gradient clipping 限制梯度范数或梯度值，防止异常大梯度导致训练不稳定。

常见方式：global norm clipping，即当整体梯度范数超过阈值时按比例缩小全部梯度。

为什么需要它：深层网络、长序列、混合精度训练或异常 batch 可能导致梯度爆炸。

在大模型中：gradient clipping 是常见训练稳定化配置，尤其用于预训练、SFT 和 RLHF/PPO 等阶段。

局限：它是保护机制，不是根因修复。频繁触发 clipping 可能说明学习率、数据、loss scale 或训练配置存在问题。

常见误区：clip 阈值越小越安全。阈值过小会长期压缩有效梯度，降低训练效率。

面试表达：gradient clipping 的价值是防止极端 batch 破坏训练，但需要结合梯度范数日志判断是否过度触发。

## Initialization

一句话定义：初始化决定训练开始时参数的分布和尺度。

为什么重要：初始化会影响激活方差、梯度传播和早期训练稳定性。

常见目标：让信号在前向和反向传播中不过快放大或衰减。

常见方法：Xavier initialization、Kaiming initialization、normal initialization，以及 Transformer 中的残差相关缩放策略。

在大模型中：初始化与 normalization、残差连接、模型深度、学习率 warmup 强相关。

常见误区：有了 LayerNorm 就不需要关心初始化。归一化能缓解尺度问题，但不能完全消除初始化对训练稳定性的影响。

面试表达：初始化不是只影响第一步 loss，它会影响整个早期优化轨迹，深层 Transformer 尤其需要匹配结构和学习率设计。

## Xavier Initialization

一句话定义：Xavier 初始化根据输入和输出维度设置参数方差，目标是保持前向和反向信号尺度稳定。

典型适用：tanh、sigmoid 等较早期激活函数网络中常见。

核心直觉：如果权重太大，激活和梯度容易爆炸；权重太小，信号容易消失。

与 Kaiming 的区别：Kaiming 初始化更针对 ReLU 类激活的半截断特性设计。

面试表达：Xavier 的核心不是某个固定公式，而是根据 fan-in 和 fan-out 控制方差，维持信号传播稳定。

## Kaiming Initialization / He Initialization

一句话定义：Kaiming 初始化是为 ReLU 类激活设计的参数初始化方法。

核心直觉：ReLU 会截断约一半输入，因此需要相应调整权重方差，保持激活尺度稳定。

典型适用：ReLU、Leaky ReLU 等激活网络。

在 Transformer 中：虽然大模型不一定直接套用最朴素 Kaiming 初始化，但“匹配激活函数和网络结构控制方差”的思想仍然重要。

面试追问：为什么不同激活函数需要不同初始化？因为激活函数会改变输出分布和梯度分布，初始化必须配合这种变化。

## Parameter Count

一句话定义：参数量是模型中可训练参数的总数，决定模型容量和存储需求的重要因素。

计算方式：线性层参数量约为 `d_in * d_out`，embedding 参数量约为 `vocab_size * hidden_size`。

在 Transformer 中：主要参数来自 embedding、attention 的 QKV/O 投影、MLP/FFN 和输出 head。

为什么重要：参数量影响模型容量、显存占用、加载速度、训练成本和部署成本。

常见误区：参数量越大不必然越好。数据质量、训练 token 数、架构、优化和对齐策略都会影响最终能力。

面试表达：估算参数量时要能拆分到 embedding、attention、MLP 和层数，而不是只背模型总参数。

## Activation Memory

一句话定义：activation memory 是训练时为反向传播保存中间激活所占用的显存。

为什么重要：训练显存不只来自参数和优化器状态，长序列、大 batch 和深层网络会让激活显存非常大。

影响因素：batch size、sequence length、hidden size、layer number、attention 实现、是否使用 activation checkpointing。

在大模型中：激活显存常是训练长上下文模型的重要瓶颈。

常见缓解：gradient checkpointing、sequence parallel、flash attention、减小 micro batch size。

常见误区：只看参数量无法判断训练显存。两个同参数量模型，如果序列长度不同，激活显存可能差异很大。

面试表达：训练显存大致由参数、梯度、优化器状态和激活组成，其中激活与 batch 和 sequence length 强相关。

## Mixed Precision Training

一句话定义：mixed precision training 是在训练中混合使用 FP32、FP16、BF16 等不同精度，以降低显存和带宽压力并提升吞吐。

核心直觉：大部分矩阵乘法适合低精度，少数数值敏感位置需要保留更高精度或额外稳定化手段。

常见组合：FP16 + `GradScaler`，或 BF16 + `autocast`。具体选择取决于硬件、框架和训练稳定性。

## FP16

一句话定义：FP16 是 16 位浮点格式，显存和带宽开销约为 FP32 的一半。

优势：存储省、吞吐高，很多 GPU 支持成熟。

风险：指数范围较小，容易 overflow / underflow，训练时常配合 loss scaling 和 `GradScaler`。

## BF16

一句话定义：BF16 是 16 位浮点格式，指数范围接近 FP32，但尾数位更少。

优势：动态范围大，通常比 FP16 更不容易溢出，大模型训练中常更稳。

风险：尾数精度比 FP16 低，且依赖硬件和框架支持。

## Autocast

一句话定义：`autocast` 是 PyTorch AMP 中自动选择算子执行 dtype 的上下文管理器。

作用：让适合低精度的算子以 FP16 / BF16 执行，让部分敏感算子保留更稳的精度，减少手动 dtype 管理。

边界：`autocast` 主要管理前向算子精度，不等于自动解决所有显存、溢出或训练稳定性问题。

## GradScaler

一句话定义：`GradScaler` 通过 loss scaling 缓解 FP16 小梯度下溢问题。

典型顺序：`scale(loss).backward()`、`unscale_(optimizer)`、gradient clipping、`step(optimizer)`、`update()`。

关键边界：裁剪前要先 unscale；BF16 通常不强依赖 GradScaler。

## Loss Scaling

一句话定义：loss scaling 是先把 loss 放大再反向传播，optimizer step 前再把梯度缩回真实尺度。

公式直觉：如果 `L_scaled=s*L`，则梯度也被放大 `s` 倍；unscale 后再用于裁剪和更新。

## Activation Checkpointing

一句话定义：activation checkpointing 是前向少存一部分激活、反向时重算部分前向，用计算换显存。

适用场景：深层 Transformer、长上下文训练、显存紧张但可接受额外计算成本的实验。

常见坑：会增加 backward 时间；涉及随机层时要关注 RNG 状态和 checkpoint API 参数。

## CUDA Memory Stats

一句话定义：CUDA memory stats 是 PyTorch 提供的 GPU 显存统计接口，用于观察 allocated、reserved、peak memory 等状态。

常见指标：`memory_allocated()`、`memory_reserved()`、`max_memory_allocated()` 和 `reset_peak_memory_stats()`。

调试用途：区分真实 tensor 占用、缓存保留、峰值显存和疑似显存泄漏。

## OOM Audit

一句话定义：OOM audit 是系统排查 out-of-memory 的流程。

常见顺序：区分训练 / 验证阶段，缩小 batch size 和 sequence length，检查 mixed precision、gradient accumulation、activation checkpointing、optimizer state、带图 tensor 缓存和验证是否关闭梯度。

## PyTorch Tensor

一句话定义：PyTorch tensor 是深度学习代码里的核心数据对象，包含数据、shape、dtype、device、stride 和梯度相关状态。

在大模型中：input ids、embedding、hidden states、attention scores、logits、loss mask、参数和梯度都是 tensor。

面试表达：调试 PyTorch 代码时先看 shape，再看 dtype、device、contiguous / stride、mask 方向和数值是否 finite。

## Tensor Shape

一句话定义：tensor shape 描述每个维度的长度，是模型实现中最重要的调试信息。

常见 LLM shape：`input_ids [B,T]`，hidden states `[B,T,d]`，logits `[B,T,V]`，attention scores `[B,H,T,T]`。

常见错误：loss、matmul、mask 或 concat 前没有对齐 shape。

## Dtype

一句话定义：dtype 是 tensor 的数据类型，决定数值精度、显存占用和某些模块的输入要求。

常见类型：token id 和 label 通常是 `torch.long`；模型激活常见 `float32`、`float16`、`bfloat16`；mask 常见 bool 或浮点加性 mask。

## Device

一句话定义：device 表示 tensor 存放在 CPU、GPU 或其他设备上。

常见坑：同一次计算里的 tensor 不能混在 CPU 和 GPU 上；batch 通常要移动到 `next(model.parameters()).device`。

## Broadcasting

一句话定义：broadcasting 允许某些 shape 不同的 tensor 做逐元素运算。

规则：从右往左对齐，每一维要么相等，要么其中一个是 1，要么某个 tensor 缺少该维。

在 LLM 中：bias、LayerNorm 参数和 attention mask 都大量依赖 broadcasting。

## Tensor Stride

一句话定义：stride 描述沿某个维度移动一步时，在底层 storage 中跳过多少个元素。

为什么重要：transpose / permute 通常改变 stride 而不复制数据，所以结果可能不是 contiguous。

## Contiguous Tensor

一句话定义：contiguous tensor 的内存布局与其 shape 顺序兼容，常可直接用 `view` 改 shape。

常见场景：多头 attention 从 `[B,H,T,d_h]` 转回 `[B,T,d]` 时，经常写 `transpose(...).contiguous().view(...)`。

## View vs Reshape

一句话定义：`view` 要求当前 tensor 的 stride 与目标 shape 兼容；`reshape` 更宽松，必要时可能创建拷贝。

工程建议：原型和调试中优先用 `reshape` 更稳；明确追求 view 语义时，要先确认 tensor 是否 contiguous。

## Tensor Mask

一句话定义：tensor mask 用 bool 或加性浮点值控制哪些位置参与计算。

在 attention 中：padding mask 常从 `[B,T]` 扩成 `[B,1,1,T]`，causal mask 常从 `[T,T]` 扩成 `[1,1,T,T]`。

常见坑：mask shape 能广播不代表语义正确，要确认 query 维和 key 维方向。

## Tensor Shape Audit

一句话定义：tensor shape audit 是系统检查 shape、dtype、device、stride、contiguous、mask 和 finite 数值的调试流程。

用途：快速定位大模型代码中的 loss shape mismatch、device mismatch、view 报错、attention mask 方向错误和 NaN / Inf。

## PyTorch Autograd

一句话定义：PyTorch autograd 是自动求导系统，在 forward 时动态构建计算图，在 backward 时按链式法则计算梯度。

核心机制：可求导操作会产生 `grad_fn`，模型参数通常是 leaf tensor，`loss.backward()` 会把梯度累积到 leaf 参数的 `.grad`。

在大模型中：pretraining、SFT、LoRA、RLHF、DPO 和训练 debug 都依赖 autograd 正确连接 loss 与可训练参数。

## Dynamic Computation Graph

一句话定义：动态计算图是在每次 forward 执行时按真实 Python 控制流构建的计算图。

优势：`if`、`for`、递归和动态 shape 可以自然参与建图。

注意：默认一次 backward 后图会释放；普通训练应重新 forward，而不是滥用 `retain_graph=True`。

## Requires Grad

一句话定义：`requires_grad` 决定 tensor 是否需要被 autograd 跟踪梯度。

工程用法：冻结 base model 时把 base 参数设为 `False`，LoRA adapter 或 head 参数保持 `True`。

边界：冻结参数不等于整段 forward 都不建图；如果后续可训练参数依赖中间结果，仍可能需要保留部分图。

## Leaf Tensor

一句话定义：leaf tensor 通常是用户直接创建且需要梯度的 tensor，模型参数也是典型 leaf tensor。

常见现象：默认只有 leaf tensor 的 `.grad` 会在 backward 后被保留；中间 tensor 要观察梯度需要 `retain_grad()` 或 hook。

## Grad Accumulation

一句话定义：PyTorch 默认把每次 backward 的梯度加到已有 `.grad` 上，而不是自动清零。

为什么需要：它支持多 loss 和 gradient accumulation；普通训练每个 optimizer step 前要 `zero_grad()`。

## Zero Grad

一句话定义：`zero_grad()` 用于清空历史梯度，避免不同 batch 的梯度意外叠加。

常见写法：`optimizer.zero_grad(set_to_none=True)`，通常比把梯度填零更省内存。

## Detach

一句话定义：`detach()` 返回一个与原 tensor 共享数据但不再连接当前计算图的新 tensor。

常见用途：日志、缓存、target network、stop-gradient 和截断长序列反传。

风险：误把模型输出或中间 hidden state detach，会导致梯度断开，参数学不到。

## No Grad

一句话定义：`torch.no_grad()` 是上下文管理器，使作用域内的新操作不记录 autograd 图。

与 `model.eval()` 区别：`eval()` 改变 dropout / batch norm 等模块行为，`no_grad()` 关闭梯度记录；验证通常两者都用。

## Inference Mode

一句话定义：`torch.inference_mode()` 是更激进的推理模式，进一步关闭部分 autograd 相关机制。

适用：纯推理、离线 embedding 抽取和验证中不需要后续参与 autograd 的张量。

## In-place Operation

一句话定义：in-place 操作会直接修改原 tensor，PyTorch 中很多以下划线结尾的方法都是 in-place。

autograd 风险：backward 可能需要 forward 保存的中间值；如果这些值被原地修改，可能报版本计数或 inplace 错误。

## Autograd Hook

一句话定义：hook 可以在反向传播时观察或处理梯度，常用于临时 debug。

用途：查看某层梯度是否为 0、NaN、Inf 或异常大；定位梯度消失 / 爆炸位置。

## Autograd Audit

一句话定义：autograd audit 是系统检查 loss 是否依赖参数、梯度是否存在、图是否被 detach / no_grad 切断、梯度是否累积和数值是否 finite 的调试流程。

常见输出：`loss.requires_grad`、`loss.grad_fn`、参数 `requires_grad`、`grad is None`、grad norm、missing grad 参数名和 anomaly detection 结果。

## nn.Module

一句话定义：`nn.Module` 是 PyTorch 组织模型的核心抽象，负责注册参数、子模块和 buffer，并提供递归遍历、device / dtype 迁移、训练/推理模式切换和保存加载接口。

为什么重要：optimizer、`state_dict`、DDP、FSDP、AMP、checkpoint 和大模型组件替换都依赖 Module 树。

常见误区：把 `nn.Module` 只理解成“必须继承的模型类”。真正关键的是注册机制和递归管理能力。

## nn.Parameter

一句话定义：`nn.Parameter` 是特殊 tensor，被赋值为 Module 属性时会自动注册为模型参数。

工程意义：注册参数会出现在 `model.parameters()`、`named_parameters()` 和 `state_dict()` 中，optimizer 通常据此更新权重。

常见坑：普通 tensor 即使 `requires_grad=True`，如果不是 `Parameter`，通常不会被 optimizer 自动拿到。

## Module Registration

一句话定义：Module registration 是把参数、子模块和 buffer 挂到 Module 树上的机制。

常见方式：属性赋值、`ModuleList`、`ModuleDict`、`ParameterList`、`ParameterDict` 和 `register_buffer`。

风险：子模块放进普通 list / dict，或者在 `forward` 中临时创建层，会导致参数不注册、权重不保存、device 不迁移、DDP 不同步。

## ModuleList

一句话定义：`ModuleList` 是保存一组已注册子模块的容器，但不规定 forward 执行顺序。

在大模型中：Transformer block 堆叠通常用 `ModuleList`，因为 forward 里还要处理 mask、cache、hidden states、gradient checkpointing 等控制逻辑。

对比：`Sequential` 适合简单流水线，`ModuleDict` 适合按名称选择 adapter、head 或分支。

## ParameterList

一句话定义：`ParameterList` 是保存一组已注册参数的容器。

使用场景：自定义可学习 prompt、加权融合、任务 bias、adapter 标量或不方便包装成完整子模块的参数集合。

常见坑：普通 list 中的 `nn.Parameter` 不会自动注册。

## Buffer

一句话定义：buffer 是不由 optimizer 更新、但属于模型状态的 tensor。

常见例子：BatchNorm running statistics、固定位置编码、固定 attention mask、量化 scale / zero point 和统计量。

工程规则：buffer 不出现在 `parameters()` 中，会随 `model.to(device)` 移动，默认保存在 `state_dict()`；可用 `persistent=False` 排除可再生成的大缓存。

## state_dict

一句话定义：`state_dict` 是 PyTorch 保存模型权重的标准字典，包含注册参数和持久化 buffer。

不包含：Python 类定义、`forward` 代码、普通属性、未注册 tensor、optimizer 状态和 `persistent=False` buffer。

面试表达：推荐保存 `model.state_dict()`，而不是直接 pickle 整个 model；完整训练恢复还要保存 optimizer、scheduler、step、epoch 和 config。

## load_state_dict

一句话定义：`load_state_dict` 用 checkpoint 字典恢复模型参数和 buffer。

关键参数：`strict=True` 要求 key 完全匹配；`strict=False` 允许部分加载，但必须检查 missing keys 和 unexpected keys 是否符合预期。

常见原因：模型结构变化、层命名变化、DDP 的 `module.` 前缀、只加载 backbone、分类头变化或词表大小变化。

## train / eval Mode

一句话定义：`model.train()` 和 `model.eval()` 递归设置 Module 的 `training` 状态。

影响层：Dropout、BatchNorm 这类依赖训练模式的层。

边界：`eval()` 不会关闭 autograd；推理通常还要配合 `torch.no_grad()` 或 `torch.inference_mode()`。

## Module Hook

一句话定义：Module hook 是在 forward 或 backward 流程中插入回调的机制。

用途：抓中间激活、统计层输出、检查 NaN / Inf、观察梯度和临时研究分析。

风险：保存激活时要 `detach()`，用完要 `remove()`，不要把 hook 当长期业务逻辑依赖。

## Module Audit

一句话定义：Module audit 是系统检查模型注册树、参数表、buffer、`state_dict`、train/eval 状态、optimizer 参数组和保存加载一致性的调试流程。

常见输出：`named_parameters()`、`named_buffers()`、`state_dict().keys()`、可训练参数量、optimizer 参数量、missing / unexpected keys、冻结参数列表和保存加载后 logits 对齐结果。

## Dataset / DataLoader

一句话定义：Dataset / DataLoader 是 PyTorch 中把原始样本稳定组织成训练 batch 的数据管线核心抽象。

分工：Dataset 负责单样本读取和轻量预处理；DataLoader 负责 batch 取样、可选 shuffle、多进程加载、调用 `collate_fn` 和配合 sampler 控制顺序。

大模型场景：常见输出包括 `input_ids [B,T]`、`labels [B,T]`、`attention_mask [B,T]`，padding 位置的 label 通常设为 `-100` 以避免参与 loss。

## Map-style Dataset

一句话定义：map-style Dataset 是支持 `__len__()` 和 `__getitem__(idx)` 的可索引数据集。

适用场景：固定离线语料、监督训练数据、可随机访问的数据文件和需要 shuffle / sampler 控制索引顺序的训练任务。

常见坑：`__getitem__` 中做过重预处理会拖慢训练；把 batch 级 padding 写进 Dataset 会浪费计算并增加调试难度。

## IterableDataset

一句话定义：IterableDataset 是通过 `__iter__()` 流式产生样本的数据集。

适用场景：超大规模流式数据、在线日志、远程对象存储逐条读取和无法提前知道长度的数据源。

常见坑：不能简单依赖 `shuffle=True` 完成全局随机；多 worker / 多 rank 场景需要额外处理切片，避免重复读同一批样本。

## Collate Function

一句话定义：`collate_fn` 是把一个样本列表整理成模型可消费 batch 的函数。

大模型场景：它通常负责变长序列 padding、构造 `attention_mask`、把 padding label 设为 `-100`、整理复杂字段和必要时做 packing。

调试重点：检查 `input_ids`、`labels`、`attention_mask` 的 shape、padding 方向、ignore index 和 next-token shift 是否一致。

## Data Sampler

一句话定义：sampler 控制 DataLoader 按什么顺序取样本索引。

常见类型：顺序采样、随机采样、加权采样、length bucket 采样、batch sampler 和分布式采样。

面试重点：`shuffle=True` 是简单随机顺序；sampler 控制单样本索引顺序；batch sampler 直接控制一整个 batch 包含哪些索引。

## DistributedSampler

一句话定义：DistributedSampler 在分布式训练中按 rank 切分数据索引，让不同进程看到不同样本子集。

关键规则：每个 epoch 通常要调用 `set_epoch(epoch)` 刷新 shuffle；`shuffle=True` 通常不要和手动 sampler 混用；数据量不能整除时要理解补样本和 `drop_last` 的取舍。

## DataLoader Worker

一句话定义：DataLoader worker 是用于并行读取和预处理数据的子进程。

参数相关：`num_workers` 控制 worker 数量，`persistent_workers` 控制 worker 是否跨 epoch 保持存活，`prefetch_factor` 控制每个 worker 预取 batch 数。

常见坑：worker 不是越多越好；CPU、磁盘 IO、解码库、多进程开销和随机种子都可能成为瓶颈或错误来源。

## Pin Memory

一句话定义：pinned memory 是页锁定 CPU 内存，常用于加速 CPU 到 GPU 的异步数据拷贝。

工程口径：`pin_memory=True` 常与 `tensor.to(device, non_blocking=True)` 搭配；它主要优化搬运路径，不改变模型数学逻辑。

## Data Pipeline Audit

一句话定义：data pipeline audit 是系统检查 Dataset、collate、DataLoader、sampler、padding、mask、worker 和分布式切分是否正确的调试流程。

常见输出：样本原始结构、batch shape、有效 token 比例、padding waste、labels ignore 检查、shuffle 可复现性、rank 间索引 overlap 和吞吐瓶颈定位。

## Training Loop

一句话定义：training loop 是把模型、数据、loss、optimizer、scheduler、日志、验证和 checkpoint 串成可持续实验系统的训练主流程。

典型顺序：batch 到 device、forward、loss、`backward()`、可选 gradient clipping、`optimizer.step()`、`scheduler.step()`、`zero_grad()`、日志、验证和 checkpoint。

工程重点：它必须可观测、可恢复、可复现，并能定位 loss spike、NaN / Inf、梯度异常、学习率异常和 checkpoint resume 问题。

## Training Step

一句话定义：training step 是一次真实参数更新，对应一次 `optimizer.step()`。

边界：在 gradient accumulation 中，多个 micro-step 才对应一个 training step；scheduler 和 global step 通常应该跟随真实 optimizer step，而不是每个 micro-batch。

## Gradient Accumulation

一句话定义：gradient accumulation 是把多个 micro-batch 的梯度累积起来，再执行一次 optimizer step，以模拟更大的 batch。

关键规则：每个 micro-step 通常使用 `loss / accum_steps` 再 backward；否则梯度尺度会放大，等价于隐式提高学习率。

## Optimizer Step

一句话定义：optimizer step 是优化器根据当前梯度和内部状态更新模型参数的动作。

工程顺序：通常在 `loss.backward()` 和 gradient clipping 之后执行；执行后要清梯度，避免下一步意外累积。

## Scheduler Step

一句话定义：scheduler step 是更新学习率调度状态的动作。

常见规则：大多数按 step 更新的 scheduler 放在 `optimizer.step()` 之后；使用梯度累积时，只在真实 optimizer step 后调用。

## Evaluation Loop

一句话定义：evaluation loop 是验证模型指标的循环，通常不更新参数。

关键规则：使用 `model.eval()` 切换模块行为，并配合 `torch.no_grad()` 或 `torch.inference_mode()` 关闭梯度记录；结束后按需要恢复 `model.train()`。

## Checkpoint Resume

一句话定义：checkpoint resume 是从保存的训练状态恢复模型训练。

完整状态：除了模型 `state_dict`，还应保存 optimizer、scheduler、epoch / global step、config 和随机状态。只保存模型权重会丢失动量、二阶矩和学习率位置。

## Non-Finite Loss

一句话定义：non-finite loss 指 loss 变成 NaN 或 Inf。

排查入口：当前 batch、learning rate、gradient norm、混合精度溢出、mask 全忽略 / 全 `-inf`、输入异常值和带图 tensor 泄漏。

## Training Loop Audit

一句话定义：training loop audit 是系统检查训练 step 顺序、loss mask、梯度、学习率、验证、checkpoint 和恢复训练是否正确的调试流程。

常见输出：raw loss、scaled loss、grad norm、optimizer step 数、scheduler step 数、learning rate、validation loss、checkpoint keys、resume 后指标一致性和非有限 loss 检查结果。

## Distributed Training

一句话定义：distributed training 是用多个进程、GPU 或机器共同完成训练，以突破单卡显存和速度限制。

常见路线：数据并行、参数分片、张量并行、流水线并行和 expert parallel。入门阶段优先掌握 DDP 数据并行。

## Rank / World Size

一句话定义：rank 是分布式训练中的全局进程编号，world size 是总进程数，local rank 是当前机器内的进程或 GPU 编号。

工程用途：`local_rank` 常用于绑定本机 GPU，`rank==0` 常用于打印日志和保存 checkpoint，`world_size` 用于计算 global batch 和通信规模。

## Process Group

一句话定义：process group 是一组可相互通信的分布式训练进程。

常见操作：初始化后可执行 `all_reduce`、`broadcast`、`barrier` 等 collective；训练结束时应销毁进程组。

## DistributedDataParallel / DDP

一句话定义：DDP 是 PyTorch 推荐的数据并行训练封装，每个进程通常持有一份模型副本，处理不同 mini-batch，并在 backward 中同步梯度。

关键机制：DDP 通过 autograd hook 和 gradient bucket 在梯度 ready 时触发同步，尽量让通信和反向计算重叠。

## All-Reduce

一句话定义：all-reduce 是把所有 rank 上的张量做归约，并把归约结果返回给所有 rank 的 collective 操作。

DDP 语境：每个 rank 的本地梯度经过 all-reduce 求平均后，所有 rank 使用相同梯度执行 optimizer step，从而保持参数一致。

## torchrun

一句话定义：`torchrun` 是 PyTorch 常用的分布式训练启动方式，会启动多个训练进程并设置 rank 相关环境。

常见变量：`RANK`、`LOCAL_RANK`、`WORLD_SIZE`。训练脚本通常读取这些变量完成进程组初始化和 GPU 绑定。

## Global Batch Size

一句话定义：global batch size 是一次真实参数更新中所有 rank、所有梯度累积 micro-step 共同贡献的样本数。

常见公式：`B_global = B_device * world_size * accum_steps`。多卡训练时如果每卡 batch 不变，global batch 会变大，学习率和 warmup 需要重新核对。

## DDP no_sync

一句话定义：`no_sync()` 是 DDP 在梯度累积中跳过部分 micro-step 梯度同步的上下文管理器。

使用边界：通常只在非最后一个 accumulation micro-step 使用，最后一个 micro-step 仍要同步梯度并执行 optimizer step。

## Distributed Training Audit

一句话定义：distributed training audit 是系统检查 rank、数据切分、global batch、梯度同步、collective、checkpoint 写入和卡住风险的调试流程。

常见输出：rank / local rank / world size、每个 rank 的数据索引、rank 间 overlap、global batch size、all-reduce 后梯度、参数一致性、通信次数、缺失 collective 的 rank 和 rank 0 checkpoint 写入检查。

## Debug Tensor Metadata

一句话定义：debug tensor metadata 是先检查 tensor 的 shape、dtype、device、requires_grad、stride 和 contiguous 状态，再看具体数值。

用途：快速定位 shape mismatch、dtype mismatch、device mismatch、非 contiguous view 报错、loss 展平错误和 mask broadcast 方向错误。

## Finite Check

一句话定义：finite check 是检查输入、logits、loss、grad 和参数中是否出现 NaN 或 Inf。

排查顺序：先定位非有限值首次出现在哪个阶段，再判断是数据异常、学习率过大、fp16 溢出、mask 全 `-inf`、loss 分母为 0 还是梯度爆炸。

## Gradient Debug

一句话定义：gradient debug 是检查可训练参数是否有梯度、梯度是否有限、梯度范数是否合理，以及参数是否进入 optimizer。

常见输出：`grad is None` 参数名、grad norm、非有限 grad、全 0 grad、optimizer 参数覆盖率和断图位置。

## Forward Hook Debug

一句话定义：forward hook debug 是临时注册 hook 捕获模块输入或输出，用于定位中间激活 shape、dtype、NaN / Inf 或异常尺度。

注意事项：保存激活时应 `detach()`，用完调用 `remove()`，避免长期缓存大 tensor 导致显存增长。

## Anomaly Detection

一句话定义：anomaly detection 是 PyTorch autograd 的调试模式，用于帮助定位 backward 异常或 in-place 破坏反传的位置。

边界：它会显著降低训练速度，只适合小范围 debug，不适合长期训练。

## CUDA Synchronize Timing

一句话定义：CUDA synchronize timing 是在计时前后调用同步，避免 CUDA 异步执行导致 Python 侧只测到 kernel launch 时间。

用途：分段测量 DataLoader、CPU-GPU copy、forward、backward、optimizer step 和通信耗时。

## torch.profiler

一句话定义：`torch.profiler` 是 PyTorch 用于收集 CPU / CUDA 算子耗时、调用次数、shape 和内存信息的 profiling 工具。

使用边界：profiler 有额外开销，通常只在少量 step 上开启；生产环境需要结合 trace、step schedule 和可观测性系统使用。

## DataLoader Bottleneck

一句话定义：DataLoader bottleneck 是数据读取、解码、预处理、collate 或 CPU-GPU 拷贝速度低于训练消耗速度，导致 GPU 等数据。

排查入口：单独计时 DataLoader 迭代、检查 `num_workers`、`pin_memory`、远程 IO、在线 tokenization 和 batch 组装逻辑。

## Debug / Profiling Audit

一句话定义：debug / profiling audit 是系统检查正确性和性能瓶颈的训练排查流程。

常见输出：tensor metadata、shift loss 对齐、valid label 数、finite report、hook 激活 shape、grad norm、optimizer 参数覆盖、step timing、CUDA memory trend、profiler top ops 和分布式 rank 日志。

## Transformer Components

一句话定义：Transformer components 是构成 decoder-only LLM 主干的基础模块，包括 token embedding、位置编码或 RoPE、causal self-attention、MLP / FFN、normalization、residual、final norm 和 LM head。

核心数据流：`input_ids [B,T] -> hidden [B,T,d] -> blocks -> logits [B,T,V]`。

面试重点：要能逐步说清每个模块的输入输出 shape、为什么需要 causal mask、attention 和 MLP 的分工、Pre-Norm 的训练稳定性，以及 loss 为什么要右移。

## Token Embedding

一句话定义：token embedding 是把离散 token id 查表映射为连续 hidden vector 的模块。

核心 shape：embedding table 是 `[V,d]`，`input_ids [B,T]` 查表后得到 `[B,T,d]`。

常见误区：token id 只是索引，不应直接当连续数值特征输入线性层。

## LM Head Weight Tying

一句话定义：LM head weight tying 是让输出词表投影权重和输入 embedding 权重共享同一份参数。

工程含义：`lm_head.weight = embed_tokens.weight` 后，两者应指向同一块参数存储。

优点与边界：它能减少参数量并让输入输出 token 表示空间更一致，但要注意 vocab resize、特殊 token 扩展和 checkpoint 加载时的权重共享关系。

## Causal Mask

一句话定义：causal mask 是自回归语言模型中阻止当前位置看到未来 token 的 attention mask。

核心形状：普通训练常见 mask 是 `[1,1,T,T]`，允许 `s <= t`，禁止 `s > t`。

KV Cache 边界：decode 阶段可能出现 `query_len=1`、`key_len>1`，mask 需要考虑历史 cache 长度，否则当前 token 可能看不到完整历史。

## Scaled Dot-Product Attention

一句话定义：scaled dot-product attention 用 `QK^T / sqrt(D_h)` 得到 score，mask 后 softmax，再乘以 `V` 得到上下文表示。

核心 shape：`Q/K/V` 常见为 `[B,H,T,D_h]`，score 是 `[B,H,T_q,T_k]`，输出回到 `[B,H,T_q,D_h]`。

面试重点：缩放是为了稳定 score 方差，mask 必须在 softmax 前进入 score。

## Multi-Head Self-Attention

一句话定义：multi-head self-attention 把 hidden state 投影成多个 head 的 Q/K/V，让不同 head 在不同表示子空间中做信息路由。

实现关键：`hidden_size` 必须能被 `num_heads` 整除；`transpose` 后合并多头前常需要 `contiguous()` 或安全使用 `reshape()`。

常见输出：输入 `[B,T,d]`，输出仍是 `[B,T,d]`，便于和 residual branch 相加。

## KV Cache

一句话定义：KV Cache 是自回归推理中缓存历史 key/value，避免每个 decode step 重复计算历史 token 的机制。

核心 shape：单层常见 cache 是 `k/v [B,H,T_cache,D_h]`。

教学边界：最小实现只需演示 prefill 生成 cache、decode 拼接新 K/V、query 看见完整历史；生产系统还要处理分页、释放、调度、prefix cache 和不同请求长度。

## Transformer Component Audit

一句话定义：Transformer component audit 是系统验证手写 Transformer 组件 shape、mask、loss shift、RoPE、权重绑定和 KV Cache 语义是否正确的教学调试流程。

常见输出：RMSNorm 输出 shape、causal mask 行方向、attention 行和、未来权重是否为 0、MHA / MLP / block 输出 shape、RoPE 是否保持范数、logits shape、LM loss、权重绑定和 cache 长度增长。

面试价值：它把“我能手写 Transformer”落到可执行检查项，避免只背公式但代码里 mask 方向、head reshape 或 shift loss 出错。

## PyTorch Engineering Interview Readiness

一句话定义：PyTorch engineering interview readiness 是把 PyTorch 机制、shape / 公式、常见坑和 debug 路径组织成可口述、可追问、可验证的面试准备状态。

核心覆盖：tensor metadata、autograd、Module 注册、DataLoader、training loop、AMP / 显存、DDP、debug / profiling 和 Transformer component。

自评门禁：回答不能只背 API 名称，应至少覆盖机制、关键 shape 或公式、失败场景、排查顺序和最小 demo 证据。

常见误区：把“我用过 PyTorch”误当成“我能解释 PyTorch 工程问题”。工程面试更看重能否定位 loss 不下降、NaN、OOM、shape mismatch、device mismatch、DDP 卡住和 mask 语义错误。

## 本章小结

本章覆盖深度学习组件中最常被面试追问的基础模块。

核心结论如下：

1. Transformer 的基础计算由线性层、激活函数、归一化、残差连接和优化器共同支撑。
2. MLP/FFN 不是简单附属模块，它承担逐 token 非线性变换和大量参数容量。
3. GeLU、SiLU、SwiGLU 的区别体现了从普通激活到门控 FFN 的演进。
4. LayerNorm 和 RMSNorm 比 BatchNorm 更适合自回归语言模型。
5. 残差连接和 Pre-LN 是深层 Transformer 稳定训练的重要基础。
6. AdamW、warmup、cosine decay、weight decay 和 gradient clipping 共同决定大模型训练稳定性。
7. 手写 Transformer 组件时，shape、mask、loss shift、RoPE 和 KV Cache 都要能被最小 demo 验证。
8. PyTorch 工程面试要把机制、公式、代码 demo 和排查路径连起来。
9. 面试中不能只背定义，要能说明每个组件解决什么问题、为什么适合大模型、有什么工程代价。
