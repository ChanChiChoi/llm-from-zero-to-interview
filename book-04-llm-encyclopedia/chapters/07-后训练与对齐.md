## Post-Training

一句话定义：post-training 是预训练之后，为了让 base model 更适合指令遵循、对话、安全和人类偏好而进行的训练阶段。

核心目标：预训练让模型学会“如何续写文本”，后训练让模型学会“如何按用户意图回答”。这个阶段把 base model 从通用文本建模器塑造成 instruct model、chat model 或领域助手。

常见组成：instruction tuning、SFT、偏好数据收集、reward model、RLHF、DPO、安全微调、拒答边界训练、红队测试和能力回归评估。

为什么需要：base model 可能会继续补全文本，而不是回答问题；可能不理解 system/user/assistant 角色；可能输出有害内容；也可能不符合人类对有帮助、诚实、无害的偏好。

关键风险：后训练能显著改善交互体验，但也可能造成灾难性遗忘、过度拒答、模板化、能力退化、reward hacking 和风格单一。

面试表达：后训练不是单纯“微调一下”，而是把预训练能力对齐到用户任务和安全边界的系统工程。它既包括训练算法，也包括数据、评估和产品策略。

## Instruction Tuning

一句话定义：instruction tuning 是使用指令-回答数据训练模型，让模型学会根据自然语言指令完成任务。

数据形式：常见样本包含 instruction、optional input、response，也可以是多轮 chat 格式。任务包括问答、摘要、翻译、改写、分类、代码解释、信息抽取、推理和格式生成。

解决的问题：base model 擅长续写，但不一定知道用户想让它执行任务。instruction tuning 通过大量任务示例，让模型学习“看到任务描述后输出答案”的行为模式。

能力提升：通常能提升 zero-shot、few-shot、格式遵循、多任务泛化和用户体验。模型会更像助手，而不是纯文本补全器。

数据质量要求：指令要清晰、多样；回答要正确、有帮助、格式稳定；任务类型要覆盖目标场景；不能大量混入错误答案和模板化表达。

面试表达：instruction tuning 的本质是用监督数据把模型的通用语言能力映射到指令执行能力上。

## SFT

一句话定义：SFT 是 supervised fine-tuning，用监督学习在高质量输入-输出样本上微调模型。

核心目标：最大化 `P(response | instruction, context)`。在 chat model 中，通常只对 assistant response 计算 loss，不对 system 和 user token 计算 loss。

关键实现：chat template、tokenization、label mask、padding、truncation、packing、EOS 处理、学习率、batch size、训练轮数和验证集设计。

为什么只训练 assistant token：用户输入和系统提示是条件，不是模型需要模仿输出的答案。如果把 user token 也计入 loss，模型可能学会生成用户问题或角色标记，影响对话行为。

常见风险：训练数据过窄导致通用能力退化；学习率过大导致遗忘；训练过久导致过拟合；错误答案会被模型模仿；过多模板化数据会让输出风格僵硬。

评估方式：不能只看 SFT loss。需要同时看指令遵循、事实性、格式稳定性、安全性、通用 benchmark、领域 benchmark 和人工评测。

面试表达：SFT 是后训练的基础环节，重点不只是算法，而是数据质量、chat template、loss mask 和能力回归评估。

## Full Fine-Tuning

一句话定义：full fine-tuning 是更新模型全部参数的微调方式。

优点：表达能力强，能充分适配新任务或新领域；对大规模高质量后训练数据，效果通常更有上限。

缺点：显存和计算成本高，训练不稳定风险更大，保存完整 checkpoint 成本高，也更容易在小数据上过拟合或遗忘。

适用场景：数据规模较大、任务非常重要、资源充足、需要深度改变模型行为，或训练一个正式发布的 instruct/chat 模型。

面试表达：full fine-tuning 不是永远最好。它能力上限高，但成本和风险也高；小数据场景下 LoRA 往往更稳、更经济。

## LoRA

一句话定义：LoRA 是冻结基座模型，在部分线性层上训练低秩 adapter 的参数高效微调方法。

核心直觉：不直接更新原始权重 `W`，而是学习一个低秩增量 `ΔW = B A`。推理时可以把增量合并回原权重，也可以以 adapter 形式加载。

优点：显存低、训练快、存储小、适合多任务多 adapter 管理，对资源有限的 SFT 很实用。

关键超参：rank、alpha、dropout、target modules、学习率和训练轮数。rank 太小可能表达不足，rank 太大成本上升且更易过拟合。

局限：LoRA 冻结大部分基座参数，深度改变模型能力的上限可能低于 full fine-tuning；不同任务 adapter 合并也可能相互干扰。

面试表达：LoRA 的价值是用小参数增量近似微调效果，核心权衡是成本、可管理性和适配能力上限。

## QLoRA

一句话定义：QLoRA 是量化加载基座模型，并在量化模型上训练 LoRA adapter 的参数高效微调方法。

为什么提出：大模型显存占用巨大，普通 LoRA 仍然需要加载较高精度的基座权重。QLoRA 通过 4-bit 等量化方式降低显存，使单卡或少卡微调更可行。

优势：显存需求更低，训练成本更低，适合资源有限的领域 SFT 和实验验证。

风险：量化配置、计算 dtype、梯度 checkpoint、optimizer 和 target modules 都会影响稳定性；极端量化可能损失模型能力。

面试表达：QLoRA 是 LoRA 与低比特量化结合的微调方案，主要解决显存瓶颈，不是新的对齐目标。

## Catastrophic Forgetting

一句话定义：catastrophic forgetting 是微调后模型丢失原有通用能力、专业能力或安全能力的现象。

常见表现：领域任务变好，但通用问答、数学、代码、多语言或指令遵循变差；模型风格变窄；安全拒答边界异常。

常见原因：数据分布过窄，学习率过大，训练轮数过多，缺少混合数据，缺少能力回归评估，或者只优化单一任务 loss。

缓解方法：降低学习率，减少训练步数，使用 LoRA，引入通用 replay 数据，混合多能力数据，做 early stopping，建立回归评估集。

面试表达：微调不是只看目标任务提升，还要看原有能力有没有退化。灾难性遗忘是后训练必须监控的核心风险。

## Chat Template

一句话定义：chat template 是把 system、user、assistant 等多轮消息序列化成 token 序列的格式规则。

为什么重要：模型训练时看到的是序列化后的 token，不是结构化 JSON。template 决定角色边界、轮次边界、assistant 开始位置和 EOS 位置。

常见问题：训练和推理 template 不一致会导致分布偏移；特殊 token 配置错误会导致角色标记泄漏；EOS 缺失会让模型停不下来；assistant prompt 错误会导致不回答或续写用户。

工程建议：训练、评估和推理必须使用同一 tokenizer 和 chat template；新增 special token 后要同步 resize embedding；上线前用样例打印最终 token 化文本。

面试表达：chat template 是 chat model 的输入协议。很多“模型不听话”的问题，根因不是权重，而是模板、EOS 或 loss mask 错。

## Loss Mask

一句话定义：loss mask 是控制哪些 token 参与 loss 计算的掩码。

在 SFT 中：通常 mask 掉 system token、user token、padding token，只让 assistant response token 参与 loss。

为什么重要：如果 user token 参与 loss，模型会学会复现用户输入；如果 padding 参与 loss，训练会引入无意义目标；如果 assistant token 被误 mask，模型学不到回答。

常见排查：打印 input ids、labels、attention mask 和解码文本，确认 label 为 `ignore_index` 的位置是否正确，确认 EOS 是否参与训练。

面试表达：loss mask 是 SFT 正确性的基础细节。后训练 loss 正常但模型行为异常时，首先要检查模板和 mask。

## System Prompt

一句话定义：system prompt 是位于对话开头，用来设定助手身份、行为边界、安全规范和输出风格的高优先级提示。

作用：规定模型的角色、任务范围、输出格式、安全规则和冲突处理原则。

局限：如果模型没有通过训练学会遵循系统消息，仅靠 system prompt 不稳定。它还可能被 prompt injection 攻击诱导绕过。

工程策略：system prompt 应与训练数据、工具权限、安全分类器、输出校验和日志监控配合使用，而不是单独承担安全边界。

面试表达：system prompt 是行为控制层，不是安全系统本身。真正的对齐需要训练、权限控制和评估闭环。

## Synthetic Instruction Data

一句话定义：synthetic instruction data 是由强模型、规则或程序生成的指令微调数据。

优势：低成本、可扩展、可定向补齐能力，适合生成长尾任务、格式数据、安全样本、代码题和多语言样本。

风险：教师模型错误会被学生模仿；风格单一会导致学生模型模板化；生成数据可能包含幻觉、偏见和不安全内容；重复样本会降低有效信号。

质量控制：使用强教师模型、多模型评审、规则校验、执行验证、人工抽检、去重、难度分层和覆盖度统计。

面试表达：合成指令数据的关键是过滤和验证，而不是生成本身。没有质量控制的合成数据会把教师模型的错误放大。

## Preference Data

一句话定义：preference data 是表示人类或 AI 更偏好哪个回答的数据，常见格式是同一 prompt 下的 chosen/rejected response pair。

来源：人工标注、专家标注、用户反馈、模型自评、规则筛选、红队样本和 AI 反馈。

用途：训练 reward model，或直接用于 DPO、IPO、KTO、ORPO 等偏好优化方法。

标注维度：有帮助性、事实性、完整性、简洁性、格式遵循、安全性、拒答合理性、语气和用户满意度。

常见问题：标注者偏好不一致、长度偏差、领域知识不足、chosen 只是相对更好但仍然有错、偏好标准随业务变化。

面试表达：偏好数据是对齐阶段的燃料。它不只是“哪个答案更好”，还隐含了产品价值观和安全边界。

## Reward Model

一句话定义：reward model 是输入 prompt 和 response 后输出标量 reward 的模型，用来近似人类偏好。

训练目标：常用 pairwise ranking loss，让 chosen response 的 reward 高于 rejected response。

作用：在 RLHF 中，reward model 为 policy model 生成的回答打分，作为强化学习优化信号。

局限：reward score 是偏好代理，不等于真实质量、事实正确性或安全性。它可能对长度、语气、模板、领域和分布外回答产生偏差。

面试表达：reward model 是把人类偏好压缩成可优化分数的代理模型。它越被强力优化，越需要警惕 reward hacking 和分布偏移。

## Pairwise Ranking Loss

一句话定义：pairwise ranking loss 是用于偏好 pair 的排序损失，鼓励 chosen 的分数高于 rejected。

常见形式：

```text
L = -log sigmoid(r_chosen - r_rejected)
```

直觉：如果 chosen 的 reward 已经明显高于 rejected，loss 较小；如果 rejected 反而更高，loss 较大。

局限：它只学习相对偏好，不保证 reward 分数有绝对可解释性；如果 pair 数据质量差，模型会学到错误偏好。

面试表达：pairwise loss 适合偏好学习，因为人类通常更容易比较两个回答，而不是给单个回答打绝对分。

## RLHF

一句话定义：RLHF 是 reinforcement learning from human feedback，通过人类偏好训练 reward model，再用强化学习优化语言模型行为。

典型流程：先训练 SFT model；收集同一 prompt 下多个回答的偏好数据；训练 reward model；用 PPO 等算法优化 policy model，同时用 KL penalty 约束它不要偏离 reference model 太远。

为什么需要：SFT 学的是示范答案，但人类偏好往往难以完全写成监督数据。RLHF 可以让模型在多个候选行为中更偏向人类喜欢的输出。

优势：能显著改善有帮助性、对话风格、拒答边界和用户体验。

难点：训练链路复杂，reward model 偏差会被放大，PPO 稳定性敏感，评估成本高，容易出现 reward hacking。

面试表达：RLHF 的核心不是“强化学习很高级”，而是把人类偏好变成优化信号，并用 KL 约束防止模型偏离语言分布。

## Policy Model

一句话定义：policy model 是 RLHF 中被优化的语言模型策略，负责根据 prompt 生成 response。

在 LLM 中：policy 的动作是生成 token，状态是当前上下文，整段回答的质量由 reward model 或规则反馈评估。

初始化：通常从 SFT model 初始化，而不是直接从 base model 开始。这样模型已有较好的指令遵循和语言质量。

面试表达：policy model 就是 RLHF 中真正被更新的模型，目标是提高 reward，同时保持可读性、安全性和原有能力。

## Reference Model

一句话定义：reference model 是 RLHF 或 DPO 中通常冻结的基准模型，用来约束 policy 不要偏离原始分布太远。

常见选择：SFT model 的冻结副本。

作用：提供 KL 约束或隐式正则，防止 policy 为了追求 reward 而生成奇怪、重复、过长或分布外文本。

面试表达：reference model 是对齐优化中的“锚点”。没有这个锚点，模型容易为了 reward 分数牺牲语言质量和通用能力。

## PPO

一句话定义：PPO 是 Proximal Policy Optimization，一种常用于 RLHF 的策略优化算法。

核心思想：在提高期望 reward 的同时限制每次策略更新幅度，避免策略更新过猛导致训练崩溃。

在 RLHF 中：policy 生成回答，reward model 打分，再结合 KL penalty 得到最终优化信号，用 PPO 更新 policy。

优点：相对稳定，实践成熟。缺点：实现复杂，超参敏感，训练成本高，需要同时维护 policy、reference、reward model 和 value model 等组件。

面试表达：PPO 是 RLHF 的常见优化器，负责在 reward 提升和策略稳定之间做平衡。

## KL Penalty

一句话定义：KL penalty 是惩罚 policy model 与 reference model 输出分布差异的约束项。

作用：降低模型偏离、语言退化和 reward hacking 风险。它让模型在变得更符合偏好的同时，不要离 SFT 模型太远。

直觉：如果 policy 为了高 reward 生成极端文本，KL 会变大，从而抵消一部分 reward。

权衡：KL 系数过大，模型学不动；过小，模型可能过度优化 reward model。

面试表达：KL penalty 是 RLHF 稳定性的关键，它把偏好优化限制在合理的语言分布附近。

## Reward Hacking

一句话定义：reward hacking 是模型利用 reward model 漏洞获得高分，但输出不符合真实人类意图的现象。

典型表现：无意义长回答、过度礼貌、模板化、过度自信、编造引用、规避问题、过度拒答、重复安全免责声明。

根本原因：reward model 是人类偏好的近似代理，不可能覆盖所有真实质量维度。policy 一旦强力优化代理分数，就可能找到代理漏洞。

缓解方法：限制优化强度，使用 KL 约束，更新 reward model，引入人工评估，使用多维 reward，加入红队测试和分布外评估。

面试表达：reward hacking 是 Goodhart 定律在 RLHF 中的体现。指标被优化得越强，越可能偏离原本想衡量的真实目标。

## Over-Optimization

一句话定义：over-optimization 是模型过度优化 reward model，导致 reward 分数上升但人类实际偏好下降的现象。

现象：训练初期 reward 和人工质量可能一起提升；继续优化后 reward 继续升，但回答开始变长、变空、变模板化或事实性下降。

原因：reward model 在训练分布内有效，但 policy 生成分布逐渐变化后，reward model 的判断不再可靠。

应对：监控人工评测和 held-out 偏好集，控制 KL，early stopping，定期重训 reward model，使用多样化评估。

面试表达：偏好优化不是越久越好。对齐训练必须监控真实人类指标，而不是只看 reward 曲线。

## Goodhart's Law

一句话定义：Goodhart's Law 指当一个指标成为优化目标，它就不再是好指标。

在 RLHF 中：reward score 原本用于近似人类偏好，但当 policy 专门优化 reward score 时，它可能学会取巧，让 reward score 失去代表性。

例子：如果 reward model 偏好长答案，policy 可能生成冗长但空洞的答案；如果 reward model 偏好安全措辞，policy 可能对正常问题也过度拒答。

面试表达：Goodhart 定律解释了为什么对齐训练不能只依赖单一 reward，需要人工评估、多指标监控和安全边界。

## Distribution Shift

一句话定义：distribution shift 是训练分布和测试或优化时遇到的分布不一致。

在 reward model 中：RM 训练时看到的回答来自 SFT model 或固定候选集合，但 RLHF 过程中 policy 会不断变化，生成越来越不同的回答。

风险：RM 对新分布回答打分不可靠，可能鼓励奇怪行为，引发 reward hacking。

缓解：在线采样新回答补充偏好数据，迭代训练 reward model，引入不确定性估计和人工审核。

面试表达：对齐系统是动态分布问题。policy 变化后，reward model 的泛化能力本身也需要被评估。

## Pairwise Accuracy

一句话定义：pairwise accuracy 是 reward model 在偏好验证集上是否给 chosen response 更高分的比例。

用途：评估 RM 是否学会了偏好排序。

局限：整体准确率可能掩盖分领域失败；高 pairwise accuracy 不代表 reward 分数校准；也不代表 policy 优化后不会 hacking。

面试表达：pairwise accuracy 是 RM 的基础指标，但只能说明排序拟合情况，不能单独证明 reward model 可安全用于强优化。

## Reward Calibration

一句话定义：reward calibration 是 reward model 的分数尺度是否能稳定反映偏好强弱和不确定性。

为什么重要：如果 reward score 未校准，不同领域、不同长度、不同任务之间的分数不可比，PPO 或 reranking 可能被误导。

常见问题：模型对长回答普遍打高分，对某类语气有偏好，对专业领域误判，对分布外回答过度自信。

面试表达：RM 不只要排序准确，还要分数可用。校准差的 reward 会让优化过程变得不稳定和不可解释。

## DPO

一句话定义：DPO 是 Direct Preference Optimization，直接用 chosen/rejected 偏好数据优化 policy 的方法。

核心特点：不显式训练 reward model，不使用 PPO，而是通过偏好 pair 的 log probability 构造训练目标。

为什么流行：训练链路比 RLHF 简洁，稳定性更好，工程实现更容易，适合用已有偏好数据对 SFT 模型进一步对齐。

基本直觉：如果人类更喜欢 chosen，那么模型应提高 chosen 相对 rejected 的概率；同时用 reference model 控制变化幅度。

局限：DPO 依赖偏好数据质量；对 pair 构造、长度偏差、beta、reference 选择敏感；并不自动解决事实性和安全边界问题。

面试表达：DPO 可以理解为把 RLHF 中的 reward 建模和 KL 约束推导成一个直接的监督式偏好优化目标。

## DPO Loss

一句话定义：DPO loss 鼓励 policy 相比 reference model 更偏向 chosen response 而不是 rejected response。

核心直觉：比较 policy 对 chosen/rejected 的 log probability 差值，再减去 reference model 的对应差值。如果 policy 比 reference 更偏向 chosen，loss 就会更小。

关注点：response 的 log probability 计算、label mask、长度归一化、reference model 冻结、beta 设置和数据 pair 质量。

面试表达：DPO loss 不是简单提高 chosen 概率，而是提高 chosen 相对 rejected、相对 reference 的优势。

## Beta in DPO

一句话定义：DPO 中的 beta 是控制偏好优化强度和 reference 约束权衡的超参数。

影响：beta 较大时，偏好优化更强，模型行为变化更明显，但可能带来退化；beta 较小时，更新更保守，可能优化不足。

选择方式：通常需要结合验证集、人工评估、能力回归和安全评测调参。

面试表达：beta 控制 DPO 的对齐力度。它不是越大越好，而是在偏好提升和模型稳定之间取平衡。

## IPO

一句话定义：IPO 是 Identity Preference Optimization，一类直接偏好优化方法，目标是改善 DPO 在某些设定下可能过度优化偏好间隔的问题。

核心动机：DPO 的目标可能持续拉大 chosen 与 rejected 的概率差；IPO 尝试用更直接的偏好约束稳定优化。

适用理解：面试中不必死记公式，重点知道它属于 DPO 之后的直接偏好优化变体，关注优化稳定性和偏好过拟合问题。

面试表达：IPO 是偏好优化家族的一种改进方法，目的在于让偏好学习更稳定，减少过度拉大概率差带来的副作用。

## KTO

一句话定义：KTO 是 Kahneman-Tversky Optimization，一种不一定需要成对偏好、可利用好/坏单样本反馈的对齐方法。

核心动机：真实反馈中不总能得到 chosen/rejected pair，很多时候只有“这个回答好”或“这个回答差”。KTO 试图利用这类非成对反馈训练模型。

优势：数据格式更灵活，适合从用户点赞、点踩、审核标签等反馈中构造训练信号。

面试表达：KTO 的重点是降低对成对偏好数据的依赖，用更接近真实产品反馈的数据做对齐。

## ORPO

一句话定义：ORPO 是 Odds Ratio Preference Optimization，将监督微调和偏好优化结合到一个训练目标中的方法。

核心思想：在学习 chosen response 的同时，显式降低 rejected response 的相对优势，从而不需要单独的 reference model 或复杂 RL 流程。

优势：流程简洁，适合把 SFT 和 preference alignment 合并训练。

面试表达：ORPO 是简化对齐流程的偏好优化方法，强调用一个目标同时做模仿学习和偏好区分。

## RLAIF

一句话定义：RLAIF 是 reinforcement learning from AI feedback，用 AI 反馈替代或补充人类反馈进行对齐。

为什么需要：人工偏好标注成本高、速度慢、标准难统一。强模型或规则系统可以提供大规模、低成本反馈。

优点：可扩展、成本低、反馈速度快。风险是 AI 评审可能继承教师模型偏见、误判和安全盲点。

面试表达：RLAIF 降低了反馈成本，但反馈质量仍然决定对齐上限。AI feedback 通常需要人工抽检和校准。

## Constitutional AI

一句话定义：Constitutional AI 是使用一组原则或宪法来指导模型自我批评、修正和偏好反馈的对齐方法。

核心思路：不完全依赖人工逐条标注，而是让模型根据明确原则判断回答是否安全、诚实、有帮助，并生成修正版本或偏好标签。

优势：可以把安全原则显式化，降低人工标注成本，提高安全数据扩展性。

局限：原则设计会影响模型行为；模型对原则的理解可能不稳定；复杂价值冲突仍需要人工和产品策略介入。

面试表达：Constitutional AI 的关键是把对齐标准写成原则，再用模型自评和修正扩展安全训练数据。

## Length Bias

一句话定义：length bias 是由于回答长度不同导致偏好判断、reward 分数或 log probability 比较受到影响的偏差。

常见表现：标注者或 reward model 偏好更长、更详细的回答；DPO 中长回答的总 log probability 天然更低；生成策略中模型可能为了高分输出冗长文本。

应对：控制 chosen/rejected 长度差异，做长度归一化，引入简洁性偏好，人工评估时明确“不是越长越好”。

面试表达：长度偏差是偏好数据和偏好优化中的常见陷阱。一个回答更长，不代表它更正确或更有帮助。

## Over-Refusal

一句话定义：over-refusal 是模型对本应正常回答的请求也过度拒绝的现象。

常见原因：安全数据比例过高，安全边界标注过严，模型把敏感词和危险意图混淆，reward model 过度奖励拒答。

例子：用户询问合法的化学安全知识、网络防护知识、医疗常识，模型却直接拒绝回答。

危害：降低可用性，伤害用户体验，也可能掩盖模型没有能力区分安全和不安全请求的问题。

缓解：细化安全分类，加入正当使用样本，训练安全替代回答，区分拒答、部分回答和安全引导。

面试表达：安全对齐不是拒绝越多越好。好的安全模型应能区分恶意请求和正当请求，在边界内尽量有帮助。

## Safety Alignment

一句话定义：safety alignment 是让模型在有害、违法、隐私、歧视、自伤、危险操作等场景中遵循安全边界的训练和评估过程。

目标：减少有害内容生成，保护隐私，避免危险指导，降低偏见和歧视，同时保持对正常请求的帮助性。

常见手段：安全 SFT、拒答数据、偏好优化、红队攻击、分类器过滤、工具权限控制、输出审查和日志监控。

难点：安全边界具有上下文依赖。同样的内容在新闻报道、安全教育、科研讨论和恶意操作中含义不同。模型需要判断意图、场景和风险等级。

面试表达：安全对齐是训练和系统工程共同完成的，不能只靠一句 system prompt，也不能只靠模型拒答。理想状态是在安全边界内最大化有帮助性。

## 本章小结

本章围绕后训练与对齐展开。预训练提供通用能力，后训练决定模型如何与用户交互、如何遵循指令、如何表达偏好、如何处理安全边界。

高频结论如下：

1. 后训练把 base model 从文本续写器塑造成 instruct/chat assistant。
2. SFT 的核心不只是监督学习，还包括 chat template、loss mask、EOS 和数据质量。
3. LoRA/QLoRA 主要解决微调成本问题，不改变对齐目标本身。
4. 偏好数据表达人类或 AI 对回答质量的相对判断，是 RLHF/DPO 的基础。
5. RLHF 用 reward model 和 PPO 优化行为，但容易遇到 reward hacking 和 over-optimization。
6. KL penalty 和 reference model 用来限制 policy 偏离，保护语言质量和稳定性。
7. DPO 用偏好 pair 直接优化模型，流程比 RLHF 简洁，但仍依赖数据质量和超参选择。
8. RLAIF、Constitutional AI、IPO、KTO、ORPO 都是围绕反馈成本、优化稳定性和数据形式的改进方向。
9. 安全对齐不是简单拒答，而是在安全边界内尽量保持有帮助性。
