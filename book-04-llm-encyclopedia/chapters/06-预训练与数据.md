条目：Pretraining、Base Model、Corpus、Web Data、Code Data、Multilingual Data、Domain Data、Data Quality、Data Cleaning、Deduplication、PII Filtering、Toxicity Filtering、Data Mixture、Data Sampling、Curriculum Learning、Synthetic Data、Benchmark Contamination、Data Packing、Document Boundary、Sequence Length、Checkpoint、Training Token、Epoch、Scaling Law、Chinchilla Scaling、Compute-Optimal Training。

## Pretraining

一句话定义：pretraining 是在大规模通用语料上训练基础模型的阶段，目标是让模型获得通用语言、知识、代码、推理和模式识别能力。

在 GPT 类模型中：预训练通常使用 causal language modeling，也就是 next-token prediction。模型输入一段 token 序列，在 causal mask 约束下预测每个位置的下一个 token。

为什么需要：下游任务的人工标注数据有限，但互联网上存在大量自然文本、代码、论文、百科、书籍和论坛内容。预训练利用这些未标注文本构造自监督信号，是大模型通用能力的来源。

训练产物：预训练结束后通常得到 base model。它能较好地续写文本，但未必能稳定遵循用户指令，也未必具备聊天助手的安全边界。

关键决策：模型规模、tokenizer、训练 token 数、上下文长度、数据混合比例、学习率计划、batch size、优化器、并行策略、checkpoint 策略和评估集设计。

常见风险：数据质量差会引入噪声和偏见；数据重复会导致记忆和训练浪费；benchmark contamination 会让评估虚高；训练不稳定会造成 loss spike、梯度异常或 checkpoint 不可用。

面试表达：预训练是 LLM 能力底座的形成阶段，本质是用海量 token 做自监督 next-token prediction。它决定模型能力上限，后训练更多是在交互方式和偏好上塑形。

## Base Model

一句话定义：base model 是经过预训练但未经过指令微调和对齐的基础模型。

典型能力：语言续写、知识补全、代码补全、模式模仿、简单问答和上下文学习。它学到的是文本分布，不是产品级助手行为。

典型不足：不会稳定遵循自然语言指令，可能继续补全用户输入而不是回答问题；安全拒答能力弱；多轮对话格式不稳定；对系统提示和角色边界没有强约束。

为什么保留 base model：它适合继续预训练、领域适配、能力分析和作为后训练底座。很多研究会优先比较 base model 的原始能力，因为它受到指令微调数据风格影响较小。

面试表达：base model 更像“文本续写器”，instruct/chat model 更像“对话助手”。两者差别来自后训练、聊天模板、偏好对齐和安全策略。

## Corpus

一句话定义：corpus 是用于训练或评估的一组文本数据集合。

常见来源：网页、书籍、论文、百科、新闻、代码仓库、问答社区、论坛、多语言语料、专业领域文档、合成数据。

核心要求：规模大、覆盖广、质量高、重复少、污染可控、版权和隐私风险可控。语料不是越多越好，低质量 token 可能浪费计算并拉低模型能力。

训练语料和评估语料：训练语料用于更新参数，评估语料用于监控泛化表现。评估集必须和训练集隔离，否则 loss 和 benchmark 分数会被污染。

面试表达：corpus 决定模型“从什么世界学习”。数据来源、清洗规则和混合比例会直接影响模型的语言、知识、代码、数学和安全能力。

## Web Data

一句话定义：web data 是从网页抓取的大规模文本数据，是很多 LLM 预训练语料的重要组成部分。

优点：规模巨大，主题覆盖广，语言和写作风格多样，成本相对较低。

问题：网页数据噪声很高，包含广告、模板页、乱码、重复内容、SEO 垃圾、低质量自动生成文本、隐私信息、仇恨内容和版权风险。

处理方法：URL 过滤、语言识别、正文抽取、质量分类器、规则清洗、去重、黑名单、PII 过滤、毒性过滤和域名级采样控制。

面试表达：web data 提供覆盖度，但也带来噪声和风险。高质量预训练不是简单“爬得越多越好”，而是要把有效信号密度做高。

## Code Data

一句话定义：code data 是来自代码仓库、文档、issue、notebook 和编程问答的数据，用于增强模型的代码理解和生成能力。

为什么重要：代码具有严格语法、长距离依赖、符号引用和可执行语义。训练代码数据不仅提升编程能力，也常常改善结构化推理和工具调用能力。

常见来源：开源代码仓库、README、API 文档、单元测试、配置文件、脚本、notebook 和技术问答。

数据治理：需要去除大文件、二进制文件、自动生成文件、重复仓库、低质量代码、敏感密钥和许可证风险内容。

工程坑：如果训练数据中包含 API key、密码或私有路径，模型可能记忆并泄漏。代码数据清洗必须包含 secret scanning 和许可证策略。

面试表达：代码数据让模型学习形式化结构和可执行模式，但必须重视去重、secret 过滤和 license 风险。

## Multilingual Data

一句话定义：multilingual data 是包含多种自然语言的训练语料，用于提升模型的多语言理解和生成能力。

核心问题：高资源语言数据多，低资源语言数据少。如果简单按原始规模采样，模型会偏向英语、中文等高资源语言，低资源语言能力较弱。

常见策略：温度采样、语言配额、低资源语言上采样、高质量平行语料、跨语言评估和语言识别清洗。

风险：低资源语言上采样过度会导致重复和过拟合；自动翻译数据可能引入翻译腔；语言识别错误会污染语料。

面试表达：多语言能力不是自然出现的附属品，它依赖语料覆盖、采样策略、tokenizer 设计和评估集建设。

## Domain Data

一句话定义：domain data 是来自特定领域的语料，例如医疗、法律、金融、教育、科研、工业和企业内部文档。

作用：增强模型在专业术语、领域知识、文档格式和任务流程上的能力。

使用方式：可以在预训练阶段加入，也可以对 base model 做 continued pretraining，或在后训练阶段加入领域指令数据。

风险：领域数据可能质量参差不齐、知识过时、包含隐私信息或与通用能力冲突。领域继续预训练还可能导致通用能力退化。

面试表达：领域数据能补专业能力，但要控制质量、比例和评估。不能只看领域 loss 下降，还要看通用能力是否被破坏。

## Data Quality

一句话定义：data quality 是训练数据的有效信号密度、准确性、多样性、可读性、安全性和任务价值。

为什么重要：预训练计算极其昂贵，同样训练一万亿 token，高质量 token 能提供更多有效学习信号，低质量 token 可能只是在消耗 GPU 时间。

评价维度：文本是否自然、信息是否密集、是否重复、是否乱码、是否广告模板、是否包含有害内容、是否事实可靠、是否覆盖目标能力。

常见方法：规则过滤、困惑度过滤、质量分类器、启发式分数、人工抽检、去重统计、下游 ablation 和小模型预实验。

面试表达：数据质量决定单位 compute 的收益。大模型训练里经常不是缺 token，而是缺高质量、低污染、覆盖目标能力的 token。

## Data Cleaning

一句话定义：data cleaning 是去除或修复低质量、重复、有害、乱码、格式异常或不适合训练的数据的过程。

常见步骤：HTML 清理、正文抽取、编码修复、语言识别、长度过滤、乱码检测、低信息文本过滤、广告模板过滤、URL/邮箱/手机号处理、非法字符清理。

为什么必要：原始数据中大量内容不适合作为训练信号。如果直接训练，模型会学习垃圾格式、错误知识、重复模板和有害表达。

难点：过滤太松会保留噪声，过滤太严会损失多样性和长尾知识。数据清洗通常要在质量和覆盖之间平衡。

面试表达：data cleaning 不是简单删脏数据，而是提高训练信号密度，同时尽量保留语言和知识多样性。

## Deduplication

一句话定义：deduplication 是移除重复或近重复数据的过程。

为什么重要：重复数据会浪费训练计算，增加模型记忆风险，放大某些观点或模板，还可能导致评估集泄漏。

粒度：可以做文档级去重、段落级去重、句子级去重、代码文件级去重，也可以用 MinHash、SimHash、embedding 相似度等方法做近重复检测。

精确去重与近似去重：精确去重删除完全相同内容，简单但覆盖有限；近似去重能处理模板页、转载、轻微改写，但误删风险更高。

工程注意：去重通常要区分训练集内部去重和训练集对评估集去重。后者对 benchmark 可信度尤其重要。

面试表达：去重的价值不只是省 token，更是降低记忆和数据污染风险。大模型数据治理中，near-dedup 往往比 exact-dedup 更关键。

## PII Filtering

一句话定义：PII filtering 是识别并过滤个人可识别信息的过程，例如身份证号、手机号、邮箱、地址、姓名组合、账号和密钥。

为什么需要：训练数据中如果含有隐私信息，模型可能在生成时泄漏，带来安全、合规和伦理风险。

常见方法：正则规则、NER 模型、敏感字段词典、secret scanner、上下文规则和人工抽检。

挑战：PII 形式多样，规则可能漏检；过度过滤又会误删正常文本，例如技术文档中的示例邮箱或公开联系信息。

面试表达：PII filtering 是大模型数据安全的底线之一，尤其在企业内部数据、代码数据和用户日志数据中必须严格处理。

## Toxicity Filtering

一句话定义：toxicity filtering 是过滤仇恨、暴力、色情、骚扰、极端主义和其他有害内容的过程。

目标：降低模型学习有害表达和危险行为的概率，提高后续对齐和安全训练的效率。

复杂性：不是所有敏感内容都应删除。安全研究、新闻报道、法律文本和医学文本可能包含敏感词，但上下文是正当的。

常见策略：分级过滤、领域白名单、上下文分类器、人工审核和安全评估集。对于安全相关任务，有时需要保留经过标注和规范化的安全数据。

面试表达：毒性过滤要避免简单关键词删除。关键是区分有害生成样本和正当讨论样本，否则会损害模型对安全问题的理解。

## Data Mixture

一句话定义：data mixture 是不同来源、语言、领域和任务类型数据在训练集中的混合比例。

影响：数据混合比例决定模型能力分布。代码比例高通常提升代码能力；数学和高质量推理文本能提升推理表现；多语言比例影响跨语言能力；网页噪声比例过高会影响输出质量。

设计方式：根据目标能力设定初始比例，通过小规模训练实验、验证集 loss、下游 benchmark 和人工评估迭代调整。

常见矛盾：专业数据多会增强领域能力，但可能降低通用聊天能力；代码数据多会提升编程，但可能改变自然语言风格；低资源语言上采样能提升覆盖，但可能引入重复。

面试表达：data mixture 是训练“配方”的核心。模型最终擅长什么，很大程度由训练 token 的来源和比例决定。

## Data Sampling

一句话定义：data sampling 是从不同数据源中抽取训练样本的策略。

常见方式：按原始数据量采样、按固定配额采样、温度采样、动态采样、按质量分数加权采样。

温度采样直觉：降低高资源数据的优势，提高低资源或高价值数据被采到的概率。温度越高，分布通常越平滑。

为什么重要：即使总语料固定，采样策略不同，模型看到的数据分布也不同，最终能力会发生显著变化。

面试表达：数据采样决定模型实际训练分布。讨论数据配方时，不能只说有哪些数据，还要说每类数据以什么概率进入训练。

## Curriculum Learning

一句话定义：curriculum learning 是按一定顺序组织训练数据，让模型从较容易或更基础的数据逐步过渡到更复杂数据的训练思想。

在 LLM 中的形式：先短序列后长序列，先高质量通用数据后专业数据，先自然语言后代码或数学，或在训练后期提高高质量数据比例。

潜在好处：提升训练稳定性，让模型先学基础语言模式，再学习复杂结构；也可能提高后期能力收敛效率。

局限：LLM 规模下 curriculum 的收益并不总是稳定，错误的数据顺序可能造成分布偏移或过拟合。实际需要实验验证。

面试表达：curriculum learning 的核心是训练顺序设计，但在大模型中它是经验性工程策略，不是一定优于随机混合。

## Synthetic Data

一句话定义：synthetic data 是由模型、程序、规则或模拟器生成的数据。

用途：补充稀缺任务、生成指令数据、构造数学题和代码题、扩充多语言数据、生成安全拒答样本、构造工具调用轨迹。

优点：成本低、可扩展、可定向补能力，适合覆盖人工数据不足的长尾场景。

风险：教师模型错误会被学生模型模仿；数据风格可能单一；过多合成数据可能降低自然性；如果缺少过滤，会放大幻觉和偏见。

质量控制：使用强模型生成、多模型交叉验证、规则校验、执行器验证、人工抽检、去重、难度分层和拒绝低质量样本。

面试表达：合成数据的关键不是“自动生成”，而是“自动生成后如何筛选和验证”。没有质量控制的合成数据很容易污染模型。

## Benchmark Contamination

一句话定义：benchmark contamination 是评估集内容泄漏进训练数据，导致评估结果虚高的问题。

典型形式：训练语料中包含 benchmark 原题、答案、解析、翻译版本、近重复版本，或代码 benchmark 的测试用例和标准解。

危害：模型看似能力强，实际可能只是记住了题目或答案，导致能力判断、模型选型和论文结论失真。

检测方法：训练集和评估集做 n-gram 匹配、MinHash 近重复检测、embedding 相似度检索、时间切分评估、私有评估集和动态评测。

应对方式：训练前过滤已知 benchmark，报告数据 cutoff，使用新题和私有测试集，做 contamination analysis。

面试表达：benchmark contamination 是大模型评估可信度的核心风险。越热门的公开 benchmark，越需要怀疑数据泄漏。

## Data Packing

一句话定义：data packing 是把多个较短文本样本拼接成固定长度训练序列的方法，用来减少 padding、提高 token 利用率。

为什么需要：预训练和微调常使用固定 sequence length。如果很多样本很短，直接 padding 会浪费大量计算。packing 可以把多个样本装进同一个序列。

常见做法：把多个文档或样本按 token 拼接，中间加入 EOS 或特殊分隔符，直到接近最大长度。

注意事项：不同文档之间是否允许 attention 互相可见，要看训练目标。预训练中跨文档拼接通常用 EOS 隔开；SFT 中 packing 还必须正确处理 loss mask，避免 user token 或 padding token 参与 loss。

工程坑：如果没有正确插入 EOS，模型可能学习错误的跨文档衔接；如果 label mask 错，模型可能在不该训练的位置计算 loss；如果边界处理错，可能导致对话模板污染。

面试表达：packing 是提高训练效率的关键工程技巧，但必须处理好 EOS、document boundary、attention mask 和 loss mask。

## Document Boundary

一句话定义：document boundary 是不同文档或样本之间的边界信息。

为什么重要：语言模型训练时经常把多个文档拼接成连续 token。如果不告诉模型文档结束，模型可能把两个无关文档当成自然连续文本。

常见处理：在文档之间加入 EOS token、特殊分隔符或重置位置。现代预训练中最常见的是用 EOS 表示文档结束。

影响：边界处理会影响模型对段落结束、回答终止、多文档上下文和生成停止的学习。

面试表达：document boundary 是预训练数据格式的一部分。它看似细节，但会影响模型是否学会正确结束文本和区分独立样本。

## Sequence Length

一句话定义：sequence length 是训练时单个样本包含的最大 token 数，也常被称为训练上下文长度。

影响：更长序列能让模型学习长距离依赖、长文档结构和长上下文任务，但 attention 成本通常随长度平方增长，训练代价更高。

常见策略：先用较短 sequence length 训练大部分 token，再用更长 sequence length 做长上下文继续训练；或者在数据中混合不同长度样本。

误区：把推理配置里的 max context 改大，不代表模型真的具备长上下文能力。长上下文能力需要位置编码、训练长度分布和评估共同支持。

面试表达：sequence length 是能力和成本之间的权衡。长上下文不是只改配置，而是训练数据、位置编码和系统成本共同决定。

## Checkpoint

一句话定义：checkpoint 是训练过程中保存的模型和训练状态快照。

内容：model weights、optimizer state、learning rate scheduler state、RNG state、training step、数据加载状态、混合精度 scaler、模型 config 和 tokenizer 信息。

作用：故障恢复、阶段评估、模型选择、回滚异常训练、继续训练、发布中间模型。

保存策略：按固定 step 保存、按验证指标保存、保留最近若干个 checkpoint、保留关键里程碑 checkpoint。大模型 checkpoint 体积极大，存储和传输本身也是工程问题。

工程坑：只保存模型权重不能完全恢复训练；如果 optimizer state 或数据迭代状态丢失，继续训练的曲线可能发生变化。tokenizer/config 与 checkpoint 不匹配也会导致加载错误。

面试表达：checkpoint 不是简单的权重文件，而是可复现训练状态的快照。大规模训练必须设计可靠的 checkpoint 和恢复机制。

## Training Token

一句话定义：training token 是训练过程中被模型看到并用于学习的 token 数量。

为什么重要：LLM 能力不仅取决于参数量，也取决于训练 token 数。参数很多但 token 不足，会出现 undertraining；参数较小但 token 充分，可能在固定 compute 下更优。

统计注意：要区分原始字符数、文档数、tokenizer 后 token 数、参与 loss 的 token 数和被 mask 掉的 token 数。不同 tokenizer 下 token 数不可直接等价。

面试表达：训练 token 数是 scaling law 的核心变量之一。讨论模型训练规模时，只说参数量是不够的，还要说训练了多少 token。

## Epoch

一句话定义：epoch 表示训练过程完整遍历一遍训练数据。

在大模型中的特殊性：LLM 预训练数据巨大，常常不按传统小数据集那样训练很多 epoch，而是更关注总训练 token 数和数据采样分布。

风险：对小规模或重复数据训练多个 epoch 会增加记忆和过拟合风险，尤其在领域继续预训练和 SFT 中更明显。

面试表达：在 LLM 预训练里，epoch 概念不如 training token 和 sampling distribution 重要；在微调里，epoch 数过多则容易过拟合和遗忘。

## Scaling Law

一句话定义：scaling law 是描述模型性能随参数量、数据量和计算量扩展而变化的经验规律。

常见变量：模型参数量 `N`、训练 token 数 `D`、计算量 `C`、验证 loss 或 perplexity。

核心用途：在正式训练大模型前，用小规模实验外推大规模模型表现，帮助决定模型规模、训练 token 数和算力预算。

典型结论：在一定范围内，增加参数、增加数据和增加计算通常能平滑降低语言建模 loss。但收益呈递减趋势，并且不同能力不一定都随 loss 同步提升。

局限：scaling law 多数基于 loss，而用户关心的指令遵循、安全、推理、工具调用和多模态能力不一定能完全由 loss 解释。

面试表达：scaling law 是大模型资源规划工具，不是绝对定律。它能指导 compute 分配，但不能替代真实下游评估。

## Chinchilla Scaling

一句话定义：Chinchilla scaling 强调在固定计算预算下，应平衡模型参数量和训练 token 数，而不是一味增大参数。

核心启发：很多早期大模型参数很大，但训练 token 不足，属于 undertrained。给较小模型训练更多高质量 token，可能在同等 compute 下获得更低 loss。

直观理解：如果模型太大但数据太少，参数没有被充分训练；如果数据很多但模型太小，容量不足。compute-optimal training 要在容量和数据之间找平衡。

影响：后来很多模型更重视 token 数、数据质量和多 epoch/多 token 训练，而不是只追求参数规模。

注意：Chinchilla 结论依赖实验设定、数据分布和目标指标。实际模型还要考虑推理成本、部署需求、能力目标和数据可得性。

面试表达：Chinchilla 的面试重点不是背比例，而是说明“固定 compute 下，大模型训练要同时扩参数和扩数据，很多模型不是参数不够，而是训练 token 不够”。

## Compute-Optimal Training

一句话定义：compute-optimal training 是在固定计算预算下选择最优模型规模、训练 token 数和训练配方的策略。

核心问题：预算应该花在更大模型上，还是更多 token 上？应该训练一个更强但推理贵的模型，还是训练一个较小但 token 更充分、部署更便宜的模型？

考虑因素：训练 compute、推理 compute、数据质量、目标任务、上下文长度、部署延迟、显存限制、服务成本和后训练需求。

与 Chinchilla 的关系：Chinchilla 给出语言建模 loss 视角下的 compute-optimal 经验启发；真实工程中的 compute-optimal 还要加入推理成本和产品目标。

例子：如果目标是高并发线上服务，较小但训练充分的模型可能比更大模型更合适；如果目标是前沿能力探索，则可能更愿意训练更大模型并接受高成本。

面试表达：compute-optimal training 不是单纯追求最大参数量，而是在训练成本、数据规模、模型容量和推理成本之间做全局最优。

## 本章小结

本章围绕预训练和数据治理展开。预训练决定 LLM 的能力底座，而数据决定模型从什么分布中学习、学习到哪些能力、带入哪些风险。

高频结论如下：

1. 预训练通常用 next-token prediction 在海量 token 上训练 base model。
2. 数据质量、去重和污染控制决定单位 compute 的训练收益。
3. data mixture 和 sampling strategy 会直接影响模型能力分布。
4. 合成数据有用，但必须经过筛选、验证和去重。
5. benchmark contamination 会让评估结果虚高，是大模型评估中的核心风险。
6. data packing 能提高训练效率，但必须处理好 EOS、边界和 loss mask。
7. checkpoint 应保存完整训练状态，而不只是模型权重。
8. scaling law 和 Chinchilla scaling 用来指导参数量、token 数和 compute 的平衡。
9. compute-optimal training 要同时考虑训练成本、推理成本、数据质量和目标能力。
