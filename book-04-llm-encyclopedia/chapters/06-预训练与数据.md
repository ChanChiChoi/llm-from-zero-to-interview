## Pretraining

一句话定义：pretraining 是在大规模通用语料上训练基础模型的阶段，目标是让模型获得通用语言、知识、代码、推理和模式识别能力。

在 GPT 类模型中：预训练通常使用 causal language modeling，也就是 next-token prediction。模型输入一段 token 序列，在 causal mask 约束下预测每个位置的下一个 token。

为什么需要：下游任务的人工标注数据有限，但互联网上存在大量自然文本、代码、论文、百科、书籍和论坛内容。预训练利用这些未标注文本构造自监督信号，是大模型通用能力的来源。

训练产物：预训练结束后通常得到 base model。它能较好地续写文本，但未必能稳定遵循用户指令，也未必具备聊天助手的安全边界。

关键决策：模型规模、tokenizer、训练 token 数、上下文长度、数据混合比例、学习率计划、batch size、优化器、并行策略、checkpoint 策略和评估集设计。

常见风险：数据质量差会引入噪声和偏见；数据重复会导致记忆和训练浪费；benchmark contamination 会让评估虚高；训练不稳定会造成 loss spike、梯度异常或 checkpoint 不可用。

面试表达：预训练是 LLM 能力底座的形成阶段，本质是用海量 token 做自监督 next-token prediction。它决定模型能力上限，后训练更多是在交互方式和偏好上塑形。

## Base Model

一句话定义：base model 是经过预训练但未经过指令微调和对齐的基础模型。

典型能力：语言续写、知识补全、代码补全、模式模仿、简单问答和上下文学习。它学到的是文本分布，不是产品级助手行为。

典型不足：不会稳定遵循自然语言指令，可能继续补全用户输入而不是回答问题；安全拒答能力弱；多轮对话格式不稳定；对系统提示和角色边界没有强约束。

为什么保留 base model：它适合继续预训练、领域适配、能力分析和作为后训练底座。很多研究会优先比较 base model 的原始能力，因为它受到指令微调数据风格影响较小。

面试表达：base model 更像“文本续写器”，instruct/chat model 更像“对话助手”。两者差别来自后训练、聊天模板、偏好对齐和安全策略。

## Corpus

一句话定义：corpus 是用于训练或评估的一组文本数据集合。

常见来源：网页、书籍、论文、百科、新闻、代码仓库、问答社区、论坛、多语言语料、专业领域文档、合成数据。

核心要求：规模大、覆盖广、质量高、重复少、污染可控、版权和隐私风险可控。语料不是越多越好，低质量 token 可能浪费计算并拉低模型能力。

训练语料和评估语料：训练语料用于更新参数，评估语料用于监控泛化表现。评估集必须和训练集隔离，否则 loss 和 benchmark 分数会被污染。

面试表达：corpus 决定模型“从什么世界学习”。数据来源、清洗规则和混合比例会直接影响模型的语言、知识、代码、数学和安全能力。

## Web Data

一句话定义：web data 是从网页抓取的大规模文本数据，是很多 LLM 预训练语料的重要组成部分。

优点：规模巨大，主题覆盖广，语言和写作风格多样，成本相对较低。

问题：网页数据噪声很高，包含营销模板、模板页、乱码、重复内容、SEO 垃圾、低质量自动生成文本、隐私信息、仇恨内容和版权风险。

处理方法：URL 过滤、语言识别、正文抽取、质量分类器、规则清洗、去重、黑名单、PII 过滤、毒性过滤和域名级采样控制。

面试表达：web data 提供覆盖度，但也带来噪声和风险。高质量预训练不是简单“爬得越多越好”，而是要把有效信号密度做高。

## Web-Scale Data Collection

一句话定义：web-scale data collection 是围绕大规模网页、代码、论文、论坛和多语言来源建立合法、可追溯、可治理训练语料的采集工程。

核心流程：source planning、license / ToS / robots 审查、公开或授权采集、raw storage、HTML/PDF/代码解析、正文抽取、语言和领域识别、质量过滤、PII / secret 扫描、benchmark contamination 检测、去重、数据配比、版本管理和审计。

关键元数据：source id、source type、license、URL hash、content hash、crawl time、language、domain、quality score、risk flags、dedup cluster id、dataset version 和 delete policy。

常见误区：技术上能访问不代表能用于训练；web-scale 不是越多越好；没有 provenance 的数据很难处理删除请求、污染排查和模型版本追责。

面试表达：我会把 web-scale 数据采集讲成数据治理系统，而不是爬虫脚本。合格 pipeline 要能证明数据从哪里来、为什么允许用、经过哪些过滤、进入哪个版本，以及未来如何审计和删除。

## Specialized Data

一句话定义：specialized data 是为了提升特定能力而单独建设、审计和采样的数据池，例如代码、数学、医学、法律、金融、科研和企业知识数据。

为什么要单独处理：专项数据的质量标准不同于普通网页。代码要看许可证、密钥、语法、测试和功能正确性；数学要看答案、过程、难度和题库污染；领域数据要看权威性、时效性、引用、隐私和专家审计。

工程做法：先给每类数据建立独立元数据和风险标记，再使用类型专属质量分、污染检测、去重、抽样审计和 data mixture 配比。不能把专项数据简单混入通用语料。

面试表达：专项数据的核心不是“多加一点专业文本”，而是用类型专属门禁提高目标能力，同时避免 license、secret、PII、benchmark contamination 和高风险错误建议进入训练。

## Code Data

一句话定义：code data 是来自代码仓库、文档、issue、notebook 和编程问答的数据，用于增强模型的代码理解和生成能力。

为什么重要：代码具有严格语法、长距离依赖、符号引用和可执行语义。训练代码数据不仅提升编程能力，也常常改善结构化推理和工具调用能力。

常见来源：开源代码仓库、README、API 文档、单元测试、配置文件、脚本、notebook 和技术问答。

数据治理：需要去除大文件、二进制文件、自动生成文件、重复仓库、低质量代码、敏感密钥和许可证风险内容。

工程坑：如果训练数据中包含 API key、密码或私有路径，模型可能记忆并泄漏。代码数据清洗必须包含 secret scanning 和许可证策略。

面试表达：代码数据让模型学习形式化结构和可执行模式，但必须重视去重、secret 过滤和 license 风险。

## Unit Test Data

一句话定义：unit test data 是和代码任务配套的测试用例、断言、输入输出样例或测试函数，用来约束代码生成的功能正确性。

为什么重要：代码评估不能只看文本相似度。单元测试能把自然语言需求转成可执行约束，帮助判断生成代码是否真的满足接口、边界条件和异常场景。

数据形态：常见组合包括题目描述、函数签名、参考实现、公开测试、隐藏测试、失败用例、文档字符串和单测文件。训练时要避免把评测集隐藏测试泄漏进训练数据。

面试表达：高质量代码数据应该尽量保留“需求-实现-测试-文档”的对齐关系；测试越贴近真实接口和边界条件，越能支撑 functional correctness。

## Functional Correctness

一句话定义：functional correctness 是代码是否按照题目或接口规范完成预期功能，而不是生成文本是否像参考答案。

核心判断：最常见方法是执行测试用例，看输出、异常、状态变化和副作用是否符合预期。通过已有测试并不等于完全正确，因为测试覆盖可能不足。

工程风险：如果训练数据包含 benchmark 题目、参考答案或隐藏测试，功能正确性评估会被污染。代码数据治理必须同时做测试质量审计和 train-eval overlap 检测。

面试表达：代码模型的关键指标不是“答案长得像不像”，而是能否在约束下运行正确。单测、静态分析、执行沙箱和污染隔离共同构成功能正确性的评估闭环。

## Math Data

一句话定义：math data 是包含数学题目、证明、推导、答案、解析和验证信号的数据，用于提升模型的计算、符号推理和多步推理能力。

为什么重要：数学数据能训练模型分解问题、保持变量一致性、执行中间计算和检查结论。它对推理能力有帮助，但也容易受到错误解析和题库污染影响。

治理重点：需要同时验证最终答案和推理过程，标记题目难度、知识点、来源、重复题、近重复解析和 benchmark contamination。只保留“看起来像推理”的长答案并不可靠。

面试表达：数学数据质量取决于可验证性。高质量数学样本要能说明题目、步骤、答案和难度为什么可信，并明确隔离评测集原题、答案和解析。

## Verifier Data

一句话定义：verifier data 是用于训练或校准验证器的数据，通常包含问题、候选解、正确性标签、评分或偏好比较。

作用：在数学、代码和复杂推理任务中，模型可以生成多个候选答案，再由 verifier 评分或重排。这样能把“生成”与“判断正确性”部分解耦。

数据要求：候选解要覆盖正确、部分正确、错误步骤、错误答案和边界情况；标签要可靠，最好经过程序验证、人工复核或多来源一致性检查。

面试表达：verifier data 的价值在于让系统学会识别可靠解法，而不是只模仿单一路径。它常和多候选采样、reranking、过程监督和回归测试一起使用。

## Multilingual Data

一句话定义：multilingual data 是包含多种自然语言的训练语料，用于提升模型的多语言理解和生成能力。

核心问题：高资源语言数据多，低资源语言数据少。如果简单按原始规模采样，模型会偏向英语、中文等高资源语言，低资源语言能力较弱。

常见策略：温度采样、语言配额、低资源语言上采样、高质量平行语料、跨语言评估和语言识别清洗。

风险：低资源语言上采样过度会导致重复和过拟合；自动翻译数据可能引入翻译腔；语言识别错误会污染语料。

面试表达：多语言能力不是自然出现的附属品，它依赖语料覆盖、采样策略、tokenizer 设计和评估集建设。

## Domain Data

一句话定义：domain data 是来自特定领域的语料，例如医疗、法律、金融、教育、科研、工业和企业内部文档。

作用：增强模型在专业术语、领域知识、文档格式和任务流程上的能力。

使用方式：可以在预训练阶段加入，也可以对 base model 做 continued pretraining，或在后训练阶段加入领域指令数据。

风险：领域数据可能质量参差不齐、知识过时、包含隐私信息或与通用能力冲突。领域继续预训练还可能导致通用能力退化。

面试表达：领域数据能补专业能力，但要控制质量、比例和评估。不能只看领域 loss 下降，还要看通用能力是否被破坏。

## Domain Authority

一句话定义：domain authority 是专业领域数据来源的可信度和适用性，例如官方指南、标准文档、同行评审论文、监管文件、教材或经专家维护的知识库。

为什么重要：医疗、法律、金融和工程安全等领域中，低权威来源可能包含过时、片面或错误建议。模型训练如果放大这些内容，会增加高风险输出概率。

审计方法：记录来源机构、作者、发布日期、更新日期、引用、适用地区、版本和撤回状态；不同领域还需要专家抽检和权限控制。

面试表达：领域数据不是按文本流畅度筛选，而是按来源权威性、时效性、引用完整性和风险边界筛选。权威性不足的数据更适合隔离、降权或只用于分析。

## Expert Audit

一句话定义：expert audit 是由具备领域资质或经验的人对专业数据、标注、模型输出或评估样本做抽检和风险复核。

适用场景：医学建议、法律条款、金融报告、工业安全、教育测评和高价值企业知识。专家审计通常用于发现普通质量分类器难以识别的事实错误、过期信息和风险表述。

工程落地：审计要有抽样策略、rubric、错误分类、严重度、复核记录和闭环修复动作。专家意见不应只停留在一次性验收，而要进入数据版本和评估报告。

面试表达：高风险领域不能只靠继续预训练或通用清洗规则。专家审计是领域数据治理的最后一道人工质量门禁，用来校准模型能学什么、不能学什么。

## Data Quality

一句话定义：data quality 是训练数据的有效信号密度、准确性、多样性、可读性、安全性和任务价值。

为什么重要：预训练计算极其昂贵，同样训练一万亿 token，高质量 token 能提供更多有效学习信号，低质量 token 可能只是在消耗 GPU 时间。

评价维度：文本是否自然、信息是否密集、是否重复、是否乱码、是否营销模板、是否包含有害内容、是否事实可靠、是否覆盖目标能力。

常见方法：规则过滤、困惑度过滤、质量分类器、启发式分数、人工抽检、去重统计、下游 ablation 和小模型预实验。

面试表达：数据质量决定单位 compute 的收益。大模型训练里经常不是缺 token，而是缺高质量、低污染、覆盖目标能力的 token。

## Quality Score

一句话定义：quality score 是把文本可读性、信息密度、重复度、风险命中、领域价值和来源可信度融合成的样本级或文档级质量分。

常见特征：长度、unique token ratio、字符比例、重复行比例、boilerplate / SEO 命中、困惑度、质量分类器分数、PII / secret / 安全风险和来源权重。

工程注意：质量分不能脱离语言、领域和来源桶。代码、数学、法律、医学和低资源语言可能天然不同于普通网页文本，应该做分桶阈值校准。

面试表达：quality score 不是“越高越好”的单一裁判，而是清洗系统里的一个排序和审计信号；最终还要叠加隐私、安全、污染、去重和覆盖率门禁。

## Quality Classifier

一句话定义：quality classifier 是用标注样本、启发式标签或高质量参考集训练出来的文本质量分类器，用来辅助判断样本是否适合进入某类训练数据池。

为什么需要：纯规则容易漏掉语义层面的低质量内容，也容易误伤长尾领域；质量分类器可以学习更复杂的文本模式，但仍需要抽样审计和阈值校准。

风险：分类器会继承标注偏好和训练集偏差，可能偏向英文、百科体或主流领域，导致论坛、口语、少数语言和专业文本被系统性低估。

面试表达：质量分类器适合作为多级过滤的一环，不适合单独决定训练集。成熟做法是按桶看分布、抽样估计误删 / 漏删，并用小模型 ablation 验证收益。

## Data Cleaning

一句话定义：data cleaning 是去除或修复低质量、重复、有害、乱码、格式异常或不适合训练的数据的过程。

常见步骤：HTML 清理、正文抽取、编码修复、语言识别、长度过滤、乱码检测、低信息文本过滤、营销模板过滤、URL/邮箱/手机号处理、非法字符清理。

为什么必要：原始数据中大量内容不适合作为训练信号。如果直接训练，模型会学习垃圾格式、错误知识、重复模板和有害表达。

难点：过滤太松会保留噪声，过滤太严会损失多样性和长尾知识。数据清洗通常要在质量和覆盖之间平衡。

面试表达：data cleaning 不是简单删脏数据，而是提高训练信号密度，同时尽量保留语言和知识多样性。

## Data Filtering Audit

一句话定义：data filtering audit 是对过滤前后数据的保留率、风险命中、质量分分布、语言 / 领域覆盖、误删和漏删进行审计。

核心指标：token 保留率、分桶保留率、重复率、PII / secret 命中率、安全风险命中率、benchmark contamination 命中率、人工审计误删率和漏删率。

为什么重要：过滤规则看起来合理，不代表对最终模型有益。过严会损失长尾知识和多样性，过松会把噪声、隐私、安全风险和评测污染带进训练。

面试表达：清洗过滤必须有审计闭环。真正可上线的数据 pipeline 要能回答“删了什么、为什么删、误删多少、漏掉多少、对各语言和领域有什么影响”。

## Data Incident

一句话定义：data incident 是因为数据来源、过滤、去重、评估污染、配比、格式、版本或治理证据缺失导致模型能力、评估可信度、上线风险或实验复现出现异常的事故。

典型表现：benchmark 分数虚高但线上退化、训练 loss 异常顺滑但泛化差、某语言或领域突然下降、合成数据风格过重、代码生成出现敏感片段、旧实验无法复现。

排查顺序：先抽样看原始样本，再查 exact / near dedup、train-eval overlap、质量过滤、风险命中、data mixture、lineage 和数据版本 diff，最后再讨论模型结构或训练超参。

面试表达：数据事故不是“某条样本脏了”，而是数据证据链断了。资深工程师要能把现象拆成可观测指标和可复现版本差异。

## Data Risk Rate

一句话定义：data risk rate 是训练数据中命中 license、secret、PII、安全、版权或评估污染等高风险标记的比例。

常见口径：可以按样本数、token 数、数据源、语言 / 领域桶分别统计。代码数据通常要单独看 license 和 secret，用户日志要单独看 PII 和权限，评估相关数据要单独看 benchmark contamination。

工程用途：风险率不是越低越好就结束，还要看漏检和误删。风险命中样本需要隔离、复核、修复规则和进入版本报告。

面试表达：data risk rate 是上线门禁，不是训练 loss 指标。只要高风险命中不可解释，就不能把这批数据直接进入正式训练。

## Data Incident Gate

一句话定义：data incident gate 是把重复、近重复、评估污染、风险命中、配比偏移、lineage 覆盖和版本可复现性合并成的数据事故门禁。

常见检查项：exact duplicate rate、near duplicate rate、benchmark contamination rate、secret / PII / license risk rate、synthetic ratio、mixture shift、lineage coverage、manifest 完整度和抽样复核结论。

为什么重要：单项指标通过不代表数据可用。例如风险率很低但 benchmark 泄漏存在，评估仍不可信；保留率正常但 code 桶全部被过滤，代码能力仍可能回退。

面试表达：数据门禁要能输出 failed gates 和 rejected reasons。好的回答不是“我们做了清洗”，而是能说清哪些门禁失败、影响什么能力、怎么回滚和复验。

## Deduplication

一句话定义：deduplication 是移除重复或近重复数据的过程。

为什么重要：重复数据会浪费训练计算，增加模型记忆风险，放大某些观点或模板，还可能导致评估集泄漏。

粒度：可以做文档级去重、段落级去重、句子级去重、代码文件级去重，也可以用 MinHash、SimHash、embedding 相似度等方法做近重复检测。

精确去重与近似去重：精确去重删除完全相同内容，简单但覆盖有限；近似去重能处理模板页、转载、轻微改写，但误删风险更高。

工程注意：去重通常要区分训练集内部去重和训练集对评估集去重。后者对 benchmark 可信度尤其重要。

面试表达：去重的价值不只是省 token，更是降低记忆和数据污染风险。大模型数据治理中，near-dedup 往往比 exact-dedup 更关键。

## Exact Deduplication

一句话定义：exact deduplication 是对规范化后的文档、段落、代码文件或样本计算 hash，删除完全相同内容的过程。

关键点：必须先做规范化，例如统一空白、大小写、Unicode、HTML 残留和模板字段；否则本质相同的文本会因为格式差异逃过去。

优点和边界：exact dedup 便宜、可解释、适合大规模第一层过滤，但不能发现轻微改写、转载加页脚、代码 fork 和语义等价题解。

面试表达：exact dedup 是必要的底层门禁，但不是大模型去重的全部。真正影响训练质量的常常是 near duplicate 和 train-eval overlap。

## Near Deduplication

一句话定义：near deduplication 是识别高度相似但不完全相同的样本，例如转载、改标题、镜像、代码 fork、模板页和轻微改写。

常见方法：n-gram Jaccard、MinHash/LSH、SimHash 汉明距离、编辑距离、embedding 相似度和人工审计。

风险：阈值过高会漏掉重复，阈值过低会误删同主题但细节不同的高价值样本。短文本、代码、数学题和多语言数据尤其需要分桶校准。

面试表达：near dedup 的核心不是“把相似内容全删掉”，而是在减少无意义重复、控制记忆风险和保留多样性之间做平衡。

## MinHash

一句话定义：MinHash 是用紧凑签名估计两个集合 Jaccard 相似度的方法，常用于文本 shingle 集合的近重复检测。

工作方式：把文档切成 n-gram shingles，生成多个 hash 下的最小值作为 signature；两个文档 signature 相同位置比例越高，Jaccard 相似度越高。

工程用途：MinHash 通常和 LSH 配合，用于 web-scale 近重复候选召回，避免对海量文档两两比较。

面试表达：MinHash 解决的是“相似度计算太贵”的问题，不是最终保留策略。最终仍要做相似度阈值、聚类、代表样本选择和抽样审计。

## SimHash

一句话定义：SimHash 是把文本特征压缩成固定长度二进制指纹，并用汉明距离判断近重复的方法。

适用场景：网页近重复、镜像页、模板页和快速候选过滤。它存储小、速度快，但特征选择和汉明阈值会影响误删 / 漏检。

和 MinHash 的区别：MinHash 更直接对应 Jaccard 集合相似度；SimHash 更像加权特征投票后的指纹压缩。两者都需要按数据类型审计阈值。

面试表达：SimHash 适合快速网页去重，但不能替代污染检测、语义相似审计或人工抽检。

## Train-Eval Overlap

一句话定义：train-eval overlap 是训练语料和评估样本之间存在原文、答案、解析、代码签名、测试样例或近重复内容。

为什么重要：训练集内部可以去重得很好，但如果包含 benchmark 题目或答案，评估仍然会虚高。

常见检测：exact match、n-gram overlap、MinHash/LSH、embedding 检索、答案泄漏检测、时间切分和人工审计。

面试表达：训练去重和评测污染检测是两件事。前者控制训练分布和记忆风险，后者保护评估可信度。

## Canary

一句话定义：canary 是受控合成探针，用于评估模型是否容易记住并复述罕见字符串或特定训练片段。

使用边界：canary 应使用非真实敏感信息，不能用真实用户隐私、真实凭据或真实业务秘密当探针。

工程用途：通过控制 canary 出现次数、上下文和数据版本，观察去重、隐私过滤和训练策略是否降低记忆风险。

面试表达：canary 的价值是做受控风险评估，而不是复原训练数据。它应和 PII / secret 过滤、去重和输出安全评估一起使用。

## PII Filtering

一句话定义：PII filtering 是识别并过滤个人可识别信息的过程，例如身份证号、手机号、邮箱、地址、姓名组合、账号和密钥。

为什么需要：训练数据中如果含有隐私信息，模型可能在生成时泄漏，带来安全、合规和伦理风险。

常见方法：正则规则、NER 模型、敏感字段词典、secret scanner、上下文规则和人工抽检。

挑战：PII 形式多样，规则可能漏检；过度过滤又会误删正常文本，例如技术文档中的示例邮箱或公开联系信息。

面试表达：PII filtering 是大模型数据安全的底线之一，尤其在企业内部数据、代码数据和用户日志数据中必须严格处理。

## Secret Scanning

一句话定义：secret scanning 是识别 API key、access token、私钥、密码、连接串和内部路径等敏感凭据的过程。

为什么需要：代码仓库、配置文件、日志和论坛样本可能意外包含真实凭据。进入训练集后，模型可能记忆或在生成中泄漏这些字符串。

常见方法：高召回正则、已知 token 前缀规则、熵检测、上下文规则、官方 secret scanner、人工抽检和命中样本隔离。

面试表达：secret scanning 要优先高召回，并把命中样本从通用训练池隔离；示例 key 和真实 key 要靠上下文、格式和来源进一步区分，不能只靠关键词。

## Toxicity Filtering

一句话定义：toxicity filtering 是过滤仇恨、暴力、色情、骚扰、极端主义和其他有害内容的过程。

目标：降低模型学习有害表达和危险行为的概率，提高后续对齐和安全训练的效率。

复杂性：不是所有敏感内容都应删除。安全研究、新闻报道、法律文本和医学文本可能包含敏感词，但上下文是正当的。

常见策略：分级过滤、领域白名单、上下文分类器、人工审核和安全评估集。对于安全相关任务，有时需要保留经过标注和规范化的安全数据。

面试表达：毒性过滤要避免简单关键词删除。关键是区分有害生成样本和正当讨论样本，否则会损害模型对安全问题的理解。

## Data Mixture

一句话定义：data mixture 是不同来源、语言、领域和任务类型数据在训练集中的混合比例。

影响：数据混合比例决定模型能力分布。代码比例高通常提升代码能力；数学和高质量推理文本能提升推理表现；多语言比例影响跨语言能力；网页噪声比例过高会影响输出质量。

设计方式：根据目标能力设定初始比例，通过小规模训练实验、验证集 loss、下游 benchmark 和人工评估迭代调整。

常见矛盾：专业数据多会增强领域能力，但可能降低通用聊天能力；代码数据多会提升编程，但可能改变自然语言风格；低资源语言上采样能提升覆盖，但可能引入重复。

工程字段：成熟数据表通常会记录 source、language、domain、quality score、risk label、dedup cluster、license、token count、sampling weight、effective epoch 和 dataset version。

面试表达：data mixture 是训练“配方”的核心。模型最终擅长什么，很大程度由训练 token 的来源和比例决定。

## Mixture Shift

一句话定义：mixture shift 是清洗、去重、过滤、采样或数据版本变更后，实际训练数据配比相对目标配比发生的偏移。

常见原因：某个数据桶被风险门禁大量过滤，近重复去重删除了热门来源，质量分类器误伤低资源语言，代码 license 审查清掉大量仓库，合成数据在过滤后占比被动升高。

工程用途：训练前要同时报告目标 mixture 和清洗后的实际 mixture。训练后如果某能力退化，优先查对应语言、领域、任务或模态的数据桶是否发生了意外偏移。

面试表达：mixture 配置不是写在 YAML 里就结束。真正影响模型的是进入 batch 的实际 token 分布，所以必须审计 mixture shift。

## Data Sampling

一句话定义：data sampling 是从不同数据源中抽取训练样本的策略。

常见方式：按原始数据量采样、按固定配额采样、温度采样、动态采样、按质量分数加权采样。

温度采样直觉：降低高资源数据的优势，提高低资源或高价值数据被采到的概率。温度越高，分布通常越平滑。

为什么重要：即使总语料固定，采样策略不同，模型看到的数据分布也不同，最终能力会发生显著变化。

面试表达：sampling strategy 是 data mixture 真正落地到训练 batch 的方式。讨论配比时要同时说明采样权重、计划 token 数、effective epoch 和风险门禁。

## Temperature Sampling

一句话定义：temperature sampling 是对原始数据池大小做平滑，让高资源数据不过度主导、低资源或高价值数据获得更多采样机会。

常见写法：可以把数据池大小 `n_k` 转成 `n_k^alpha` 后归一化，其中 `alpha=1` 接近自然分布，`alpha=0` 接近均匀分布。也有资料用 temperature 的倒数来表示同一思想。

风险：过度平滑会让小数据池被反复采样，带来过拟合、记忆和污染风险；平滑不足又会让低资源语言或高价值领域被淹没。

面试表达：temperature sampling 的目标是折中覆盖和重复 exposure，不是让所有语言或领域完全等比例。

## Effective Epoch

一句话定义：effective epoch 是某个数据池计划采样 token 数除以该池清洗后 token 数，用来衡量它被重复使用的程度。

为什么重要：上采样小而高质量的数据池可以补能力短板，但 effective epoch 过高会增加记忆、题库污染和风格同质化风险。

工程做法：对代码、数学、专业文档、合成数据和低资源语言单独监控 effective epoch，并结合去重、污染检测、canary 和小模型 ablation 调阈值。

面试表达：上采样不能只报比例，还要报 effective epoch。否则看似只是提高数学比例，实际可能是在重复训练同一批题。

## Mixture Ablation

一句话定义：mixture ablation 是固定数据版本和训练 recipe，系统改变某类数据采样比例，观察能力、风险和成本变化的实验。

常见实验：提高代码比例、提高数学比例、提高多语言比例、加入合成推理数据、降低低质量网页比例、调整安全数据比例。

评估维度：通用问答、代码、数学、多语言、安全、事实性、长上下文、记忆风险、污染风险、人工样例和验证 loss。

面试表达：data mixture 没有标准答案，必须用 ablation 找 trade-off，并在更大模型和更长训练上复验小模型趋势。

## Curriculum Learning

一句话定义：curriculum learning 是按一定顺序组织训练数据，让模型从较容易或更基础的数据逐步过渡到更复杂数据的训练思想。

在 LLM 中的形式：先短序列后长序列，先高质量通用数据后专业数据，先自然语言后代码或数学，或在训练后期提高高质量数据比例。

潜在好处：提升训练稳定性，让模型先学基础语言模式，再学习复杂结构；也可能提高后期能力收敛效率。

局限：LLM 规模下 curriculum 的收益并不总是稳定，错误的数据顺序可能造成分布偏移或过拟合。实际需要实验验证。

面试表达：curriculum learning 的核心是训练顺序设计，但在大模型中它是经验性工程策略，不是一定优于随机混合。

## Synthetic Data

一句话定义：synthetic data 是由模型、程序、规则或模拟器生成的数据。

用途：补充稀缺任务、生成指令数据、构造数学题和代码题、扩充多语言数据、生成安全拒答样本、构造工具调用轨迹、生成内部回归测试样本。

优点：成本低、可扩展、可定向补能力，适合覆盖人工数据不足的长尾场景。

风险：teacher 错误会被 student 模仿；数据风格可能单一；过多合成数据可能降低自然性；如果缺少过滤，会放大幻觉、偏见、评测污染和安全边界错误。

质量控制：使用强模型生成、多模型交叉验证、规则校验、执行器验证、人工抽检、去重、难度分层、授权审查、污染扫描和拒绝低质量样本。

面试表达：合成数据的关键不是“自动生成”，而是“自动生成后如何筛选和验证”。没有质量控制的合成数据很容易污染模型。

## Synthetic Instruction Data

一句话定义：synthetic instruction data 是由规则、模板或 LLM 生成的 instruction-input-output 样本，用于指令微调、格式跟随、工具调用和多轮对话训练。

典型流程：先准备种子任务，再生成多样指令和回答，然后过滤无效、重复、相似、污染和低质量样本，最后用小规模训练实验验证收益。

风险：如果任务模板单一，模型会学到固定话术；如果 teacher 输出未验证，student 会继承错误；如果合成题和 benchmark 太接近，评估会虚高。

面试表达：合成指令数据适合补覆盖，但必须记录 seed、teacher、prompt、sampling 参数、过滤规则、质量分和数据版本。

## Self-Instruct

一句话定义：Self-Instruct 是用模型自举生成 instruction、input 和 output，再经过过滤后用于 instruction tuning 的方法。

核心启发：它不是让模型无限生成数据，而是强调生成、过滤、去重、任务多样性和评估闭环。生成质量和过滤策略比原始数量更重要。

工程边界：Self-Instruct 风格数据适合教学和开源模型训练思路，但真实项目还要处理授权、隐私、评测污染、teacher 偏差和人工抽检。

面试表达：Self-Instruct 的价值是降低人工指令数据门槛；它的风险是错误自举和模板化，所以必须配合验证和 ablation。

## Evol-Instruct

一句话定义：Evol-Instruct 是从较简单指令出发，通过增加约束、深度、复杂输入或多步骤要求，演化出更复杂指令的合成数据方法。

解决问题：普通合成指令容易偏简单，复杂任务覆盖不足。Evol-Instruct 试图主动提高指令复杂度，训练模型处理组合约束和多步骤任务。

风险：复杂不等于高质量。约束冲突、不可回答、答案不满足指令、伪复杂和重复改写都需要过滤。

面试表达：Evol-Instruct 的关键是复杂度控制和质量验证，而不是机械堆限制条件。

## Distillation Data

一句话定义：distillation data 是 teacher model 生成或标注的数据，用来训练 student model 学习 teacher 的回答、推理、偏好、评分或格式。

常见形式：teacher 回答、分步解释、代码实现、工具调用轨迹、chosen/rejected 偏好对、评分标签、错误分析和过程监督数据。

工程门禁：必须确认 teacher 输出使用授权，记录 teacher 版本、prompt、sampling 参数、质量验证、污染扫描和安全过滤。无授权复制闭源模型能力不是合规的数据工程方案。

面试表达：蒸馏数据关注 teacher-student 能力迁移；它可以降低标注成本，但会继承 teacher 偏差、错误和风格。

## Teacher Model

一句话定义：teacher model 是用于生成答案、解释、偏好或评分的较强模型、规则系统、模拟器或专家标注系统。

作用：teacher 可以补充人工标注不足，生成高质量初稿，也可以给 student 提供评分和反馈信号。

风险：teacher 不等于真值。它可能 hallucinate、偏向某种表达、违反目标安全边界，或输出授权不允许用于训练的数据。

面试表达：teacher 的输出必须被审计，而不是直接当 gold label。真正可信的是 teacher 加验证、过滤、人工抽检和实验收益。

## Student Model

一句话定义：student model 是使用 teacher 生成或标注数据训练的目标模型，通常容量更小、成本更低或面向特定场景。

能力边界：student 学到的是训练数据中的行为模式，受模型容量、数据覆盖、优化目标和评估体系约束，不等于完整复制 teacher。

常见目标：降低推理成本、提升特定任务能力、稳定输出格式、增强工具调用或构建私有部署模型。

面试表达：student 可以在窄分布上接近甚至超过 teacher 的局部表现，但全局能力仍要靠独立评估验证。

## Model Collapse

一句话定义：model collapse 是模型生成数据被反复用于训练后，数据分布逐渐变窄、长尾信息丢失、错误累积和输出同质化的风险。

形成原因：合成数据如果逐代替代自然数据，生成器偏差会被不断放大，真实世界的复杂性和少数模式会被压缩。

防范：保留高质量自然数据和人工数据，控制 synthetic ratio，做去重和多样性审计，加入真实用户分布评估，并监控长尾能力和事实性回归。

面试表达：合成数据是分布编辑，不是免费扩容。比例、自然数据锚点和验证闭环决定它是增强能力还是加速退化。

## Preference Data

一句话定义：preference data 是对同一 prompt 下多个回答的比较数据，用于训练 reward model、DPO 或其他偏好优化方法。

常见结构：prompt、chosen response、rejected response、rubric、标注者、语言、风险类别、标注理由和数据版本。排序列表也可以转成多个成对偏好样本。

质量风险：标注者不一致、rejected 太弱、长度偏置、迎合偏置、安全过度偏置、LLM judge 偏差和隐私数据混入。

面试表达：偏好数据不是“哪个好看选哪个”，而是对 helpful、honest、harmless 等行为准则的稀疏采样。必须审计一致性、margin、长度偏置和训练后回归。

## Chosen Response

一句话定义：chosen response 是偏好样本中被标注为更好的回答。

判断依据：通常来自 helpfulness、factuality、safety、format following、tone、citation、professional boundary 和用户意图匹配等维度。

工程注意：chosen 不一定是完美答案，只是在当前候选中更好。训练前要检查它是否安全、事实可靠、未包含隐私和未泄漏评测内容。

面试表达：chosen response 是相对偏好信号，不是绝对 gold label。高质量数据要记录为什么 chosen 胜出。

## Rejected Response

一句话定义：rejected response 是偏好样本中较差的回答，用来和 chosen 构成比较信号。

数据价值：好的 rejected 应该接近真实模型错误模式，且和 chosen 有可解释差异。太弱的 rejected 会让模型只学到粗糙偏好。

常见问题：明显胡说、过短、故意不安全、和 chosen 差异太大或只因长度较短被拒。此类样本要降权或剔除。

面试表达：DPO / RM 的数据质量很大程度取决于 rejected 的质量。rejected 要有难度，而不是随便造一个坏答案。

## Reward Model Data

一句话定义：reward model data 是用于训练奖励模型的偏好排序数据，通常包含 prompt、多个回答、排序或 chosen/rejected 对。

训练目标：让 reward model 给 chosen 更高分，给 rejected 更低分，从而近似人类偏好。它学的是标注规范下的相对排序，不是事实真值。

审计指标：pairwise accuracy、标注一致性、分桶准确率、长度偏置、reward calibration、OOD 泛化、安全类别表现和 reward hacking 风险。

面试表达：reward model 只是偏好的代理目标。它必须和人工评估、安全评估和事实性评估一起看，不能只追 reward score。

## Safety Data

一句话定义：safety data 是用于训练和评估模型安全行为的数据，覆盖风险分类、拒答、边界允许、误拒修复、红队回归和专业高风险场景。

核心目标：让模型区分该帮助、该拒绝、该澄清、该给安全替代建议的场景。安全数据不是简单把风险关键词都映射为拒答。

数据分层：明确安全、敏感但可答、高风险、边界模糊、紧急风险、专业高风险、多语言和隐私场景都要单独标注。

面试表达：好的安全数据同时降低漏拒和误拒。只训练拒答会导致模型保守，缺少边界允许数据会损害 helpfulness。

## Over-Refusal

一句话定义：over-refusal 是模型把本应安全回答的请求错误拒绝，也叫误拒。

常见原因：安全训练样本只有拒答，没有边界允许数据；关键词过滤太粗；标注规范过度保守；多语言安全数据不足。

治理方式：构造安全可答样本、教育性 / 防御性 / 高层次解释样本、专业边界样本和误拒回归集，并按语言和风险类别评估。

面试表达：安全模型不是拒得越多越好。误拒会降低可用性，甚至阻断正常教育和防御场景。

## Unsafe Leak

一句话定义：unsafe leak 是模型在应拒绝或安全改写的高风险请求上给出了不安全回答，也叫漏拒。

常见来源：安全数据覆盖不足、红队失败模式未回流、policy 版本不一致、多轮上下文中边界丢失或模型过度追求 helpfulness。

治理方式：红队回归、风险分层、安全拒答和安全替代建议数据、边界样本专家审计、多语言测试和上线监控。

面试表达：漏拒是安全底线风险，但只压漏拒会带来误拒。必须用成对数据和双指标一起优化。

## Red Team Data

一句话定义：red team data 是系统性安全测试中发现的模型失败样本、风险类别、失败模式、安全响应和回归测试数据。

使用边界：红队数据应服务于防御性训练、评估和治理。进入训练时要保留必要标签和安全回答，避免传播可操作风险细节。

工程用途：发现模型弱点、补安全数据、构建回归集、跟踪策略版本变化和评估上线风险。

面试表达：红队数据不是攻击样本库，而是安全治理闭环的一部分。关键是把失败模式转化为可复测、可修复的数据资产。

## Multimodal Data

一句话定义：multimodal data 是同时包含图像、文本、OCR、音频、视频、字幕、时间戳或多模态问答的数据。

治理重点：相比纯文本，多模态数据必须同时检查媒体质量、跨模态对齐、时序同步、隐私、版权来源、评测污染和任务配比。图像清晰但 caption 错、音频好但 transcript 错、视频有字幕但时间错位，都可能让模型学到错误关联。

面试表达：多模态数据工程的核心不是“把图片、音频和视频拼进训练集”，而是证明不同模态确实互相解释，并且来源、隐私和评测隔离都可审计。

## Image-Text Pair

一句话定义：image-text pair 是图像与 caption、alt text、标题、周围文本或问答文本组成的训练样本。

常见问题：网页文本不一定描述图片，装饰图、营销素材图、模板图和新闻配图都可能造成弱监督噪声。高质量图文对要记录来源、图像 hash、文本来源、语言、质量分、对齐分和许可证状态。

面试表达：图文对不是图片加一句话，而是一个需要证明“文本真的在解释图像”的弱监督样本。

## Image-Text Alignment

一句话定义：image-text alignment 是图像内容和文本描述之间的一致性。

常见做法：可以用 CLIP 类 embedding 相似度、OCR 命中、对象 / 属性规则、人工抽样和下游 VQA 评测共同判断。相似度只能辅助过滤，不能替代人工或任务级验证。

面试表达：对齐分低的样本会让 VLM 学错视觉概念；对齐分高也不保证 caption 完整，因此需要分桶抽检和任务回归。

## OCR Data

一句话定义：OCR data 是从图片、截图、文档页、票据、表格或网页截图中识别出来的文字及其版面结构。

关键字段：识别文本、bounding box、页码、阅读顺序、置信度、语言、表格结构和脱敏标记。只保留纯文本会丢失版面、列关系和图文位置关系。

面试表达：OCR 数据治理要同时看识别准确率和 layout 保真度，否则文档问答、表格理解和截图理解会受到明显影响。

## Caption Data

一句话定义：caption data 是描述图像、音频或视频片段内容的自然语言文本。

质量要求：caption 应具体、可验证、不过度幻觉，并且和媒体片段时间范围一致。过泛的“一个人在做事”或带营销口吻的描述通常训练价值有限。

面试表达：caption 数据的价值在于把感知内容变成语言监督，但 caption 错、空泛或时间错位会直接污染多模态对齐。

## Audio-Transcript Pair

一句话定义：audio-transcript pair 是音频片段与对应转录文本组成的训练样本。

治理重点：需要记录采样率、语言、说话人、时间戳、噪声、转录版本、WER / CER、VAD 切片和隐私脱敏状态。音频清晰度和 transcript 准确性要一起审计。

面试表达：语音数据不是只看音频质量，还要看转录是否准确、时间是否对齐、多语言和口音是否覆盖。

## ASR Data

一句话定义：ASR data 是用于训练或评估自动语音识别系统的音频、转录、时间戳和质量标签。

常见指标：英文或空格分词语言常看 WER，中文或字符级任务常看 CER；工程上还要看分语种、分口音、分噪声场景和分时长的错误分布。

面试表达：ASR 数据质量决定语音模型的鲁棒性，平均 WER 低不代表长音频、噪声、口音和多说话人场景都可靠。

## Video-Text Data

一句话定义：video-text data 是视频片段与字幕、caption、事件描述、动作标签或视频问答组成的数据。

治理重点：视频要处理片段切分、帧采样、镜头变化、音频、字幕、事件时间范围、版权来源和隐私风险。一个全视频标题通常不能当作每个片段的精确标签。

面试表达：视频数据的难点在时间结构，训练样本必须说明文本描述对应哪个片段、哪段声音或哪个事件。

## Temporal Alignment

一句话定义：temporal alignment 是音频、字幕、视频帧、事件描述和问答证据在时间轴上的同步关系。

常见风险：字幕延迟、ASR 切片错位、视频 caption 只描述开头或结尾、长视频标题泛化到全部片段。时间错位会让模型把错误画面、声音和文本绑定在一起。

面试表达：多模态数据治理要把时间戳当作核心字段，而不是附属元数据；音频和视频任务尤其需要 forced alignment 或抽样复核。

## Grounded Multimodal QA

一句话定义：grounded multimodal QA 是答案必须由给定图像、文档、音频或视频证据支持的多模态问答数据。

质量要求：问题要可由媒体内容回答，答案要引用或对齐到可见文字、物体、动作、声音或时间片段。无法从媒体中验证的答案应标记为 hallucinated 或 unsupported。

面试表达：多模态指令数据不能只看回答是否流畅，还要检查答案是否被视觉、OCR、音频或视频证据支持。

## Visual Privacy

一句话定义：visual privacy 是图像和视频数据中的人脸、车牌、证件、住址、病历、儿童影像、屏幕截图和其他可识别个人信息风险。

治理方式：训练数据需要记录隐私风险标签、脱敏状态、删除策略、来源许可和人工复核。高风险媒体不能因为文本侧没有 PII 就直接进入训练。

面试表达：多模态隐私比文本隐私更难，因为敏感信息可能藏在像素、画面背景、OCR 和声音里，必须在媒体解析阶段就做风险门禁。

## Data Attribution

一句话定义：data attribution 是追溯模型行为、错误、能力变化或评估结果可能来自哪些训练数据、数据源或数据版本的过程。

常见证据：相似训练样本、数据源统计、版本变化、训练日志、embedding / gradient 近似、消融实验和人工审计。大模型里的 attribution 多数是证据链，不是严格因果证明。

面试表达：data attribution 主要服务错误诊断和数据治理，目标是把“模型为什么这样表现”追溯到可审计的数据来源、版本和处理规则。

## Data Valuation

一句话定义：data valuation 是估计数据对目标模型指标、能力、风险或产品价值的贡献，用于决定保留、删除、采购、标注、上采样或降权。

关键点：数据价值不是固定常数，而依赖目标模型、训练阶段、已有数据分布、评估指标、风险约束和成本。一个数据源可能提升代码能力，同时损害安全或风格。

面试表达：数据估值不是只算准确率提升，而是把收益、成本、风险、副作用和可治理性放进同一个决策框架。

## Influence Function

一句话定义：influence function 用来估计训练样本权重发生微小变化时，对某个测试样本 loss 或预测的影响。

工程边界：经典公式涉及 Hessian inverse，深度模型和大模型中直接计算成本高且近似不稳定。实际更常用小模型、梯度相似、训练轨迹或检索近邻做局部诊断。

面试表达：Influence function 是训练数据归因的重要思想，但在 LLM 规模下通常不能被当作精确逐条训练数据贡献计算器。

## Data Shapley

一句话定义：Data Shapley 用 Shapley value 思想衡量数据在不同子集中的平均边际贡献。

优点和限制：它理论性质好，能考虑数据交互；但精确计算需要大量子集训练或近似训练，通常只适合小规模数据池、源级估值或作为估值思想。

面试表达：Data Shapley 的价值在于定义“公平边际贡献”，但大模型里更现实的是近似 Shapley、源级 ablation 和 proxy 实验。

## Gradient Similarity

一句话定义：gradient similarity 是比较训练样本或数据源梯度方向与目标任务梯度方向是否一致的近似归因信号。

使用方式：如果某批数据的梯度和目标验证集方向接近，它可能对目标能力有帮助；如果方向相反，可能带来负贡献。它仍需结合风险、污染和消融验证。

面试表达：梯度相似是弱信号，不是最终决策。高相似数据如果有 benchmark contamination 或隐私风险，仍然必须阻断。

## Source-Level Ablation

一句话定义：source-level ablation 是删除、降权、加入或上采样一个数据源后，通过训练实验观察多维指标变化的数据估值方法。

适用场景：用于高价值数据池、可疑数据源、采购决策、配比调整和上线前关键验证。它比逐条估值贵，但解释性强，和真实训练目标更一致。

面试表达：大模型里源级 ablation 常比样本级估值更实用，因为训练数据规模大、单条样本影响小、数据源和数据分布影响更明显。

## Proxy Model

一句话定义：proxy model 是用较小模型、短训练或低成本实验近似评估数据价值的替代模型。

风险：proxy 结论不一定外推到大模型，尤其在能力涌现、长上下文、多模态和安全行为上。需要用多尺度实验或最终小比例验证校准。

面试表达：proxy model 是数据估值的成本控制工具，不是最终真相；它适合筛候选和排雷，关键决策仍要用更接近目标训练的验证。

## Negative Value Data

一句话定义：negative value data 是对目标效用产生负贡献的数据，例如错误标注、过时知识、低质合成、污染样本、隐私风险和导致误拒的数据。

治理方式：通过质量分、异常 loss、错误归因、源级 ablation、人工审计、安全回归和污染检测定位，再决定重洗、降权、隔离或删除。

面试表达：数据工程不只是找高价值数据，删掉负价值数据有时比增加新数据更有效。

## Active Learning

一句话定义：active learning 是在标注预算有限时，优先选择最值得标注或复核的样本。

选择信号：不确定性、模型分歧、用户频率、风险等级、目标能力薄弱、代表性、多样性和标注成本。大模型数据工程中常用于 SFT、偏好、安全、领域和多模态标注。

面试表达：主动学习不是只挑最难样本，而是在价值、覆盖、风险和成本之间做标注优先级排序。

## Data Selection

一句话定义：data selection 是在 token、标注、训练和风险预算下选择哪些数据进入训练、上采样、降权或删除。

常见约束：目标能力、质量分、来源可信度、多样性、语言 / 领域配比、隐私版权风险、benchmark contamination、成本和版本可追溯性。

面试表达：数据选择最终是带约束的优化问题。不能只选最像评测集的数据，否则会过拟合 benchmark 并损害泛化。

## Dataset Versioning

一句话定义：dataset versioning 是把训练数据快照、manifest、pipeline、schema、过滤配置、采样配置和审计记录绑定成可复现版本的工程机制。

为什么重要：训练结果依赖具体数据版本。数据目录被重写、过滤规则变化、shard 顺序变化或采样权重变化，都会让模型结果不可复现。

面试表达：数据版本不是文件名加日期，而是数据、代码、配置、hash、统计和审计记录的组合。

## Data Governance

一句话定义：data governance 是围绕数据来源、权限、隐私、license、删除请求、审计、文档和训练可复现性的治理体系。

工程重点：治理不是训练结束后的合规表格，而是嵌入数据 pipeline 的控制面。它要支持追溯、回滚、隔离、删除、审计和发布说明。

面试表达：大模型数据治理决定团队能否说清模型用了什么数据、为什么允许用、经过哪些处理、进入哪个版本，以及未来如何删除和复核。

## Data Lineage

一句话定义：data lineage 是数据从原始来源到最终训练样本、训练版本和模型版本的血缘记录。

典型字段：source id、raw object hash、processing steps、filter versions、dedup cluster、dataset version、training run id 和 deletion status。

面试表达：lineage 让问题样本可追源，也让删除请求和污染排查能从来源追到所有受影响版本。

## Lineage Coverage

一句话定义：lineage coverage 是具备来源、处理路径、版本、hash、风险标记和训练去向等血缘字段的样本或 token 比例。

为什么重要：lineage 覆盖不足时，数据事故很难定位到来源和版本，也很难处理删除请求、污染追溯、回滚和复现实验。

工程用途：按数据源、数据桶和风险等级统计 lineage coverage。高风险、授权、代码、用户日志和评估相关数据应使用更严格阈值。

面试表达：lineage coverage 低说明团队不是缺一个字段，而是缺事故追溯能力。它应该进入数据版本报告和训练上线门禁。

## Dataset Manifest

一句话定义：dataset manifest 是描述某个数据版本包含哪些 shard、hash、样本数、token 数、统计分布和风险摘要的清单。

作用：manifest 是训练可复现的入口。训练日志应记录 manifest、data mixture、sampler、tokenizer 和评测版本，而不是只记录一个数据目录。

面试表达：没有 manifest，就很难证明两次训练使用的是同一份数据。

## Immutable Snapshot

一句话定义：immutable snapshot 是一旦用于训练就不再原地修改的数据快照。

做法：用内容 hash、对象存储版本、shard checksum、数据库快照和配置版本保证快照稳定。如果发现错误，应创建新版本而不是偷改旧版本。

面试表达：不可变快照的价值是让实验结果、审计记录和删除追溯有稳定参照。

## Schema Evolution

一句话定义：schema evolution 是数据元数据字段、标签体系和字段含义随时间演进时的版本管理。

注意事项：新增字段要有默认值，旧数据要能回填；字段含义变化必须版本化，例如 quality score v1 和 v2 不能混当同一个分数。

面试表达：schema 演进如果不治理，会让过滤规则、采样器和评估分析使用不一致字段，导致结论失真。

## Deletion Request

一句话定义：deletion request 是用户、数据提供方、版权方或合规团队要求删除或撤回某些数据的请求。

工程难点：要定位 raw data、清洗数据、shard、索引、embedding、旧版本和已训练模型是否受影响，并记录执行证明。它依赖 source id、hash、lineage 和版本记录。

面试表达：删除不是删当前目录，而是一个跨版本、跨缓存、跨派生数据的追踪和证明流程。

## Dataset Datasheet

一句话定义：dataset datasheet 是数据集说明书，记录创建动机、组成、采集、处理、用途、限制、风险、维护和删除策略。

价值：datasheet 让数据集透明、可沟通、可复核，减少“训练时没人记得数据怎么来的”的问题。

面试表达：datasheet 是数据治理文档，不是宣传材料；它要能支撑风险讨论和使用边界。

## Dataset Card

一句话定义：dataset card 是面向数据集使用者的数据说明卡，通常包含数据来源、字段、任务、划分、许可、偏差、风险和使用限制。

使用场景：开源数据、内部数据平台和合作数据交付都可以用 dataset card 帮助用户理解数据能做什么、不能做什么。

面试表达：dataset card 更偏数据集使用说明，datasheet 更偏完整治理说明；两者都要求数据版本和元数据可靠。

## Model Card

一句话定义：model card 是模型说明卡，描述模型用途、训练概况、评估结果、适用范围、限制和风险。

与数据治理关系：model card 中的训练数据、适用语言、偏差、风险和删除策略，都依赖 dataset versioning、lineage、datasheet 和审计结果提供事实基础。

面试表达：没有数据版本和治理，model card 很容易变成泛泛而谈，无法支撑真实风险边界。

## Benchmark Contamination

一句话定义：benchmark contamination 是评估集内容泄漏进训练数据，导致评估结果虚高的问题。

典型形式：训练语料中包含 benchmark 原题、答案、解析、翻译版本、近重复版本，或代码 benchmark 的测试用例和标准解。

危害：模型看似能力强，实际可能只是记住了题目或答案，导致能力判断、模型选型和论文结论失真。

检测方法：训练集和评估集做 n-gram 匹配、MinHash 近重复检测、embedding 相似度检索、时间切分评估、私有评估集和动态评测。

应对方式：训练前过滤已知 benchmark，报告数据 cutoff，使用新题和私有测试集，做 contamination analysis。

面试表达：benchmark contamination 是大模型评估可信度的核心风险。越热门的公开 benchmark，越需要怀疑数据泄漏。

## Data Interview Readiness

一句话定义：Data Interview Readiness 是候选人能否把大模型数据工程问题讲成目标、来源、处理、配比、评估、治理和 trade-off 闭环的准备度。

核心维度：answer coverage、metric coverage、engineering evidence、risk coverage、trade-off depth、red flags 和 time budget。

常用指标：回答覆盖度、指标覆盖度、token 保留率、风险命中率、contamination 命中率、effective epoch、lineage 覆盖率、datasheet 完成度和 mock interview 平均得分。

复盘方法：每次模拟面试后，记录题目、原回答、缺失框架、缺失指标、缺失风险、错误表述、超时点和下一轮 revision plan。复盘重点不是追求固定话术，而是让回答能被工程证据支撑。

面试表达：数据工程面试不是背某个数据集比例，而是证明你能设计可审计、可复现、可评估、可治理的数据系统。

## Data Packing

一句话定义：data packing 是把多个较短文本样本拼接成固定长度训练序列的方法，用来减少 padding、提高 token 利用率。

为什么需要：预训练和微调常使用固定 sequence length。如果很多样本很短，直接 padding 会浪费大量计算。packing 可以把多个样本装进同一个序列。

常见做法：把多个文档或样本按 token 拼接，中间加入 EOS 或特殊分隔符，直到接近最大长度。

注意事项：不同文档之间是否允许 attention 互相可见，要看训练目标。预训练中跨文档拼接通常用 EOS 隔开；SFT 中 packing 还必须正确处理 loss mask，避免 user token 或 padding token 参与 loss。

工程坑：如果没有正确插入 EOS，模型可能学习错误的跨文档衔接；如果 label mask 错，模型可能在不该训练的位置计算 loss；如果边界处理错，可能导致对话模板污染。

面试表达：packing 是提高训练效率的关键工程技巧，但必须处理好 EOS、document boundary、attention mask 和 loss mask。

## Document Boundary

一句话定义：document boundary 是不同文档或样本之间的边界信息。

为什么重要：语言模型训练时经常把多个文档拼接成连续 token。如果不告诉模型文档结束，模型可能把两个无关文档当成自然连续文本。

常见处理：在文档之间加入 EOS token、特殊分隔符或重置位置。现代预训练中最常见的是用 EOS 表示文档结束。

影响：边界处理会影响模型对段落结束、回答终止、多文档上下文和生成停止的学习。

面试表达：document boundary 是预训练数据格式的一部分。它看似细节，但会影响模型是否学会正确结束文本和区分独立样本。

## Sequence Length

一句话定义：sequence length 是训练时单个样本包含的最大 token 数，也常被称为训练上下文长度。

影响：更长序列能让模型学习长距离依赖、长文档结构和长上下文任务，但 attention 成本通常随长度平方增长，训练代价更高。

常见策略：先用较短 sequence length 训练大部分 token，再用更长 sequence length 做长上下文继续训练；或者在数据中混合不同长度样本。

误区：把推理配置里的 max context 改大，不代表模型真的具备长上下文能力。长上下文能力需要位置编码、训练长度分布和评估共同支持。

面试表达：sequence length 是能力和成本之间的权衡。长上下文不是只改配置，而是训练数据、位置编码和系统成本共同决定。

## Checkpoint

一句话定义：checkpoint 是训练过程中保存的模型和训练状态快照。

内容：model weights、optimizer state、learning rate scheduler state、RNG state、training step、数据加载状态、混合精度 scaler、模型 config 和 tokenizer 信息。

作用：故障恢复、阶段评估、模型选择、回滚异常训练、继续训练、发布中间模型。

保存策略：按固定 step 保存、按验证指标保存、保留最近若干个 checkpoint、保留关键里程碑 checkpoint。大模型 checkpoint 体积极大，存储和传输本身也是工程问题。

工程坑：只保存模型权重不能完全恢复训练；如果 optimizer state 或数据迭代状态丢失，继续训练的曲线可能发生变化。tokenizer/config 与 checkpoint 不匹配也会导致加载错误。

面试表达：checkpoint 不是简单的权重文件，而是可复现训练状态的快照。大规模训练必须设计可靠的 checkpoint 和恢复机制。

## Training Token

一句话定义：training token 是训练过程中被模型看到并用于学习的 token 数量。

为什么重要：LLM 能力不仅取决于参数量，也取决于训练 token 数。参数很多但 token 不足，会出现 undertraining；参数较小但 token 充分，可能在固定 compute 下更优。

统计注意：要区分原始字符数、文档数、tokenizer 后 token 数、参与 loss 的 token 数和被 mask 掉的 token 数。不同 tokenizer 下 token 数不可直接等价。

面试表达：训练 token 数是 scaling law 的核心变量之一。讨论模型训练规模时，只说参数量是不够的，还要说训练了多少 token。

## Epoch

一句话定义：epoch 表示训练过程完整遍历一遍训练数据。

在大模型中的特殊性：LLM 预训练数据巨大，常常不按传统小数据集那样训练很多 epoch，而是更关注总训练 token 数和数据采样分布。

风险：对小规模或重复数据训练多个 epoch 会增加记忆和过拟合风险，尤其在领域继续预训练和 SFT 中更明显。

面试表达：在 LLM 预训练里，epoch 概念不如 training token 和 sampling distribution 重要；在微调里，epoch 数过多则容易过拟合和遗忘。

## Scaling Law

一句话定义：scaling law 是描述模型性能随参数量、数据量和计算量扩展而变化的经验规律。

常见变量：模型参数量 `N`、训练 token 数 `D`、计算量 `C`、验证 loss 或 perplexity。

核心用途：在正式训练大模型前，用小规模实验外推大规模模型表现，帮助决定模型规模、训练 token 数和算力预算。

典型结论：在一定范围内，增加参数、增加数据和增加计算通常能平滑降低语言建模 loss。但收益呈递减趋势，并且不同能力不一定都随 loss 同步提升。

局限：scaling law 多数基于 loss，而用户关心的指令遵循、安全、推理、工具调用和多模态能力不一定能完全由 loss 解释。

面试表达：scaling law 是大模型资源规划工具，不是绝对定律。它能指导 compute 分配，但不能替代真实下游评估。

## Chinchilla Scaling

一句话定义：Chinchilla scaling 强调在固定计算预算下，应平衡模型参数量和训练 token 数，而不是一味增大参数。

核心启发：很多早期大模型参数很大，但训练 token 不足，属于 undertrained。给较小模型训练更多高质量 token，可能在同等 compute 下获得更低 loss。

直观理解：如果模型太大但数据太少，参数没有被充分训练；如果数据很多但模型太小，容量不足。compute-optimal training 要在容量和数据之间找平衡。

影响：后来很多模型更重视 token 数、数据质量和多 epoch/多 token 训练，而不是只追求参数规模。

注意：Chinchilla 结论依赖实验设定、数据分布和目标指标。实际模型还要考虑推理成本、部署需求、能力目标和数据可得性。

面试表达：Chinchilla 的面试重点不是背比例，而是说明“固定 compute 下，大模型训练要同时扩参数和扩数据，很多模型不是参数不够，而是训练 token 不够”。

## Compute-Optimal Training

一句话定义：compute-optimal training 是在固定计算预算下选择最优模型规模、训练 token 数和训练配方的策略。

核心问题：预算应该花在更大模型上，还是更多 token 上？应该训练一个更强但推理贵的模型，还是训练一个较小但 token 更充分、部署更便宜的模型？

考虑因素：训练 compute、推理 compute、数据质量、目标任务、上下文长度、部署延迟、显存限制、服务成本和后训练需求。

与 Chinchilla 的关系：Chinchilla 给出语言建模 loss 视角下的 compute-optimal 经验启发；真实工程中的 compute-optimal 还要加入推理成本和产品目标。

例子：如果目标是高并发线上服务，较小但训练充分的模型可能比更大模型更合适；如果目标是前沿能力探索，则可能更愿意训练更大模型并接受高成本。

面试表达：compute-optimal training 不是单纯追求最大参数量，而是在训练成本、数据规模、模型容量和推理成本之间做全局最优。

## 本章小结

本章围绕预训练和数据治理展开。预训练决定 LLM 的能力底座，而数据决定模型从什么分布中学习、学习到哪些能力、带入哪些风险。

高频结论如下：

1. 预训练通常用 next-token prediction 在海量 token 上训练 base model。
2. 数据质量、去重和污染控制决定单位 compute 的训练收益。
3. data mixture 和 sampling strategy 会直接影响模型能力分布。
4. 合成数据有用，但必须经过筛选、验证和去重。
5. benchmark contamination 会让评估结果虚高，是大模型评估中的核心风险。
6. data packing 能提高训练效率，但必须处理好 EOS、边界和 loss mask。
7. checkpoint 应保存完整训练状态，而不只是模型权重。
8. scaling law 和 Chinchilla scaling 用来指导参数量、token 数和 compute 的平衡。
9. compute-optimal training 要同时考虑训练成本、推理成本、数据质量和目标能力。
