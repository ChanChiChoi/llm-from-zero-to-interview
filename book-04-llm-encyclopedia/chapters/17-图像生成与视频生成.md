# Q. 图像生成与视频生成

条目：Diffusion Model、Forward Diffusion、Reverse Denoising、Noise Scheduler、U-Net、VAE、Latent Diffusion、Stable Diffusion、Classifier-Free Guidance、DALL·E、Autoregressive Image Generation、Video Diffusion、Video Token、Spatiotemporal Patch、World Model、Sora、Temporal Consistency、Motion Prior、Text-to-Image、Text-to-Video。

## 本章速览

一句话定义：图像生成与视频生成是让模型根据文本、图像、布局、草图、参考图或其他条件，生成符合语义、视觉质量和时空一致性的视觉内容。

大模型语境下，图像生成的主流路线包括扩散模型、自回归图像生成和多模态统一建模。视频生成在图像生成基础上进一步要求时间连续、运动合理、物理一致和长程可控。

核心问题：生成模型不仅要“画得像”，还要满足用户指令、保持主体一致、避免文字和手部错误、控制风格与构图，并在视频中保持帧间一致、动作自然和镜头逻辑合理。

常见技术栈：文本 encoder 理解 prompt，生成 backbone 负责从噪声或 token 中生成视觉表示，VAE 或 tokenizer 负责在像素空间和潜空间之间转换，scheduler 或采样器控制生成过程，guidance 机制控制条件遵循程度。

面试表达：图像生成主要看语义对齐、视觉保真和可控性；视频生成还要看时序一致性、运动建模和长视频稳定性。扩散模型是当前图像和视频生成的重要基础，但并不是唯一路线。

## Diffusion Model

Diffusion Model，扩散模型，是通过逐步加噪和逐步去噪学习数据分布的生成模型。

直观理解：训练时把真实图片一步步加噪，直到接近纯噪声；模型学习在任意噪声程度下如何预测噪声或还原干净图像。生成时从随机噪声开始，经过多步去噪，逐渐生成清晰图片。

扩散模型包含两个过程：

| 过程 | 作用 | 是否学习 |
| --- | --- | --- |
| Forward Diffusion | 把真实数据逐步加噪 | 通常固定，不需要学习 |
| Reverse Denoising | 从噪声逐步恢复数据 | 需要神经网络学习 |

为什么提出扩散模型：GAN 生成速度快，但训练容易不稳定、模式崩塌；VAE 稳定但生成质量容易偏模糊；自回归模型似然建模清晰但高分辨率图像生成成本高。扩散模型用稳定的去噪训练目标获得高质量生成效果，因此成为图像生成的重要路线。

优点：训练稳定、生成质量高、覆盖模式较好、易于条件控制、适合图像编辑和超分辨率等任务。

缺点：多步采样导致推理慢；高分辨率生成成本高；文本和图像对齐依赖数据和条件机制；精确结构、文字、数量、手部和复杂空间关系仍可能出错。

面试表达：扩散模型的本质是学习反向去噪过程。训练时模型看到带噪样本和噪声强度，学习预测噪声或干净样本；生成时从随机噪声迭代去噪。

## Forward Diffusion

Forward Diffusion 是从真实样本到噪声的加噪过程。

给定真实图像 `x_0`，逐步加入高斯噪声得到 `x_1, x_2, ..., x_T`。当 T 足够大时，`x_T` 接近标准高斯噪声。

常见形式可以写成：

$$
q(x_t\mid x_{t-1})=\mathcal{N}\left(\sqrt{1-\beta_t}\,x_{t-1},\beta_t I\right)
$$

其中 `beta_t` 是第 t 步的噪声强度。噪声强度如何随时间变化，就是 noise schedule。

一个重要性质是可以从 `x_0` 直接采样任意时刻 `x_t`：

$$
x_t=\sqrt{\bar{\alpha}_t}\,x_0+\sqrt{1-\bar{\alpha}_t}\,\epsilon
$$

这使训练更高效，因为模型不需要真的一步步加噪，而是随机采样一个时间步 t，直接构造带噪样本。

面试表达：正向扩散通常是固定的马尔可夫加噪过程，它定义了训练样本如何被破坏；模型真正要学的是反向去噪。

## Reverse Denoising

Reverse Denoising 是从噪声恢复数据的生成过程。

训练时，模型接收带噪样本 `x_t`、时间步 `t` 和条件 `c`，预测噪声 `epsilon`、干净样本 `x_0` 或 velocity `v`。最经典的训练目标是噪声预测：

$$
L=\left\|\epsilon-\epsilon_\theta(x_t,t,c)\right\|_2^2
$$

生成时，从随机噪声 `x_T` 开始，模型反复预测并去除噪声，得到 `x_{T-1}, x_{T-2}, ..., x_0`。

常见预测参数化：

| 参数化 | 模型预测 | 特点 |
| --- | --- | --- |
| epsilon prediction | 噪声 | 经典 DDPM 常用，直观简单 |
| x0 prediction | 干净样本 | 直接预测最终图像，某些场景更直观 |
| v prediction | velocity | 在部分高质量扩散模型中常用，稳定性好 |

去噪过程中的条件可以来自文本、类别、图片、mask、深度图、边缘图、姿态图等。Text-to-Image 中最常见条件是文本 embedding。

面试表达：反向去噪不是一次生成完整图片，而是在多个噪声等级上逐步修正。每一步都利用模型预测当前噪声中应该去掉什么。

## Noise Scheduler

Noise Scheduler 定义扩散过程中的噪声强度、时间步和采样更新规则。

训练 scheduler 决定不同时间步加多少噪声。采样 scheduler 决定如何从 `x_t` 更新到 `x_{t-1}`，以及使用多少步生成最终图像。

常见采样器或 scheduler 包括 DDPM、DDIM、DPM-Solver、Euler、Heun、UniPC 等。不同 scheduler 会影响生成速度、细节、稳定性和 prompt 遵循。

核心取舍：

| 选择 | 优点 | 缺点 |
| --- | --- | --- |
| 多步采样 | 质量更稳，细节更好 | 推理慢 |
| 少步采样 | 延迟低，成本低 | 容易细节差或不稳定 |
| 随机采样 | 多样性好 | 可复现性和稳定性弱 |
| 确定性采样 | 结果更可控 | 多样性可能降低 |

工程中常见参数包括 sampling steps、CFG scale、seed、scheduler 类型、分辨率、negative prompt 等。调参时不能只看单张样例，要固定 seed 和测试 prompt 集合做对比。

面试表达：scheduler 不改变模型权重，但会明显影响采样速度和质量。扩散模型部署时，采样器选择和步数压缩是重要优化点。

## U-Net

U-Net 是早期扩散模型中最常见的去噪网络结构。

它包含下采样路径、瓶颈层和上采样路径，并通过 skip connection 保留多尺度细节。图像生成需要同时理解全局语义和局部纹理，U-Net 的多尺度结构很适合这个任务。

在文本到图像扩散模型中，U-Net 通常会接收三类输入：带噪 latent 或图像、时间步 embedding、文本条件 embedding。文本条件常通过 cross-attention 注入 U-Net。

U-Net 的优势：多尺度建模强、局部细节好、在图像任务上成熟稳定。

U-Net 的局限：扩展到很大规模和长程依赖时不如 Transformer 自然；视频生成中需要额外加入时间维度或时序模块；高分辨率和长视频成本较高。

发展趋势：很多新模型开始使用 Diffusion Transformer，也就是 DiT，用 Transformer 处理图像 patch 或 latent token，以获得更好的 scaling 能力。

面试表达：Stable Diffusion 早期核心去噪网络是 U-Net，文本通过 cross-attention 影响去噪过程。新一代模型越来越多使用 Transformer 化的扩散 backbone。

## VAE

VAE 在 Stable Diffusion 类模型中常用于像素空间和 latent space 的转换。

直接在像素空间做扩散计算量很大。例如 512 x 512 x 3 的图片维度很高。Latent Diffusion 先用 VAE encoder 把图片压缩到较低分辨率 latent，再在 latent 空间扩散，最后用 VAE decoder 还原为图片。

简化流程：

```text
training image -> VAE encoder -> latent z
latent z -> add noise -> denoising model -> clean latent
clean latent -> VAE decoder -> generated image
```

VAE 的作用不是理解 prompt，而是压缩和重建图像。文本理解通常由 text encoder 完成，去噪由 U-Net 或 DiT 完成。

优点：显著降低扩散计算成本，使高分辨率生成更可行。

缺点：VAE 压缩会带来重建损失，可能影响细节、文字、纹理和边缘；VAE decoder 质量也会限制最终图片质量。

面试表达：Stable Diffusion 之所以叫 latent diffusion，是因为扩散发生在 VAE 压缩后的 latent 空间，而不是原始像素空间。

## Latent Diffusion

Latent Diffusion 是在潜空间中进行扩散建模的路线。

核心思想：先训练一个 autoencoder 或 VAE，把图片压缩为 latent 表示；再在 latent 上训练扩散模型；生成完成后解码回像素空间。

相比 pixel diffusion，latent diffusion 的优势是计算成本低、显存占用小、生成速度更快，可以用较低成本训练和部署高分辨率图像生成模型。

代价是 latent 表示不是无损的。对于文字、细小图案、复杂纹理和精确几何，latent 压缩可能导致信息损失。

面试表达：Latent Diffusion 的核心 trade-off 是用压缩潜空间换取效率，但压缩器质量会成为生成质量上限的一部分。

## Stable Diffusion

Stable Diffusion 是开源影响力最大的文本到图像生成模型路线之一，典型结构是 text encoder + latent diffusion U-Net + VAE。

基本生成流程：

```text
prompt -> tokenizer -> text encoder -> text embeddings
random noise latent + text embeddings -> denoising U-Net -> clean latent
clean latent -> VAE decoder -> image
```

主要组件：

| 组件 | 作用 |
| --- | --- |
| Text Encoder | 编码 prompt，提供文本条件 |
| U-Net / DiT | 在 latent 空间预测噪声或去噪方向 |
| VAE Encoder | 训练时把图片压缩到 latent |
| VAE Decoder | 生成时把 latent 解码成图片 |
| Scheduler | 控制采样更新过程 |
| CFG | 增强文本条件遵循 |

Stable Diffusion 的影响：开源生态丰富，支持 LoRA、DreamBooth、ControlNet、Inpainting、Outpainting、img2img、风格迁移、局部重绘和各类插件。

常见能力扩展：

| 扩展 | 目标 |
| --- | --- |
| LoRA | 低成本学习人物、风格或特定概念 |
| DreamBooth | 个性化主体生成 |
| ControlNet | 用边缘、深度、姿态等控制结构 |
| Inpainting | 根据 mask 重绘局部区域 |
| img2img | 基于参考图生成变体 |
| IP-Adapter | 参考图风格或主体条件注入 |

面试表达：Stable Diffusion 的核心不是一个单独网络，而是一套 latent diffusion 系统。文本条件控制语义，去噪模型生成 latent，VAE 负责像素重建，scheduler 控制采样。

## Classifier-Free Guidance

Classifier-Free Guidance，简称 CFG，是扩散模型中常用的条件引导方法，用于增强生成结果对 prompt 的遵循。

训练时，模型有时接收真实条件，有时把条件置空，学会条件生成和无条件生成。采样时，同时计算 conditional prediction 和 unconditional prediction，再组合：

```text
pred = pred_uncond + scale * (pred_cond - pred_uncond)
```

其中 `scale` 是 CFG scale。scale 越大，模型越强烈地朝 prompt 条件方向生成。

CFG scale 的影响：

| CFG scale | 结果倾向 |
| --- | --- |
| 太低 | prompt 遵循弱，可能更随机 |
| 适中 | 语义和视觉质量平衡 |
| 太高 | 画面可能过饱和、僵硬、细节异常或 artifacts 增多 |

CFG 的价值：不需要额外训练 classifier，就能增强条件控制，是文本到图像扩散模型的核心技巧之一。

面试表达：CFG 本质是在采样时放大“有条件预测”和“无条件预测”的差异，让生成结果更贴近文本条件，但过强 guidance 会牺牲自然度和多样性。

## DALL·E

DALL·E 是 OpenAI 的图像生成系列模型，代表了从文本生成图像的重要路线。

早期 DALL·E 使用离散图像 token 和自回归 Transformer，把图像生成看成 token 序列生成问题。DALL·E 2 引入 CLIP 表示和扩散 decoder。后续系统更强调 prompt 遵循、文字渲染、图像编辑和与 ChatGPT 式交互结合。

在面试中，不需要死记每个版本细节，关键是理解 DALL·E 代表了文本到图像生成的标志性系统，并且图像生成路线可以是自回归 token 建模，也可以是扩散建模，也可以是混合式建模。

面试表达：DALL·E 系列说明文本到图像生成并不只有 Stable Diffusion 一种实现。核心都是把自然语言条件转化为视觉生成过程的控制信号。

## Autoregressive Image Generation

Autoregressive Image Generation 是把图像离散化为 token，然后像语言模型一样逐 token 生成图像。

基本流程：先用 VQ-VAE、VQGAN 或视觉 tokenizer 把图片编码成离散 token 序列，再用 Transformer 根据文本条件或上下文自回归预测下一个图像 token，最后解码成图片。

优点：训练目标和语言模型统一，天然适合多模态 token 统一建模；可以和文本 token 放在同一序列中；概率建模清晰。

缺点：高分辨率图像 token 数量巨大，逐 token 生成慢；视觉细节依赖 tokenizer；长序列建模成本高。

与扩散模型对比：

| 方面 | 自回归图像生成 | 扩散图像生成 |
| --- | --- | --- |
| 建模方式 | 逐 token 预测 | 逐步去噪 |
| 训练目标 | next token prediction | denoising loss |
| 生成速度 | token 多时较慢 | 多步采样较慢，可蒸馏加速 |
| 优势 | 与 LLM 统一，概率建模清晰 | 图像质量高，编辑和控制成熟 |
| 难点 | tokenizer 和长序列成本 | 采样速度和精确控制 |

面试表达：自回归图像生成把图片当成离散语言来生成，扩散模型把生成看成连续去噪过程。二者各有优势，未来多模态统一模型可能融合两类思想。

## Text-to-Image

Text-to-Image 是根据文本 prompt 生成图像。

好的 Text-to-Image 模型需要同时满足四类要求：

| 要求 | 含义 |
| --- | --- |
| Prompt Alignment | 是否符合文本描述 |
| Visual Quality | 是否清晰、美观、无明显 artifacts |
| Compositionality | 是否理解多个对象、属性和关系 |
| Controllability | 是否能稳定控制风格、构图、主体和细节 |

常见 prompt 组成包括主体、属性、动作、场景、风格、镜头、光照、构图、质量词和负向约束。

常见失败模式：多个实体关系混乱；数量不准；左右位置错；文字生成错误；手指异常；小物体缺失；风格压过语义；过度依赖训练集常见构图。

工程建议：评估 Text-to-Image 不要只看随机好图，要用固定 prompt 集覆盖人物、物体、文字、空间关系、计数、长 prompt、多语言和安全边界。

面试表达：Text-to-Image 的难点不是生成漂亮图片，而是在复杂 prompt 下同时保证语义、结构、细节和可控性。

## Image Editing

Image Editing 是根据原图和指令修改图像。

常见形式包括 inpainting、outpainting、局部重绘、风格迁移、主体替换、背景替换、姿态控制、图像增强和参考图编辑。

和纯 Text-to-Image 相比，图像编辑更强调保留未修改区域、保持主体身份一致、精准遵循编辑指令，并避免全图漂移。

关键技术：

| 技术 | 作用 |
| --- | --- |
| Mask conditioning | 指定修改区域 |
| img2img strength | 控制保留原图和重绘程度 |
| ControlNet | 用结构条件控制边缘、深度、姿态 |
| Instruct editing data | 学会自然语言编辑指令 |
| Reference adapter | 保持人物、物体或风格一致 |

面试表达：图像编辑比文生图更考验条件控制。模型既要改该改的地方，又不能破坏不该改的地方。

## Video Diffusion

Video Diffusion 是将扩散模型扩展到视频生成的路线。

视频可以看作一组连续帧，因此模型需要同时建模空间内容和时间变化。最直接的方法是把视频表示为 `frames x height x width` 的时空张量，然后在这个张量上加噪和去噪。

视频生成比图像生成更难，原因包括：

| 难点 | 说明 |
| --- | --- |
| 计算量大 | 多帧带来更高 token 数和显存成本 |
| 时序一致 | 主体外观不能帧间闪烁 |
| 运动合理 | 动作要自然，不能突然跳变 |
| 物理一致 | 物体运动、遮挡、光影要符合直觉 |
| 长程依赖 | 长视频需要保持场景、角色和事件连续 |
| 数据成本 | 高质量视频文本对更难清洗和标注 |

常见做法：在图像扩散模型基础上加入 temporal attention、3D convolution、时序模块、帧间 latent 建模或先生成关键帧再插帧。

面试表达：视频扩散不是简单逐帧生成图片。逐帧生成会导致闪烁和身份漂移，必须显式建模时间维度和运动先验。

## Video Token

Video Token 是视频在模型中的离散或连续表示单元。

如果使用自回归路线，视频可以通过 video tokenizer 压缩成离散 token。模型像生成文本一样生成视频 token，再由 decoder 还原为视频。

如果使用扩散路线，视频 token 可以是 latent patch、spatiotemporal patch 或连续特征。模型在这些 token 上进行去噪或预测。

视频 token 设计的关键是压缩率和信息保真之间的平衡。压缩太弱，token 太多，训练和推理成本爆炸；压缩太强，运动细节、人物身份、文字和小物体容易丢失。

面试表达：视频生成模型的效率很大程度取决于视频表示。好的 video tokenizer 或 latent 表示要在时空压缩和细节保真之间取得平衡。

## Spatiotemporal Patch

Spatiotemporal Patch 是把视频切成同时覆盖空间和时间的小块。

图像 ViT 把图片切成二维 patch。视频模型可以把连续帧切成三维 patch，例如覆盖若干帧、若干高度和若干宽度。每个 patch 被映射成一个 token，供 Transformer 或扩散 backbone 处理。

优势：能统一建模空间纹理和时间变化，适合 Transformer 化视频生成模型。

挑战：视频 patch 数量巨大。假设 16 帧、每帧 32 x 32 个空间 patch，如果不做时间压缩，就有 16384 个 patch token，成本非常高。

常见优化包括时空下采样、局部 attention、factorized attention、窗口 attention、latent compression 和分层生成。

面试表达：spatiotemporal patch 是把视频看成时空 token 序列。它让 Transformer 能处理视频，但也带来长序列计算瓶颈。

## Temporal Consistency

Temporal Consistency，时序一致性，是视频生成质量的核心指标。

它要求同一个主体在不同帧中保持身份、外观、颜色、位置变化和光照合理连续。比如人物的脸不能一帧一个样，衣服颜色不能闪烁，背景不能无缘无故变形。

常见时序问题：

| 问题 | 表现 |
| --- | --- |
| Flickering | 帧间闪烁，纹理不稳定 |
| Identity Drift | 人物或物体身份逐渐变掉 |
| Shape Deformation | 形状扭曲，手脚异常 |
| Motion Jitter | 运动抖动，不平滑 |
| Scene Inconsistency | 背景和光照突然变化 |
| Object Permanence Failure | 被遮挡后物体消失或变形 |

缓解方法：使用视频级训练数据；引入 temporal attention；加入光流或运动约束；用关键帧控制；使用更强 latent 视频表示；训练时加入时序一致性损失或偏好数据。

面试表达：视频生成的核心挑战是时间维度。单帧好看不代表视频好，时序一致性是区分图像模型和视频模型的重要标准。

## Motion Prior

Motion Prior 是模型对运动规律的先验。

视频不是随机帧序列，而包含动作、速度、加速度、遮挡、相机运动和物理约束。模型需要学习人如何走路、车如何移动、水如何流动、镜头如何推进等规律。

运动先验来源：大规模视频数据、动作标注数据、光流、姿态序列、物理模拟数据、相机轨迹数据和人类偏好反馈。

常见控制方式：文本描述动作、给定起止帧、给定姿态骨架、给定相机轨迹、给定深度或边缘序列、给定参考视频。

面试表达：视频生成不仅是空间生成，还要建模运动。运动先验不足会导致动作不自然、速度不稳定和物理违背。

## Text-to-Video

Text-to-Video 是根据文本 prompt 生成视频。

相比 Text-to-Image，Text-to-Video 的 prompt 不仅描述画面，还描述动作、时间变化和镜头语言。例如“一个穿红衣服的人从左向右跑过雨中的街道，镜头缓慢跟随”。

评估维度：

| 维度 | 关注点 |
| --- | --- |
| 文本对齐 | 场景、主体、动作是否符合 prompt |
| 单帧质量 | 每帧是否清晰自然 |
| 时序一致 | 主体和背景是否稳定 |
| 运动质量 | 动作是否自然、速度是否合理 |
| 镜头控制 | 推拉摇移、视角变化是否符合描述 |
| 物理合理 | 遮挡、碰撞、重力和光影是否合理 |
| 长程连贯 | 长视频事件是否持续一致 |

常见失败模式：开头符合 prompt，后面漂移；主体变形；运动停滞；相机和物体运动混乱；文字和手部错误；复杂多主体交互失败。

面试表达：Text-to-Video 的难点是把静态语义扩展到动态事件。模型必须同时理解“有什么”和“怎么变化”。

## World Model

World Model 在生成视频语境中指模型对世界状态变化和因果规律的内部建模能力。

如果模型只是拟合像素分布，它可能生成看起来合理但物理不一致的视频。更强的 world model 应该理解物体持续存在、遮挡后仍然存在、动作有因果顺序、物体运动受物理约束、角色目标和环境交互会影响后续状态。

World Model 与视频生成的关系：高质量视频生成需要一定世界建模能力，因为视频天然包含时间、空间、因果和物理。尤其在机器人、自动驾驶、游戏、仿真和交互式 agent 中，视频生成不只是内容创作，也可能用于预测未来状态。

需要注意：当前很多视频生成模型能生成逼真片段，但不等于具备可靠可控的世界模型。面试中要避免把“视频很真实”直接等同于“理解物理世界”。

面试表达：World Model 强调模型是否能预测世界状态如何随动作和时间演化。视频生成是观察这种能力的窗口，但逼真生成不必然代表可靠推理。

## Sora

Sora 是 OpenAI 展示的文本到视频生成模型，代表了长时长、高质量视频生成的重要进展。

公开信息中，Sora 被描述为可以生成较长视频，并能处理复杂场景、相机运动、多个主体和一定程度的物理一致性。它常被讨论为“视频生成模型是否正在学习世界模型”的代表案例。

理解 Sora 时应关注几个关键词：

| 关键词 | 含义 |
| --- | --- |
| Video as patches | 将视频表示为时空 patch 或 token |
| Scaling | 利用大规模数据和模型扩展提升生成能力 |
| Long horizon | 生成更长时间范围的视频 |
| Physical consistency | 更好保持物体、运动和场景连续 |
| General simulator | 朝通用视觉世界模拟器方向探索 |

面试中不需要编造未公开细节。更稳妥的回答是：Sora 体现了视频生成从短片段、低一致性向长时长、高一致性和世界模拟方向发展，但真实架构、数据和训练细节以公开资料为准。

面试表达：Sora 的意义不只是生成视频更清晰，而是显示了大规模视频生成模型在时空一致性、复杂运动和场景持续性上的潜力。

## 图像生成评估

图像生成评估不能只看“好不好看”。

常见评估维度：

| 维度 | 说明 |
| --- | --- |
| 文本对齐 | 是否符合 prompt 中主体、属性、关系和风格 |
| 视觉质量 | 清晰度、纹理、光照、构图和美感 |
| 多样性 | 同一 prompt 是否能生成不同合理结果 |
| 真实性 | 是否接近真实图像或目标风格 |
| 结构正确 | 手、脸、物体形状和空间关系是否正确 |
| 文字能力 | 图中文字是否正确可读 |
| 安全性 | 是否生成违规、侵权或敏感内容 |
| 可控性 | 是否能稳定遵循参考图、mask、姿态或布局 |

常见自动指标包括 FID、IS、CLIPScore、美学评分和人类偏好模型评分。但自动指标存在局限：FID 不一定反映 prompt 对齐；CLIPScore 可能偏语义而忽略细节；美学评分可能鼓励“漂亮但不准确”的图片。

实际评估建议：建立 prompt suite，覆盖简单主体、复杂组合、空间关系、计数、文字、人物、风格、禁生成边界和业务场景。人工评测时分项打分，不要只给总分。

面试表达：图像生成评估要拆成文本对齐、视觉质量、结构正确性和安全可控性。公开指标只能辅助，不能替代场景化人工评测。

## 视频生成评估

视频生成评估比图像更复杂，因为每帧质量和帧间关系都重要。

常见维度：

| 维度 | 说明 |
| --- | --- |
| Prompt Alignment | 视频内容和动作是否符合文本 |
| Frame Quality | 单帧清晰度和美观度 |
| Temporal Consistency | 主体、背景、光照是否稳定 |
| Motion Naturalness | 运动是否平滑自然 |
| Identity Preservation | 人物和物体是否保持一致 |
| Physical Plausibility | 是否符合基本物理和因果 |
| Long-range Coherence | 长视频事件是否连贯 |
| Safety | 是否生成有害或侵权内容 |

自动评估很难完全可靠。视频可能单帧好看但运动异常，也可能运动合理但 prompt 对齐差。真实项目中通常需要人工评测、偏好比较和任务化评估结合。

面试表达：视频生成评估必须同时看空间质量和时间质量。只抽帧看图是不够的，因为模型最大的问题往往出现在帧间一致性和运动逻辑上。

## 安全与版权

图像和视频生成涉及明显的安全、版权和合规问题。

常见风险：生成虚假新闻图、深度伪造、人脸冒用、版权风格模仿、商标滥用、色情暴力内容、未成年人风险、医疗金融误导、政治敏感内容等。

治理方式：训练数据过滤；prompt 和输出审核；水印和 provenance；人脸与名人保护；版权数据管理；拒答策略；使用政策分级；企业场景保留日志和追溯能力。

面试表达：生成模型落地不能只追求质量，还要考虑滥用风险。尤其是视频生成，真实感越强，伪造和误导风险越高。

## 工程取舍

图像和视频生成落地常见取舍如下：

| 取舍 | 选择 A | 选择 B | 判断标准 |
| --- | --- | --- | --- |
| 模型路线 | 扩散模型 | 自回归生成 | 质量、速度、统一建模需求 |
| 生成空间 | 像素空间 | latent 空间 | 保真度 vs 成本 |
| Backbone | U-Net | Transformer/DiT | 成熟稳定 vs scaling 能力 |
| 采样步数 | 多步 | 少步/蒸馏 | 质量 vs 延迟 |
| 控制方式 | prompt | ControlNet/参考图/布局 | 简单性 vs 精确控制 |
| 个性化 | 全量微调 | LoRA/Adapter | 效果 vs 训练成本 |
| 视频生成 | 逐帧 | 时序建模 | 简单性 vs 一致性 |
| 评估 | 公开指标 | 场景人工评测 | 横向比较 vs 真实可用性 |

常见优化方向：使用更快 scheduler、减少采样步数、模型蒸馏、量化、缓存文本 embedding、批量推理、使用低秩适配、分辨率分阶段生成、先低清后超分、视频关键帧加插帧。

面试表达：生成模型工程中最常见的 trade-off 是质量、速度、成本和可控性。图像任务可以用扩散生态快速落地，视频任务还要额外处理时序一致性和算力成本。

## 高频面试题

### 1. 扩散模型和 GAN 有什么区别？

GAN 通过生成器和判别器对抗训练，生成速度快，但训练可能不稳定，容易模式崩塌。扩散模型通过加噪和去噪学习数据分布，训练更稳定，生成质量高，但通常需要多步采样，推理较慢。

一句话：GAN 是对抗生成，扩散模型是逐步去噪生成。

### 2. Stable Diffusion 为什么要在 latent 空间生成？

因为像素空间维度太高，直接扩散成本大。Stable Diffusion 用 VAE 把图片压缩到 latent 空间，在低维 latent 上去噪，最后再解码成图片。这显著降低训练和推理成本。

代价是 VAE 压缩可能损失细节，尤其是文字、小物体和复杂纹理。

### 3. CFG 的作用是什么？

CFG 用于增强条件遵循。采样时同时计算有条件和无条件预测，并放大二者差异，让生成结果更贴近 prompt。

CFG scale 太低会导致 prompt 遵循弱，太高可能导致过饱和、细节异常和多样性下降。

### 4. U-Net 和 DiT 有什么区别？

U-Net 是卷积和多尺度结构为主的去噪网络，适合图像局部细节建模，Stable Diffusion 早期常用。DiT 是 Diffusion Transformer，把 latent patch 当 token 处理，更符合 Transformer scaling 路线。

一句话：U-Net 成熟稳定，DiT 更适合大规模 Transformer 化扩展。

### 5. 视频生成为什么比图像生成难？

视频不仅要求每帧好看，还要求帧间一致、动作自然、主体身份稳定、物理和镜头逻辑合理。多帧带来更大计算量和更长依赖，训练数据也更难清洗和标注。

### 6. 什么是时序一致性？

时序一致性是指视频中主体、背景、光照、纹理和运动在不同帧之间保持合理连续。常见失败包括闪烁、身份漂移、形状变形和运动抖动。

### 7. Sora 的意义是什么？

Sora 展示了大规模视频生成模型在长时长、高质量、复杂运动和场景一致性上的潜力。它也推动了关于视频生成模型是否具备世界模拟能力的讨论。

稳妥回答是：Sora 代表视频生成向通用视觉世界模拟方向迈进，但不能简单把逼真视频等同于可靠世界模型。

### 8. 如何评估一个文生图模型？

要分项评估文本对齐、视觉质量、多样性、结构正确性、文字能力、安全性和可控性。自动指标可以参考，但必须配合固定 prompt suite 和人工评测。

### 9. 如何评估一个文生视频模型？

除了 prompt 对齐和单帧质量，还要评估时序一致性、运动自然度、身份保持、物理合理性和长程连贯性。只抽帧看图不够，因为视频模型的关键问题常出现在时间维度。

## 本章小结

图像生成和视频生成是生成式 AI 的核心方向。扩散模型通过正向加噪和反向去噪实现高质量生成，Stable Diffusion 用 latent diffusion 降低成本，CFG 增强条件遵循，自回归图像生成则提供了与语言模型统一的 token 建模思路。

视频生成在图像生成基础上增加了时间维度，需要解决时序一致性、运动先验、长程连贯和物理合理性。Sora 等模型展示了大规模视频生成的潜力，也让 world model 成为高频讨论概念。

面试最终表达：图像生成看语义、质量和控制；视频生成还要看时间、运动和世界一致性。工程落地时，质量、速度、成本、安全和版权必须一起考虑。
