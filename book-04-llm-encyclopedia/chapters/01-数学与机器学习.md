# A. 基础数学与机器学习

条目：线性代数、向量、矩阵乘法、内积、范数、投影、特征值、矩阵秩、SVD、低秩近似、PCA、低秩压缩、LoRA、QLoRA、概率论、随机变量、概率分布、条件概率、概率链式法则、期望、方差、信息论、熵、条件熵、互信息、交叉熵、KL 散度、负对数似然、困惑度、优化基础、最大似然估计、最大后验估计、贝叶斯公式、贝叶斯更新、不确定性、校准、ECE、Brier score、选择性预测、强化学习数学、MDP、return、value function、advantage、policy gradient、PPO clipped objective、RLHF KL penalty、reward model loss、DPO loss、梯度下降、SGD、Momentum、Adam、AdamW、学习率调度、梯度裁剪、Hessian、loss spike、反向传播、过拟合、泛化、正则化、Bias-Variance Trade-off。

## 线性代数 Linear Algebra

一句话定义：线性代数是研究向量、矩阵、线性变换、分解和几何关系的数学工具。

在大模型中：embedding 是向量，linear layer 是矩阵乘法，attention 是内积相似度和加权求和，LoRA 是低秩矩阵更新。

面试表达：讲线性代数不要停在定义，要主动连接到 `QK^T` shape、embedding 检索、gradient norm、SVD 和 LoRA 参数量。

## 向量 Vector

一句话定义：向量是一组有方向和大小的数，常用于表示 token、句子、图片或参数梯度。

在大模型中：一个 token hidden state 常写成 `h_i in R^d`，其中 `d` 是 hidden size。

面试表达：向量方向常对应语义或特征组合，向量长度常对应尺度或激活强度；检索中常先归一化再比较方向。

## 内积 Dot Product

一句话定义：内积是两个向量对应元素相乘再求和，用于衡量匹配程度。

核心公式：`x^T y = sum_i x_i y_i`。

在大模型中：attention score 来自 query 和 key 的内积；embedding 检索也常用 dot product 或归一化后的 dot product。

面试表达：dot product 同时受方向和长度影响；如果只想比较方向，要用 cosine similarity 或先做 L2 normalize。

## 向量范数 Vector Norm

一句话定义：范数衡量向量大小，最常见的是 L2 norm。

核心公式：`||x||_2 = sqrt(sum_i x_i^2)`。

在大模型中：gradient norm 用于训练稳定性监控，embedding norm 影响检索相似度，RMSNorm / LayerNorm 也和尺度控制有关。

面试表达：gradient norm 异常升高常提示学习率、数据、loss mask、数值精度或分布式状态需要排查。

## 矩阵乘法 Matrix Multiplication

一句话定义：矩阵乘法把一组向量线性变换到新的空间，关键是中间维度必须匹配。

常见 shape：`X [B, d_in] @ W [d_in, d_out] -> Y [B, d_out]`。

在大模型中：Q/K/V projection、MLP up/down projection、输出词表投影和张量并行都围绕大矩阵乘法展开。

面试表达：Transformer shape 推导的核心是矩阵乘法。`Q [B,H,T,d_h] @ K^T [B,H,d_h,T]` 得到 `[B,H,T,T]`。

## 投影 Projection

一句话定义：投影表示一个向量在某个方向或子空间上的分量。

核心公式：如果 `u` 不是零向量，`proj_u(x) = (x^T u / u^T u)u`。

在大模型中：PCA、表示分析、steering direction、低秩近似和 embedding 可视化都需要投影直觉。

面试表达：投影帮助解释“高维表示里哪些方向最重要”，但不能把任意方向都解释成稳定语义。

## 特征值与特征向量 Eigenvalue and Eigenvector

一句话定义：如果 `Av = lambda v`，则 `v` 是矩阵 `A` 的特征向量，`lambda` 是对应特征值。

直觉：矩阵作用在特征向量方向上时，只改变长度，不改变方向。

在大模型中：特征值直觉常出现在 PCA、优化曲率、Hessian 分析、稳定性和表示空间主方向中。

面试表达：特征值不用背得很抽象，重点说清“哪些方向被放大或压缩”。

## 矩阵秩 Matrix Rank

一句话定义：矩阵秩表示矩阵中线性独立方向的数量。

直觉：如果某些行或列可以由其他行列线性组合得到，它们没有提供新的独立方向，rank 会更低。

在大模型中：rank 直觉用于理解低秩近似、PCA、LoRA rank、模型压缩和表示空间自由度。

## 奇异值分解 Singular Value Decomposition / SVD

一句话定义：SVD 把矩阵分解成 `W = U Sigma V^T`，其中奇异值表示不同方向的重要程度。

关键指标：rank-r 近似保留前 `r` 个最大奇异值；Frobenius 误差由被丢弃奇异值平方和决定，能量保留率可写成 `sum_{i<=r} sigma_i^2 / sum_i sigma_i^2`。

在大模型中：SVD 可用于理解低秩近似、表示压缩、PCA 直觉和 LoRA 的低秩更新假设。

面试表达：SVD 说明矩阵信息可能集中在少数重要方向上，因此可以用 rank-r 近似保留主要结构。

## 低秩近似 Low-Rank Approximation

一句话定义：低秩近似用更少的方向近似原矩阵，减少参数量、存储或计算。

常见形式：`W_r = U_r Sigma_r V_r^T`。

在大模型中：LoRA 用 `Delta W = A B` 约束微调更新位于低秩子空间，参数量从 `d_in * d_out` 降到 `r * (d_in + d_out)`。

面试表达：低秩不是免费午餐。rank 太小表达能力不足，rank 太大参数节省变少，也可能更容易过拟合。

## 主成分分析 Principal Component Analysis / PCA

一句话定义：PCA 寻找数据中方差最大的正交方向，并把数据投影到这些方向上。

核心口径：先中心化数据，再分析协方差矩阵或对中心化数据矩阵做 SVD；explained variance ratio 表示保留主方向解释了多少方差。

在大模型中：PCA 常用于 embedding / hidden state 可视化、表示分析、异常点探索和微调前后表示变化观察。

常见误区：PCA 是线性分析工具，二维可视化不能单独证明模型理解或真实能力。

## 低秩压缩 Low-Rank Compression

一句话定义：用两个较小矩阵近似原始权重矩阵，减少参数量或计算量。

和 LoRA 的区别：低秩压缩近似原始 `W`，可能直接损伤原模型能力；LoRA 通常冻结原始 `W`，只用低秩方式学习增量 `Delta W`。

## LoRA Low-Rank Adaptation

一句话定义：LoRA 冻结原始权重，只训练低秩增量 `Delta W = A B`。

参数量：`r * (d_in + d_out)`，当 `r` 远小于 `d_in` 和 `d_out` 时，可训练参数和优化器状态大幅减少。

面试表达：LoRA 不是让原权重低秩，而是假设下游任务的权重更新可以低秩表示。

## QLoRA Quantized LoRA

一句话定义：QLoRA 把 base model 量化存储，同时训练 LoRA adapter。

在大模型中：它适合显存受限微调，但要额外关注量化误差、计算 dtype、scale 开销、activation 峰值和任务质量回归。

## 概率论 Probability Theory

一句话定义：概率论是描述不确定事件、随机变量和概率分布的数学工具。

在大模型中：下一个 token 是随机变量，模型输出条件概率分布，训练最大化真实 token 概率，解码从分布中选择 token。

面试表达：LLM 的训练、解码、PPL、uncertainty、calibration 和 hallucination 都离不开概率论。概率高表示“在模型分布下更可能”，不等于事实必真。

## 随机变量 Random Variable

一句话定义：随机变量把不确定结果映射成可以计算的数学对象。

在语言模型中：`X_t` 可以表示第 `t` 个位置的 token，取值范围是整个 vocabulary。

面试表达：给定上下文后，模型预测的不是一个固定 token，而是 `X_t` 在词表上的条件概率分布。

## 概率分布 Probability Distribution

一句话定义：概率分布给出每个可能结果发生的概率，离散分布要求概率非负且总和为 1。

在大模型中：logits 经过 softmax 后成为 token 分布；temperature、top-k 和 top-p 都是在这个分布上做解码控制。

面试表达：logits 不是概率。softmax 才把它们变成可采样、可算 NLL 和 entropy 的概率分布。

## 条件概率 Conditional Probability

一句话定义：条件概率描述在已知事件 `B` 后事件 `A` 发生的概率。

核心公式：`P(A | B) = P(A, B) / P(B)`。

在语言模型中：核心对象是 `P(x_t | x_{1:t-1})`，即给定前文预测下一个 token。

面试表达：语言模型不是学 token 的无条件频率，而是学上下文条件下的 token 分布。

## 概率链式法则 Chain Rule of Probability

一句话定义：任意联合概率都可以分解成一系列条件概率的乘积。

核心公式：`P(x_{1:T}) = product_t P(x_t | x_{1:t-1})`。

在大模型中：自回归语言模型正是利用链式法则，把长序列建模拆成逐 token 的 next-token prediction。

面试表达：next-token prediction 不是经验口号，而是概率链式法则给出的序列建模方式。

## 期望 Expectation

一句话定义：期望是随机变量在长期平均意义下的值。

核心公式：离散情况下 `E[X] = sum_x x P(X=x)`。

在大模型中：训练目标可以看作数据分布上的期望损失，mini-batch loss 是对这个期望的估计。

面试表达：batch size 影响梯度噪声，因为 mini-batch 只是对总体期望的近似。

## 方差 Variance

一句话定义：方差衡量随机变量围绕均值波动的程度。

核心公式：`Var(X) = E[X^2] - E[X]^2`。

在大模型中：mini-batch 梯度、随机种子、采样解码、人评和 benchmark 子样本都会带来方差。

面试表达：一次实验提升不一定可靠，要结合样本量、置信区间、随机种子和错误分析判断。

## 信息论 Information Theory

一句话定义：信息论把概率分布和不确定性连接起来，回答“一个事件多意外、一个分布多不确定、两个分布差多少、一个变量能减少另一个变量多少不确定性”。

在大模型中：信息论直接对应 token NLL、cross entropy loss、perplexity、KL penalty、DPO reference model、uncertainty、RAG 证据质量和多模态表示对齐。

面试表达：讲信息论不要只背公式，要主动连接 `-log p(token)`、`PPL=exp(avg NLL)`、`H(p,q)=H(p)+KL(p||q)`、RLHF 的 KL 约束和 RAG 中“证据能否降低答案不确定性”。

## 交叉熵 Cross Entropy

一句话定义：交叉熵衡量用模型分布 `q` 表示真实分布 `p` 时的平均代价。

核心公式：`H(p, q) = -Σ_x p(x) log q(x)`。

在语言模型中：如果真实标签是 one-hot，交叉熵就是 `-log P(真实下一个 token)`。

与 KL 关系：`H(p, q) = H(p) + KL(p || q)`；真实数据分布固定时，最小化交叉熵等价于让模型分布更接近真实分布。

为什么需要它：语言模型训练要让真实 token 的概率更高，交叉熵正好提供这个优化目标。

常见误区：交叉熵低不等于模型回答一定真实、有用或安全。

## 最大似然估计 Maximum Likelihood Estimation / MLE

一句话定义：选择让观测数据概率最大的模型参数。

在语言模型中：最大化训练文本的概率 `P_θ(data)`。

与交叉熵关系：最大化 log likelihood 等价于最小化 negative log likelihood；在 one-hot 标签下就是最小化交叉熵。

## 负对数似然 Negative Log-Likelihood / NLL

一句话定义：对真实数据概率取负对数得到的损失。

核心公式：`NLL = -Σ log P_θ(x_t | x_{1:t-1})`。

为什么使用 log：概率连乘容易下溢，取 log 后把乘法变成加法，数值更稳定。

## KL 散度 KL Divergence

一句话定义：衡量两个概率分布差异的非对称指标。

核心公式：`KL(p || q) = Σ_x p(x) log(p(x) / q(x))`。

与交叉熵关系：`KL(p || q) = H(p, q) - H(p)`。

在大模型中：RLHF 常用 KL penalty 限制 policy 偏离 reference；DPO 虽不显式跑 PPO，也通过 reference model 约束偏好变化。

面试追问：为什么最小化交叉熵等价于最小化 KL？为什么 KL 方向不同会带来不同优化行为？

## 困惑度 Perplexity

一句话定义：语言模型平均交叉熵的指数形式，用来衡量模型对文本的困惑程度。

核心公式：如果 loss 使用自然对数，`perplexity = exp(loss)`。

直觉：perplexity 越低，模型对下一个 token 越确定。

局限：perplexity 受 tokenizer、验证集、截断方式、loss mask 和污染影响，不直接等价于下游任务能力、事实性、安全性或用户满意度。

## 梯度 Gradient

一句话定义：梯度表示函数对参数变化的敏感程度，以及函数值上升最快的方向。

在训练中：梯度告诉我们参数如何变化会让 loss 增大，梯度下降则沿负梯度方向更新参数。

相关概念：learning rate、backpropagation、gradient norm。

## 梯度下降 Gradient Descent

一句话定义：沿着 loss 负梯度方向更新参数的优化方法。

核心公式：`θ_new = θ_old - η * ∇_θ L`。

常见问题：学习率太大可能发散，太小训练缓慢。

## 优化基础 Optimization Basics

一句话定义：优化基础研究如何根据 loss、梯度和优化器状态更新模型参数。

在大模型中：预训练、SFT、DPO 和 reward model 训练都依赖优化；常见稳定性问题包括 learning rate 过大、warmup 太短、grad norm 爆炸、optimizer state 恢复错误和 mixed precision overflow。

面试表达：不要只说“用 AdamW”。要能解释梯度、学习率、global batch、momentum、一阶矩 / 二阶矩、decoupled weight decay、warmup、gradient clipping 和 loss spike 排查链路。

## SGD Stochastic Gradient Descent

一句话定义：SGD 用 mini-batch 梯度近似全量梯度，降低每步计算成本。

在大模型中：即使用 AdamW，梯度本身也来自 mini-batch 估计；global batch size 会影响梯度噪声、吞吐和 learning rate 选择。

面试误区：单卡 micro batch 不等于 global batch；global batch 还要乘以 gradient accumulation steps 和 data parallel worker 数。

## Momentum

一句话定义：Momentum 用历史梯度的移动平均平滑更新方向。

核心直觉：连续同向梯度会累积加速，来回震荡的梯度会被抵消一部分。

在大模型中：Adam 的一阶矩可以看作 momentum 思想的自适应版本。

## Adam

一句话定义：Adam 同时维护梯度一阶矩和二阶矩，用自适应缩放更新参数。

核心机制：一阶矩决定较稳定的方向，二阶矩估计梯度尺度，梯度大的方向步长被缩小。

常见误区：Adam 不是自动免调参。学习率、warmup、weight decay、epsilon 和 batch size 仍会影响稳定性。

## AdamW

一句话定义：AdamW 是把 weight decay 从 Adam 梯度更新中解耦的优化器。

核心区别：Adam + L2 会把衰减项混入梯度并被自适应缩放；AdamW 单独对参数做衰减，更符合 weight decay 的正则化直觉。

在大模型中：AdamW 是预训练和微调常见默认选择，但通常不对 bias、LayerNorm / RMSNorm 参数做 weight decay。

## 学习率调度 Learning Rate Schedule

一句话定义：学习率调度控制不同训练阶段每一步的学习率。

常见组合：linear warmup 后接 cosine decay 或 linear decay。

在大模型中：warmup 缓解早期不稳定，decay 让后期小步收敛；schedule 要和总 token、batch size、optimizer、数据质量一起调。

## 反向传播 Backpropagation

一句话定义：利用链式法则从 loss 反向计算每个参数梯度的方法。

为什么需要它：神经网络是复杂计算图，手动求每个参数梯度不可行，反向传播可以高效自动计算。

## 链式法则 Chain Rule

一句话定义：复合函数求导规则。

核心公式：如果 `y = f(z)` 且 `z = g(w)`，则 `dy/dw = dy/dz * dz/dw`。

在神经网络中：反向传播就是链式法则在计算图上的系统应用。

## 梯度消失 Vanishing Gradient

一句话定义：梯度在反向传播过程中逐层变小，导致底层参数几乎无法更新。

常见缓解：残差连接、归一化、合理初始化。

## 梯度爆炸 Exploding Gradient

一句话定义：梯度在反向传播过程中变得过大，导致参数更新不稳定甚至 NaN。

常见缓解：gradient clipping、learning rate warmup、数值精度检查。

## Gradient Clipping

一句话定义：限制梯度范数，防止梯度爆炸导致训练不稳定的方法。

核心公式：若 global grad norm 超过阈值，就按比例缩小所有梯度，使裁剪后范数不超过阈值。

局限：它能缓解异常大梯度，但不能替代对数据、loss 和学习率问题的根因排查。

## Hessian

一句话定义：Hessian 是 loss 对参数的二阶导数矩阵，描述 loss surface 的曲率。

在大模型中：通常不会显式计算完整 Hessian，但曲率直觉能解释为什么高曲率方向对 learning rate 敏感，为什么过大学习率会震荡。

## Loss Spike

一句话定义：loss spike 是训练过程中 loss 突然异常升高的现象。

优化排查：对齐 spike step 的 learning rate、warmup、grad norm、clipping、AdamW state、mixed precision scaler、异常 batch 和 checkpoint resume 状态。

## 熵 Entropy

一句话定义：熵衡量一个概率分布的不确定性。

核心公式：`H(p) = -Σ_x p(x) log p(x)`。

直觉：如果一个分布非常确定，例如某个事件概率接近 1，熵就低；如果很多事件概率接近均匀，熵就高。

在语言模型中：下一个 token 分布越分散，熵越高；模型越不确定，生成越容易出现多样性，也更容易不稳定。

为什么重要：采样温度、top-k、top-p、困惑度、校准和不确定性分析都和熵有关。

常见误区：熵高不一定坏。开放式创作需要一定熵；事实问答、代码生成和结构化抽取通常希望输出分布更集中。

面试表达：熵描述的是分布本身的不确定性，交叉熵描述的是用一个分布去编码另一个分布时的代价，KL 散度描述两个分布之间的额外代价。

## 条件熵 Conditional Entropy

一句话定义：条件熵衡量在已知变量 `Y` 后，变量 `X` 仍然剩余的不确定性。

核心公式：`H(X | Y) = -Σ_{x,y} p(x, y) log p(x | y)`。

直觉：如果 `Y` 能很好预测 `X`，那么 `H(X | Y)` 就低；如果 `Y` 对 `X` 几乎没有帮助，条件熵就接近 `H(X)`。

在语言模型中：给定上下文 `x_{1:t-1}` 后，下一个 token `x_t` 的不确定性可以理解为一种条件熵。

面试追问：语言模型为什么建模 `P(x_t | x_{1:t-1})`？因为自然语言序列可以按链式法则分解为一系列条件概率。

## 互信息 Mutual Information

一句话定义：互信息衡量两个随机变量之间共享的信息量。

核心公式：`I(X; Y) = H(X) - H(X | Y)`。

等价形式：`I(X; Y) = KL(p(x, y) || p(x)p(y))`。

直觉：如果知道 `Y` 能显著减少对 `X` 的不确定性，互信息就大。

在大模型中：互信息思想常出现在表示学习、对比学习、检索增强、特征选择和数据去重中。

RAG 直觉：一个 chunk 和 query 词面相似不够；它必须真正减少答案的不确定性，才算提供高信息量证据。

常见误区：互信息是对称的，`I(X; Y) = I(Y; X)`；KL 散度一般不是对称的。

## JS 散度 Jensen-Shannon Divergence

一句话定义：JS 散度是 KL 散度的对称、平滑版本，用于衡量两个分布差异。

核心公式：`JS(p || q) = 1/2 KL(p || m) + 1/2 KL(q || m)`，其中 `m = (p + q) / 2`。

特点：相对 KL 更稳定，并且是对称的。

在生成模型中：GAN 原始目标可以和 JS 散度联系起来；分布对齐、数据分布比较时也可能提到它。

和 KL 的区别：KL 关注用 `q` 近似 `p` 的额外代价，方向不同含义不同；JS 同时考虑两边，更适合做对称分布差异描述。

## 最大后验估计 Maximum A Posteriori / MAP

一句话定义：MAP 在观测数据之外引入参数先验，选择后验概率最大的参数。

核心公式：`θ_MAP = argmax_θ p(θ | D) = argmax_θ p(D | θ)p(θ)`。

和 MLE 的区别：MLE 只最大化数据似然 `p(D | θ)`；MAP 还考虑先验 `p(θ)`。

在机器学习中：很多正则化可以从 MAP 角度解释。例如 L2 正则可以看作对参数加入高斯先验，L1 正则可以看作加入拉普拉斯先验。

面试表达：MLE 更依赖数据本身，MAP 通过先验限制参数空间，在数据较少或需要正则化时更稳健。

常见误区：MAP 不是贝叶斯推断的全部。它只取后验最大点，并没有完整保留后验分布的不确定性。

## 贝叶斯公式 Bayes' Theorem

一句话定义：贝叶斯公式描述如何用观测数据更新对假设的信念。

核心公式：`p(θ | D) = p(D | θ)p(θ) / p(D)`。

概念对应：`p(θ)` 是先验，`p(D | θ)` 是似然，`p(θ | D)` 是后验，`p(D)` 是证据。

在大模型中：虽然主流 LLM 训练通常不是完整贝叶斯训练，但贝叶斯思想有助于理解不确定性、校准、先验、正则化和在线更新。

面试追问：为什么后验正比于似然乘先验？因为 `p(D)` 对参数优化来说是常数。

## 先验 Prior

一句话定义：先验是在看到当前证据之前，对假设或参数的已有信念。

在大模型中：模型参数、训练数据分布、系统默认策略和历史评估结果都可以提供某种先验，但先验不能替代当前证据。

面试表达：LLM 的语言先验能生成合理文本，但事实性问题还需要 RAG、工具或引用证据来更新判断。

## 似然 Likelihood

一句话定义：似然衡量在某个假设成立时，观察到当前证据的可能性。

核心口径：Bayes rule 中 `p(E | H)` 越高，说明证据 `E` 越支持假设 `H`。

在大模型中：检索到强证据、工具校验通过、引用支持结论，都可以提高“回答可靠”这个假设的似然。

## 后验 Posterior

一句话定义：后验是结合先验和证据之后，对假设的更新信念。

核心口径：`posterior proportional to likelihood times prior`。

在大模型中：可靠系统不应只看模型原始回答，而应结合证据质量、风险等级和校准结果决定回答、验证、澄清或拒答。

## 贝叶斯更新 Bayesian Update

一句话定义：贝叶斯更新是用新证据把先验变成后验的过程。

常见例子：如果某类任务先验可靠率是 70%，RAG 找到强支持证据并且引用检查通过，后验可靠率应上升；如果检索为空或证据冲突，后验可靠率应下降。

面试表达：Bayesian update 在 LLM 系统里更多是一种决策框架，不要求整个模型训练都是贝叶斯式的。

## 不确定性 Uncertainty

一句话定义：不确定性描述模型或系统对当前判断缺少把握的程度。

常见来源：输入歧义、数据噪声、模型知识不足、分布外问题、证据缺失、工具失败和 judge 偏差。

在大模型中：不确定性管理决定模型什么时候回答、检索、调用工具、澄清、拒答或交给人工。

## Aleatoric Uncertainty

一句话定义：数据本身随机性、噪声或歧义带来的不确定性。

例子：用户问题含糊、标注多解、图片模糊、语音噪声大。

处理方式：澄清问题、表达条件、输出多个可能答案或降低结论强度。

## Epistemic Uncertainty

一句话定义：模型知识不足、数据覆盖不足或分布外输入带来的不确定性。

例子：模型没见过某个新领域、RAG 没召回证据、工具结果缺失、任务超出能力边界。

处理方式：检索、工具验证、人审、拒答或补充训练数据。

## Bias-Variance Trade-off

一句话定义：偏差-方差权衡描述模型误差来源于欠拟合和对训练数据扰动敏感两方面。

偏差：模型假设太简单，无法拟合真实规律。

方差：模型对训练数据变化过于敏感，容易过拟合。

典型现象：小模型可能高偏差、低方差；大模型可能低偏差、高方差，但现代深度学习中还存在双下降等更复杂现象。

平方损失下常见分解：`expected error = bias^2 + variance + noise`。这个公式用于定位误差来源，不表示真实项目能直接精确分解每个 LLM 输出。

在大模型中：模型规模、数据规模、正则化、训练轮数、数据质量都会影响泛化。大模型并不自动泛化好，仍需要数据治理、评估集和对齐过程。

面试表达：偏差-方差权衡不是让模型越小越好，而是提醒我们从模型容量、数据量和正则化三个角度分析泛化问题。

## 过拟合 Overfitting

一句话定义：模型在训练集上表现很好，但在未见数据上表现明显变差。

常见诊断：训练分数高、验证分数低，或训练 loss 下降但验证 loss / 人评 / 线上指标变差。若用 accuracy，可看 `G_score = S_train - S_val`。

常见原因：模型容量过大、训练数据太少、数据噪声大、训练轮数过多、验证集泄漏、评估集与真实分布不一致。

在大模型微调中：过拟合可能表现为训练 loss 持续下降，但验证集 loss 或真实任务指标变差；也可能表现为回答风格僵硬、记忆训练样例、泛化能力下降。

常见缓解：更多高质量数据、early stopping、weight decay、dropout、数据增强、降低学习率、减少训练轮数、混入通用数据。

面试表达：过拟合不能只看训练 loss，要看验证集和真实任务切片；对大模型还要看是否出现能力退化和安全行为变化。

## 欠拟合 Underfitting

一句话定义：模型连训练集都学不好，说明模型能力、特征、训练配置或数据处理存在问题。

常见原因：模型太小、训练不够、学习率不合适、特征不足、数据标签噪声过大、优化器配置错误。

在大模型中：微调欠拟合可能表现为模型仍然像 base model 一样回答，不遵循目标格式，目标任务指标没有提升。

排查方向：先确认数据格式、label mask、loss 是否正确，再看学习率、batch size、训练步数和 LoRA rank 等配置。

和过拟合区别：欠拟合是训练集和验证集都差；过拟合是训练集好、验证集差。

## 泛化 Generalization

一句话定义：泛化指模型在未见样本、未见场景或真实分布上的表现能力。

核心公式：真实风险 `R(f)=E_{(x,y)~P}[ell(f(x),y)]`，经验风险 `R_hat(f)=1/n sum_i ell(f(x_i),y_i)`，泛化问题关心两者之间的差距。

为什么重要：面试和线上系统关心的不是训练集表现，而是模型在新问题上的可靠性。

在大模型中：泛化包括任务泛化、领域泛化、长度泛化、指令泛化、格式泛化和安全泛化。

评估方式：使用独立测试集、时间切分测试集、跨领域测试集、难例集和线上 A/B 测试。

常见误区：benchmark 高分不等于真实业务泛化好。benchmark 可能被污染，也可能和业务分布差异很大。

## 正则化 Regularization

一句话定义：正则化通过限制模型复杂度或训练行为，降低过拟合风险。

核心形式：`L_reg(theta)=R_hat(f_theta)+lambda ||theta||_2^2`。正则项越强，模型越不容易贴合训练噪声，但过强会欠拟合。

常见方法：L1、L2、weight decay、dropout、early stopping、数据增强、label smoothing、噪声注入。

在大模型中：常见正则化包括 weight decay、dropout、数据混合、训练步数控制、KL 约束和 replay 数据。

为什么需要：模型容量越大，越容易记住训练数据中的噪声或偏差，正则化提供约束。

常见误区：正则化不是越强越好。过强正则可能导致欠拟合或目标任务学不进去。

## L1 正则 Lasso Regularization

一句话定义：L1 正则在 loss 中加入参数绝对值之和，鼓励参数稀疏。

核心形式：`L_total = L_task + λ ||θ||_1`。

特点：容易产生稀疏解，即很多参数变成 0。

适用场景：传统机器学习中的特征选择、稀疏线性模型。

在大模型中：直接对全部参数使用 L1 并不常见，但稀疏化、剪枝和结构化压缩中有类似思想。

## L2 正则 Ridge Regularization

一句话定义：L2 正则在 loss 中加入参数平方和，鼓励参数不要过大。

核心形式：`L_total = L_task + λ ||θ||_2^2`。

特点：让参数更平滑、更小，但通常不会精确变成 0。

和 weight decay 关系：在普通 SGD 下，L2 正则和 weight decay 形式接近；在 Adam 这类自适应优化器中，AdamW 的 decoupled weight decay 更常用。

面试高频点：AdamW 相比 Adam + L2 的关键差异是把权重衰减从梯度更新中解耦出来。

## Dropout

一句话定义：Dropout 在训练时随机屏蔽部分神经元或激活，降低模型对特定路径的依赖。

直觉：相当于训练很多共享参数的子网络，提升鲁棒性。

训练和推理区别：训练时随机 dropout；推理时通常关闭 dropout，并使用缩放后的整体网络。

在 Transformer 中：dropout 可能出现在 attention weight、residual branch、MLP 激活等位置。

常见误区：大模型训练中 dropout 不一定总是开启或很大，具体取决于模型规模、数据规模和训练 recipe。

## Early Stopping

一句话定义：当验证集指标不再提升时提前停止训练，避免继续训练导致过拟合。

适用场景：监督微调、小数据训练、传统深度学习任务。

在大模型微调中：如果训练 loss 下降但验证任务指标开始变差，可以考虑 early stopping。

注意事项：验证集必须干净、独立，并且和目标任务分布一致；否则 early stopping 的判断会误导训练。

## 校准 Calibration

一句话定义：校准衡量模型输出概率和真实正确率是否一致。

例子：如果模型对一批样本都给出 80% 置信度，那么其中大约 80% 应该真的正确，这才是校准好的模型。

为什么重要：在安全、医疗、金融、拒答、路由和人工兜底场景中，模型不仅要回答，还要知道自己不确定。

常见方法：temperature scaling、Platt scaling、isotonic regression、置信度阈值、选择性预测。

在 LLM 中的困难：生成概率不等价于事实正确概率；模型可能高置信度胡说，也可能低概率生成正确答案。

面试表达：校准关注的是概率可信度，不是单纯准确率。一个模型可以准确率高但校准差。

## Expected Calibration Error / ECE

一句话定义：ECE 把样本按置信度分桶，衡量每个桶的平均置信度和真实准确率之间的加权差距。

核心形式：`ECE = sum_b |S_b| / n * |acc(S_b) - conf(S_b)|`。

局限：ECE 依赖分桶方式、样本量和置信度定义。LLM 的自然语言置信表达不能直接当概率。

面试表达：ECE 可以发现“模型说 90% 把握，但真实只有 70% 正确”这类过度自信问题。

## Brier Score

一句话定义：Brier score 衡量概率预测和 0/1 真实标签之间的均方误差。

核心形式：`Brier = 1/n sum_i (p_i - y_i)^2`。

用途：常用于二分类概率质量评估，既反映准确性，也反映概率校准。

## 选择性预测 Selective Prediction / Abstention

一句话定义：模型在低置信或高风险时选择不自动回答，而是澄清、检索、验证、拒答或转人工。

在大模型中：高风险医疗、法律、金融、权限操作和安全请求不能只按最高概率 token 自动回答，要结合证据和风险门禁。

面试表达：可靠系统不是永远回答，而是在证据不足或风险过高时知道如何停止、验证或升级。

## 置信度 Confidence

一句话定义：置信度是模型对某个预测的自信程度。

在分类中：通常用最大类别概率作为置信度。

在语言模型中：可以用 token 概率、序列 logprob、熵、self-consistency 一致性、外部 verifier 分数等近似衡量。

局限：LLM 的自然语言“我很确定”不一定可信，生成文本中的自信语气也不等于真实置信度。

工程用法：低置信度时可以拒答、要求用户补充信息、触发检索、调用工具或转人工。

## ROC-AUC

一句话定义：ROC-AUC 衡量二分类模型在不同阈值下区分正负样本的能力。

ROC 曲线：横轴是假阳性率 FPR，纵轴是真阳性率 TPR。

AUC 直觉：随机抽一个正样本和一个负样本，模型给正样本更高分的概率。

适用场景：类别不平衡时比单一 accuracy 更稳定，但仍要结合业务阈值看 precision 和 recall。

在大模型中：可用于 reward model、分类器、风险识别、安全检测器等二分类或打分任务。

## Precision / Recall / F1

一句话定义：Precision 关注预测为正的样本有多少是真的正，Recall 关注真实正样本有多少被找出来，F1 是二者调和平均。

核心公式：`Precision = TP / (TP + FP)`，`Recall = TP / (TP + FN)`，`F1 = 2PR / (P + R)`。

适用场景：分类、抽取、检索、风险识别、工具调用成功检测。

权衡：提高阈值通常提升 precision、降低 recall；降低阈值通常提升 recall、降低 precision。

面试表达：指标选择要看业务代价。安全审核可能更重 recall，自动执行工具可能更重 precision。

## Accuracy

一句话定义：Accuracy 是预测正确样本占总样本的比例。

核心公式：`Accuracy = correct / total`。

优点：简单直观。

局限：类别极不平衡时可能误导。例如 99% 都是负样本，模型全预测负也有 99% accuracy。

在大模型评估中：选择题、分类任务、格式固定任务可以用 accuracy；开放式生成不能只用 accuracy。

## Label Smoothing

一句话定义：Label smoothing 把 one-hot 标签变软，避免模型对训练标签过度自信。

核心思想：真实类别不再是 1，其他类别不再是 0，而是分配一个很小概率。

优点：改善泛化和校准，降低过拟合到噪声标签的风险。

局限：如果任务需要极强置信度或标签完全确定，过强 label smoothing 可能伤害性能。

在语言模型中：部分训练 recipe 可能使用 label smoothing，但主流 causal LM 训练通常重点还是 next-token cross entropy。

## 数据泄漏 Data Leakage

一句话定义：训练过程或特征中包含了测试时不应该知道的信息，导致评估结果虚高。

常见形式：测试集混入训练集、时间穿越、标签泄漏、重复样本、benchmark 污染、prompt 中意外包含答案。

在大模型中：预训练语料可能包含 benchmark；微调数据可能和评估集重复；RAG 检索库可能直接包含标准答案。

危害：模型看似效果很好，但上线后真实表现下降。

排查方法：去重、时间切分、相似度检索、数据来源审计、评估集隔离。

## 数据分布偏移 Distribution Shift

一句话定义：训练数据分布和测试或线上数据分布不一致。

常见诊断：验证集分数高但线上目标分布分数低，可用 `Delta_score = S_val - S_target` 近似衡量性能掉落。

常见类型：领域偏移、时间偏移、用户群体偏移、输入格式偏移、语言风格偏移、难度偏移。

在大模型中：模型在公开 benchmark 上表现好，但在企业内部文档、方言语音、低质量图片或垂直领域术语上表现差，就是分布偏移。

缓解方法：收集目标域数据、持续评估、领域自适应、数据增强、RAG、在线监控。

面试表达：泛化问题通常不是单纯模型问题，也可能是数据分布和评估设计问题。

## 损失函数 Loss Function

一句话定义：损失函数把模型预测和目标之间的差异转化为可优化的标量。

常见类型：交叉熵、MSE、hinge loss、contrastive loss、ranking loss、DPO loss、reward model loss。

在大模型中：预训练常用 next-token cross entropy；偏好优化可能使用 DPO loss、PPO 目标或 reward ranking loss。

为什么重要：损失函数决定了模型被优化成什么样，而评估指标决定了我们真正关心什么。二者不一致时需要特别小心。

## 经验风险最小化 Empirical Risk Minimization / ERM

一句话定义：ERM 在训练集上最小化平均损失，用训练数据近似真实分布风险。

核心形式：`theta_hat = arg min_theta 1/n sum_i L(f_theta(x_i), y_i)`。

直觉：真实数据分布不可完全知道，所以用训练集样本估计期望损失。

局限：如果训练集不代表真实分布，ERM 学到的模型可能泛化不好。

在大模型中：预训练、SFT、分类器训练都可以看作不同形式的经验风险最小化。

## 优化器 Optimizer

一句话定义：优化器根据梯度和内部状态决定如何更新参数。

常见优化器：SGD、Momentum、Adagrad、RMSProp、Adam、AdamW。

在大模型中：AdamW 是非常常见的训练优化器，因为它结合了一阶矩、二阶矩自适应更新和解耦 weight decay。

面试重点：优化器不仅影响收敛速度，也影响稳定性、泛化和显存占用。

## 学习率 Learning Rate

一句话定义：学习率控制每次参数更新的步长。

过大：训练可能发散、loss 震荡、出现 NaN。

过小：训练很慢，可能学不到目标任务。

在大模型中：学习率通常配合 warmup、cosine decay、linear decay 使用；微调学习率通常比从头预训练小得多。

面试表达：学习率是训练最敏感的超参数之一，debug 训练不稳定时应优先检查。

## Warmup

一句话定义：Warmup 在训练初期逐步升高学习率，避免刚开始梯度和优化器状态不稳定导致发散。

为什么需要：大模型训练初期参数、激活、梯度和 Adam 统计量都还不稳定，直接使用峰值学习率可能导致 loss spike。

常见设置：按总步数的 1% 到 5% 做 warmup，具体取决于模型规模、batch size 和训练 recipe。

局限：warmup 只能缓解训练初期不稳定，不能修复错误数据、错误 mask 或过高峰值学习率。

## Batch Size

一句话定义：batch size 是一次参数更新使用的样本数量。

影响：batch 越大，梯度估计通常越稳定，但显存需求更高；batch 太小，梯度噪声更大。

在大模型中：常用 gradient accumulation 实现更大的 global batch size。

相关概念：micro batch size、global batch size、gradient accumulation steps。

公式：`global batch size = micro batch size * data parallel size * gradient accumulation steps`。

面试误区：显卡上设置的 per_device_train_batch_size 不等于全局 batch size。

## 随机种子 Random Seed

一句话定义：随机种子控制随机过程的初始化，使实验更可复现。

影响范围：参数初始化、数据 shuffle、dropout、采样、负样本选择。

在深度学习中：即使设置随机种子，GPU 并行、非确定性算子和分布式训练仍可能带来微小差异。

工程建议：重要实验记录 seed、代码版本、数据版本、依赖版本和硬件环境。

## 超参数 Hyperparameter

一句话定义：超参数是在训练前设定、通常不直接由梯度学习得到的配置。

例子：学习率、batch size、训练轮数、weight decay、dropout、LoRA rank、warmup ratio、max sequence length。

为什么重要：相同模型和数据，不同超参数可能得到完全不同的结果。

面试表达：调参不能只靠玄学，应结合 loss 曲线、验证集指标、bad case 和资源约束进行系统分析。

## 训练集 / 验证集 / 测试集

一句话定义：训练集用于学习参数，验证集用于调参与选择模型，测试集用于最终无偏评估。

常见错误：用测试集反复调 prompt 或调超参，会让测试集变成验证集，导致最终结果偏乐观。

在大模型项目中：还可以额外保留 hidden test、时间外测试集、困难样本集和线上灰度指标。

面试表达：数据切分是评估可信度的基础，不能只看模型分数，还要看分数从哪里来。

## 泛化审计 Generalization Audit

一句话定义：泛化审计是同时检查 train / validation / test gap、分布偏移、污染重叠、关键切片和 checkpoint 选择的评估流程。

核心检查：训练分数不能单独汇报；验证集用于调参与 early stopping；最终测试集不能反复调 prompt 或调超参；线上切片要覆盖目标用户、目标领域和长尾输入。

常见指标：`G_score = S_train - S_val`、`Delta_score = S_val - S_target`、exact overlap rate、worst-slice score、regression count。

在大模型中：SFT、DPO、RAG prompt 和 benchmark 迭代都可能过拟合固定评测集，所以泛化审计要包含改写、组合、分布外、时间切分和污染检测。

面试表达：我不会只说“验证集涨了”。我会说明数据切分、是否污染、关键切片、线上分布差异、early stopping 依据和残余风险。

## 强化学习数学 Reinforcement Learning Math

一句话定义：强化学习数学研究 policy 如何在状态中选择动作，并根据 reward 优化长期回报。

在大模型中：RLHF 把语言模型看成 policy，把生成 token 看成 action，把完整回答看成 trajectory，把 reward model 分数看成偏好代理奖励。

面试表达：讲 RLHF 不能只背流程，要能解释 policy、reward、value、advantage、PPO、KL 和 reward hacking 各自解决什么问题。

## Markov Decision Process / MDP

一句话定义：MDP 用状态、动作、转移、奖励和折扣因子描述序贯决策问题。

核心形式：`M = (S, A, P, r, gamma)`。

LLM 映射：状态是 prompt 加已生成 token，动作是下一个 token，转移是追加 token，奖励通常来自完整回答后的 reward model。

## Return

一句话定义：return 是从当前时刻开始的累计折扣奖励。

核心形式：`G_t = sum_k gamma^k r_{t+k}`。

注意：return 不是当前 reward，它包含未来奖励。RLHF 中 reward 往往在完整回答后才出现，所以 token 级 credit assignment 更难。

## Value Function

一句话定义：value function 估计某个状态未来能获得的期望 return。

核心形式：`V^pi(s) = E_pi[G_t | s_t=s]`。

在 RLHF 中：value model 提供 baseline，帮助计算 advantage，降低 policy gradient 方差。

## Q Function

一句话定义：Q function 估计在某个状态采取某个动作后未来能获得的期望 return。

核心形式：`Q^pi(s,a) = E_pi[G_t | s_t=s, a_t=a]`。

与 value 的关系：value 是按 policy 对 action value 的加权平均。

## Advantage

一句话定义：advantage 衡量某个动作比当前状态的平均预期好多少。

核心形式：`A^pi(s,a) = Q^pi(s,a) - V^pi(s)`。

在 PPO 中：正 advantage 的动作应提高概率，负 advantage 的动作应降低概率。

## Policy Gradient

一句话定义：policy gradient 直接沿着提高期望 reward 的方向更新 policy 参数。

核心直觉：`grad J` 与 `grad log pi(a|s) * advantage` 成正比。

面试表达：policy gradient 不是直接回传 reward 到每个 token，而是用 log probability trick 和 advantage 估计更新方向。

## PPO Clipped Objective

一句话定义：PPO 用新旧 policy 概率比限制一次更新幅度，避免 policy 变化过大。

核心对象：`rho = pi_new(a|s) / pi_old(a|s)`，clip 范围通常围绕 `1 +/- epsilon`。

在 RLHF 中：PPO 用 reward model 信号更新 policy，同时配合 KL penalty 和 value baseline 提高稳定性。

## RLHF KL Penalty

一句话定义：KL penalty 限制 RLHF policy 不要偏离 reference model 太远。

核心形式：`reward - beta * KL(policy || reference)`。

面试表达：KL 约束能降低 reward hacking 和语言退化风险，但不能保证每个输出都真实、安全或最优。

## Reward Model Pairwise Loss

一句话定义：reward model 用 chosen / rejected 偏好对学习哪个回答更好。

核心形式：`L_RM = -log sigmoid(r_chosen - r_rejected)`。

局限：reward model 学的是偏好 proxy，不是真实人类价值本身，过度优化会带来 reward hacking。

## DPO Loss

一句话定义：DPO 直接用 preference pair 优化 policy，使 policy 相对 reference 更偏好 chosen 而不是 rejected。

核心口径：DPO 比较 policy 和 reference 的 chosen / rejected log probability gap，并用 `beta` 控制偏离强度。

面试表达：DPO 省掉显式 reward model 和 PPO loop，但不等于没有约束；reference model 和 beta 仍然提供分布锚点。

## Evaluation Statistics

一句话定义：评估统计把有限评估集上的分数看成随机估计，用均值、方差、标准误、置信区间和显著性检验判断提升是否可信。

在 LLM 中：accuracy、win rate、judge score、human rubric、latency、cost 和 safety violation 都应该报告样本量和不确定性，而不是只报告单点平均值。

面试表达：统计显著不等于值得上线，还要结合 effect size、成本、延迟、安全、分层结果和失败样本。

## Standard Error

一句话定义：标准误衡量样本均值本身的不确定性。

核心形式：`SE(mean)=s/sqrt(n)`。

在评估中：样本量越小、样本方差越大，标准误越大，置信区间也越宽。

## Confidence Interval

一句话定义：置信区间给出效果大小的合理不确定范围。

常见形式：`mean +/- 1.96 * SE(mean)`。

面试表达：如果 lift 的置信区间跨过 0，就不能只因为点估计为正而断言新模型更好。

## p-value

一句话定义：p-value 是在零假设成立时，观察到当前或更极端结果的概率。

常见误区：它不是“新模型无效的概率”，也不是“新模型更好的概率”。

在项目中：p-value 必须和预注册主指标、样本量、effect size、置信区间、多重比较和护栏指标一起解释。

## Bootstrap

一句话定义：bootstrap 通过有放回重采样估计指标分布和置信区间。

在 LLM 评估中：适合 pairwise win rate、LLM judge score、RAG faithfulness、多维人工评分等难写解析方差的复杂指标。

边界：bootstrap 不能修复样本偏差；样本本身不代表真实用户分布时，置信区间也会给出偏的结论。

## Paired Evaluation

一句话定义：paired evaluation 让两个模型在同一批样本上比较，直接统计每条样本上的差异。

为什么需要：它控制了样本难度差异，比两个模型分别在不同样本集合上比较更稳。

在 LLM 中：pairwise human eval、同题 accuracy 对比、同 prompt judge 对比和 RAG ablation 都优先采用 paired 设计。

## McNemar Test

一句话定义：McNemar test 用于比较两个模型在同一批二分类样本上的正确 / 错误差异。

核心对象：`b` 是旧错新对，`c` 是旧对新错；两个模型都对或都错的样本不参与差异检验。

面试表达：当二分类结果是 paired 时，不应只比较两个 accuracy 的独立比例检验；McNemar 更关注同一题上的反转。

## Sample Size

一句话定义：样本量决定评估能否检测到你关心的最小效果。

关键概念：minimum detectable effect、显著性水平、统计功效、基线比例、方差和分层流量。

在 A/B test 中：样本量应在实验前估算，不能看到趋势后临时决定停止。

## Statistical Power

一句话定义：统计功效是在真实存在某个效果时，实验能检测到它的概率。

常见风险：样本量太小会导致“没显著”不等于“没效果”，只是实验不够有力。

## Multiple Comparisons

一句话定义：同时尝试很多模型、prompt、指标、分层或中途查看，会增加偶然显著的概率。

缓解方法：预注册主指标、保留独立 holdout、区分探索性和确认性分析，并对多重检验做校正。

## Bonferroni Correction

一句话定义：Bonferroni 用更严格的 `alpha / m` 阈值控制多重比较下的 family-wise error rate。

特点：简单、保守，适合确认性实验中的少量关键假设。

## Benjamini-Hochberg

一句话定义：Benjamini-Hochberg 通过排序 p-value 控制 false discovery rate。

特点：通常比 Bonferroni 不那么保守，适合探索性分析中的多个候选发现，但探索性结果仍需要独立验证。

## Experiment Gate

一句话定义：experiment gate 把主指标、置信区间、显著性、样本量、护栏指标、污染检查和多重比较规则合成上线门禁。

在 LLM 中：即使主指标提升，也要同时检查 latency、cost、safety regression、over-refusal、关键用户分层和 bad case 类型。

## Math Interview Readiness

一句话定义：数学面试准备度衡量候选人是否能把公式、直觉、LLM 场景、边界条件和最小代码同时讲清。

核心维度：topic coverage、formula accuracy、intuition clarity、LLM connection、caveat coverage、demo coverage。

面试表达：数学题不是只背公式；好的回答要能解释公式变量、适用条件、工程场景和常见误区。

## Math Interview Rubric

一句话定义：数学面试 rubric 是对回答质量的结构化评分规则。

常见评分项：公式是否正确，直觉是否清楚，是否连接到 pretraining / SFT / RLHF / RAG / evaluation，是否说明边界，是否能写出最小 demo。

用途：复盘时把“我好像懂”变成可量化的缺口，例如 Bayes/MAP 手算薄弱、DPO/PPO 边界不清、泛化题缺少代码例子。

## Formula Coverage

一句话定义：formula coverage 检查一轮复习是否覆盖核心公式族，而不是只会少数高频题。

数学基础中的核心族：线性代数、概率、信息论、优化、低秩、泛化、贝叶斯、RLHF 数学和评估统计。

## Formula Audit

一句话定义：formula audit 用手算或最小代码验证公式是否能真的算出数值。

例子：验证 `H(p,q)=H(p)+KL(p||q)`、softmax cross entropy 梯度 `p-y`、LoRA 参数量、DPO margin、paired eval lift 和 bootstrap CI。

## Mock Interview Retrospective

一句话定义：模拟面试复盘把每道回答拆成得分、薄弱点、缺失主题和下一步修订计划。

在本书系中：它用于把第十三册数学公式、代码 demo 和面试表达连成闭环，避免只看章节不练输出。
