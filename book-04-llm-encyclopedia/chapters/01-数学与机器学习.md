# A. 基础数学与机器学习

条目：熵、交叉熵、KL 散度、最大似然估计、最大后验估计、梯度下降、反向传播、过拟合、泛化、正则化、Bias-Variance Trade-off、校准。

## 交叉熵 Cross Entropy

一句话定义：交叉熵衡量用模型分布 `q` 表示真实分布 `p` 时的平均代价。

核心公式：`H(p, q) = -Σ_x p(x) log q(x)`。

在语言模型中：如果真实标签是 one-hot，交叉熵就是 `-log P(真实下一个 token)`。

为什么需要它：语言模型训练要让真实 token 的概率更高，交叉熵正好提供这个优化目标。

常见误区：交叉熵低不等于模型回答一定真实、有用或安全。

## 最大似然估计 Maximum Likelihood Estimation / MLE

一句话定义：选择让观测数据概率最大的模型参数。

在语言模型中：最大化训练文本的概率 `P_θ(data)`。

与交叉熵关系：最大化 log likelihood 等价于最小化 negative log likelihood；在 one-hot 标签下就是最小化交叉熵。

## 负对数似然 Negative Log-Likelihood / NLL

一句话定义：对真实数据概率取负对数得到的损失。

核心公式：`NLL = -Σ log P_θ(x_t | x_{1:t-1})`。

为什么使用 log：概率连乘容易下溢，取 log 后把乘法变成加法，数值更稳定。

## KL 散度 KL Divergence

一句话定义：衡量两个概率分布差异的非对称指标。

核心公式：`KL(p || q) = Σ_x p(x) log(p(x) / q(x))`。

与交叉熵关系：`KL(p || q) = H(p, q) - H(p)`。

面试追问：为什么最小化交叉熵等价于最小化 KL？

## 困惑度 Perplexity

一句话定义：语言模型平均交叉熵的指数形式，用来衡量模型对文本的困惑程度。

核心公式：如果 loss 使用自然对数，`perplexity = exp(loss)`。

直觉：perplexity 越低，模型对下一个 token 越确定。

局限：perplexity 不直接等价于下游任务能力、事实性、安全性或用户满意度。

## 梯度 Gradient

一句话定义：梯度表示函数对参数变化的敏感程度，以及函数值上升最快的方向。

在训练中：梯度告诉我们参数如何变化会让 loss 增大，梯度下降则沿负梯度方向更新参数。

相关概念：learning rate、backpropagation、gradient norm。

## 梯度下降 Gradient Descent

一句话定义：沿着 loss 负梯度方向更新参数的优化方法。

核心公式：`θ_new = θ_old - η * ∇_θ L`。

常见问题：学习率太大可能发散，太小训练缓慢。

## 反向传播 Backpropagation

一句话定义：利用链式法则从 loss 反向计算每个参数梯度的方法。

为什么需要它：神经网络是复杂计算图，手动求每个参数梯度不可行，反向传播可以高效自动计算。

## 链式法则 Chain Rule

一句话定义：复合函数求导规则。

核心公式：如果 `y = f(z)` 且 `z = g(w)`，则 `dy/dw = dy/dz * dz/dw`。

在神经网络中：反向传播就是链式法则在计算图上的系统应用。

## 梯度消失 Vanishing Gradient

一句话定义：梯度在反向传播过程中逐层变小，导致底层参数几乎无法更新。

常见缓解：残差连接、归一化、合理初始化。

## 梯度爆炸 Exploding Gradient

一句话定义：梯度在反向传播过程中变得过大，导致参数更新不稳定甚至 NaN。

常见缓解：gradient clipping、learning rate warmup、数值精度检查。

## Gradient Clipping

一句话定义：限制梯度范数，防止梯度爆炸导致训练不稳定的方法。

局限：它能缓解异常大梯度，但不能替代对数据、loss 和学习率问题的根因排查。

## 熵 Entropy

一句话定义：熵衡量一个概率分布的不确定性。

核心公式：`H(p) = -Σ_x p(x) log p(x)`。

直觉：如果一个分布非常确定，例如某个事件概率接近 1，熵就低；如果很多事件概率接近均匀，熵就高。

在语言模型中：下一个 token 分布越分散，熵越高；模型越不确定，生成越容易出现多样性，也更容易不稳定。

为什么重要：采样温度、top-k、top-p、困惑度、校准和不确定性分析都和熵有关。

常见误区：熵高不一定坏。开放式创作需要一定熵；事实问答、代码生成和结构化抽取通常希望输出分布更集中。

面试表达：熵描述的是分布本身的不确定性，交叉熵描述的是用一个分布去编码另一个分布时的代价，KL 散度描述两个分布之间的额外代价。

## 条件熵 Conditional Entropy

一句话定义：条件熵衡量在已知变量 `Y` 后，变量 `X` 仍然剩余的不确定性。

核心公式：`H(X | Y) = -Σ_{x,y} p(x, y) log p(x | y)`。

直觉：如果 `Y` 能很好预测 `X`，那么 `H(X | Y)` 就低；如果 `Y` 对 `X` 几乎没有帮助，条件熵就接近 `H(X)`。

在语言模型中：给定上下文 `x_{1:t-1}` 后，下一个 token `x_t` 的不确定性可以理解为一种条件熵。

面试追问：语言模型为什么建模 `P(x_t | x_{1:t-1})`？因为自然语言序列可以按链式法则分解为一系列条件概率。

## 互信息 Mutual Information

一句话定义：互信息衡量两个随机变量之间共享的信息量。

核心公式：`I(X; Y) = H(X) - H(X | Y)`。

等价形式：`I(X; Y) = KL(p(x, y) || p(x)p(y))`。

直觉：如果知道 `Y` 能显著减少对 `X` 的不确定性，互信息就大。

在大模型中：互信息思想常出现在表示学习、对比学习、检索增强、特征选择和数据去重中。

常见误区：互信息是对称的，`I(X; Y) = I(Y; X)`；KL 散度一般不是对称的。

## JS 散度 Jensen-Shannon Divergence

一句话定义：JS 散度是 KL 散度的对称、平滑版本，用于衡量两个分布差异。

核心公式：`JS(p || q) = 1/2 KL(p || m) + 1/2 KL(q || m)`，其中 `m = (p + q) / 2`。

特点：相对 KL 更稳定，并且是对称的。

在生成模型中：GAN 原始目标可以和 JS 散度联系起来；分布对齐、数据分布比较时也可能提到它。

和 KL 的区别：KL 关注用 `q` 近似 `p` 的额外代价，方向不同含义不同；JS 同时考虑两边，更适合做对称分布差异描述。

## 最大后验估计 Maximum A Posteriori / MAP

一句话定义：MAP 在观测数据之外引入参数先验，选择后验概率最大的参数。

核心公式：`θ_MAP = argmax_θ p(θ | D) = argmax_θ p(D | θ)p(θ)`。

和 MLE 的区别：MLE 只最大化数据似然 `p(D | θ)`；MAP 还考虑先验 `p(θ)`。

在机器学习中：很多正则化可以从 MAP 角度解释。例如 L2 正则可以看作对参数加入高斯先验，L1 正则可以看作加入拉普拉斯先验。

面试表达：MLE 更依赖数据本身，MAP 通过先验限制参数空间，在数据较少或需要正则化时更稳健。

常见误区：MAP 不是贝叶斯推断的全部。它只取后验最大点，并没有完整保留后验分布的不确定性。

## 贝叶斯公式 Bayes' Theorem

一句话定义：贝叶斯公式描述如何用观测数据更新对假设的信念。

核心公式：`p(θ | D) = p(D | θ)p(θ) / p(D)`。

概念对应：`p(θ)` 是先验，`p(D | θ)` 是似然，`p(θ | D)` 是后验，`p(D)` 是证据。

在大模型中：虽然主流 LLM 训练通常不是完整贝叶斯训练，但贝叶斯思想有助于理解不确定性、校准、先验、正则化和在线更新。

面试追问：为什么后验正比于似然乘先验？因为 `p(D)` 对参数优化来说是常数。

## Bias-Variance Trade-off

一句话定义：偏差-方差权衡描述模型误差来源于欠拟合和对训练数据扰动敏感两方面。

偏差：模型假设太简单，无法拟合真实规律。

方差：模型对训练数据变化过于敏感，容易过拟合。

典型现象：小模型可能高偏差、低方差；大模型可能低偏差、高方差，但现代深度学习中还存在双下降等更复杂现象。

在大模型中：模型规模、数据规模、正则化、训练轮数、数据质量都会影响泛化。大模型并不自动泛化好，仍需要数据治理、评估集和对齐过程。

面试表达：偏差-方差权衡不是让模型越小越好，而是提醒我们从模型容量、数据量和正则化三个角度分析泛化问题。

## 过拟合 Overfitting

一句话定义：模型在训练集上表现很好，但在未见数据上表现明显变差。

常见原因：模型容量过大、训练数据太少、数据噪声大、训练轮数过多、验证集泄漏、评估集与真实分布不一致。

在大模型微调中：过拟合可能表现为训练 loss 持续下降，但验证集 loss 或真实任务指标变差；也可能表现为回答风格僵硬、记忆训练样例、泛化能力下降。

常见缓解：更多高质量数据、early stopping、weight decay、dropout、数据增强、降低学习率、减少训练轮数、混入通用数据。

面试表达：过拟合不能只看训练 loss，要看验证集和真实任务切片；对大模型还要看是否出现能力退化和安全行为变化。

## 欠拟合 Underfitting

一句话定义：模型连训练集都学不好，说明模型能力、特征、训练配置或数据处理存在问题。

常见原因：模型太小、训练不够、学习率不合适、特征不足、数据标签噪声过大、优化器配置错误。

在大模型中：微调欠拟合可能表现为模型仍然像 base model 一样回答，不遵循目标格式，目标任务指标没有提升。

排查方向：先确认数据格式、label mask、loss 是否正确，再看学习率、batch size、训练步数和 LoRA rank 等配置。

和过拟合区别：欠拟合是训练集和验证集都差；过拟合是训练集好、验证集差。

## 泛化 Generalization

一句话定义：泛化指模型在未见样本、未见场景或真实分布上的表现能力。

为什么重要：面试和线上系统关心的不是训练集表现，而是模型在新问题上的可靠性。

在大模型中：泛化包括任务泛化、领域泛化、长度泛化、指令泛化、格式泛化和安全泛化。

评估方式：使用独立测试集、时间切分测试集、跨领域测试集、难例集和线上 A/B 测试。

常见误区：benchmark 高分不等于真实业务泛化好。benchmark 可能被污染，也可能和业务分布差异很大。

## 正则化 Regularization

一句话定义：正则化通过限制模型复杂度或训练行为，降低过拟合风险。

常见方法：L1、L2、weight decay、dropout、early stopping、数据增强、label smoothing、噪声注入。

在大模型中：常见正则化包括 weight decay、dropout、数据混合、训练步数控制、KL 约束和 replay 数据。

为什么需要：模型容量越大，越容易记住训练数据中的噪声或偏差，正则化提供约束。

常见误区：正则化不是越强越好。过强正则可能导致欠拟合或目标任务学不进去。

## L1 正则 Lasso Regularization

一句话定义：L1 正则在 loss 中加入参数绝对值之和，鼓励参数稀疏。

核心形式：`L_total = L_task + λ ||θ||_1`。

特点：容易产生稀疏解，即很多参数变成 0。

适用场景：传统机器学习中的特征选择、稀疏线性模型。

在大模型中：直接对全部参数使用 L1 并不常见，但稀疏化、剪枝和结构化压缩中有类似思想。

## L2 正则 Ridge Regularization

一句话定义：L2 正则在 loss 中加入参数平方和，鼓励参数不要过大。

核心形式：`L_total = L_task + λ ||θ||_2^2`。

特点：让参数更平滑、更小，但通常不会精确变成 0。

和 weight decay 关系：在普通 SGD 下，L2 正则和 weight decay 形式接近；在 Adam 这类自适应优化器中，AdamW 的 decoupled weight decay 更常用。

面试高频点：AdamW 相比 Adam + L2 的关键差异是把权重衰减从梯度更新中解耦出来。

## Dropout

一句话定义：Dropout 在训练时随机屏蔽部分神经元或激活，降低模型对特定路径的依赖。

直觉：相当于训练很多共享参数的子网络，提升鲁棒性。

训练和推理区别：训练时随机 dropout；推理时通常关闭 dropout，并使用缩放后的整体网络。

在 Transformer 中：dropout 可能出现在 attention weight、residual branch、MLP 激活等位置。

常见误区：大模型训练中 dropout 不一定总是开启或很大，具体取决于模型规模、数据规模和训练 recipe。

## Early Stopping

一句话定义：当验证集指标不再提升时提前停止训练，避免继续训练导致过拟合。

适用场景：监督微调、小数据训练、传统深度学习任务。

在大模型微调中：如果训练 loss 下降但验证任务指标开始变差，可以考虑 early stopping。

注意事项：验证集必须干净、独立，并且和目标任务分布一致；否则 early stopping 的判断会误导训练。

## 校准 Calibration

一句话定义：校准衡量模型输出概率和真实正确率是否一致。

例子：如果模型对一批样本都给出 80% 置信度，那么其中大约 80% 应该真的正确，这才是校准好的模型。

为什么重要：在安全、医疗、金融、拒答、路由和人工兜底场景中，模型不仅要回答，还要知道自己不确定。

常见方法：temperature scaling、Platt scaling、isotonic regression、置信度阈值、选择性预测。

在 LLM 中的困难：生成概率不等价于事实正确概率；模型可能高置信度胡说，也可能低概率生成正确答案。

面试表达：校准关注的是概率可信度，不是单纯准确率。一个模型可以准确率高但校准差。

## 置信度 Confidence

一句话定义：置信度是模型对某个预测的自信程度。

在分类中：通常用最大类别概率作为置信度。

在语言模型中：可以用 token 概率、序列 logprob、熵、self-consistency 一致性、外部 verifier 分数等近似衡量。

局限：LLM 的自然语言“我很确定”不一定可信，生成文本中的自信语气也不等于真实置信度。

工程用法：低置信度时可以拒答、要求用户补充信息、触发检索、调用工具或转人工。

## ROC-AUC

一句话定义：ROC-AUC 衡量二分类模型在不同阈值下区分正负样本的能力。

ROC 曲线：横轴是假阳性率 FPR，纵轴是真阳性率 TPR。

AUC 直觉：随机抽一个正样本和一个负样本，模型给正样本更高分的概率。

适用场景：类别不平衡时比单一 accuracy 更稳定，但仍要结合业务阈值看 precision 和 recall。

在大模型中：可用于 reward model、分类器、风险识别、安全检测器等二分类或打分任务。

## Precision / Recall / F1

一句话定义：Precision 关注预测为正的样本有多少是真的正，Recall 关注真实正样本有多少被找出来，F1 是二者调和平均。

核心公式：`Precision = TP / (TP + FP)`，`Recall = TP / (TP + FN)`，`F1 = 2PR / (P + R)`。

适用场景：分类、抽取、检索、风险识别、工具调用成功检测。

权衡：提高阈值通常提升 precision、降低 recall；降低阈值通常提升 recall、降低 precision。

面试表达：指标选择要看业务代价。安全审核可能更重 recall，自动执行工具可能更重 precision。

## Accuracy

一句话定义：Accuracy 是预测正确样本占总样本的比例。

核心公式：`Accuracy = correct / total`。

优点：简单直观。

局限：类别极不平衡时可能误导。例如 99% 都是负样本，模型全预测负也有 99% accuracy。

在大模型评估中：选择题、分类任务、格式固定任务可以用 accuracy；开放式生成不能只用 accuracy。

## Label Smoothing

一句话定义：Label smoothing 把 one-hot 标签变软，避免模型对训练标签过度自信。

核心思想：真实类别不再是 1，其他类别不再是 0，而是分配一个很小概率。

优点：改善泛化和校准，降低过拟合到噪声标签的风险。

局限：如果任务需要极强置信度或标签完全确定，过强 label smoothing 可能伤害性能。

在语言模型中：部分训练 recipe 可能使用 label smoothing，但主流 causal LM 训练通常重点还是 next-token cross entropy。

## 数据泄漏 Data Leakage

一句话定义：训练过程或特征中包含了测试时不应该知道的信息，导致评估结果虚高。

常见形式：测试集混入训练集、时间穿越、标签泄漏、重复样本、benchmark 污染、prompt 中意外包含答案。

在大模型中：预训练语料可能包含 benchmark；微调数据可能和评估集重复；RAG 检索库可能直接包含标准答案。

危害：模型看似效果很好，但上线后真实表现下降。

排查方法：去重、时间切分、相似度检索、数据来源审计、评估集隔离。

## 数据分布偏移 Distribution Shift

一句话定义：训练数据分布和测试或线上数据分布不一致。

常见类型：领域偏移、时间偏移、用户群体偏移、输入格式偏移、语言风格偏移、难度偏移。

在大模型中：模型在公开 benchmark 上表现好，但在企业内部文档、方言语音、低质量图片或垂直领域术语上表现差，就是分布偏移。

缓解方法：收集目标域数据、持续评估、领域自适应、数据增强、RAG、在线监控。

面试表达：泛化问题通常不是单纯模型问题，也可能是数据分布和评估设计问题。

## 损失函数 Loss Function

一句话定义：损失函数把模型预测和目标之间的差异转化为可优化的标量。

常见类型：交叉熵、MSE、hinge loss、contrastive loss、ranking loss、DPO loss、reward model loss。

在大模型中：预训练常用 next-token cross entropy；偏好优化可能使用 DPO loss、PPO 目标或 reward ranking loss。

为什么重要：损失函数决定了模型被优化成什么样，而评估指标决定了我们真正关心什么。二者不一致时需要特别小心。

## 经验风险最小化 Empirical Risk Minimization / ERM

一句话定义：ERM 在训练集上最小化平均损失，用训练数据近似真实分布风险。

核心形式：`min_θ 1/n Σ_i L(f_θ(x_i), y_i)`。

直觉：真实数据分布不可完全知道，所以用训练集样本估计期望损失。

局限：如果训练集不代表真实分布，ERM 学到的模型可能泛化不好。

在大模型中：预训练、SFT、分类器训练都可以看作不同形式的经验风险最小化。

## 优化器 Optimizer

一句话定义：优化器根据梯度和内部状态决定如何更新参数。

常见优化器：SGD、Momentum、Adagrad、RMSProp、Adam、AdamW。

在大模型中：AdamW 是非常常见的训练优化器，因为它结合了一阶矩、二阶矩自适应更新和解耦 weight decay。

面试重点：优化器不仅影响收敛速度，也影响稳定性、泛化和显存占用。

## 学习率 Learning Rate

一句话定义：学习率控制每次参数更新的步长。

过大：训练可能发散、loss 震荡、出现 NaN。

过小：训练很慢，可能学不到目标任务。

在大模型中：学习率通常配合 warmup、cosine decay、linear decay 使用；微调学习率通常比从头预训练小得多。

面试表达：学习率是训练最敏感的超参数之一，debug 训练不稳定时应优先检查。

## Warmup

一句话定义：Warmup 在训练初期逐步升高学习率，避免刚开始梯度和优化器状态不稳定导致发散。

为什么需要：大模型训练初期参数、激活、梯度和 Adam 统计量都还不稳定，直接使用峰值学习率可能导致 loss spike。

常见设置：按总步数的 1% 到 5% 做 warmup，具体取决于模型规模、batch size 和训练 recipe。

局限：warmup 只能缓解训练初期不稳定，不能修复错误数据、错误 mask 或过高峰值学习率。

## Batch Size

一句话定义：batch size 是一次参数更新使用的样本数量。

影响：batch 越大，梯度估计通常越稳定，但显存需求更高；batch 太小，梯度噪声更大。

在大模型中：常用 gradient accumulation 实现更大的 global batch size。

相关概念：micro batch size、global batch size、gradient accumulation steps。

公式：`global batch size = micro batch size * data parallel size * gradient accumulation steps`。

面试误区：显卡上设置的 per_device_train_batch_size 不等于全局 batch size。

## 随机种子 Random Seed

一句话定义：随机种子控制随机过程的初始化，使实验更可复现。

影响范围：参数初始化、数据 shuffle、dropout、采样、负样本选择。

在深度学习中：即使设置随机种子，GPU 并行、非确定性算子和分布式训练仍可能带来微小差异。

工程建议：重要实验记录 seed、代码版本、数据版本、依赖版本和硬件环境。

## 超参数 Hyperparameter

一句话定义：超参数是在训练前设定、通常不直接由梯度学习得到的配置。

例子：学习率、batch size、训练轮数、weight decay、dropout、LoRA rank、warmup ratio、max sequence length。

为什么重要：相同模型和数据，不同超参数可能得到完全不同的结果。

面试表达：调参不能只靠玄学，应结合 loss 曲线、验证集指标、bad case 和资源约束进行系统分析。

## 训练集 / 验证集 / 测试集

一句话定义：训练集用于学习参数，验证集用于调参与选择模型，测试集用于最终无偏评估。

常见错误：用测试集反复调 prompt 或调超参，会让测试集变成验证集，导致最终结果偏乐观。

在大模型项目中：还可以额外保留 hidden test、时间外测试集、困难样本集和线上灰度指标。

面试表达：数据切分是评估可信度的基础，不能只看模型分数，还要看分数从哪里来。
