# 第二十册：Agent Harness、Coding Agent Runtime 与智能体工程框架

## 本书定位

本书专门讲 Harness。这里的 Harness 不只是 evaluation harness，而是更广义的大模型智能体运行框架：它负责把模型、工具、技能（skill）、文件系统、终端、权限、安全策略、任务状态、日志追踪、评估回放和用户交互组织成一个可运行、可控、可观测、可复现的系统。

本书会重点分析当前主流 coding agent 和 agent runtime 的设计思想，包括 Claude Code、OpenCode、Codex、Cursor 类产品、Aider、SWE-agent、OpenHands 等。

其中 Claude Code 可参考公开源码研究资料，例如：

```text
https://github.com/luyao618/Claude-Code-Source-Study
```

后续扩写案例章节时，可以借鉴其中关于 Claude Code 架构、命令系统、工具调用、上下文管理、权限和执行链路的分析方式。

## 本书要回答的问题

1. Harness 到底是什么？
2. Coding agent 为什么不能只是一个模型 API？
3. Claude Code、OpenCode、Codex 这类系统的核心架构是什么？
4. Agent 如何读写文件、执行命令、调用工具、维护上下文？
5. Skill 在 harness 中和 tool、agent、prompt、memory 的边界是什么？
6. 如何设计权限、安全沙箱和用户确认机制？
7. 如何记录 trace、支持 replay、做 evaluation harness？
8. 如何衡量一个 coding agent 是否真的有效？
9. 如何设计一个可扩展的 agent runtime？
