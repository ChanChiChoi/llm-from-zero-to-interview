# 第五部分：面试与复习

这一部分把前面内容转成面试能力。第 29-30 讲已经讲过开放研究问题和模拟面试方法，这里继续补两个更贴近真实面试收尾的主题：OpenAI 风格开放题的系统拆解，以及 12 周总复习计划。

## 第 37 讲：OpenAI 风格开放题

### 本讲目标

OpenAI 风格开放题通常没有唯一标准答案。它考察的不是你能不能背出某篇论文，而是你能否把一个模糊问题拆成可验证、可实验、可取舍的研究或工程方案。

本讲你需要掌握：

1. 开放题和知识题的区别。
2. 如何用结构化框架拆题。
3. 如何讨论数据、模型、训练、推理、评估和安全。
4. 如何在不知道答案时仍然给出高质量分析。
5. 多模态、reasoning、安全和评估类开放题的回答模板。

### 开放题考什么

基础题常问：

```text
什么是 KV Cache？
```

开放题常问：

```text
如果模型数学推理能力差，你会如何 debug 和改进？
```

区别在于，开放题更关注：

1. 问题定义。
2. 假设拆解。
3. 实验设计。
4. 评估指标。
5. trade-off。
6. 风险控制。
7. 迭代路线。

它不是考你“知道什么”，而是考你“如何思考”。

### 通用回答框架

遇到开放题，可以用下面的框架：

```text
1. 澄清目标和约束
2. 拆分可能原因或子问题
3. 给出优先级
4. 设计实验验证假设
5. 提出改进方案
6. 说明评估指标
7. 讨论风险和 trade-off
```

这个框架几乎适用于训练、推理、多模态、评估、安全和产品化问题。

### 示例 1：模型数学推理能力差，如何 debug？

可以从六个层面拆。

第一，数据。

1. 数学数据比例是否足够。
2. 数据是否覆盖不同难度和题型。
3. 推理步骤是否正确。
4. 合成数据是否有错误。
5. 是否存在 benchmark contamination。

第二，模型。

1. 模型容量是否不足。
2. context length 是否限制长推理。
3. tokenizer 对数字、公式是否不友好。
4. 是否需要 verifier 或 tool use。

第三，训练。

1. SFT 是否只学答案不学过程。
2. 偏好优化是否偏好长但错误的解释。
3. 是否需要过程监督。
4. 是否需要 curriculum。

第四，推理。

1. greedy 是否过早锁定错误路径。
2. self-consistency 是否提升稳定性。
3. test-time compute 是否不足。
4. 是否可以结合代码执行或 symbolic solver。

第五，评估。

1. 是否只看最终答案。
2. 是否区分算术、代数、几何、证明。
3. 是否做错误归因。
4. 是否有分布外测试。

第六，安全和产品约束。

1. 高风险场景是否需要拒答或提示不确定。
2. 是否需要引用工具结果。
3. 延迟和成本是否可接受。

一个较好的面试回答不是只说“加数学数据”，而是能给出排查顺序和验证方法。

### 示例 2：如何训练下一代更强模型？

不要只回答“更大模型、更多数据”。

可以从多个维度回答：

1. 数据质量和数据配比。
2. 模型架构和 context length。
3. 训练稳定性和 scaling。
4. 后训练和偏好优化。
5. Reasoning 和 test-time compute。
6. 多模态和工具使用。
7. 评估、红队和安全。
8. 推理成本和产品体验。

推荐表达：

```text
我会先明确目标能力，比如数学、代码、多模态还是低延迟助手。然后围绕数据、模型、训练、后训练、推理和评估建立闭环。更大的模型可能有帮助，但如果数据质量、评估污染、后训练目标和推理策略没处理好，单纯 scale 不一定带来目标能力提升。
```

### 示例 3：如何判断模型是否真的具备推理能力？

回答要避免只说 benchmark。

可以讲：

1. 构造未见过的新题。
2. 做题面扰动和反事实改写。
3. 检查中间步骤是否可验证。
4. 用 verifier 或执行器验证答案。
5. 做组合泛化测试。
6. 比较不同 test-time compute 下的表现。
7. 做错误类型分析。

关键句：

```text
推理能力不能只靠静态 benchmark 分数判断，还要看模型能否在新表述、新组合和可验证任务上稳定泛化。
```

### 示例 4：如何平衡安全性和有用性？

安全题不要答成“危险就拒绝”。

更好的回答是分级：

1. 明确定义风险 taxonomy。
2. 区分正常请求、边界请求和危险请求。
3. 对危险请求拒绝并给安全替代。
4. 对正常请求避免 over-refusal。
5. 用红队、安全评估和人工抽检持续校准。
6. 监控上线后误拒率和漏拒率。

面试中可以强调：

```text
安全不是简单降低 helpfulness，而是在不同风险等级下给出合适行为。
```

### 示例 5：如何设计统一处理文本、图像、语音、视频的模型？

回答可以按系统拆：

1. 每种模态先有 encoder 或 tokenizer。
2. 用 connector 或统一 token space 对齐表示。
3. 用 LLM 或 multimodal transformer 做联合推理。
4. 输出端支持文本、语音或图像生成。
5. 训练上需要跨模态对齐、指令微调和偏好优化。
6. 评估上分别覆盖单模态、多模态组合和实时交互。
7. 安全上处理图像注入、语音冒充和视频伪造。

不要声称某个闭源模型内部一定怎么做，而要说清楚“可能路线”和“需要验证的点”。

### 不知道答案怎么办

开放题中，承认不确定比硬编更好。

可以这样说：

```text
这个具体实现我不确定，但我会从机制上这样分析。首先定义目标，然后列出几个可能原因，再用实验逐个排除。
```

好处是：

1. 不会编造事实。
2. 展示工程排查能力。
3. 让面试官看到你的思考过程。

### 真实项目中的坑

#### 坑 1：一上来给方案，不定义问题

开放题先问清目标、约束和指标，否则方案没有意义。

#### 坑 2：只谈模型，不谈数据

很多能力问题首先来自数据质量、数据覆盖和评估污染。

#### 坑 3：只谈 benchmark，不谈误差分析

平均分不能告诉你模型为什么失败。

#### 坑 4：不讲 trade-off

更高质量、更低延迟、更低成本、更安全通常不能同时最优。

#### 坑 5：把社区猜测当确定事实

对闭源模型、未公开 recipe、新模型细节，要明确区分官方披露、论文结论和推测。

### 面试官会怎么问

#### 问题 1：如果模型幻觉严重，你如何改进？

回答框架：

1. 先区分事实幻觉、引用幻觉、推理幻觉和工具幻觉。
2. 从数据、训练、解码、RAG、工具和 verifier 排查。
3. 建立事实性评估和引用准确率评估。
4. 对高风险场景引入拒答和人工审核。

#### 问题 2：如果多模态模型看图经常答错，如何 debug？

回答框架：

1. 检查 vision encoder 是否能提取足够特征。
2. 检查 connector 对齐是否充分。
3. 检查 instruction data 覆盖和质量。
4. 按 OCR、计数、空间关系、小目标做错误归因。
5. 用 grounding 或局部标注增强验证。

#### 问题 3：如果要降低推理成本，你会怎么做？

回答框架：

1. 模型侧：量化、蒸馏、稀疏、投机解码。
2. 系统侧：continuous batching、KV Cache 管理、prefix cache。
3. 产品侧：限制输出长度、缓存、路由小模型。
4. 评估质量、延迟和成本 trade-off。

### 小练习

1. 用通用框架回答“模型代码能力差如何改进”。
2. 设计一个评估模型是否真的会推理的实验。
3. 写一个 safety 和 helpfulness 平衡的回答模板。
4. 用 5 分钟回答“如何设计下一代多模态助手”。

### 本讲总结

本讲最重要的结论：

1. 开放题考察问题拆解和实验意识。
2. 好回答要包含目标、原因、实验、方案、指标和 trade-off。
3. 不确定时不要编造，要回到机制和验证路径。
4. 数据、评估和安全常常比单纯模型结构更关键。

## 第 38 讲：模拟面试与总复习

### 本讲目标

最后一讲的目标是把第一册 38 讲变成一个可执行复习系统。你要从“看过内容”升级到“能在面试中稳定表达”。

本讲你需要掌握：

1. 如何组织 12 周冲刺计划。
2. 如何准备自我介绍和项目讲述。
3. 如何做 mock interview。
4. 如何复盘错题。
5. 如何判断第一册是否真正掌握。

### 第一册知识地图

第一册可以分成五大块。

第一块：基础与语言模型。

1. LLM 在学什么。
2. Next-token prediction。
3. Cross entropy、MLE、perplexity。
4. Backpropagation、AdamW、LR schedule。

第二块：Transformer 核心。

1. Tokenization。
2. Embedding 和 position encoding。
3. Self-attention。
4. MHA、causal mask、Transformer block。
5. LayerNorm、RMSNorm、RoPE、miniGPT。

第三块：训练、对齐与推理。

1. Pretraining。
2. Scaling law。
3. Distributed training。
4. Instruction tuning、SFT、RLHF、DPO。
5. Decoding、KV Cache、FlashAttention、量化。
6. 幻觉、评估、安全、开放研究和模拟面试。

第四块：多模态基础。

1. 多模态模型概览。
2. CLIP。
3. VLM。
4. Diffusion。
5. 语音模型。
6. 视频生成。

第五块：面试综合。

1. 开放题拆解。
2. 总复习和 mock interview。

### 12 周冲刺计划

第 1-2 周：基础与语言模型。

目标：能讲清 LLM、next-token prediction、cross entropy、perplexity、反向传播和 AdamW。

验收：

1. 5 分钟讲清 LLM 在学什么。
2. 手写 next-token input/labels 构造。
3. 手写 cross entropy。
4. 解释 loss 不下降的排查方向。

第 3-4 周：Transformer 核心。

目标：能手写 attention，并讲清 Transformer block。

验收：

1. 推导 attention scores shape。
2. 实现 causal mask。
3. 比较 MHA、MQA、GQA。
4. 解释 RoPE 和长上下文问题。

第 5-6 周：训练主线。

目标：理解预训练、数据、scaling law 和分布式训练。

验收：

1. 画出预训练 pipeline。
2. 解释 Chinchilla scaling 的启发。
3. 比较 DDP、ZeRO/FSDP、TP、PP。
4. 列出训练 loss spike 排查清单。

第 7-8 周：后训练与对齐。

目标：讲清 SFT、RLHF、DPO 和 Reward Model。

验收：

1. 构造 chat template 样本。
2. 手写 SFT label mask。
3. 解释 PPO 中 reference model 的作用。
4. 手写 DPO loss 的直觉。
5. 举例说明 reward hacking。

第 9-10 周：推理、部署、评估和安全。

目标：讲清模型上线和可靠性问题。

验收：

1. 解释 prefill/decode。
2. 估算 KV Cache 显存。
3. 解释 FlashAttention 为什么省显存。
4. 比较 INT8、INT4 量化风险。
5. 设计一个事实性和安全评估表。

第 11 周：多模态基础。

目标：能从文本 LLM 扩展到图像、语音和视频。

验收：

1. 讲清 CLIP 对比学习。
2. 画出 VLM 架构。
3. 解释 diffusion 加噪和去噪。
4. 比较级联语音助手和端到端语音模型。
5. 说明视频生成为什么难。

第 12 周：开放题与模拟面试。

目标：把知识变成稳定表达。

验收：

1. 做一次 45 分钟 mock interview。
2. 准备 2 分钟自我介绍。
3. 准备一个项目的 5 分钟讲稿。
4. 从错题中整理 10 个高频薄弱点。

### 自我介绍模板

自我介绍不要堆经历，要围绕岗位能力组织。

推荐结构：

```text
1. 我是谁，以及当前背景
2. 我为什么转向或深入 LLM
3. 我做过哪些相关项目或系统学习
4. 我最强的能力点是什么
5. 我希望在这个岗位解决什么问题
```

示例骨架：

```text
我目前主要准备大模型算法岗，系统学习了语言模型、Transformer、预训练、后训练、推理优化、评估安全和多模态基础。我重点做过 miniGPT、SFT/LoRA、DPO、RAG 或推理服务相关项目。相比只会调用 API，我更关注模型训练目标、系统瓶颈和评估闭环。接下来希望在真实团队中继续深入模型能力改进和可靠评估。
```

### 项目讲述模板

每个项目按六段准备：

1. 问题：为什么做这个项目。
2. 数据：数据从哪里来，如何清洗和构造。
3. 方法：模型、训练、推理或系统方案。
4. 实验：指标、baseline 和 ablation。
5. 失败：遇到什么问题，如何 debug。
6. 改进：如果继续做，会怎么升级。

面试官通常不怕你项目小，怕你讲不出为什么和失败分析。

### Mock Interview 流程

一次 45 分钟模拟面试可以这样安排：

1. 5 分钟：自我介绍和背景追问。
2. 10 分钟：项目深挖。
3. 10 分钟：Transformer 或训练基础。
4. 10 分钟：推理、评估或安全。
5. 10 分钟：开放研究题。

每次 mock 后不要只记录“答得不好”，要写清楚：

1. 哪道题没答好。
2. 卡在定义、公式、代码还是 trade-off。
3. 正确回答应该怎么组织。
4. 下次如何用 2 分钟答清楚。

### 错题复盘模板

```text
题目：
我的原回答：
问题类型：概念不清 / 公式不会 / 代码不会 / trade-off 漏掉 / 表达混乱
正确答案框架：
我需要补的知识：
下次 2 分钟回答：
```

复盘的关键是把错误变成可训练项。

### 最终验收标准

学完第一册，不是“读完”就算完成，而是要达到：

1. 任意基础概念能 2 分钟讲清。
2. 高频公式能说明直觉和 shape。
3. 核心代码能手写或解释。
4. 训练、推理、评估、安全能串成系统。
5. 多模态问题能讲出主线和边界。
6. 开放题能给出结构化分析。
7. 项目能讲出问题、方法、结果、失败和改进。

### 本讲总结

本讲最重要的结论：

1. 面试准备要从阅读转向表达和复盘。
2. 第一册 38 讲构成大模型算法岗的最小完整闭环。

4. Mock interview 和错题复盘是把知识转成面试竞争力的关键。
